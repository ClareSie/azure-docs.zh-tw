## <a name="access-your-account-settings"></a>訪問您的帳戶設置

如果您尚未這樣做，您（或您的組織的管理員）應訪問合作夥伴中心帳戶的[帳戶設置](https://partner.microsoft.com/dashboard/account/management)，以便：
- 檢查您公司的帳戶驗證狀態
- 確認您的賽門鐵克 ID、賣家 ID、MPN ID、發行者 ID 和聯繫資訊，包括公司審批人和賣家連絡人
- 為將在合作夥伴中心使用您業務帳戶的任何人創建使用者帳戶

### <a name="open-developer-settings"></a>打開開發人員設置

帳戶設置位於合作夥伴中心[商業市場儀表板](https://partner.microsoft.com/dashboard/commercial-marketplace)的右上角。 選擇齒輪圖示（靠近儀表板右上角），然後選擇 **"開發人員"設置**。

![合作夥伴中心中的帳戶設置功能表](./media/dashboard-developer-settings.png)

在**帳戶設置**中，您將能夠查看您的：
- **帳戶詳細資訊**：帳戶類型和帳戶狀態
- **發行者 ID：** 賣家 ID、使用者 ID、發行者 ID、Azure AD 租戶等。
- **聯繫資訊**： 發行者顯示名稱、賣家連絡人姓名、電子郵件、電話和位址
- **跟蹤 GUID：** 任何跟蹤 GUID 與您的帳戶相關聯

### <a name="account-details"></a>帳戶詳細資訊

在"帳戶詳細資訊"部分中，您可以看到基本資訊，例如**您的帳戶類型**（公司或個人）和帳戶的**驗證狀態**。 在帳戶驗證過程中，這些設置將顯示所需的每個步驟，包括電子郵件驗證、雇傭驗證和業務驗證。 您也可以在此處更新電子郵件，並根據需要重新發送驗證。

### <a name="publisher-ids"></a>發行者代

在發行者 ID 部分中，您可以看到**您的賽門鐵克 ID、****賣家 ID、****使用者 ID、MPN** **ID**和 Azure **AD 租戶**。 這些值由 Microsoft 分配，以唯一標識您的開發人員帳戶，並且無法編輯。

### <a name="contact-info"></a>連絡資訊

在"聯繫資訊"部分中，您可以看到您的**Publisher 顯示名稱**、**賣家聯繫資訊**（公司賣家的連絡人姓名、電子郵件、電話號碼和位址）**和公司審批人**（有權批准公司決策的個人的姓名、電子郵件和電話號碼）。

#### <a name="payout-account"></a>支付帳戶

付款帳戶是從銷售中匯到收益的銀行帳戶。 此銀行帳戶必須位於您註冊合作夥伴中心帳戶的同一國家/地區。

要設置您的支出帳戶，您需要**關聯您的 Microsoft 帳戶**：
1. 轉到合作夥伴中心[中的"商業市場概述"頁面](https://partner.microsoft.com/dashboard/commercial-marketplace/overview)。
2. 在"設定檔"部分下，選擇 **"關聯您的 Microsoft 帳戶**"。
3. 出現提示後，請使用 Microsoft 帳戶 （MSA） 登錄。 此帳戶不能已與其他合作夥伴中心帳戶關聯。
4. 要完成支出帳戶的設置，請完全登出合作夥伴中心，然後使用 Microsoft 帳戶（而不是您的工作帳戶）重新登錄。

現在，您的 Microsoft 帳戶已關聯，要添加支出帳戶，您需要：
- **選擇付款條件**：銀行帳戶或PayPal
- **添加付款資訊**：這可能包括選擇帳戶類型（支票或儲蓄）、輸入帳戶持有人姓名、帳號和路由號碼、帳單位址、電話號碼或 PayPal 電子郵件地址。 *有關使用PayPal作為您的帳戶付款條件，並瞭解其是否支援您的市場區域的詳細資訊，請參閱[PayPal資訊](https://docs.microsoft.com/windows/uwp/publish/setting-up-your-payout-account-and-tax-forms#paypal-info)。

> [!IMPORTANT]
> 變更您的付款帳戶會延遲您的付款最多一次付款週期。 出現此延遲的原因是我們需要驗證帳戶更改，就像我們首次設置支出帳戶時一樣。 您仍然會在帳戶確認之後收到全額款項。當期付款週期未付的款項會新增至下一期。  

#### <a name="tax-profile"></a>稅金設定檔

查看您當前的稅務設定檔狀態，確認顯示正確的**實體類型**和**稅務證書資訊**。 選擇 **"編輯"** 以更新或完成任何必需的表單。

為了確立您的稅務身份，您必須指定您的居住國和公民身份，並填寫與您的國家/地區相關的相應納稅表格。

無論您的居住國還是公民權，您必須填寫美國稅務表格才能通過 Microsoft 銷售任何優惠。 滿足某些美國居住要求的合作夥伴必須填寫 IRS W-9 表格。 美國以外的其他合作夥伴必須填寫 IRS W-8 表格。 完成稅金設定檔之後，您就可以在線上填寫這些表單。

向 Microsoft 收到付款或要求稅務協定優惠時不需要美國個人納稅識別號碼 (或 ITIN)。

您可以在合作夥伴中心以電子方式填寫和提交您的納稅申報表;在大多數情況下，您不需要列印和郵寄任何表單。

不同國家和地區的稅金要求不同。 您必須在納稅中支付的確切金額取決於您銷售優惠的國家和地區。 在某些國家/地區，Microsoft 代表您徵收銷售和使用稅。 這些國家/地區將在列出您的報價的過程中被識別。 在其他國家/地區，根據您註冊的地點，您可能需要將銷售匯出，並將銷售用稅直接匯給當地稅務機關。 此外，您收到的銷售收入可能作為收入納稅。 我們強烈建議您聯繫您所在國家或地區的有關當局，以便最好地説明您確定適用于 Microsoft 銷售交易的稅務資訊。

##### <a name="withholding-rates"></a>扣繳率
您在納稅申報表中送出的資訊將決定適當的稅金扣繳率。 扣繳率只適用於您在美國完成的銷售；非美國地區的銷售不適用扣繳率。 扣繳率視情況而有不同，但對於在美國以外地區註冊的大多數開發人員而言，預設扣繳率為 30%。 如果您的國家/地區和美國已簽訂所得稅協定，您可以選擇降低此扣繳率。

##### <a name="tax-treaty-benefits"></a>稅務協定優惠
如果您在美國以外的地區，可能可以享有稅務協定優惠。 這些福利因國家/地區而異，並可能允許您減少 Microsoft 扣繳的稅款金額。 您可以完成 W-8BEN 表單的 Part II 來要求稅務協定優惠。 建議您與您國家或地區的適當資源聯繫，以確認您是否適用這些優惠。

[詳細瞭解 Windows 應用/遊戲開發人員和 Azure 應用商店發佈商的稅務詳細資訊](https://docs.microsoft.com/windows/uwp/publish/tax-details-for-paid-apps)。

#### <a name="payout-hold-status"></a>支付保留狀態

預設情況下，Microsoft 每月發送一次付款。 但是，您可以選擇暫停付款，這將阻止將付款發送到您的帳戶。 如果您選擇暫停您的支出，我們將繼續記錄您賺取的任何收入，並在**您的付款摘要**中提供詳細資訊。 但是，在您刪除保留之前，我們不會向您的帳戶發送任何付款。 

若要保留付款，請移至 [**帳戶設定**]。 在 [**財務詳細資料**] 的 [**支付保留狀態**] 區段中，將滑桿切換到 [**開啟**]。 您隨時都可以變更您的支付保留狀態，但是請注意您決定將會影響後續的每月支付。 例如，如果您希望保留 4 月份的付款，請確保在 3 月底之前將付款保留狀態設置為 **"打開**"。

將支付保留狀態設為 [**開啟**] 之後，除非將滑桿切換回 [**關閉**]，否則會保留所有支付。 當您這樣做時，您將在下一個月的支付週期中被包括在內（前提是任何適用的付款閾值都已滿足）。 例如，如果您已暫停付款，但希望 6 月生成支出，請確保在 5 月底之前將付款保留狀態切換為 **"關閉**"。

> [!NOTE]
> **您的"支出保留"狀態**選擇適用于通過 Microsoft 合作夥伴中心支付**的所有**收入來源，包括 Azure 應用商店、AppSource、Microsoft 應用商店、廣告等。 您無法針對個別收入來源選取不同保留狀態。

### <a name="tracking-guids"></a>跟蹤 GUID

全域唯一識別碼 （GUID） 是唯一的參考編號（具有 32 個十六進位數位），可用於跟蹤 Azure 使用方式。 

若要建立 GUID 來進行追蹤，您應該使用 GUID 產生器。 Azure 儲存體小組已建立 [GUID 產生器表單](https://aka.ms/StoragePartners)，其會透過電子郵件傳送正確格式的 GUID 給您，且可跨不同追蹤系統重複使用。

建議您為每個產品的每個供應項目與散發通道建立唯一的 GUID。 如果您不想要分割報告，您可以讓產品的多個散發通道使用單一 GUID。

若您使用範本部署產品，且該產品在 Azure Marketplace 與 GitHub 上都有提供，您必須建立並註冊 2 個相異的 GUID：

*   Azure Marketplace 中的產品 A
*   GitHub 上的產品 A

報告是由合作夥伴值 (Microsoft 合作夥伴 ID) 與 GUID 所完成。 您還可以在更精細的級別跟蹤 GUID，以便與產品/服務中的每個計畫保持一致。

有關詳細資訊，請參閱[使用 GUID 跟蹤 Azure 客戶使用方式常見問題 。](https://docs.microsoft.com/azure/marketplace/azure-partner-customer-usage-attribution#faq)

帳單位址從法人預先填充，您可以稍後更新此位址。 "稅務"和"加值稅 ID"欄位是可選的。  無法編輯國家/地區名稱和公司名稱。

## <a name="multi-user-account-management"></a>多使用者帳戶管理

合作夥伴中心利用[Azure 活動目錄](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-whatis)（Azure AD） 進行多使用者帳戶訪問和管理。 作為註冊過程的一部分，組織的 Azure AD 將自動與合作夥伴中心帳戶關聯。

## <a name="manage-users"></a>管理使用者

合作夥伴中心**的使用者**部分（在**帳戶設置**）下，讓我們使用 Azure AD 來管理有權訪問合作夥伴中心帳戶的使用者、組和 Azure AD 應用程式。 為了管理使用者，您必須使用[工作帳戶](./company-work-accounts.md)（關聯的 Azure AD 租戶）登錄。 要管理其他工作帳戶/租戶中的使用者，您需要登出，然後以具有該工作帳戶/租戶的**Manager**許可權的使用者身份重新登錄。

使用工作帳戶（Azure AD 租戶）登錄後，可以：
- [新增或移除使用者](#add-or-remove-users)
- [變更使用者密碼。](#change-a-user-password)
- [添加或刪除組](#add-or-remove-users)
- [添加或刪除 Azure AD 應用程式](#add-new-azure-ad-applications)
- [管理 Azure AD 應用程式的金鑰](#manage-keys-for-an-azure-ad-application)
- [定義使用者角色和權限](#define-user-roles-and-permissions)

請記住，所有合作夥伴中心使用者（包括組和 Azure AD 應用程式）都必須在與合作夥伴中心帳戶關聯的[Azure AD 租戶](#manage-tenants)中具有活動工作帳戶。

### <a name="add-or-remove-users"></a>新增或移除使用者

您的帳戶必須具有要在其中添加或編輯使用者[的工作帳戶（Azure AD 租戶）](./company-work-accounts.md)[**的管理器級**](#define-user-roles-and-permissions)許可權。

#### <a name="add-existing-users"></a>新增現有使用者

要將使用者添加到公司[工作帳戶（Azure AD 租戶）](./company-work-accounts.md)中已存在的合作夥伴中心帳戶：

1. 轉到**使用者**（在 **"帳戶設置**"下）並選擇"**添加使用者**"。
2. 從出現的清單中選取一個或多個使用者。 您可以使用搜尋方塊來搜尋特定的使用者。
•如果選擇要添加到合作夥伴中心帳戶的多個使用者，則必須為其分配相同的角色或自訂許可權集。 要添加具有不同角色/許可權的多個使用者，請為每個角色或自訂許可權集重複這些步驟。
3. 完成選擇使用者後，按一下"**添加選定的**"。
4. 在 **\[角色\]** 區段中，指定所選使用者的角色或自訂權限。
5. 選取 [儲存]****。

#### <a name="create-new-users"></a>建立新的使用者

要創建新使用者帳戶，您必須具有具有[**全域管理員**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)許可權的帳戶。 

1. 轉到**使用者**（在 **"帳戶設置**"下），選擇 **"添加使用者**"，然後選擇"**創建新使用者**"。
1. 為每個新使用者輸入名字、姓氏和使用者名。 
1. 如果希望新使用者在組織的目錄中具有全域管理員帳戶，請選中標記為 **"使此使用者成為 Azure AD 中的全域管理員"核取方塊，並完全控制所有目錄資源**。 這將可讓使用者完整存取您公司的 Azure AD 中的所有系統管理功能。 他們將能夠添加和管理組織的工作帳戶（Azure AD 租戶）中的使用者，儘管不在合作夥伴中心，除非您授予該帳戶適當的角色/許可權。
1. 如果選中此框**以使此使用者成為全域管理員**，則需要提供**密碼恢復電子郵件**，以便使用者在必要時恢復其密碼。
1. 在 **"組成員身份**"部分中，選擇要新使用者所屬的任何組。
1. 在 **\[角色\]** 區段中，指定使用者的角色或自訂權限。
1. 選取 [儲存]****。

在合作夥伴中心創建新使用者還將在您登錄的工作帳戶（Azure AD 租戶）中為該使用者創建一個帳戶。 更改合作夥伴中心中的使用者名將在組織的工作帳戶（Azure AD 租戶）中進行相同的更改。

#### <a name="invite-new-users-by-email"></a>通過電子郵件邀請新使用者

要通過電子郵件邀請當前不是公司工作帳戶（Azure AD 租戶）一部分的使用者，您必須具有具有[**全域管理員**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)許可權的帳戶。

1. 轉到**使用者**（在 **"帳戶設置**"下），選擇 **"添加使用者**"，然後選擇**通過電子郵件邀請使用者**。
2. 輸入一個或多個電子郵件地址 (最多十個)，以逗號或分號分隔。
3. 在 **\[角色\]** 區段中，指定使用者的角色或自訂權限。
4. 選取 [儲存]****。

您邀請的使用者將獲得加入合作夥伴中心帳戶的電子郵件邀請。 將在工作帳戶（Azure AD 租戶）中創建新的來賓使用者帳戶。 使用者必須接受邀請才能存取您的帳戶。

如果需要重新發送邀請，請訪問 **"使用者"** 頁面，在使用者清單中查找邀請，選擇其電子郵件地址（或顯示 *"邀請待定*"的文本）。 然後，在頁面底部，選擇 **"重新發送邀請**"。

> [!NOTE]
> 如果組織使用[目錄集成](https://go.microsoft.com/fwlink/p/?LinkID=724033)將本地目錄服務與 Azure AD 同步，則無法在合作夥伴中心創建新使用者、組或 Azure AD 應用程式。 您需要（或本地目錄中的另一個管理員）需要在本地目錄中直接創建它們，然後才能在合作夥伴中心中查看和添加它們。

#### <a name="remove-a-user"></a>移除使用者

要從工作帳戶（Azure AD 租戶）中刪除使用者，請轉到 **"使用者**"（在 **"帳戶設置**"下），使用最右側列中的核取方塊選擇要刪除的使用者，然後從可用操作中選擇 **"刪除**"。 將顯示一個快顯視窗，以確認要刪除選定的使用者。

#### <a name="change-a-user-password"></a>變更使用者密碼。

如果您的一個使用者需要更改其密碼，如果您在創建使用者帳戶時提供了**密碼恢復電子郵件**，他們可以自己更改密碼。 您也可以依照下列步驟來更新使用者的密碼。 要更改公司工作帳戶（Azure AD 租戶）中的使用者密碼，必須在具有[**全域管理員**](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles)許可權的帳戶上登錄。 請注意，這將更改 Azure AD 租戶中的使用者的密碼以及他們用於訪問合作夥伴中心的密碼。

1. 從 **"使用者"** 頁（在 **"帳戶設置**"下）選擇要編輯的使用者帳戶的名稱。
2. 選擇頁面底部的 **"重置密碼**"按鈕。
3. 將顯示一個確認頁，顯示使用者的登錄資訊，包括臨時密碼。 請務必列印或複製此資訊，並將它提供給該使用者，因為您在離開此頁面之後將無法存取該暫時密碼。

## <a name="manage-groups"></a>管理群組

組允許您一起控制多個使用者角色和許可權。

#### <a name="add-an-existing-group"></a>添加現有組

要將組織工作帳戶（Azure AD 租戶）中已存在的組添加到合作夥伴中心帳戶：請執行：

1. 從 **"使用者"** 頁（在 **"帳戶設置**"下）選擇 **"添加組**"。
2. 從出現的清單中選取一個或多個群組。 您可以使用搜尋方塊來搜尋特定的群組。
如果選擇要添加到合作夥伴中心帳戶的多個組，則必須為其分配相同的角色或自訂許可權集。 要添加具有不同角色/許可權的多個組，請為每個角色或一組自訂許可權重複這些步驟。
3. 完成選擇組後，按一下"**添加所選**"。
4. 在 **\[角色\]** 區段中，指定所選群組的角色或自訂權限。 組的所有成員將能夠訪問您的合作夥伴中心帳戶，您向該組申請的許可權，而不考慮與其個人帳戶關聯的角色和許可權。
5. 選取 [儲存]****。

添加現有組時，該組成員的每個使用者將能夠訪問您的合作夥伴中心帳戶，並具有與該組分配的角色關聯的許可權。

#### <a name="add-a-new-group"></a>新增群組

要向合作夥伴中心帳戶添加全新組，請：

1. 從 **"使用者"** 頁（在 **"帳戶設置**"下）選擇 **"添加組**"。
2. 在下一頁上，選擇 **"新群組**"。
3. 輸入新群組的顯示名稱。
4. 指定群組的角色或自訂權限。 無論與個人帳戶關聯的角色/許可權如何，組所有成員都可以使用您在此處申請的許可權訪問您的合作夥伴中心帳戶。
5. 從顯示的清單中為新組選擇使用者。 您可以使用搜尋方塊來搜尋特定的使用者。
6. 完成選取使用者時，按一下 **\[新增選取的項目\]**，將其新增到新群組。
7. 選取 [儲存]****。

請注意，此新組也將在組織的工作帳戶（Azure AD 租戶）中創建，而不僅僅是在合作夥伴中心帳戶中。

#### <a name="remove-a-group"></a>刪除組

要從工作帳戶（Azure AD 租戶）中刪除組，請轉到 **"使用者**"（在 **"帳戶設置"** 下），使用最右側列中的核取方塊選擇要刪除的組，然後從可用操作中選擇 **"刪除**"。 將顯示一個快顯視窗，以確認要刪除選定的組。

## <a name="manage-azure-ad-applications"></a>管理 Azure AD 應用程式

您可以允許屬於公司 Azure AD 的應用程式或服務訪問合作夥伴中心帳戶。

#### <a name="add-existing-azure-ad-applications"></a>添加現有的 Azure AD 應用程式

要添加公司 Azure 活動目錄中已存在的應用程式，請執行：

1. 從 **"使用者"** 頁（在 **"帳戶設置**"下）選擇 **"添加 Azure AD 應用程式**"。
2. 從出現的清單中選取一個或多個 Azure AD 應用程式。 您可以使用搜尋方塊來搜尋特定的 Azure AD 應用程式。 如果選擇要添加到合作夥伴中心帳戶的多個 Azure AD 應用程式，則必須為其分配相同的角色或自訂許可權集。 要添加具有不同角色/許可權的多個 Azure AD 應用程式，請為每個角色或一組自訂許可權重複這些步驟。
3. 選擇完 Azure AD 應用程式後，按一下"**添加選定的**"。
4. 在 **\[角色\]** 區段中，指定所選 Azure AD 應用程式的角色或自訂權限。
5. 選取 [儲存]****。

#### <a name="add-new-azure-ad-applications"></a>添加新的 Azure AD 應用程式

如果要授予合作夥伴中心對全新 Azure AD 應用程式帳戶的存取權限，可以在 **"使用者"** 部分創建一個。 請注意，這將在公司工作帳戶（Azure AD 租戶）中創建新帳戶，而不僅僅是在合作夥伴中心帳戶中。 如果您主要使用此 Azure AD 應用程式進行合作夥伴中心身份驗證，並且不需要使用者直接存取它，則可以為**回復 URL** **和應用 ID URI**輸入任何有效位址，只要這些值不由目錄中的任何其他 Azure AD 應用程式使用。

1. 從 **"使用者"** 頁（在 **"帳戶設置**"下）選擇 **"添加 Azure AD 應用程式**"。
2. 在下一頁上，選擇**新的 Azure AD 應用程式**。
3. 輸入新 Azure AD 應用程式的**回復 URL。** 使用者可以從此 URL 登入並使用您的 Azure AD 應用程式 (也稱為 App URL 或登入 URL)。 **回覆 URL** 長度不可超過 256 個字元，而且在您的目錄中必須是唯一的。
4. 輸入新 Azure AD 應用程式的 **\[應用程式識別碼 URI\]**。 這是 Azure AD 應用程式的邏輯識別碼，在將單個登錄要求傳送到 Azure AD 時顯示。 請注意，對於目錄中的每個 Azure AD 應用程式，**應用 ID URI**必須是唯一的。 此 ID 不能超過 256 個字元。 有關應用 ID URI 的詳細資訊，請參閱[將應用程式與 Azure 活動目錄集成](https://docs.microsoft.com/azure/active-directory/develop/quickstart-modify-supported-accounts#change-the-application-registration-to-support-different-accounts)。
5. 在 **\[角色\]** 區段中，指定 Azure AD 應用程式的角色或自訂權限。
6. 選取 [儲存]****。

新增或建立 Azure AD 應用程式之後，您可以返回 **\[使用者\]** 區段，然後選取應用程式名稱以檢閱應用程式的設定，包括租用戶識別碼、用戶端識別碼、回覆 URL，以及 App 識別碼 URI。

#### <a name="remove-an-application"></a>移除應用程式

要從工作帳戶（Azure AD 租戶）中刪除應用程式，請轉到 **"使用者**"（在 **"帳戶設置"** 下），使用最右側列中的核取方塊選擇要刪除的應用程式，然後從可用操作中選擇 **"刪除**"。 將顯示一個快顯視窗，以確認要刪除選定的應用程式。

#### <a name="manage-keys-for-an-azure-ad-application"></a>管理 Azure AD 應用程式的金鑰

如果您的 Azure AD 應用程式會在 Microsoft Azure AD 讀取或寫入資料，則它需要金鑰。 通過在合作夥伴中心中編輯其資訊，可以為 Azure AD 應用程式創建金鑰。 您也可以移除不再需要的金鑰。

1. 從 **"使用者"** 頁（在 **"帳戶設置**"下）選擇 Azure AD 應用程式的名稱。 您將看到 Azure AD 應用程式的所有活動金鑰，包括創建金鑰的日期和何時過期。 
2. 要刪除不再需要的鍵，請選擇 **"刪除**"。
3. 要添加新鍵，請選擇"**添加新鍵**"。
4. 您將看到一個螢幕，顯示**用戶端 ID**和**鍵值**。 請務必列印或複製此資訊，因為離開此頁面後將無法再次訪問此資訊。
5. 如果要創建更多鍵，請選擇"**添加其他鍵**"。

## <a name="define-user-roles-and-permissions"></a>定義使用者角色和權限

可以為合作夥伴中心上的商業市場計畫分配以下角色和許可權：

- **Manager**
  - 可以訪問除稅費和付款設置之外的所有 Microsoft 帳戶功能
  - 可以管理使用者、角色和工作帳戶（租戶）
- **開發 人員**
  - 可以管理和發佈優惠
  - 可以查看某些發行者報告

> [!NOTE]
> 對於商業市場計畫，不使用全域管理員、業務貢獻者、財務貢獻者和行銷人員角色。 將這些角色指派給使用者不起作用。 只有管理器和開發人員角色向使用者授予許可權。

有關在合作夥伴中心的其他區域（如 Azure 活動目錄 （AD）、雲解決方案供應商 （CSP）、控制台供應商 （CPV）、來賓使用者或 Microsoft 合作夥伴網路 （MPN））中管理角色和許可權的詳細資訊，請參閱[在合作夥伴中心中分配使用者角色和許可權](https://docs.microsoft.com/partner-center/permissions-overview)。

## <a name="manage-tenants"></a>管理租戶

Azure 活動目錄 （AD） 租戶（在本文檔中也稱為"工作帳戶"）是您在 Azure 門戶中設置的組織的表示形式，可説明您為內部管理 Microsoft 雲服務的特定實例和外部使用者。 如果您的組織訂閱了 Microsoft 雲服務（如 Azure、Microsoft Intune 或 Office 365），則為您建立了 Azure AD 租戶。

您可以設置多個租戶以與合作夥伴中心一起使用。 在合作夥伴中心帳戶中具有**Manager**角色的任何使用者都可以選擇從帳戶中添加和刪除 Azure AD 租戶。  

### <a name="add-an-existing-tenant"></a>添加現有租戶

要將另一個 Azure AD 租戶與您的合作夥伴中心帳戶關聯，請執行：

1. 從 **"租戶"** 頁（在 **"帳戶設置**"下）選擇 **"關聯其他 Azure AD 租戶**"。
2. 輸入您想要建立關聯之租用戶的 Azure AD 認證。
3. 檢閱 Azure AD 租用戶的組織和網域名稱。 若要完成關聯，請選取 **\[確認\]**。

如果關聯成功，您將準備好在合作夥伴中心中的 **"使用者"** 部分添加和管理帳戶使用者。

### <a name="create-a-new-tenant"></a>創建新租戶

要使用合作夥伴中心帳戶創建新的 Azure AD 租戶，請執行：

1. 從 **"租戶"** 頁（在 **"帳戶設置**"下）選擇 **"創建新的 Azure AD 租戶**"。
2. 輸入新 Azure AD 的目錄資訊︰
    - **功能變數名稱**：我們將用於 Azure AD 域的唯一名稱，以及".onmicrosoft.com"。 例如，如果輸入"示例"，則 Azure AD 域將為"example.onmicrosoft.com"。
    - **連絡人電子郵件**：我們可以在需要時針對帳戶相關資訊與您連絡的電子郵件地址。
    - **全域管理員使用者帳戶資訊**：您想要針對新的全域管理員帳戶使用的名字、姓氏、使用者名稱及密碼。
3. 選擇 **"創建**"以確認新域和帳戶資訊。
4. 使用新的 Azure AD 全域管理員使用者名和密碼登錄，開始[添加和管理使用者](#manage-users)。

有關在 Azure 門戶（而不是通過合作夥伴中心門戶）中創建新租戶的詳細資訊，請參閱在 Azure[活動目錄中創建新租戶](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-access-create-new-tenant)的文章。

### <a name="remove-a-tenant"></a>刪除租戶

要從合作夥伴中心帳戶中刪除租戶，請在 **"租戶"** 頁上查找其名稱（在 **"帳戶設置**"中），然後選擇"**刪除**"。 系統將提示您確認要刪除租戶。 執行此操作後，該租戶中的任何使用者都無法登錄到合作夥伴中心帳戶，並且您為這些使用者配置的任何許可權都將被刪除。

刪除租戶時，從該租戶添加到合作夥伴中心帳戶的所有使用者將不再能夠登錄到該帳戶。

> [!TIP]
> 如果您當前使用同一租戶中的帳戶登錄到合作夥伴中心，則無法刪除租戶。 要刪除租戶，您必須以與帳戶關聯的其他租戶**的經理**身份登錄到合作夥伴中心。 如果只有一個租用戶關聯至帳戶，則必須以開啟該帳戶的 Microsoft 帳戶登入後，才能移除該租用戶。

## <a name="agreements"></a>合約

合作夥伴中心**的協定**部分（在**帳戶設置**下）允許您查看您授權的發佈協定的清單。 這些協定按名稱和版本號列出，包括接受協定的日期和接受協定的使用者的名稱。

如果需要**的操作**可能會顯示在此頁面頂部，如果有需要您注意的協定更新。 要接受更新的協定，請先閱讀連結的協定版本，然後選擇 **"接受協定**"。
