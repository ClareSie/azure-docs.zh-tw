## <a name="azure-maps-supported-views"></a>Azure 地圖支援的視圖

> [!Note]
> 2019 年 8 月 1 日，Azure 地圖在以下國家/地區發佈：
>  * 阿根廷
>  * 印度
>  * 摩洛哥
>  * 巴基斯坦
>
> 2019 年 8 月 1 日之後，"**視圖"** 參數將定義上述新區域/國家/地區的返回的地圖內容。 Azure 地圖**視圖**參數（也稱為"使用者區域參數"）是一個兩個字母的 ISO-3166 國家/地區代碼，它將顯示該國家/地區的正確地圖，指定通過 Azure 地圖服務返回哪組具有地緣政治爭議的內容，包括地圖上顯示的邊界和標籤。 

請確保根據服務正在使用的 REST API 和 SDK 的要求設置**視圖**參數。
>  
>
>  **休息 API：**
>  
>  確保已根據需要設置"視圖"參數。 視圖參數指定通過 Azure 地圖服務返回的地緣政治爭議內容集。 
>
>  受影響的 Azure 映射 REST 服務：
>    
>    * 獲取地圖磁貼
>    * 獲取地圖圖像 
>    * 獲取搜索模糊
>    * 獲取搜索 POI
>    * 獲取搜索 POI 類別
>    * 獲取搜索附近
>    * 獲取搜索位址
>    * 獲取結構化搜索位址
>    * 取得搜尋位址反轉
>    * 獲取搜索位址反向交叉街
>    * Post 幾何內搜尋
>    * 發佈搜索位址批量預覽
>    * 發佈搜索位址反向批次處理預覽
>    * 沿路線發佈搜索
>    * 後搜索模糊批次處理預覽
>
>    
>  **Sdk：**
>
>  確保已根據需要設置 **"視圖"** 參數，並且具有最新版本的 Web SDK 和 Android SDK。 受影響的 SDK：
>
>    * Azure 地圖服務 Web SDK
>    * Azure 地圖服務 Android SDK

預設情況下，View 參數設置為**Unified**，即使您尚未在請求中定義它。 確定使用者的位置。 然後，為該位置正確設置 **"視圖"** 參數。 或者，您可以設置"View_Auto"，它將基於請求的 IP 位址返回地圖資料。  Azure 地圖中的 **"視圖"** 參數必須符合適用法律，包括有關地圖、圖像和其他資料以及通過 Azure 地圖有權訪問的協力廠商內容的地圖國家/地區的地圖的法律。
