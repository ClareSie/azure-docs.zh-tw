# <a name="customer-dashboard-in-commercial-marketplace-analytics"></a>商業市集分析中的客戶儀表板

本文提供合作夥伴中心的 [客戶儀表板] 相關資訊。 此儀表板會顯示包括成長趨勢在內的客戶資訊，以圖形化及可下載的格式呈現。

若要存取 [客戶儀表板]，請開啟 [商業市集] 下的 **[[分析儀表板]](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** 。

>[!NOTE]
> 如需分析術語的詳細定義，請參閱[商業市集分析的常見問題集與術語](./faq-terminology.md)。

## <a name="customer-dashboard"></a>客戶儀表板

[分析] 功能表的 [客戶儀表板] 會顯示已取得供應項目的客戶資料。 您可檢視以圖形表示的下列項目：

- [客戶摘要](#customer-summary)
- [依地理位置排列的客戶](#customer-by-geography)
- [客戶趨勢](#customer-trends)
- [依訂單和使用量排列的客戶](#customers-by-orders-and-usage)
- [依 SKU 排列的客戶](#customers-by-skus)
- [依客戶類型排列的訂單和使用量](#orders-and-usage-by-customer-type)
- [客戶詳細資料表](#customer-details-table)
- [客戶頁面篩選](#customer-page-filters)

### <a name="customer-summary"></a>客戶摘要

[客戶摘要] 區段會顯示所選日期範圍內的所有客戶計數，包括新舊客戶及流失的客戶。

- 客戶總數的定義是：曾購買過供應項目，且至少有一份訂單成立的所有客戶計數。
- 與上個月相比的客戶成長百分比，會以數字和綠色向上指標或紅色向下指標表示。
- 成長趨勢會以橫條圖表示，將滑鼠暫留在圖表的資料行上即會顯示每個月的值。

有三種**客戶類型**：新客戶、舊客戶和流失的客戶。

- 新客戶是在選取的月份內，第一次取得一或多個供應項目的客戶。
- 舊客戶是在選取的月份前，取得一或多個供應項目的客戶。
- 所流失客戶是取消之前所購買全部供應項目的客戶。

### <a name="customer-by-geography"></a>依地理位置排列的客戶

**依地理位置排列的客戶**圖會顯示所有客戶計數，以及在選取日期範圍內取得並根據客戶國家/地區對應的客戶計數。 地圖上由淺到深色彩代表由低到高的客戶計數值。 按一下資料表中的記錄，以放大國家/地區。

熱度圖會顯示客戶計數和依客戶國家/地區排列的百分比。 您可移動地圖以檢視確切的位置，並放大特定的位置。 此地圖有輔助線，其可供依位置檢視客戶的百分比，以及該位置的新增客戶。

### <a name="customer-trends"></a>客戶趨勢

**客戶趨勢**圖會逐月顯示所有客戶計數的成長趨勢，包括新舊客戶及流失的客戶。

- 折線圖表示整體客戶成長的百分比。
- [月份] 資料行代表由新舊客戶及所流失客戶堆疊而成的客戶計數。
- 流失的客戶計數會顯示在 Y 軸負值方向上。
- 您可選取特定圖例項目以顯示更詳細的檢視。 例如，從圖例中選取新客戶，以只顯示新客戶。
- 您可使用圖表上方的滑桿，沿著 X 軸左右捲動，將焦點放在特定資料點上來檢視詳細資料。
- 將滑鼠暫留在圖表的資料行上，只會顯示該月份的詳細資料。

### <a name="customers-by-orders-and-usage"></a>依訂單和使用量排列的客戶

**依訂單/使用量排列的客戶**圖有三個索引標籤：[訂單]、[標準化的使用量] 和 [未處理使用量]。 沿著 X 軸會顯示**最上位客戶的百分位數**，由其訂單數決定。 Y 軸顯示客戶的訂單計數。 次要 Y 軸 (折線圖) 會顯示總訂單數的累計百分比。 您可將滑鼠暫留在折線圖的點上，以顯示詳細資料。

請參閱以下範例圖的標準化使用量：客戶其前 30 個百分位數貢獻了累計 87% 的標準化使用量。 客戶其第 30 個百分位數只貢獻了157 萬個小時的使用量。

### <a name="customers-by-skus"></a>依 SKU 排列的客戶

**依 SKU/使用量排列的客戶**圖如下所示。

1. 大戶排行榜面板顯示訂單計數排名前 50 位客戶的詳細資料。 選取客戶之後，此大戶排行榜面板的第 2、3 和 4 區段即會顯示客戶的詳細資料。
2. 當發行者以擁有者角色登入時，客戶設定檔詳細資料就會顯示在此位置。 如果發行者是以參與者角色登入，即無法取得此區段中的詳細資料。
3. 依 SKU 排列的訂單環圈圖會顯示 SKU 購物訂單明細。 顯示訂單計數最高的前 5 項 SKU，其餘的訂單則合計在「其他全部」項下。
4. 依 SKU 排列的座椅環圈圖會顯示 SKU 座椅訂單明細。 顯示座椅數最高的前 5 項 SKU，其餘的訂單則合計在「其他全部」項下。

### <a name="orders-and-usage-by-customer-type"></a>依客戶類型排列的訂單和使用量

**依客戶類型排列的訂單/使用量**圖會分別根據圖中所選訂單、標準化使用量和未處理使用量來顯示按新客戶與舊客戶其訂單數目、標準化使用量和未處理使用量的時數份額。

- 新客戶是在同一行事曆月份內 (Y 軸)，第一次取得一或多個供應項目的客戶。
- 舊客戶是在此行事曆月份 (Y 軸) 前，已取得供應項目或回報使用量的客戶。

### <a name="customer-details-table"></a>客戶詳細資料表

**客戶詳細資料**表會依日期排序來顯示初次取得其中一項供應項目的前 1000 名客戶編號清單。

- 客戶的個人資訊必須經過客戶同意才會顯示。 您只有在以擁有者角色層級的權限登入時，才可以檢視此資訊。 深入了解使用者角色和權限。
- 方格中的每個資料行皆可排序。
- 如果記錄計數小於 1000 項，則可將資料擷取至 TSV 檔案。
- 如果記錄數目超過 1000 項，則匯出的資料會在接下來 30 天內以非同步方式放置在下載頁面中。
- 您可在資料表中套用篩選，以只顯示感興趣的資料。 資料篩選條件包括公司名稱、客戶識別碼、市集訂閱識別碼、Azure 授權類型、取得日期、遺失日期、客戶電子郵件、客戶國家/地區/州/城市/郵遞區號、客戶語言等。
- 當受保護的客戶購買供應項目時，將會遮罩 (************) [Customer Detailed Data] \(客戶詳細資料\) 中的資訊。
- 客戶維度詳細資料，例如公司名稱、客戶姓名和客戶電子郵件屬於組織識別碼層級，而不屬於 Azure Marketplace 或 AppSource 交易層級。

### <a name="customer-page-filters"></a>客戶頁面篩選

**客戶頁面**篩選會套用至客戶頁面層級。 您可選取多個篩選條件來顯示所選檢視條件的圖表，以及要查看的 [詳細訂單資料] 方格/匯出資料。 篩選條件會套用至在訂單頁面右上角選取的資料範圍內所擷取資料。

>[!NOTE]
> [客戶] 方格中每個欄位的詳細定義、頁面篩選，以及其可能的選擇範圍，請參閱[商業市集分析的常見問題集與術語](./faq-terminology.md)。
