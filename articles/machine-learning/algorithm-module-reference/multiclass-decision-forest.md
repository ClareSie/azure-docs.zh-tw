# <a name="multiclass-decision-forest-module"></a>多類決策林模組

本文介紹 Azure 機器學習設計器（預覽）中的模組。

使用此模組可以創建基於*決策林*演算法的機器學習模型。 決策林是一種集合模型，它快速生成一系列決策樹，同時從標記的資料中學習。

## <a name="more-about-decision-forests"></a>有關決策林的更多

決策林演算法是一種綜合學習方法。 該演算法的工作原理是構建多個決策樹，然後對最受歡迎的輸出類*進行投票*。 投票是一種聚合形式，其中分類決策林中的每個樹輸出標籤的非正常化頻率長條圖。 聚合過程匯總這些長條圖並正常化結果，以獲得每個標籤的"概率"。 具有高預測信心的樹木在合唱團的最終決定中具有更大的份量。

決策樹通常是非參數模型，這意味著它們支援具有不同分佈的資料。 在每個樹狀結構中，對每一級執行一系列簡單的測試，同時增加樹狀結構的層級，直到抵達葉節點 (決策) 為止。

決策樹有許多優點：

+ 可以代表非線性決策界限。
+ 在定型和預測期間，執行運算和記憶體使用都很有效率。
+ 它們會執行整合式特徵選取和分類。
+ 它們可彈性呈現雜訊特徵。

Azure 機器學習中的決策林分類器由一組決策樹組成。 一般而言，集團模型比單一決策樹的涵蓋範圍更廣、精確度更高。 有關詳細資訊，請參閱[決策樹](https://go.microsoft.com/fwlink/?LinkId=403677)。

## <a name="how-to-configure-multiclass-decision-forest"></a>如何配置多類決策林

1. 將**多類決策林**模組添加到設計器中的管道中。 您可以在**機器學習**、**初始化模型**和**分類**下找到此模組。

2. 按兩下模組以打開"**屬性"** 窗格。

3. 對於**重採樣方法**，請選擇用於創建單個樹的方法。  您可以選擇封袋或複寫。

    + **裝袋**： 裝袋也稱為*引導*聚合。 在此方法中，每個樹都生長在新樣本上，通過隨機採樣原始資料集進行替換創建，直到資料集具有原始資料集的大小。 模型的輸出通過*投票*（一種聚合形式）進行組合。 有關詳細資訊，請參閱 Bootstrap 聚合的維琪百科條目。

    + **複製**：在複製中，每個樹都接受完全相同的輸入資料的培訓。 為每個樹節點使用拆分謂詞的確定保持隨機狀態，從而創建不同的樹。

   

4. 通過設置 **"創建培訓師模式"** 選項，指定如何訓練模型。

    + **單一參數**：如果您知道如何配置模型，並提供一組值作為參數，請選擇此選項。

    + **參數範圍**：如果不確定最佳參數，並且想要運行參數掃描，請選擇此選項。 選擇要反覆運算的值範圍，[調諧模型 Hyper 參數](tune-model-hyperparameters.md)反覆運算您提供的所有可能組合設置，以確定生成最佳結果的超參數。   

5. **決策樹的數量**：鍵入可在集合中創建的決策樹的最大數量。 通過創建更多決策樹，您可以獲得更好的覆蓋率，但培訓時間可能會增加。

    在視覺化訓練的模型時，此值還控制結果中顯示的樹數。 要查看或列印單個樹，可以將該值設置為 1;將 值設置為 1。但是，這意味著只能生成一個樹（具有初始參數集的樹），並且不執行進一步的反覆運算。

6. **決策樹的最大深度**：鍵入一個數位以限制任何決策樹的最大深度。 增加樹狀結構的深度可增加有效位數，但可能會有過度配適及定型時間增加的風險。

7. **每個節點的隨機拆分數**：鍵入生成樹的每個節點時使用的拆分數。 *拆分*表示樹（節點）的每個級別中的要素被隨機分割。

8. **每個葉節點的最小樣本數**：指示在樹中創建任何終端節點（葉）所需的最小案例數。 藉由增加此值，您會增加建立新規則的臨界值。

    例如，若預設值是 1，即使單一案例可能會造成新規則的建立。 如果您將此值增加為 5，則定型資料必須至少包含五個符合相同條件的案例。



10. 連接標記的資料集以及其中一個訓練模組：

    + 如果將 **"創建訓練器模式"** 設置為 **"單一參數**"，請使用[訓練器模型](./train-model.md)模組。

11. 提交管道。
