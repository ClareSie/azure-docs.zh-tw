# <a name="getting-paid"></a>取得付款
以下是有關接收優惠、附加元件和廣告收入付款的重要資訊。

> [!IMPORTANT]
> 在商業市場從優惠銷售中收到資金之前，您需要[設置您的支付帳戶並填寫必要的稅務表格](set-up-your-payout-account-tax-forms.md)。

## <a name="store-fee"></a>市集費用

註冊[開發人員帳戶](https://go.microsoft.com/fwlink/p/?LinkID=615100)時，接受[Microsoft Azure 應用商店發行者協定](https://go.microsoft.com/fwlink/p/?LinkID=699560)。 本協定解釋了您與 Microsoft 之間的關係，因為它與商業市場中的銷售報價有關，包括 Microsoft 每次銷售收取的應用商店費用。

費用在微軟 Azure[應用商店發佈商協定中](https://go.microsoft.com/fwlink/p/?LinkID=699560)正式定義。 如果您有任何問題，請一律檢閱該文件。

商店費用適用于商業市場收集的所有產品/服務銷售，包括附加服務。

## <a name="price-tiers"></a>價格區間

您選擇的價格層設置您選擇分發產品/服務的所有國家/地區的銷售價格。 您還可以使用其他定價功能，例如為不同的市場選擇不同的價格。

您可以免費提供你的應用，也可以選擇客戶獲得您的報價必須支付的價格。 價格區間從 .99 美元開始，逐額遞增 (1.09 美元、1.19 美元等)。 價格區間之間的遞增值會隨著價格變高而增加。

> [!NOTE] 
> 這些價格等級也適用于您在產品/服務範圍內提供的任何附加元件。

每個價格區間對於市集所提供的貨幣都有對應值。 我們使用這些值説明您以全球可比價格點銷售您的產品/服務。 不過，由於貨幣匯率隨時都在變化，因此確切的銷售金額可能會隨貨幣種類的不同而有些微差異。

您也有選項可以輸入您選擇以特定市場當地貨幣表示的自由格式價格。 這樣做時，除非您提交新價格的更新，否則不會調整價格 (即使轉換率變更)。 

請記住，您所選取的價格可能會包含客戶必須支付的銷售或增值稅。 有關詳細資訊[，請參閱稅務詳細資訊，瞭解已付優惠](tax-details-paid-transactions.md)。


## <a name="payout-reporting"></a>支付報告

您可以在[合作夥伴中心的](https://partner.microsoft.com/dashboard)**付款摘要**中訪問有關付款資訊的詳細資訊並下載報告。 如需此處所顯示的詳細資訊，以及我們將您所賺取金額分類的方式，請參閱[支付摘要](payout-summary.md)。


## <a name="payout-timeframe"></a>支付時間範圍

付款是每月進行一次 (假設已符合適用的付款門檻，而您尚未進行保留支付，如下所述)。 我們通常會在某個指定月份，於該月份的 15 日之前傳送任何付款。 請注意，付款通常需要 3 到 10 個額外工作天，才能送達您的支付帳戶。 如需詳細資訊，請參閱[付款門檻、方法和時間範圍](payment-thresholds-methods-timeframes.md)。


##  <a name="payout-hold-status"></a>支付保留狀態

我們預設會每個月傳送付款，如上所述。 但是，您可以選擇暫停您的計畫支出，這將阻止我們向您的帳戶發送付款。 如果您選擇保留支付，我們會繼續記錄任何您賺到的收入，並在 [**支付摘要**] 中提供詳細資料。 不過，除非您移除保留，否則我們不會將任何付款傳送到您的帳戶。

要將付款置於保留狀態，請轉到**開發人員設置**。 在 **"支付和納稅**"下，在 **"支付和稅務設定檔分配**"部分中，找到您希望持有的付款的計畫。 按一下"**保留我的付款**"核取方塊以保留此計畫的付款。 您隨時都可以變更您的支付保留狀態，但是請注意您決定將會影響後續的每月支付。 例如，如果您想要保留年四月的支付，請務必在三月底之前將支付保留狀態設為 [**開啟**]。

一旦你將你的付款保留狀態設置為**On**，此程式的所有支出將暫停，直到您切換滑塊回**關閉**。 這樣做時，在下一個每月支付週期將會包含您 (假設已符合適用的付款門檻)。 例如，如果您已經保留支付，但想要產生六月的支付，則請確定在五月底之前將支付保留狀態切換為[**關閉**]。

> [!NOTE]
> **您的支出保留狀態**將分別應用於每個程式（Microsoft 應用商店、廣告、Azure 應用商店等）。 如果您希望對所有計劃保留付款，則必須單獨保留每個計畫的付款。
