# <a name="azure-white-papers-about-mainframe-topics"></a>關於大型主機主題的 Azure 技術白皮書

取得有關大型主機遷移、重新裝載和將 IBM Z 型系統移至 Microsoft Azure 的資源。

### <a name="demystifying-mainframe-to-azure-migration"></a>[將大型主機揭開至 Azure 遷移](https://azure.microsoft.com/resources/demystifying-mainframe-to-azure-migration/)

Azure 提供的大型主機可以提供同等的功能和功能。 這份檔是由 AzureCAT 小組的 Larry Mead 所組成，涵蓋了主要的 IBM z/OS 大型主機元件和 Azure 對等專案。 此外，它也提供藍圖，讓您開始與訂閱過時大型主機觀念的 IT 決策者進行交談。

### <a name="move-mainframe-compute-and-storage-to-azure"></a>[將大型主機計算和儲存體移至 Azure](https://azure.microsoft.com/resources/move-mainframe-compute-and-storage-to-azure/)

若要在 Microsoft Azure 上執行大型主機工作負載，您需要知道您的大型主機功能與 Azure 的比較方式。 根據 IBM z14 大型主機，《 Larry Mead for AzureCAT 》的本指南將探索如何在 Azure 上取得可比較的結果。 Azure 上可大幅調整的計算和儲存體資源可為執行大型主機工作負載的組織提供立即的權益。

### <a name="microsoft-azure-government-cloud-for-mainframe-applications"></a>[適用于大型主機應用程式的 Microsoft Azure Government 雲端](https://azure.microsoft.com/resources/microsoft-azure-government-cloud-for-mainframe-applications/)

規劃應用程式遷移是將價值和靈活性新增至甚至是妥善建立之大型主機工作負載的理想時機。 在本快速指南中，AzureCAT 的 Larry Mead 會示範美國政府機構及其合作夥伴如何將 Azure Government 用於大型主機應用程式，而遷移可能不像您想像的那麼困難。 Azure Government 在更符合成本效益和敏捷的環境中提供大型主機的優點。 此外，Azure Government 獲得暫時性的授權單位，以 (ATO) 來 FedRAMP 高影響。

### <a name="deploy-ibm-db2-purescale-on-azure"></a>[在 Azure 上部署 IBM DB2 pureScale](https://azure.microsoft.com/resources/deploy-ibm-db2-purescale-on-azure/)

透過在 Azure 上最近重新裝載 IBM DB2 環境的企業經驗來學習。 本指南是由 AzureCAT 和 DMJ 團隊成員撰寫，作者： Larry Mead、Benjamin Guinebertière、Alessandro Vozza 和 Jonathon Frost。 它們會描述在此遷移過程中所採取的步驟。 IBM Db2 pureScale 團隊的成員會檢查其結果。 安裝腳本（可在 GitHub 上取得）是以小組用於一般中型 OLTP 工作負載的架構為基礎。

### <a name="install-tmaxsoft-openframe-on-azure"></a>[在 Azure 上安裝 TmaxSoft OpenFrame](https://azure.microsoft.com/resources/install-tmaxsoft-openframe-on-azure/)

在雲端規模現代化您的基礎結構。 TmaxSoft OpenFrame 可讓您輕鬆地提起現有的大型主機資產，並將其轉移至 Azure。 在本文中，Steve 閱讀 (AzureCAT) 和 Manoj Aerroju (TmaxSoft) 說明如何設定適用于開發、示範、測試和生產工作負載的 OpenFrame 環境。

### <a name="ibm-mainframe-to-microsoft-azure-reference-architecture"></a>[IBM 大型主機來 Microsoft Azure 參考架構](https://www.astadia.com/whitepaper/ibm-mainframe-to-microsoft-azure)

這份白皮書反映 Astadia 超過25年的大型大型大型主機平臺現代化專長。 其中說明現代化工作的優點和挑戰。 本指南概述 IBM 大型主機和 IBM 大型主機到 Azure 的參考架構。 也提供 Astadia 的成功方法。

### <a name="deploying-mainframe-applications-to-microsoft-azure"></a>[將大型主機應用程式部署至 Microsoft Azure](https://www.microfocus.com/media/white-paper/deploying_mainframe_applications_to_microsoft_azure_wp.pdf)

微的解決方案可讓您免于擁有專屬大型主機硬體和軟體的限制。 在本指南中，微焦點說明如何改為將您在 IBM 大型主機上執行的 COBOL 和 PL/I 應用程式部署至雲端。

 大型主機對企業來說越來越具挑戰性。 Infosys 和 Microsoft 這份白皮書強調了成功的大型主機遷移策略。 使用案例和範例說明這些選項。
