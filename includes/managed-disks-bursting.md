高級 SSD 支援磁碟爆發。 <= 512 GiB(P20 或以下)的任何高級 SSD 磁碟大小都支援爆破。 這些磁碟大小支援盡最大努力的爆破,並利用信用系統來管理爆破。 每當磁碟流量低於其磁碟大小的預配性能目標時,積分都會在突發存儲桶中累積,當流量超出目標時,將消耗積分。 磁碟流量根據預配目標中的 IOPS 和頻寬進行追蹤。 磁碟爆爆不會繞過虛擬機器 (VM) 對 IOPS 或輸送量的大小限制。

默認情況下,在支援它的磁碟大小的新部署上啟用磁碟爆發。 現有磁碟大小(如果支援磁碟爆發)可通過以下任一方法啟用突發:

- 分離並重新連接磁碟。
- 停止並啟動 VM。

## <a name="burst-states"></a>突發狀態

當磁碟連接到虛擬機器時,所有突發適用的磁碟大小都將從完整的突發信用儲存桶開始。 爆破的最大持續時間由突發信用存儲桶的大小決定。 您只能累積未使用的積分,直到信用存儲桶的大小。 在任何時候,磁碟突發信用存儲桶可以處於以下三種狀態之一: 

- 應計時,當磁碟流量使用小於預配性能目標時。 如果磁碟流量超出 IOPS 或頻寬目標或兩者,則可以累積積分。 當您使用全磁碟頻寬時,您仍然可以累積 IO 積分,反之亦然。  

- 當磁碟流量使用超過預配性能目標時,拒絕。 突發流量將獨立使用來自IOPS或頻寬的配額。 

- 保持恆定,當磁碟流量正好達到預配的性能目標時。 

下表匯總了提供突發支援以及突發規範的磁碟大小。

## <a name="regional-availability"></a>區域可用性

磁碟爆發在公共雲中的所有區域都可用。

## <a name="example-scenarios"></a>範例案例

為了讓您更好地瞭解其工作原理,下面是一些示例方案:

- 從磁碟爆發中獲益的一個常見方案是在 OS 磁碟上更快地啟動 VM 和應用程式啟動。 以帶有 8 GiB 作業系統映射的 Linux VM 為例。 如果我們使用 P2 磁碟作為 OS 磁碟,則預配的目標是 120 IOPS 和 25 MiB。 當 VM 啟動時,操作系統磁碟將載入啟動檔。 引入爆破後,您可以讀取最大突發速度為 3500 IOPS 和 170 MiB,將負載時間至少提高 6 倍。 VM 啟動後,OS 磁碟上的流量級別通常較低,因為應用程式的大多數數據操作將針對附加的數據磁碟。 如果流量低於預配目標,您將累積積分。

- 如果要託管遠端虛擬桌面環境,每當活動用戶啟動 AutoCAD 等應用程式時,作業系統磁碟的讀取流量就會顯著增加。 在這種情況下,突發流量將消耗累積的積分,使您能夠超出預配目標,並更快地啟動應用程式。

- P1 磁碟的預配目標為 120 IOPS 和 25 MiB。 如果磁碟上的實際流量在過去 1 秒間隔中為 100 IOPS 和 20 MiB,則未使用的 20 個 IO 和 5 MB 將記入磁碟的突發存儲桶。 稍後,當流量超過預配目標時,可以使用突發存儲桶中的配額,最高為最大突發限制。 最大突發限制定義磁碟流量的上限,即使您有突發積分需要使用。 在這種情況下,即使您在信用存儲桶中有 10,000 個 IO,P1 磁碟的發貨量也不能超過每秒 3,500 IO 的最大突發數。
