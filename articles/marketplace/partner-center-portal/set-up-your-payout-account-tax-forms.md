# <a name="set-up-your-payout-account-and-tax-forms"></a>設定支出帳戶和稅賦形式

為了從商業市場的報價銷售中收到資金，您需要設置您的支付帳戶，並在[合作夥伴中心](https://partner.microsoft.com/dashboard)填寫必要的稅務表格。

如果您只計畫列出免費優惠（並且不打算提供應用內購買或使用 Microsoft 廣告），則無需設置支出帳戶或填寫任何稅單。 如果您稍後改變主意，並決定要銷售優惠（或附加服務），則可以設置支出帳戶並填寫稅務表格。 在完成您的支出帳戶和稅務資料之前，您將無法提交任何已支付的優惠或附加服務。

> [!NOTE]
> 在某些市場，出版商只能提交免費優惠。 如果您的帳戶在上述其中一個市場中註冊，就無法選擇設定支付帳戶。

設置帳戶後，您需要執行兩件事，然後才能在商業市場銷售要約（或附加項）：：

- [填寫您的納稅申報表](#tax-forms)
- [設定您的支付帳戶](#payout-account)

> [!NOTE]
> 有關如何以及何時支付您的報價所賺的錢的詳細資訊，請參閱[付款](get-paid.md)。

## <a name="tax-forms"></a>納稅申報表

### <a name="filling-out-your-tax-forms"></a>填寫您的納稅申報表

首先，您需要創建稅務設定檔並將其分配給您參與的計畫。 您可以通過完成以下步驟為商業市場創建*您的稅務設定檔*：

- 指定您的居住國家/地區和國籍。
- 填寫適當的納稅申報表。

您可以在合作夥伴中心以電子方式填寫和提交您的納稅申報表;在大多數情況下，您不需要列印和郵寄任何表單。

> [!IMPORTANT]
> 不同國家和地區的稅金要求不同。 您必須在納稅中支付的確切金額取決於您銷售報價的國家和地區。 請參閱[Microsoft Azure 應用商店發佈商協定](https://go.microsoft.com/fwlink/p/?LinkID=699560)，瞭解 Microsoft 代表您匯出銷售和使用稅的國家/地區。 在其他國家/地區，根據您註冊的位置，您可能需要將銷售情況匯出，並將產品/服務銷售使用稅款直接匯給當地稅務機關。 此外，您收到的應用程式銷售收入可能會列為應稅收入。 我們強烈建議您聯繫您所在國家或地區的相關機構，以便最好地説明您確定適合您的商業市場活動的稅務資訊。

1. 在[合作夥伴中心](https://partner.microsoft.com/dashboard)中，選擇右上角的 **"帳戶設置"** 圖示，然後選擇 **"開發人員設置**"。
2. 在左側導航功能表中，選擇 **"付款和稅金**"，然後選擇 **"付款和稅金分配**"。

3. 選擇要為其配置稅務資訊的程式和賣家 ID 組合。

4. 如果要使用現有稅務設定檔，請從下拉清單中選擇它。 否則，選擇 **"創建新設定檔**"並按"**提交**"。 您將被帶到稅務檔案頁面。
5. 按一下 **"編輯"** 按鈕編輯您的稅務資訊。
6. 選擇相應的選項按鈕，如果出現提示，請選擇您所在的國家/地區。 此步驟確定將用於在您的帳戶上付款的 Microsoft 業務實體。

7. 根據您在步驟 6 中的選擇，系統將提示您提供您所在國家/地區所需的稅務資訊。

> [!NOTE]
> 無論您的居住國還是公民權，您必須填寫美國稅務表格，才能通過商業市場銷售任何優惠或附加服務。 滿足某些美國居住要求的發行者必須填寫 IRS W-9 表格。 美國以外的其他出版商必須填寫 IRS W-8 表格。 完成稅金設定檔之後，您就可以在線上填寫這些表單。

### <a name="withholding-rates"></a>扣繳率

您在納稅申報表中送出的資訊將決定適當的稅金扣繳率。 扣繳率只適用於您在美國完成的銷售；非美國地區的銷售不適用扣繳率。 預扣率各不相同，但對於大多數在美國境外註冊的出版商，違約率為 30%。 如果您的國家/地區和美國已簽訂所得稅協定，您可以選擇降低此扣繳率。

### <a name="tax-treaty-benefits"></a>稅務協定優惠

如果您在美國以外的地區，可能可以享有稅務協定優惠。 這些福利因國家/地區而異，並可能允許您減少商業市場扣繳的稅款。 您可以完成 W-8BEN 表單的 Part II 來要求稅務協定優惠。 建議您與您國家或地區的適當資源聯繫，以確認您是否適用這些優惠。

> [!NOTE]
> 向 Microsoft 收到付款或要求稅務協定優惠時不需要美國個人納稅識別號碼 (或 ITIN)。

## <a name="payout-account"></a>支付帳戶

支付帳戶是我們用來將銷售收益支付給您的銀行帳戶。 您可以查看您在"設定檔"頁上輸入的所有付款帳戶。

> [!NOTE]
> 在某些市場中，PayPal 可以用於支付帳戶。 要瞭解特定市場是否支援 PayPal，請參閱[付款閾值、方法和時間範圍](payment-thresholds-methods-timeframes.md)以及下面的[PayPal 資訊](#paypal-info)，瞭解更多詳細資訊。

### <a name="create-a-payment-profile"></a>創建付款設定檔

1. 在[合作夥伴中心](https://partner.microsoft.com/dashboard)中，選擇右上角的 **"設置**"齒輪圖示，然後選擇 **"開發人員"設置**。
2. 在 *"付款和納稅*"標題下，選擇 **"付款"和"稅務設定檔分配**"。

    > [!NOTE]
    > 這是機密資訊，所以會提示您再次登入。

3. 選擇要配置的付款條件。

4. 選擇現有付款設定檔，或按一下"**創建新付款設定檔**"以為所選付款條件創建新設定檔。

### <a name="create-a-bank-based-payment-profile"></a>創建基於銀行的付款設定檔

如果您選擇使用銀行帳戶接收付款，您將完成以下過程來配置您的銀行帳戶。

1. 在 *"銀行設定檔"* 頁上，提供有關您的銀行的必要資訊。
2. 提供您的銀行帳戶詳細資訊。

    > [!NOTE]
    > 用來提供您帳戶資訊的欄位僅接受英數字元。

3. 提供受益人詳細資訊。
4. 返回 *"設定檔分配"* 頁面，選擇您希望我們在發放付款時使用的貨幣。

    > [!WARNING]
    > 確保您的銀行接受您選擇的支付貨幣。

5. 您需要為參與的每個計畫選擇付款設定檔，但您可以對多個程式使用相同的設定檔。

6. 按一下"提交"以保存更改。

> [!NOTE]
> Microsoft 可能需要長達 48 小時才能驗證您個人資料中的資訊。 此過程完成後 *，驗證狀態*將顯示 **"完成"**

為確保您的支出成功，請確保您：

- 在合作夥伴中心為您的支出帳戶輸入的**帳戶持有人姓名**必須與您的銀行帳戶完全相同。 例如，如果您的銀行帳戶名稱包含中間名，請在 **帳戶持有人姓名** 中加入中間名。
- 付款會直接從 Microsoft 轉入您的銀行帳戶，貨幣單位為美元 (USD)。
- 在合作夥伴中心輸入的拉丁字元的銀行資訊將轉換為西瑞爾字元。

### <a name="editing-existing-payment-profiles"></a>編輯現有付款設定檔

如果需要進行更改或更正任何不正確的資訊，可以編輯現有的付款設定檔。

1. 在[合作夥伴中心](https://partner.microsoft.com/dashboard)中，選擇右上角的 **"設置**"齒輪圖示，然後選擇 **"開發人員"設置**。
2. 在 *"付款和納稅*"標題下方，選擇 **"付款"和"納稅檔案**"。
3. 您的付款設定檔將連同其狀態一起列出。 查找要編輯的設定檔，然後按一下最右側**的編輯**

> [!IMPORTANT]
> 變更您的付款帳戶會延遲您的付款最多一次付款週期。 會發生此延遲是因為我們必須確認帳戶變更，就像我們在您第一次設定付款帳戶時所做的一樣。 您仍然會在帳戶確認之後收到全額款項。當期付款週期未付的款項會新增至下一期。 如需詳細資訊，請參閱[獲得報酬](get-paid.md)。

### <a name="paypal-info"></a>PayPal 資訊

在特定國家與地區，您可以透過輸入您的 PayPal 資訊來建立付款帳戶。 但是，請在選擇 PayPal 做為付款帳戶選項之前：

- 檢查[付款閾值、方法和時間範圍](payment-thresholds-methods-timeframes.md)，以確認 PayPal 是否是您所在國家或地區支援的付款條件。
- 檢閱下列常見問題集。 根據您的情況，PayPal 可能不是最適合您的付款帳戶選項，也許銀行帳戶才是最佳選項。

使用 PayPal 做為付款方式的常見問題：

- **我需要設定哪些 PayPal 設定才能接收付款？** 確保您的PayPal帳戶不會阻止電子支票付款。 您可以在 PayPal 的 Payment Receiving Preferences \(付款接收喜好設定\) 頁面管理此設定。 如需詳細資訊，請參閱 [PayPal 的帳戶設定頁面](https://go.microsoft.com/fwlink/p/?linkid=513139)。
- **我的國家/地區是否受支援？** 請參閱[付款閾值、方法和時間範圍](payment-thresholds-methods-timeframes.md)，瞭解 PayPal 是受支援的付款條件的位置。
- **我的 PayPal 帳戶是否必須與合作夥伴中心帳戶在同一國家/地區註冊？** 否。 當您設定 PayPal 帳戶時，您可以接受預設設定。 接收來自其他國家/地區與其他貨幣的付款時，應該不會有問題，除非您已封鎖使用某些貨幣付款。 您可以在 PayPal 的 Payment Receiving Preferences \(付款接收喜好設定\) 頁面管理此設定。
- **我是否必須手動接受 PayPal 付款？** 否。 PayPal 帳戶的預設設定會要求使用者手動接受付款，這表示若您未在 30 天內接受付款，該款項將被退回。 您可以在 PayPal 的 \[More Settings\] \(更多設定\) 頁面關閉 \[Ask Me\] \(詢問我\) 來變更此設定。
- **PayPal支援哪些貨幣？** 有關當前清單，請參閱[PayPal 的支援頁面](https://developer.paypal.com/docs/classic/api/currency-codes/#paypal)

### <a name="specific-requirements-for-certain-countriesregions"></a>某些國家/地區的特定需求

在某些國家和地區，必須遵守支付帳戶的其他需求。 如果您是巴基斯坦、俄羅斯或烏克蘭的居民，請注意以下要求。

#### <a name="pakistan"></a>巴基斯坦

R 表格 (Form-R) 是巴基斯坦的金融規範需求。 它用於指示從海外接受款項的目的和原因。 因此，每當您有資格從 Microsoft 接受每月支付的時候，您必須在支付款項可匯入您的帳戶之前，提交 R 表格 (Form-R) 給您的銀行。 請連絡您當地的銀行分行，以獲得如何取得一份 R 表格 (Form-R) 的指示。

您有資格接受支付款項的每個月都必須提交 R 表格 (Form-R) 給您的銀行。 例如，如果您預期該年內的每個月都會收到支付款項，您就必須提交 12 次 (每個月一次) R 表格 (Form-R)。

當支付款項提交至您的銀行後，您有 30 天可以提交 R 表格 (Form-R)。 若未在 30 天內提交，該款項會退還至 Microsoft。

#### <a name="russia"></a>俄羅斯

如果您是居住在俄羅斯的出版商，您可能需要向銀行提供檔，然後才能將資金存入您的帳戶。 當您具備受款資格之後，我們將透過電子郵件為您提供下列資訊：

- 接受證書 (AC) - 包含轉帳到您帳戶的付款金額。
- [Microsoft Azure 應用商店發佈商協定](https://go.microsoft.com/fwlink/p/?LinkID=699560)- 需要反簽名的發行者協定的已簽名副本。

為確保您的支出成功，請確保您：

- 在合作夥伴中心為您的支出帳戶輸入的**帳戶持有人姓名**必須與您的銀行帳戶完全相同。 例如，如果您的銀行帳戶名稱包含中間名，請在 **帳戶持有人姓名** 中加入中間名。
- 付款會直接從 Microsoft 轉入您的銀行帳戶，貨幣單位為盧布 (RUB)。
- 在合作夥伴中心輸入的拉丁字元的銀行資訊將轉換為西瑞爾字元。
- 支付的款項會進入銀行帳戶，而不是銀行卡。

#### <a name="ukraine"></a>烏克蘭

如果您是居住在烏克蘭的出版商，您可能需要向銀行提供檔，然後才能將資金存入您的帳戶。 當您具備受款資格之後，我們將透過電子郵件為您提供下列資訊：

- 接受證書 (AC) - 包含轉帳到您帳戶的付款金額。
- [Microsoft Azure 應用商店發佈商協定](https://go.microsoft.com/fwlink/p/?LinkID=699560)- 需要反簽名的發行者協定的已簽名副本。
- 增修條款合約 (AA) - 這份文件可協助您的銀行用來識別您的支付款項。

嘗試支付您的第一筆款項時，Microsoft 會提供所有三份文件。 針對任何後續的支付，您只會收到 AC 文件。 保留 ADA 和 AA 文檔，以防您需要它們從您的銀行接收未來付款。

### <a name="create-a-paypal-payment-profile"></a>創建 PayPal 付款設定檔

如果您選擇使用銀行帳戶接收付款，您將完成以下過程來配置您的銀行帳戶。

1. 在*PayPal*頁面上，提供有關您的PayPal帳戶的必要資訊。
2. 提供您的PayPal帳戶詳情。

    > [!NOTE]
    > 用來提供您帳戶資訊的欄位僅接受英數字元。

3. 提供受益人詳細資訊。
4. 返回 *"設定檔分配"* 頁面，選擇您希望我們在發放付款時使用的貨幣。
5. 您需要為參與的每個計畫選擇付款設定檔，但您可以對多個程式使用相同的設定檔。
6. 按一下"提交"以保存更改。
