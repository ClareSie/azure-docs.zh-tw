# <a name="manage-users-and-roles-in-your-iot-central-application"></a>管理 IoT 中央應用程式中的使用者和角色

本文介紹作為管理員如何在 Azure IoT 集中應用程式中添加、編輯和刪除使用者。 本文還介紹了如何在 Azure IoT 中央應用程式中管理角色。

若要存取並使用 [管理]**** 區段，您必須屬於 Azure IoT Central 應用程式的**管理員**角色。 如果創建 Azure IoT 中心應用程式，則會自動添加到該應用程式的**管理員**角色。

## <a name="add-users"></a>新增使用者

每個使用者都必須具有使用者帳戶，才能登入並存取 Azure IoT Central 應用程式。 Azure IoT 中心支援 Microsoft 帳戶和 Azure 活動目錄帳戶。 Azure IoT Central 目前不支援 Azure Active Directory 群組。

如需詳細資訊，請參閱 [Microsoft 帳戶說明](https://support.microsoft.com/products/microsoft-account?category=manage-account)和[快速入門：將使用者新增至 Azure Active Directory](https://docs.microsoft.com/azure/active-directory/add-users-azure-active-directory)。

1. 若要將使用者新增至 IoT Central 應用程式，請前往 [管理]**** 區段中的 [使用者]**** 頁面。
    
    > [!div class="mx-imgBorder"]
    >![管理使用者](media/howto-manage-users-roles/manage-users-pnp.png)

1. 若要新增使用者，在 [使用者]**** 頁面上，選擇 [+ 新增使用者]****。

1. 從 [角色]**** 下拉式清單中選擇使用者的角色。 請在本文的[管理角色](#manage-roles)一節中深入了解角色。

    > [!div class="mx-imgBorder"]
    >![添加使用者並選擇角色](media/howto-manage-users-roles/add-user-pnp.png)

    > [!NOTE]
    > 處於授予其添加其他使用者許可權的自訂角色中的使用者只能將使用者添加到具有相同或更少許可權的角色中。該角色的許可權與其自己的角色相同或更少。

如果 IoT 中心使用者 ID 從 Azure 活動目錄中刪除，然後重新添加，則使用者將無法在 IoT 中央應用程式中登錄。 要重新啟用存取權限，IoT 管理中心管理員應刪除並讀取應用程式中的使用者。

### <a name="edit-the-roles-that-are-assigned-to-users"></a>編輯指派給使用者的角色

角色在分配後無法更改。 若要變更指派給使用者的角色，請刪除該使用者，然後以不同的角色再次新增該使用者。

> [!NOTE]
> 分配的角色特定于 IoT 中心應用程式，無法從 Azure 門戶進行管理。

## <a name="delete-users"></a>刪除使用者

若要刪除使用者，請選取 [使用者]**** 頁面上的一或多個核取方塊。 然後選取 [刪除]****。

## <a name="manage-roles"></a>管理角色

角色使您能夠控制允許組織內的人員在 IoT Central 中執行各種任務。 您可以為應用程式的使用者分配三個內置角色。 如果需要更細細微性的控制項，還可以[創建自訂角色](#create-a-custom-role)。

> [!div class="mx-imgBorder"]
> ![管理角色選擇](media/howto-manage-users-roles/manage-roles-pnp.png)

### <a name="administrator"></a>系統管理員

**管理員**角色中的使用者可以管理和控制應用程式的每個部分，包括計費。

系統會將建立應用程式的使用者自動指派給**管理員**角色。 必須至少有一個使用者擔任 [系統管理員]**** 角色。

### <a name="builder"></a>建立器

**產生器**角色中的使用者可以管理應用的每個部分，但不能對"管理"或"連續資料匯出"選項卡進行更改。

### <a name="operator"></a>運算子

**操作員**角色中的使用者可以監視設備運行狀況和狀態。 不允許他們更改設備範本或管理應用程式。 操作員可以添加和刪除設備、管理設備集以及運行分析和作業。 

## <a name="create-a-custom-role"></a>建立自訂角色

如果解決方案需要更細細微性的存取控制，則可以使用自訂許可權集創建自訂角色。 要創建自訂角色，請導航到應用程式 **"管理**"部分中的 **"角色**"頁。 然後選擇 **= 新角色**，並為角色添加名稱和說明。 選擇角色所需的許可權，然後選擇 **"保存**"。

您可以將使用者添加到自訂角色，就像將使用者添加到內置角色一樣。

> [!div class="mx-imgBorder"]
> ![構建自訂角色](media/howto-manage-users-roles/create-custom-role-pnp.png)

### <a name="custom-role-options"></a>自訂角色選項

定義自訂角色時，可以選擇使用者作為角色成員時授予的許可權集。 某些許可權依賴于其他許可權。 例如，如果將 **"更新應用程式儀表板"** 許可權添加到角色，則"**查看應用程式儀表板"** 許可權將自動添加。 下表匯總了創建自訂角色時可以使用的可用許可權及其依賴項。
