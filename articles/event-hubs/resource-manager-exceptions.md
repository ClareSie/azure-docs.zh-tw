# <a name="azure-event-hubs---resource-manager-exceptions"></a>Azure 事件中心 - 資源管理器例外
本文列出了使用 Azure 資源管理器與 Azure 事件中心交互時生成的異常 - 通過範本或直接調用。

> [!IMPORTANT]
> 本文檔經常更新。 請回來查看更新。

以下各節提供了通過 Azure 資源管理器顯示的各種異常/錯誤。

## <a name="error-code-conflict"></a>錯誤代碼：衝突

| 錯誤碼 | 錯誤子代碼 | 錯誤訊息 | 描述 | 建議 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 衝突 | 40300 | 已達到或超過事件 Hub 類型的最大資源數。 實際： *， 允許的最大：# | 命名空間已達到其可以包含的事件中心數的[配額](event-hubs-quotas.md)。 | 從命名空間中刪除任何未使用的或無關的事件中心，或考慮升級到[專用群集](event-hubs-dedicated-overview.md)。 |
| 衝突 | 無 | 無法刪除災害復原 （DR） 配置，因為複製正在進行中。 在嘗試刪除 DR 配置之前，請容錯移轉或中斷配對。 | [GeoDR 複製](event-hubs-geo-dr.md)正在進行中，因此此時無法刪除配置。 | 要取消阻止刪除配置，請等待複製完成、觸發容錯移轉或中斷 GeoDR 配對。 |
| 衝突 | 無 | 命名空間更新失敗，後端發生衝突。 | 當前正在此命名空間上執行另一個操作。 | 等待當前操作完成，然後重試。 |

## <a name="error-code-429"></a>錯誤代碼： 429

| 錯誤碼 | 錯誤子代碼 | 錯誤訊息 | 描述 | 建議 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| 429 | 無 | 過渡中的命名空間預配 | 當前正在此命名空間上執行另一個操作。 | 等待當前操作完成，然後重試。 |
| 429 | 無 | 災害復原操作正在進行中。 | 當前正在此命名空間或配對上執行[GeoDR](event-hubs-geo-dr.md)操作。 | 等待當前 GeoDR 操作完成，然後重試。 |

## <a name="error-code-badrequest"></a>錯誤代碼：錯誤請求

| 錯誤碼 | 錯誤子代碼 | 錯誤訊息 | 描述 | 建議 |
| ---------- | ------------- | ------------- | ----------- | -------------- |
| BadRequest | 40000 | 無法為事件中心更改分區計數。 | Azure 事件中心的基本或標準層不支援更改分區。 | 使用基本或標準層命名空間中想要的分區數創建新的事件中心。 [專用群集](event-hubs-dedicated-overview.md)支援分區橫向擴展。 |
| BadRequest | 40000 | 消息保留日的值"*"對基本層無效。 該值不能超過"1"天。 | 基本層事件中心命名空間僅支援最多保留 1 天的消息。 | 如果需要超過一天的消息保留，[請創建一個標準事件中心命名空間](event-hubs-create.md)。 | 
| BadRequest | 無 | 指定的名稱不可用。 | 命名空間名稱必須是唯一的，並且已獲取指定的名稱。 | 如果您是具有指定名稱的現有命名空間的擁有者，則可以將其刪除，這將導致資料丟失。 然後，請重試同名。 如果命名空間不安全刪除（或者您不是擁有者），請選擇其他命名空間名稱。 |
| BadRequest | 無 | 指定的訂閱已達到其命名空間配額。 | 您的訂閱已達到其可以保留的命名空間數的[配額](event-hubs-quotas.md)。 | 請考慮刪除此訂閱中未使用的命名空間、創建另一個訂閱或升級到[專用群集](event-hubs-dedicated-overview.md)。 |
| BadRequest | 無 | 無法更新輔助命名空間 | 無法更新命名空間，因為它是[GeoDR 配對](event-hubs-geo-dr.md)中的輔助命名空間。 | 如果合適，請改為更改此配對中的主命名空間。 否則，中斷 GeoDR 配對以進行更改。 |
| BadRequest | 無 | 無法在基本 SKU 中設置自動膨脹 | 無法在基本層事件中心命名空間上啟用自動膨脹。 | 要在命名空間上[啟用自動膨脹](event-hubs-auto-inflate.md)，請確保它是標準層。 |
| BadRequest | 無 | 沒有足夠的容量來創建命名空間。 請與您的活動中心管理員聯繫。 | 所選區域處於容量狀態，無法創建更多命名空間。 | 選擇其他區域來容納您的命名空間。 |
| BadRequest | 無 | 無法在實體類型"消費者群體"上執行操作，因為命名空間"命名空間名稱"使用"基本"層。  | 基本層事件中心命名空間具有一個消費者組的 [配額]（事件-中心-配額.md_事件-中心-基本和標準---配額和限制）的一個消費者組（預設值）。 不支援創建更多消費者組。 | 繼續使用預設消費者組（$Default），或者如果需要更多，請考慮改用標準層事件中心命名空間。 | 
| BadRequest | 無 | 命名空間"命名空間名稱"不存在。 | 找不到提供的命名空間。 | 仔細檢查命名空間名稱是否正確，並且可在訂閱中找到。 如果不是，[請創建事件中心命名空間](event-hubs-create.md)。 | 
| BadRequest | 無 | 資源的位置屬性與其包含的命名空間不匹配。 | 在特定區域中創建事件中心失敗，因為它與命名空間的區域不匹配。 | 嘗試在與命名空間相同的區域中創建事件中心。 |
