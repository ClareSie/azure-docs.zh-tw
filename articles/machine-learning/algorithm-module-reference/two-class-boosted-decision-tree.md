# <a name="two-class-boosted-decision-tree-module"></a>雙類提升決策樹模組

本文介紹 Azure 機器學習設計器（預覽）中的模組。

使用此模組可以創建基於增強決策樹演算法的機器學習模型。 

提升決策樹是一種集合學習方法，其中第二樹糾正第一個樹的錯誤，第三個樹糾正第一和第二樹的錯誤，依此類推。  預測基於整個樹群一起做出預測。
  
通常，如果配置正確，提升的決策樹是在各種機器學習任務上獲得最佳性能的最簡單方法。 但是，它們也是記憶體密集型的學習者之一，並且當前實現包含記憶體中的所有內容。 因此，提升的決策樹模型可能無法處理某些線性學員可以處理的大型資料集。

## <a name="how-to-configure"></a>如何設定

此模組創建未經訓練的分類模型。 由於分類是一種受監督的學習方法，因此要訓練模型，您需要一個*標記的資料集*，其中包含一個標籤列，其中包含所有行的值。

您可以使用[訓練模型](././train-model.md)來訓練這種類型的模型。 

1.  在 Azure 機器學習中，將 **"提升決策樹"** 模組添加到管道中。
  
2.  通過設置 **"創建培訓師模式"** 選項，指定如何訓練模型。
  
    + **單一參數**：如果您知道如何配置模型，則可以提供一組特定的值作為參數。
  
    + **參數範圍**：如果您不確定最佳參數，則可以使用[Tune 模型 Hyper 參數](tune-model-hyperparameters.md)模組找到最佳參數。 提供一些值範圍，培訓師會對多個設置組合進行遍舍，以確定生成最佳結果的值的組合。
  
3.  對於**每個樹的最大葉數**，指示可在任何樹中創建的最大終端節點數（葉）。
  
     增加此值，您可能會增加樹狀結構的大小，並取得較佳的精確度，風險是過度配適和較長的定型時間。
  
4.  對於**每個葉節點的最小樣本數**，指示在樹中創建任何終端節點（葉）所需的案例數。  
  
     藉由增加此值，您會增加建立新規則的臨界值。 例如，若預設值是 1，即使單一案例可能會造成新規則的建立。 如果您將此值增加為 5，則定型資料必須至少包含五個符合相同條件的案例。
  
5.  對於**學習速率**，鍵入 0 和 1 之間的數位，用於定義學習時的步驟大小。  
  
     學習速率決定學員在最佳解決方案上收斂的速度或速度。 如果步長過大，則可能會超過最佳解決方案。 如果步長太小，則訓練需要更長的時間才能融合到最佳解決方案上。
  
6.  對於**構造的樹數**，指示要在集合中創建的決策樹的總數。 藉由建立多個決策樹，您或許能夠有較佳的涵蓋範圍，但是定型時間會拉長。
  
     此值還控制視覺化訓練的模型時顯示的樹數。 如果要查看或列印單個樹，將值設置為 1。 但是，執行此操作時，將只生成一個樹（具有初始參數集的樹），並且不執行進一步的反覆運算。
  
7.  對於**亂數種子**，可以選擇鍵入非負整數以用作隨機種子值。 指定種子可確保具有相同資料和參數的運行之間的可重現性。  
  
     預設情況下，隨機種子設置為 0，這意味著初始種子值是從系統時鐘獲取的。  使用隨機種子連續運行可以有不同的結果。
  

9. 將模型定型。
  
    + 如果將 **"創建訓練器模式"** 設置為 **"單個參數**"，請連接標記的資料集和[訓練模型](./train-model.md)模組。  
   
## <a name="results"></a>結果

培訓完成後：

+ 要保存已訓練的模型的快照，請在 **"訓練"模型**模組的右側面板中選擇 **"輸出**"選項卡。 選擇 **"註冊"資料集**圖示以將模型保存為可重用模組。

+ 要使用模型進行評分，請使用 **"分數模型"** 模組添加到管道中。
