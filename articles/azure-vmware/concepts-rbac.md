# <a name="role-based-access-control-rbac-for-azure-vmware-solution-avs"></a>Azure VMware 解決方案（AVS）的角色型存取控制（RBAC）

在 vCenter 和 ESXi 內部部署中，系統管理員可以存取 vCenter administrator@vsphere.local 帳戶，而且可能會指派額外的 Active Directory （AD）使用者/群組。 不過，在 Azure VMware 解決方案（AVS）部署中，系統管理員沒有系統管理員使用者帳戶的存取權，但可以將 AD 使用者和群組指派給 vCenter 上的 CloudAdmin 角色。  此外，AVS 私用雲端使用者無權存取或設定由 Microsoft 支援和管理的特定管理元件，例如叢集、主機、資料存放區和分散式虛擬交換器。


在 AVS 中，vCenter 有一個內建的本機使用者，稱為 cloudadmin，它會指派給內建的 CloudAdmin 角色。 本機 cloudadmin 使用者可用來設定 AD 中的其他使用者。 一般來說，CloudAdmin 角色具有在私人雲端（虛擬機器、資源集區、資料存放區和網路）中建立和管理工作負載的許可權。 AVS 中的 CloudAdmin 角色具有一組特定的 vCenter 許可權，與其他 VMware 雲端解決方案不同。   

> [!NOTE]
> AVS 目前並未在 vCenter 或 AVS 入口網站上提供自訂角色。 

## <a name="avs-cloudadmin-role-on-vcenter"></a>VCenter 上的 AVS CloudAdmin 角色

您可以在您的 AVS 私用雲端 vCenter 上，查看授與 AVS CloudAdmin 角色的許可權。

1. 登入 SDDC vSphere 用戶端，並移至 [**功能表**] [  >  **管理**]。
1. 在 [**存取控制**] 底下，選取 [**角色**]。
1. 從角色清單中選取 [ **CloudAdmin** ]，然後選取 [**許可權**]。
