# <a name="azure-security-benchmark-introduction"></a>Azure 安全基準簡介

您可能擁有數年甚至幾十年的本地計算經驗。 您知道如何保護這些部署;但是，您可以保護這些部署。但雲是不同的。 您如何知道雲部署是否安全？ 本地系統的安全實踐和雲部署的安全實踐有何區別？

白皮書、最佳實踐、參考體系結構、Web 指南、開源工具、商業解決方案、智慧源等大量可用於説明保護雲的安全。 您應該使用哪個選項？ 您可以做些什麼來獲得雲中可接受的安全級別？ 

保護雲部署的最佳方法之一是關注雲安全基準建議。 保護任何服務的基準建議首先從對網路安全風險以及如何管理網路安全風險的基本理解開始。 然後，您可以通過從雲服務提供者處採用基準安全建議來説明選擇環境中的特定安全配置設置來使用此理解。 

Azure 安全基準測試包括一系列高影響安全建議，可用於説明保護在 Azure 中使用的大多數服務。 您可以將這些建議視為"常規"或"組織"，因為它們適用于大多數 Azure 服務。 然後為每個 Azure 服務自訂 Azure 安全基準建議，並且此自訂指南包含在服務建議文章中。 

Azure 安全基準文檔指定安全控制和服務建議。

- **安全控制**：Azure 安全基準建議按安全控制項分類。 安全控制表示與供應商無關的高級安全要求，如網路安全和資料保護。 每個安全控制都有一組安全建議和說明，可説明您實現這些建議。 
- **服務建議**：如果可用，Azure 服務的基準建議將包括專為該服務量身定制的 Azure 安全基準建議。 

Azure 安全基準文檔中經常使用術語"控制"、"基準"和"基準"，瞭解 Azure 如何使用這些術語非常重要。 

| 詞彙 | 描述 | 範例 |
|--|--|--|
| 控制 | **控制項**是需要處理且不特定于技術或實現的功能或活動的高級描述。 | 資料保護是安全控制之一。 此控制項包含需要處理的特定操作，以説明確保資料受到保護。 |
| 基準測試 | **基準測試**包含特定技術（如 Azure）的安全建議。 建議按它們所屬的控制項進行分類。 | Azure 安全基準包括特定于 Azure 平臺的安全建議  |
| 基準 | **基線**是組織的安全要求。 安全要求基於基準建議。 每個組織決定在其基線中包括哪些基準建議。 | Contoso 公司通過在 Azure 安全基準中選擇要求特定建議來創建其安全基準。 |
