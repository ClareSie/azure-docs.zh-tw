# <a name="best-practices-for-creating-assessments"></a>創建評估的最佳做法

[Azure Migrate](migrate-overview.md) 會提供工具中樞，協助您探索和評估應用程式、基礎結構和工作負載，並且將這些項目遷移至 Microsoft Azure。 此中樞包含 Azure Migrate 工具和第三方獨立軟體廠商 (ISV) 供應項目。

本文總結了使用 Azure 遷移伺服器評估工具創建評估時的最佳做法。

## <a name="about-assessments"></a>關於評估

使用 Azure 遷移伺服器評估創建的評估是資料的時間點快照。 Azure 遷移中有兩種類型的評估。

**評量類型** | **詳細資料** | **資料**
--- | --- | ---
**以效能為基礎** | 根據收集的績效資料提出建議的評估 | VM 大小建議基於 CPU 和記憶體利用率資料。<br/><br/> 磁片類型建議（標準 HDD/SSD 或高級管理磁片）基於 IOPS 和本地磁片的輸送量。
**本地** | 不使用效能資料提出建議的評估。 | VM 大小建議基於本地 VM 大小<br/><br> 建議的磁片類型基於您在評估的存儲類型設置中選擇的內容。

### <a name="example"></a>範例
例如，如果您有一個具有四個內核的本地 VM 利用率為 20%，記憶體為 8 GB，利用率為 10%，則評估如下：

- **基於績效的評估**：
    - 根據內核 （4 x 0.20 = 0.8） 和記憶體 （8 GB x 0.10 = 0.8） 利用率識別有效的內核和記憶體。
    - 應用評估屬性中指定的舒適係數（例如 1.3x），以獲得用於調整大小的值。 
    - 建議 Azure 中支援 ±1.04 內核 （0.8 x 1.3） 和 ±1.04 GB （0.8 x 1.3） 記憶體的最近 VM 大小。

- **按本地（本地）評估**：
    -  建議使用具有四個內核的 VM;8 GB 記憶體。

## <a name="best-practices-for-creating-assessments"></a>創建評估的最佳做法

Azure 遷移設備持續分析本地環境，並將中繼資料和效能資料發送到 Azure。 遵循以下最佳實踐來評估使用設備發現的伺服器：

- **創建"正在進行"評估**：一旦電腦顯示在 Azure 遷移門戶中，即可立即創建"正在執行"的評估。
- **創建基於績效的評估**：設置發現後，我們建議您至少等待一天才能運行基於績效的評估：
    - 收集效能資料需要時間。 等待至少一天可確保在運行評估之前有足夠的效能資料點。
    - 運行基於績效的評估時，請確保在評估持續時間內分析環境。 例如，如果創建績效持續時間設置為一周的評估，則需要在開始發現後至少等待一周，以便收集所有資料點。 如果你不這樣做，評估將不會得到五星級。
- **重新計算評估**：由於評估是時間點快照，因此不會使用最新資料自動更新。 要使用最新資料更新評估，您需要重新計算它。

遵循這些最佳做法，對通過 將導入 Azure 遷移的伺服器進行評估。CSV 檔：

- **創建"正在進行"評估**：一旦電腦顯示在 Azure 遷移門戶中，即可立即創建"正在執行"的評估。
- **創建基於性能的評估**：這有助於獲得更好的成本估算，尤其是在本地過度預配伺服器容量時。 但是，基於性能的評估的準確性取決於您為伺服器指定的效能資料。 
- **重新計算評估**：由於評估是時間點快照，因此不會使用最新資料自動更新。 要使用最新的導入資料更新評估，您需要重新計算它。

## <a name="best-practices-for-confidence-ratings"></a>信心評級的最佳做法

當您運行基於績效的評估時，評估將授予從 1 星（最低）到 5 星（最高）的信心評級。 要有效地使用置信度：
- Azure 遷移伺服器評估需要 VM CPU/記憶體的利用率資料。
- 對於連接到本地 VM 的每個磁片，它需要讀/寫 IOPS/輸送量資料。
- 對於連接到 VM 的每個網路介面卡，它需要網路進出資料。

根據所選持續時間可用的資料點的百分比，評估的置信度如下表中的匯總結果。

   **資料點可用性** | **信賴評等**
   --- | ---
   0%-20% | 1 顆星
   21%-40% | 2 顆星
   41%-60% | 3 顆星
   61%-80% | 4 顆星
   81%-100% | 5 顆星


## <a name="common-assessment-issues"></a>共同評估問題

下面將如何解決影響評估的一些常見環境問題。

###  <a name="out-of-sync-assessments"></a>不同步評估

如果在創建評估後從組添加或刪除電腦，則創建的評估將標記為**不同步**。再次運行評估 （**重新計算**） 以反映組更改。

### <a name="outdated-assessments"></a>過時的評估

如果對已評估的組中的 VM 的本地更改，則評估標記為**過時**。 要反映更改，再次運行評估。

### <a name="low-confidence-rating"></a>低置信度

由於多種原因，評估可能沒有所有資料點：

- 您未針對正在建立評量的持續時間剖析環境。 例如，如果要創建基於*績效的評估*，績效持續時間設置為一周，則需要在開始發現後至少等待一周才能收集所有資料點。 您可以隨時按一下 **"重新計算"** 以查看最新的適用置信度評級。 僅在創建*基於績效*的評估時，才適用置信度評級。

- 少數虛擬機器在評量計算期間關閉。 如果有某些 VM 在某段期間內關閉電源，伺服器評量將無法收集這段期間的效能資料。

- 有少數 VM 是在伺服器評量中的探索啟動後建立的。 例如，如果您要建立過去一個月的效能記錄評量，但是少數虛擬機器在一週前才建立在環境中。 在此情況下，將無法取得新的 VM 在這整段期間內的效能資料，且信賴評等將會偏低。
