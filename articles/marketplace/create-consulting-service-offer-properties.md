# <a name="how-to-configure-your-consulting-service-offer-properties"></a>如何設定您的諮詢服務供應專案屬性

本文說明如何在 Microsoft 商業市場中設定諮詢服務供應專案的屬性。

在合作夥伴中心中您供應專案的 [ **屬性** ] 索引標籤上，您將會在諮詢服務、服務類型和適用產業的核心定義主要產品。 您也會指出您所贏得的相關 Microsoft 專長認證。 此索引標籤中的資訊可確保您的諮詢服務在商業 marketplace 中正確顯示，並提供給正確的客戶集合。

## <a name="select-the-primary-product"></a>選取主要產品

在 [ **主要產品** ] 下拉式清單中，選取將成為諮詢服務供應專案焦點的 Microsoft 產品。 您選取的產品將會決定您的供應專案是否會發佈至 Microsoft AppSource 或 Azure Marketplace。

* 如果您選取 Azure，供應專案將會列在 Azure Marketplace 上。 最多可以選取三個 **解決方案區域** 。 這些可讓客戶更輕鬆地找到您的供應專案。
* 如果主要產品 *不* 是 Azure，供應專案會在 AppSource 上發佈。 至少選取一個和最多三個 **適用的產品** 。 這些可讓客戶更輕鬆地找到您的供應專案。

## <a name="select-the-consulting-service-type"></a>選取諮詢服務類型

在 [ **諮詢服務類型** ] 下拉式清單中，選取您要提供給客戶的諮詢服務類型。 可用的類型為：

* **評** 量：評估客戶的環境，以判斷解決方案的適用性，並評估其實行的成本和時程表。
* **簡報** ：使用架構、示範和客戶範例的解決方案或服務簡介。
* **實作** ：完整安裝以產生完全可行的解決方案。
* **概念證明** ：判斷解決方案是否符合客戶需求的有限範圍執行。
* **研討會** ：在客戶的內部部署進行互動的互動。 其可以包含以客戶資料或環境為基礎的訓練、簡報、評量或示範。

## <a name="select-relevant-industries"></a>選取相關產業

本節中的資訊可協助我們分類您的供應專案，讓客戶可以在 Microsoft 線上商店輕鬆找到。

* 如果您選取 Azure 作為主要產品，請選取最多六個 **產業** 。 請參閱供應專案清單 [最佳做法](./gtm-offer-listing-best-practices.md)中的完整產業清單。
* 如果您選取的是 Azure 以外的主要產品，則可以選擇為每個產業選取最多兩個 **產業** 和兩 **個子產業** 。 請參閱 [供應專案清單最佳做法](./gtm-offer-listing-best-practices.md)中的完整產業和子產業清單。

## <a name="select-your-competencies"></a>選取您的專長認證

最多可選取三項與您供應專案的服務區域相關的專長認證。 在供應專案清單中，客戶會看到您的選取專案。

> [!IMPORTANT]
> 您必須擁有至少一個專長認證，才能發佈您的供應專案，視其主要產品而定。 如需詳細資訊，請參閱 [認證原則](https://docs.microsoft.com/legal/marketplace/certification-policies#800-consulting-services) 。
