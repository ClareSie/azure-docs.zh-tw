# <a name="azure-white-papers-about-mainframe-topics"></a>有關大型機主題的 Azure 白皮書

獲取有關大型機遷移、重新託管和將基於 IBM Z 的系統移動到 Microsoft Azure 的資源。

### <a name="demystifying-mainframe-to-azure-migration"></a>[將大型機與 Azure 遷移解密](https://azure.microsoft.com/resources/demystifying-mainframe-to-azure-migration/)

Azure 提供了一種大型機替代方案，能夠提供等效的功能和功能。 本文由 AzureCAT 團隊的 Larry Mead 撰寫，介紹了主要的 IBM z/OS 大型機元件和 Azure 等效項。 它還提供了一個路線圖，用於開始與訂閱過時的大型機理念的 IT 決策者進行對話。

### <a name="move-mainframe-compute-and-storage-to-azure"></a>[將大型機計算和存儲移動到 Azure](https://azure.microsoft.com/resources/move-mainframe-compute-and-storage-to-azure/)

要在 Microsoft Azure 上運行大型機工作負荷，您需要瞭解大型機的功能與 Azure 的比較情況。 基於 IBM z14 大型機，AzureCAT 的 Larry Mead 本指南探討了如何在 Azure 上獲取可比較的結果。 Azure 上可大規模擴展的計算和存儲資源為運行大型機工作負荷的組織提供了直接的好處。

### <a name="microsoft-azure-government-cloud-for-mainframe-applications"></a>[適用于大型機應用程式的 Microsoft Azure 政府雲](https://azure.microsoft.com/resources/microsoft-azure-government-cloud-for-mainframe-applications/)

規劃應用程式遷移是為甚至成熟的大型機工作負載增加價值和敏捷性的理想時機。 在此快速指南中，AzureCAT 的 Larry Mead 展示了美國政府機構及其合作夥伴如何使用 Azure 政府進行大型機應用程式，遷移可能不像您想像的那麼困難。 Azure 政府功能在更具成本效益和敏捷的環境中提供了大型機的優勢。 此外，Azure 政府還獲得了運營臨時管理局 （P-ATO），以增強 FedRAMP 的高影響力。

### <a name="deploy-ibm-db2-purescale-on-azure"></a>[在 Azure 上部署 IBM DB2 pureScale](https://azure.microsoft.com/resources/deploy-ibm-db2-purescale-on-azure/)

學習我們最近在 Azure 上重新託管其 IBM DB2 環境的企業的經驗。 本指南由 AzureCAT 和 DMJ 團隊成員編寫：拉裡·米德、本傑明·吉內貝蒂埃、亞曆山德羅·沃紮和喬納森·弗羅斯特。 它們描述了他們在遷移過程中採取的步驟。 IBM Db2 純Scale 團隊成員對其發現進行了審查。 GitHub 上提供的安裝腳本基於團隊用於典型中型 OLTP 工作負載的體系結構。

### <a name="install-tmaxsoft-openframe-on-azure"></a>[在 Azure 上安裝 TmaxSoft 開放幀](https://azure.microsoft.com/resources/install-tmaxsoft-openframe-on-azure/)

以雲規模實現基礎架構現代化。 TmaxSoft OpenFrame 可輕鬆提升現有大型機資產並將其轉移到 Azure。 在本文中，Steve Read （AzureCAT） 和 Manoj Aerroju （TmaxSoft） 解釋了如何設置適合開發、演示、測試和生產工作負載的 OpenFrame 環境。

### <a name="ibm-mainframe-to-microsoft-azure-reference-architecture"></a>[IBM 大型機到微軟 Azure 參考體系結構](https://www.astadia.com/whitepaper/ibm-mainframe-to-microsoft-azure)

本白皮書反映了 Astadia 超過 25 年的大型機平臺現代化專業知識。 它們解釋了現代化努力的好處和挑戰。 本指南概述了 IBM 大型機和 IBM 大型機到 Azure 參考體系結構。 它還介紹了 Astadia 成功方法。

### <a name="deploying-mainframe-applications-to-microsoft-azure"></a>[將大型機應用程式部署到 Microsoft Azure](https://www.microfocus.com/media/white-paper/deploying_mainframe_applications_to_microsoft_azure_wp.pdf)

Micro Focus 的解決方案讓您擺脫專有大型機硬體和軟體的限制。 在本指南中，Micro Focus 介紹了如何將 IBM 大型機中的 COBOL 和 PL/I 應用程式部署到雲中。

### <a name="breathe-new-life-into-mainframes"></a>[為大型機注入新的活力](https://www.infosys.com/services/modernization/breathe-new-life-mainframes.html)

 大型機對於業務維護來說越來越具有挑戰性。 Infosys 和 Microsoft 的本白皮書強調了成功大型機遷移的制勝策略。 用例和示例說明了這些選項。
