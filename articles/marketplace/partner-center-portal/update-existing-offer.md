# <a name="update-an-existing-offer-in-the-commercial-marketplace"></a>更新商業市集中的現有供應項目

您可以在合作夥伴中心的[商業 Marketplace 入口網站](https://partner.microsoft.com/dashboard/commercial-marketplace/offers)的 [**總覽**] 索引標籤上，查看您現有的供應專案。

若要更新目前在 marketplace 中的現有供應專案：

1. 選取您想要更新的供應專案名稱。 供應專案的狀態可能會列為 [ **Live**]、[**進行中發佈**]、[**草稿**]、[**需要注意**] 或 **[未在 marketplace 中銷售**] （如果您先前已選擇停止銷售供應專案）。 選取之後，即會開啟該供應專案的**供應專案總覽**頁面。
2. 從 [總覽] 卡片中選取 [**更新**]，或從左側導覽中選取您想要更新之區域的功能表項目。 您可能想要更新**供應專案設定**、**屬性**、**供應專案清單**、**預覽**、**技術**設定、**計畫總覽**或**試用**產品。 
3. 進行變更，然後選取 [**儲存**]。 重複此程式，直到您所有的變更都完成為止。

## <a name="review-and-publish-an-updated-offer"></a>審查及發佈更新的供應專案

當您準備好要發佈更新的供應專案時，請選取 [從任何頁面**發佈**]。 [**審查和發行**] 頁面隨即開啟。 在此頁面上，您可以：

- 請參閱您更新之供應專案區段的完成狀態： 
    - **未**發行的變更：區段已更新且已完成。 已提供所有必要的資料，且更新中未引進任何錯誤。
    - **不完整**：對區段所做的更新導入了需要修正的錯誤，或需要提供更多的資訊。
- 提供其他資訊給認證測試小組，以確保測試順利進行。
- 選取 [**提交**] 以提交更新的供應專案以供發佈。  當更新供應專案的預覽版本可供您審查和登出時，我們會傳送電子郵件給您。

> [!IMPORTANT]
> 您必須在供應專案預覽正式運作後加以審查，**並選取 [上線]** 將更新的供應專案發佈給您的目標物件（公用或私用）。

## <a name="add-a-plan-to-an-existing-offer"></a>將方案新增至現有的供應專案

若要在您已發行的現有供應專案內加入新的方案：

1. 在您現有供應專案的 [**供應專案總覽**] 頁面開啟時，移至 [計畫] [**總覽**] 頁面，然後選取 [**建立新方案**]。
1. 使用**現有的方案定價模型**，根據[方針](./create-new-saas-offer.md#plan-overview)建立新的方案。
1. 變更方案名稱之後，請選取 [**儲存草稿**]。 
1. 當您準備好要發佈更新時，請選取 [**發佈**]。 [**[審查與發佈](#review-and-publish-an-updated-offer)**] 頁面隨即開啟，並提供更新的完成狀態。

## <a name="update-a-plan-within-an-existing-offer"></a>更新現有供應專案中的方案

若要在您已發行的現有供應專案中變更方案：

1. 在您現有供應專案的 [**供應專案總覽**] 頁面開啟的情況下，選擇您要變更的方案。 如果計畫無法從**計畫總覽**清單存取，請選取 [**查看所有方案**]。
1. 您可以選擇 [方案**名稱**]、[**計價模式**] 或 [**可用性**]。 *目前，方案僅適用于英文（美國）*。
1. 對方案名稱、描述或物件可用性進行任何必要的變更後，請選取 [**儲存**]。 
1. 當您準備好要發佈更新時，請選取 [**發佈**]。 [**[審查與發佈](#review-and-publish-an-updated-offer)**] 頁面隨即開啟，並提供更新的完成狀態。

## <a name="compare-changes-to-marketplace-offers"></a>比較 marketplace 供應專案的變更

您可以使用 [**比較**] 讓您對[已發行](#compare-changes-to-published-offer)或[預覽](#compare-changes-to-a-preview-offer)的供應專案進行變更，然後再將其上線。

> [!NOTE]
> 已發佈的供應專案是已成功發行至預覽或即時狀態的供應專案。

如需一般的審核資訊，請參閱以下內容：

- 您可以在編輯過程中的任何時間點使用 [**比較**]。
- 在 [**比較**] 頁面上選取欄位，以流覽至您想要修改的值。
- 如果您已經準備好發佈更新，請選取 [**審查併發布**]。
- 若要查看所有欄位的值（即使是未更新的欄位），請選取 [**所有欄位**] 篩選準則。 您可以藉由選取 [**修改的欄位**]，然後選取下列其中一個篩選準則，在這些欄位中修改篩選：
    - [**已移除的值**] 篩選會顯示您已發行的欄位，而且您現在已經完全移除。
    - [**已新增值**] 篩選會顯示您原本未發行且現在正在新增的欄位。
    - [已**編輯的值**] 篩選會顯示已發佈的欄位，但您現在已更新內容。

      >[!NOTE]
      > 如果其中一個篩選器無法使用，則表示您未進行該類型的更新。

- 若只要查看尚未更新的值，請選取 [**未**變更的欄位] 篩選準則。 針對已發行和草稿版本所顯示的域值將會相同。

  ![用來比較已發行或預覽供應專案更新的篩選準則](./media/compare-changes-marketplace.png)

>[!NOTE]
> 下列頁面目前不支援**比較**：
>- CSP 轉銷商物件
>- 試用產品技術設定
>- 試用產品 Marketplace 清單
>- 共同銷售
>- 補充檔案

### <a name="compare-changes-to-published-offer"></a>比較已發佈供應專案的變更

請依照下列指示來比較已發佈供應專案的變更：

1. 在頁面的命令列中選取 [**比較**]。 [**比較**] 頁面提供此供應專案的已儲存變更和已發佈 marketplace 供應專案的並存版本。
2. 當您從供應專案的特定頁面存取 [**比較**] 時，預設只會顯示在該頁面上所做的變更。
    - 如果您想要比較所有頁面上的變更，請將頁面從 [**選取頁面**] 變更為 [比較]。  
    - 如果您想要比較所有頁面上的供應專案變更，請選取 [**所有頁面**]。

做為預設值，**比較**您的新變更與 live marketplace 供應專案的對比。

請記得在更新之後重新發佈您的供應專案，讓變更生效。

### <a name="compare-changes-to-a-preview-offer"></a>比較預覽供應專案的變更

如果您的預覽變更不是即時的，您可以將新的變更與預覽 marketplace 供應專案進行比較。

請依照下列指示，比較新的變更與您的預覽 marketplace 供應專案：

1. 在頁面的命令列中選取 [**比較**]。
2. 選取 [**包含**] 下拉式清單，並將它從 [ **Live 供應**專案] 變更為 [**預覽供應**專案
3. [**比較**] 頁面提供顯示變更的並存版本。

>[!NOTE]
>如果您的供應專案尚未上線，但您已發佈至預覽版本，您就無法選擇與 live 供應專案進行比較。

請記得在更新之後重新發佈您的供應專案，讓變更生效。

## <a name="delete-a-draft-offer"></a>刪除草稿供應專案

您可以從**供應專案總覽**頁面中選取 [**刪除草稿**]，以刪除草稿供應專案（尚未發行的供應專案）。 如果您先前已發佈供應專案，將無法使用此選項。

在您確認要刪除草稿供應專案之後，供應專案將不會再顯示或可在合作夥伴中心存取，且 [**所有**供應專案] 頁面將會開啟。

## <a name="delete-a-draft-plan"></a>刪除草稿計畫

若要刪除尚未發行的方案，請從**計畫的 [總覽**] 頁面選取 [**刪除草稿**]。 如果您先前已發佈供應專案，將無法使用此選項。

在您確認要刪除草稿計畫之後，在合作夥伴中心中將不會再看到或存取方案。

## <a name="cancel-publishing"></a>取消發行

若要取消具有 [**發行進行中**] 狀態的供應專案：

1. 選取供應專案名稱，以開啟 [**供應專案總覽**] 頁面。
1. 從頁面右上角選取 [**取消發佈**]。
1. 確認您想要停止發佈供應專案。

如果您稍後想要發佈供應專案，您必須在上啟動發佈程式。

> [!NOTE]
> 只有當供應專案尚未進入「發行者登出」步驟時，您才可以將供應專案從發佈中停止。 **選取 [上線]** 之後，您將無法選擇取消 [發行]。

## <a name="stop-selling-an-offer-or-plan"></a>停止銷售供應專案或方案

若要在發佈供應專案之後停止銷售，請從供應專案的 **[總覽**] 頁面選取 [**停止銷售**]。 若要停止銷售計畫，請從 [**計畫總覽**] 頁面選取 [**停止銷售**]。 （只有當您在供應專案中有多個方案時，才可以使用停止銷售計畫的選項。 您可以選擇停止銷售一個方案，而不會影響供應專案內的其他方案）。

一旦確認您想要停止銷售供應專案或方案，在幾小時內就不會再顯示在 marketplace 中，而且不會有任何新的客戶下載。

先前取得供應專案或方案的任何客戶仍可使用它。 他們可以再次下載，但如果您稍後更新並重新發佈供應專案或方案，將不會取得更新。 

在您停止銷售供應專案/方案的要求完成後，您仍然可以在合作夥伴中心的商業 Marketplace 入口網站中看到它。 若要刪除草稿，請從 [**供應專案總覽**] 或 [**計畫總覽**] 頁面選取 [**刪除草稿**]。 

如果您決定要再次列出或銷售此供應專案或方案，請遵循指示來[更新現有的供應](#update-an-existing-offer-in-the-commercial-marketplace)專案。 別忘了，在進行任何變更之後，您將需要再次**發佈**供應專案或方案。

## <a name="remove-offers-from-existing-customers"></a>移除現有客戶提供的優惠

若要移除現有客戶提供的供應專案，請[記錄支援要求](https://support.microsoft.com/supportforbusiness/productselection?sapId=48734891-ee9a-5d77-bf29-82bf8d8111ff)。 從 [**問題類型**] 清單中選取您的特定供應專案類型，然後選取 [**類別**] 清單中的 [**移除已發佈的供應**專案]。 提交要求。 支援小組會引導您完成供應專案移除程式。

## <a name="sync-private-plan-audiences"></a>同步私人方案物件

如果您的供應專案包含一或多個設定為僅供私用限制的物件使用的方案，則可以只更新可存取該私人方案的物件，而不需發佈供應專案的其他變更。 

若要更新並同步處理您方案的私用物件：

- 使用 [+**新增識別碼**] 或 [匯**入客戶（csv）** ] 按鈕，在一或多個私人方案中修改物件，然後儲存變更。
- 從 [**計畫總覽**] 頁面選取 [**同步處理私用物件**]。

**同步私用物件**只會將變更發佈給您的私用使用者，而不會發佈您可能對草稿供應專案進行的任何其他更新。
