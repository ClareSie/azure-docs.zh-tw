# <a name="advanced-multistage-attack-detection-in-azure-sentinel"></a>Azure 哨兵中的高級多階段攻擊檢測


> [!IMPORTANT]
> Azure 哨兵中的某些融合功能當前處於公共預覽版中。
> 這些功能在沒有服務等級協定的情況下提供，不建議用於生產工作負載。 可能不支援特定功能，或可能已經限制功能。 如需詳細資訊，請參閱 [Microsoft Azure 預覽版增補使用條款](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)。



通過使用基於機器學習的 Fusion 技術，Azure Sentinel 可以通過將異常行為和在殺傷鏈的各個階段觀察到的可疑活動相結合，自動檢測多階段攻擊。 然後，Azure Sentinel 會生成否則很難捕獲的事件。 這些事件包含兩個或多個警報或活動。 根據設計，這些事件體積低、逼真度高、嚴重性高。

此檢測可針對您的環境進行定制，不僅可降低誤報率，還可以檢測資訊有限或缺失的攻擊。

## <a name="configuration-for-advanced-multistage-attack-detection"></a>進階多階段攻擊偵測組態

預設情況下，在 Azure Sentinel 中啟用此檢測。 要檢查狀態，或禁用狀態，可能是因為您正在使用替代解決方案基於多個警報創建事件，請使用以下說明：

1. 如果尚未這樣做，請登錄到[Azure 門戶](https://portal.azure.com)。

2. 導航到**Azure 哨兵** > **配置** > **分析**

3. 選擇**活動規則**，並在**NAME**列中查找**高級多階段攻擊檢測**。 檢查**狀態**列以確認此檢測是啟用還是禁用。

4. 要更改狀態，請選擇此條目，並在**高級多階段攻擊檢測**邊欄選項卡上選擇 **"編輯**"。

5. 在 **"規則創建"嚮導**邊欄選項卡上，將自動為您選擇狀態更改，因此選擇 **"下一步：審閱**"，然後**保存**。 

規則範本不適用於高級多階段攻擊檢測。

> [!NOTE]
> Azure Sentinel 目前使用 30 天的歷史資料來訓練機器學習系統。 當資料通過機器學習管道時，始終使用 Microsoft 的金鑰進行加密。 但是，如果在 Azure Sentinel 工作區中啟用了 CMK，則不會使用[客戶託管金鑰 （CMK）](customer-managed-keys.md)加密訓練資料。 要退出宣告融合，請導航到**Azure 哨兵** \> **配置分析** \> **\>活動\>規則高級多階段攻擊檢測**，並在 **"狀態**"列中選擇 **"禁用"。**

## <a name="fusion-using-palo-alto-networks-and-microsoft-defender-atp"></a>融合使用帕洛阿爾托網路和微軟後衛ATP

這些方案結合了安全分析師使用的兩個基本日誌：來自帕洛阿爾托網路的防火牆日誌和來自 Microsoft Defender ATP 的端點檢測日誌。 在下面列出的所有方案中，在涉及外部 IP 位址的端點中檢測到可疑活動，然後，從外部 IP 位址回防火牆的異常流量隨後檢測到。 在 Palo Alto 日誌中，Azure Sentinel 側重于[威脅日誌](https://docs.paloaltonetworks.com/pan-os/8-1/pan-os-admin/monitoring/view-and-manage-logs/log-types-and-severity-levels/threat-logs)，當允許威脅時，流量被視為可疑（可疑資料、檔、洪水、資料包、掃描、間諜軟體、URL、病毒、漏洞、野火病毒、野火）。

### <a name="network-request-to-tor-anonymization-service-followed-by-anomalous-traffic-flagged-by-palo-alto-networks-firewall"></a>對 TOR 匿名服務的網路請求，然後是帕洛阿爾托網路防火牆標記的異常流量。

然後，帕洛阿爾托網路防火牆在 [Time生成] 處檢測到異常活動。 這表示惡意流量進入您的網路 網路流量的目標 IP 位址是 [目標 IP]。

此方案當前處於公共預覽版中。
