# <a name="get-started-with-azure-security-center-for-iot"></a>開始使用適用於 IoT 的 Azure 資訊安全中心

本文介紹了適用于 IoT 服務的 Azure 安全中心的不同構建基塊，並解釋了如何使用兩個可能的部署選項開始使用該服務。  

## <a name="deployment-options"></a>部署選項

選擇最符合 IoT 設備和環境要求的服務方案。 

### <a name="built-in-deployment"></a>內置部署
使用無縫的內置部署選項，可以快速將 IoT Azure 安全中心集成到 IoT 中心中，並提供 IoT 中心配置、設備標識和管理以及中心設備通訊模式的安全分析。

啟動包含 IoT 中心監視和建議[的內置部署](iot-hub-integration.md)。 
    <br>

### <a name="enhanced-deployment"></a>增強部署
為了增強安全功能，為 IoT 代理部署 Azure 安全中心，除了啟用 IoT 中心安全外，還提供基於代理的事件收集、分析和威脅檢測 IoT 設備的關鍵安全資料，以及全面安全狀態管理功能。

啟動[增強部署](security-agents.md)，採用基於代理的全面威脅保護和安全態勢管理解決方案。
