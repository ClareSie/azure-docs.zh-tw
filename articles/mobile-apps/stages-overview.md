# <a name="different-stages-in-mobile-application-development"></a>移動應用程式開發的不同階段
構建移動應用程式涉及多個階段。 您可以輕鬆地利用 Microsoft 服務、工具和技術構建高級移動應用程式。

## <a name="app-development"></a>應用程式開發
您可以使用 Objective-C 和 JAVA 等語言構建本機單平臺用戶端應用程式。 您可以使用 Xamarin、反應本機和 Unity 構建跨平臺應用。 您還可以使用科爾多瓦構建混合應用程式。 要構建用戶端應用程式，請使用 ID 和代碼編輯器，例如 Visual Studio、適用于 Mac 的視覺化工作室或 PC 和 Mac 上的視覺化工作室代碼。

## <a name="plan-and-design-with-back-end-services"></a>使用後端服務進行規劃和設計
 Microsoft 提供各種服務，用於為您的移動應用程式設定安全、可擴展和可靠的後端。 您可以創建無伺服器 API 和觸發器，而無需維護自己的伺服器。 使用推送通知、離線資料同步、身份驗證和資料服務來形成一個全面而廣泛的移動後端，支援各種應用構建方案。 
 
 構建後端服務有許多要求。 您必須：
   - 限制設備上的資料存儲。
   - 跨多個設備同步資料。
   - 確保應用程式在沒有網路連接的情況下運行。
   - 向使用者發送通知。
   - 減少電池消耗。
   - 構建可擴展、可靠且安全的後端。
   - 自動執行伺服器維護。

## <a name="devops-and-continuous-monitoring"></a>DevOps 和持續監控
要創建以移動和麵向雲為中心的應用程式，請利用持續集成 （CI）、持續交付 （CD） 和持續監視服務為所有平臺生成本機生成。 您可以在數千個物理設備上運行 UI 自動化測試，通過 Beta 測試管道自動執行其發佈，或者直接部署到應用商店。 您還可以為其應用程式收集使用方式分析、故障和異常資訊。

## <a name="additional-services"></a>其他服務
根據正在構建的應用程式的類型和性質，可以使用許多 Azure 服務：
  - Azure 認知搜索以快速查詢搜索所有類型的內部資料。
  - Azure 認知服務在應用程式中使用人工智慧。
  - Azure SignalR 在應用程式中啟用即時通信。
