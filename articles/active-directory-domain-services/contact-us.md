# <a name="azure-ad-domain-services---contact-us"></a>Azure AD 網域服務 - 與我們連絡

## <a name="feedback"></a>意見反應
如果您有關于我們的服務或非技術問題的回饋，則可以通過發送電子郵件給我們[：Azure AD 域服務回饋](mailto:aaddsfb@microsoft.com)與我們共用您的請求。 將審核發送到此位址的電子郵件。 產品組的成員**可以**回復請求更多資訊。 將不會答覆對此電子郵件地址進行技術支援的請求。

## <a name="technical-assistance"></a>技術援助
如果對您的受控網域有問題，請檢查以了解 [疑難排解指南](troubleshoot.md) 中所述的步驟是否可解決問題。 如果仍有問題，請登錄到 Azure 門戶。 打開**Azure AD 域服務**主頁，並從 **"支援 + 故障排除**"下的執行窗格中選擇 **"新支援請求**"。

![新增支援要求](./media/contact-us/supportRequest.png) 

盡可能準確地填寫支援請求的每個部分。 選擇服務時，請確保選擇**Azure 活動目錄域服務（VM - 網域控制站）** 以將請求路由到適當的支援專業人員。

> [!IMPORTANT]
> 從服務清單中選擇**Azure 活動目錄目錄、域和物件，** 以便借助 Azure 活動目錄進行域驗證。
>
> 

## <a name="feature-requests"></a>功能要求
您是否對如何改進 Azure 活動目錄域服務有想法？ 我們想聽聽。 轉到 Azure 回饋論壇，導航到[域服務](https://feedback.azure.com/forums/169401-azure-active-directory?category_id=160593)類別並分享您的想法。 產品組的成員會審核這些請求。
