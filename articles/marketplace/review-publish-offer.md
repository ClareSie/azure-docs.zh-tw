# <a name="review-and-publish-an-offer-to-the-commercial-marketplace"></a>審查供應專案並將其發佈至商用 marketplace

本文說明如何使用合作夥伴中心，將您的供應專案提交給預覽、預覽您的供應專案，然後將其發佈至 Microsoft 商業 marketplace。 我們也會討論如何檢查發佈狀態，因為它會繼續進行發佈步驟。 您必須已建立要發佈的供應專案。

## <a name="go-to-your-offer-in-the-commercial-marketplace"></a>前往商業 marketplace 中的供應專案

1. 登入[合作夥伴中心](https://partner.microsoft.com/dashboard/home)。
2. 在左側導覽功能表中，選取 [**商用 Marketplace**  >  **總覽**]。
3. 在 [**總覽**] 索引標籤上的 [供應專案]**底下，每**個供應專案的 [**狀態**] 欄中會顯示下列其中一個狀態指示器。

    | 狀態 | 描述 |
    | ------------ | ------------- |
    | 草稿 | 已建立供應專案，但尚未發佈。 |
    | 正在發行 | 供應專案會透過發佈程式來運作。 |
    | 需要注意 | 我們在認證期間或在另一個發佈階段發現嚴重的問題。 |
    | 預覽 | 我們已認證供應專案，現在會等待發行者進行最終驗證。 選取 **[上線]** 以即時發佈供應專案。 |
    | 即時 | 供應專案會在 marketplace 中上線，並可供客戶查看和取得。 |
    | 暫止停止銷售 | 發行者選取了供應專案或方案的「停止銷售」，但該動作尚未完成。 |
    | Marketplace 中未提供 | 市集中先前發佈的供應項目已經被移除。 |
    |||

4. 在 [**供應專案別名**] 欄中，選取您想要預覽併發布的供應專案。

## <a name="submit-your-offer-to-preview"></a>提交您的供應專案以進行預覽

1. 若要提交您的供應專案以進行預覽，請選取入口網站右上角的 [**審查併發布**]。 [**審查與發行**] 頁面隨即出現。
1. 請確定每個頁面的 [**狀態**] 欄顯示 [**完成**]。 三種可能的狀態如下：

   - **未啟動**–此頁面不完整。
   - **不完整**–頁面遺漏必要的資訊，或有需要修正的錯誤。 您必須回到頁面並加以更新。
   - **完成**–此頁面已完成。 已提供所有必要的資料，而且沒有任何錯誤。

1. 如果有任何頁面的狀態不是 [**完成**]，請在 [**頁面**] 資料行中選取頁面名稱、更正問題、儲存頁面，然後選取 [**檢查] 並再次發佈**以返回此頁面。
1. 所有頁面都完成之後，請在 [憑證的**附注**] 方塊中提供測試指示給認證小組，以確保您的應用程式已正確測試。 提供有助於瞭解應用程式的任何補充附注。
1. 若要提交供應項目進行發佈，請選取 [發佈]。 [**供應專案總覽**] 頁面隨即出現，並顯示發佈狀態。

## <a name="validation-and-publishing-steps"></a>驗證和發行步驟

選取 [**發行**] 之後，驗證和發佈程式會按照順序繼續進行。 最常見的發佈程式如下表所示：

| 階段 | 會發生的事 |
| ------------ | ------------- | ------------- |
| 自動化驗證 | 我們會處理一組自動化驗證。 |
| 認證 | 我們會進行手動驗證。 |
| 預覽建立 | 具有 [預覽] 連結的任何人都可以使用供應專案預覽的 [清單] 頁面。 如果您的供應專案將透過 Microsoft (transactable) 銷售，則只有您在供應專案的 [**預覽物件**] 頁面上指定的物件可以購買及存取供應專案以進行測試。 |
| 發佈者登出 | 我們會傳送一封電子郵件給您，要求您預覽並核准您的供應專案。 |
| 發佈 | 我們會執行一系列的步驟，以確認預覽供應專案已即時發佈到商業 marketplace。 |
|||

## <a name="automated-validation-phase"></a>自動化驗證階段

發行程式的第一個步驟是一組自動驗證。 每個驗證步驟都會對應至您在建立供應專案時所選擇的功能。 每個驗證檢查都必須完成，供應專案才能前進到發佈程式中的下一個步驟。

- **提供採購流程設定** ( # B0 10 分鐘) 

   我們確保您的供應專案可在客戶透過 Azure 入口網站購買時完成。 此步驟僅適用于透過 Microsoft 銷售的供應專案。

- **測試磁片磁碟機資料驗證** (~ 5 分鐘) 

   我們會驗證您在供應專案的 [技術設定] 頁面上所提供的資料。 我們會測試並核准試用產品的功能。 此步驟僅適用于已啟用試用產品的供應專案。

- **測試磁片磁碟機**布建 (~ 30 分鐘) 

    在您于上一個步驟中驗證試用產品的資料和功能之後，我們會部署並複寫試用產品的實例，讓客戶可以開始使用。 此步驟僅適用于已啟用試用產品的供應專案。

- **潛在客戶管理驗證和註冊** ( # B0 15 分鐘) 

    我們會根據您在 [**供應專案設定**] 頁面上提供的詳細資料，確認您的潛在客戶管理系統可以接收客戶的潛在客戶。 此步驟僅適用于已啟用潛在客戶管理的供應專案。

## <a name="certification-phase"></a>認證階段

提交給商業 marketplace 的供應專案必須經過認證才能發佈。 提供嚴格的測試，部分自動化，另一種是手動。 若要深入瞭解，請參閱[商業 marketplace 認證原則](https://aka.ms/commercial-marketplace-certification-policies)。

### <a name="types-of-validation-that-take-place-during-certification"></a>認證期間所發生的驗證類型

每個提交供應專案的認證程式中包含三個層級的驗證。

- 發行者商務資格
- 內容驗證
- 技術驗證

#### <a name="publisher-business-eligibility"></a>發行者商務資格

每個供應專案類型都會檢查一組必要的基本資格準則。 此準則可能包括發行者 MPN 狀態、保留的專長認證層級等等。

#### <a name="content-validation"></a>內容驗證

當您建立供應專案時所輸入的資訊，會核取 [品質與相關性]。 這些檢查會檢查您的專案中是否有 marketplace 清單詳細資料、定價、可用性、相關聯的計畫等等。 為了符合 Azure Marketplace 和 Microsoft AppSource 的清單準則，我們會驗證您的供應專案包括：

- 準確描述供應專案的標題
- 提供完整總覽和價值主張的書面描述
- 品質螢幕擷取畫面和影片
- 提供供應專案如何利用 Microsoft 平臺和工具的說明。

閱讀[一般清單原則](https://aka.ms/commercial-marketplace-certification-policies#100-general)，以深入瞭解內容驗證準則。

#### <a name="technical-validation"></a>技術驗證

在技術驗證期間，供應專案 (封裝或二進位) 會經歷下列檢查。

- 已掃描是否有惡意程式碼
- 受監視的網路呼叫
- 已分析封裝
- 徹底掃描供應專案的功能

此供應專案會跨各種平臺和版本進行測試，以確保其健全。

### <a name="certification-failure-report"></a>認證失敗報告

如果您的供應專案未通過任何清單、技術或原則檢查，或如果您沒有提交該類型供應專案的資格，我們會以電子郵件將認證失敗報告傳送給您。

這份報告包含任何失敗原則的描述，以及審核注意事項。 請參閱此電子郵件報告、解決任何問題、視需要更新您的供應專案，然後使用合作夥伴中心的[商業 marketplace 入口網站](https://partner.microsoft.com/dashboard/commercial-marketplace/offers)重新提交供應專案。 您可以視需要多次重新提交供應專案，直到通過認證為止。

## <a name="preview-creation-phase"></a>預覽建立階段

在預覽建立階段，我們會建立您供應專案的版本，只有您在供應專案的 [**預覽物件**] 頁面上指定的物件可以存取（如果有的話）。 您的供應專案預覽版本將無法供預覽觀眾以外的任何人使用，直到您正式發行供應專案為止。

> [!NOTE]
> 請不要使用預覽物件，讓組織外部的人員能夠看到供應專案。 請改用私用供應專案選項。 此時，您的供應專案尚未經過完整測試和驗證，而且尚未準備好進行外部散發。

## <a name="publisher-sign-off-phase"></a>發行者登出階段

當供應專案準備好供您審查並登出時，我們會傳送一封電子郵件給您，要求您審查並核准您的供應專案預覽。 您也可以在瀏覽器中重新整理**供應專案總覽**頁面，以查看您的供應專案是否已達到發行者登出階段。 如果有的話，就可以**使用 [上線**] 按鈕和 [預覽] 連結。

下列螢幕擷取畫面顯示 SaaS 供應專案的**供應專案總覽**頁面。 您會在此頁面上看到的驗證步驟，會根據您在建立供應專案時所做的供應專案類型和選擇而有所不同。

![說明合作夥伴中心供應專案的供應專案總覽頁面。 [上線] 按鈕和預覽連結會顯示。](./partner-center-portal/media/publish-status-publisher-signoff.png)

### <a name="to-preview-your-offer-and-sign-off"></a>預覽您的供應專案並登出

1. 在 [**供應專案總覽**] 頁面上，若要預覽您的供應專案，請選取 [**上線**] 按鈕底下的連結。

   > [!NOTE]
   > 根據您在建立供應專案時所選擇的選項而定，會有 AppSource preview、Azure Marketplace preview 或兩者的連結。 如果您選擇透過 Microsoft 銷售您的供應專案，任何已加入預覽物件的人都可以測試您的供應專案的取得與部署，以確保它符合您在此階段的需求。

1. 如果您想要在預覽供應專案後進行變更，您可以編輯並重新提交，以發佈新的預覽。 如需詳細資訊，請參閱在[商業 marketplace 中更新現有的供應](./partner-center-portal/update-existing-offer.md)專案。

1. 核准預覽之後，若要將您的供應專案即時發佈到商業 marketplace，**請選取 [上線]**。

   > [!TIP]
   > 如果您的供應專案已上線並可供 marketplace 中的公眾使用，則您所做的任何更新都不會上線，直到**您選取 [上線]** 為止。

## <a name="publish-phase"></a>發行階段

既然您已選擇讓供應專案上線，使其可在商業 marketplace 中使用，我們會執行一系列的最終驗證檢查，以確保即時供應專案的設定就如同供應專案的預覽版本。

- **提供採購流程設定** ( # B0 10 分鐘) 

    我們確保您的供應專案可在客戶透過 Azure 入口網站購買時完成。 此步驟僅適用于透過 Microsoft 銷售的供應專案。

- **測試磁片磁碟機資料驗證** (~ 5 分鐘) 

    我們會驗證您在供應專案的 [技術設定] 頁面上所提供的資料。 我們會測試並核准試用產品的功能。 此步驟僅適用于已啟用試用產品的供應專案。

- **測試磁片磁碟機**布建 (~ 30 分鐘) 

    我們會部署並複寫您試用產品的實例，讓它們可供客戶使用。 此步驟僅適用于已啟用試用產品的供應專案。

- **潛在客戶管理驗證和註冊** ( # B0 15 分鐘) 

    我們會確認您的潛在客戶管理系統可以根據您**供應專案設定**頁面上提供的詳細資料，來接收客戶潛在客戶。 此步驟僅適用于已啟用潛在客戶管理的供應專案。

- **最終發行 ( # B0 30 分鐘) **

    我們確保您的供應專案在 marketplace 中公開上市。

這些驗證檢查完成之後，您的供應專案就會在 marketplace 中上線。

### <a name="reporting-usage-of-preview-and-other-internal-usage"></a>報告預覽和其他內部使用方式的使用方式

若要驗證端對端購買和設定流程，請在您的供應專案處於預覽狀態時購買。 請先向 Microsoft 通知[支援票證](https://aka.ms/marketplacesupport)，以確保我們不會處理費用。
