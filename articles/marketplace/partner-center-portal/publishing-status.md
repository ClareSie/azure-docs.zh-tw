# <a name="check-the-publishing-status-of-your-commercial-marketplace-offer"></a>檢查商業市場產品/服務的產品發佈狀態

您可以在合作夥伴中心["商業市場"門戶](https://partner.microsoft.com/dashboard/commercial-marketplace/offers)的 **"產品/服務概覽"** 選項卡上查看當前**發佈狀態**。

每個供應項目應該都會顯示下列狀態指標的其中一個。

| **狀態**    | **描述**  |
| :---------- | :-------------------|
| **Draft**(英文\) | 優惠已創建，但未發佈。 |
| **正在發行** | 優惠/計畫正在通過發佈過程的步驟進行。 |
| **需要注意** | 在 Microsoft 或任何發佈步驟認證期間發現了一個關鍵問題。 |
| **預覽** | 優惠由微軟認證，現在等待發佈商的最終驗證。 選擇"上線"使優惠成為即時優惠。 |
| **即時** | 優惠是活在市場上，可以看到和收購的客戶。 |
| **待定止損銷售** | 發佈商選擇"停止銷售"報價或計畫，但操作尚未完成。 |
| **市場不可用** | 市場上以前發佈的要約/計畫已被刪除。 |

## <a name="automated-validation"></a>自動驗證

發佈過程的第一步是一組自動驗證。 每個驗證步驟都對應于您在創建產品/服務時選擇啟用的功能。 如果未啟用該功能，驗證將跳到下一個發佈步驟。 在批准發佈狀態之前，必須完成每次驗證檢查。

- **提供購買流程設置（<10 分鐘）**

在此步驟中，我們確保客戶通過 Azure 門戶購買時可以完成您的產品/服務。 此步驟僅適用于通過 Microsoft 銷售的產品優惠。

- **試駕資料驗證（+5 分鐘）**

在此步驟中，我們將驗證您在產品/服務中的試駕技術配置部分中提供的資料。 測試磁碟機功能經過測試和批准。 此步驟僅適用于啟用了試駕的產品。

- **試駕配置（約30分鐘）**

在此步驟中，在上一步驟中驗證了試駕的資料和功能後，我們將部署和複製您的試駕實例，以便它們可供客戶使用。  此步驟僅適用于啟用了試駕的產品。

- **鉛管理驗證和註冊（<15 分鐘）**

在此步驟中，我們確認您的潛在客戶管理系統可以根據產品/服務設置中提供的詳細資訊接收客戶潛在客戶。 此步驟僅適用于啟用了潛在客戶管理的產品/

## <a name="certification"></a>認證

在發佈之前，提交給合作夥伴中心商業市場的報價必須經過認證。 提交的產品/服務經過嚴格的測試，一些自動化和其他手冊，包括針對[Azure 應用商店認證策略](https://docs.microsoft.com/legal/marketplace/general-policies)的檢查。 要價提交必須標記為符合認證條件，然後才能繼續發佈流程中的下一步。

### <a name="types-of-validation-that-take-place-during-certification"></a>認證期間發生的驗證類型

There are three levels of validation included in the certification process for each offer submitted.

- 發佈商業務資格
- 內容驗證
- 技術驗證

#### <a name="publisher-business-eligibility"></a>發佈商業務資格

每個產品/服務類型檢查發行者必須滿足的一組基本資格條件。 資格標準可能包括發佈商的 MPN 狀態、所持有的能力、能力級別等。

#### <a name="content-validation"></a>內容驗證

在內容驗證期間，將檢查創建產品/服務時輸入的資訊的品質和相關性。 這些檢查將檢查您的條目，瞭解市場清單詳細資訊、定價、可用性、相關計畫等。為了滿足 Azure 應用商店和/或 AppSource 清單條件，我們將驗證您的產品/服務是否包括：

- 準確描述要約的標題;
- 寫得很好的描述，提供全面的概述和價值主張;
- 品質截圖和隨附的視頻;和
- 說明產品/服務如何利用 Microsoft 平臺和工具。

通過閱讀[常規清單策略](https://docs.microsoft.com/legal/marketplace/certification-policies#100-general)，瞭解有關內容驗證標準的資訊。

#### <a name="technical-validation"></a>技術驗證

在技術驗證期間，報價（包或二進位）會進行以下檢查。
- 掃描惡意軟體
- 監控的網路呼叫
- 包分析
- 徹底掃描產品/服務的實際功能

該產品/服務在各種平臺和版本中進行測試，以確保其穩健性。

在本文檔的技術配置部分中查看產品/服務所需的特定配置詳細資訊。

### <a name="certification-failure-report"></a>認證失敗報告

審核完成後，如果您的報價已通過認證，則它將進入發佈過程中的下一步。 如果您的報價未能通過任何清單、技術或策略檢查，或者您沒有資格提交該類型的報價，則生成認證失敗報告並通過電子郵件發送給您。

此報告包含任何失敗策略的說明以及審閱說明。 查看此電子郵件報告，解決任何問題，根據需要更新您的產品/服務，並使用合作夥伴中心[中的商業市場門戶](https://partner.microsoft.com/dashboard/commercial-marketplace/offers)重新提交產品/服務。 （您可以根據需要多次重新提交產品/服務，直到通過認證）。

## <a name="preview-creation"></a>預覽創建

在**預覽創建**步驟中，我們創建僅可供您在"預覽"產品/服務"部分中指定的受眾訪問的報價版本。

>[!Note]
> 不要使用此步驟來讓組織外部人員瞭解產品/服務。 請改為使用 **"私人優惠**"選項。 此時，您的產品尚未經過全面測試和驗證，尚未準備好進行外部分發。

## <a name="publisher-approval"></a>發佈商批准

在此步驟中，您將通過電子郵件請求在最終發佈步驟之前審核和批准產品/服務預覽。

如果您已選擇通過 Microsoft 銷售產品/服務，您將能夠測試產品/服務的採購和部署，以確保此預覽審批階段滿足您的要求。 您的優惠尚不在公開市場提供。 測試和批准此預覽後，您需要在[**"優惠概述"儀表板**](https://partner.microsoft.com/dashboard/commercial-marketplace/overview)上選擇 **"上線**"。

如果要在此預覽階段更改產品/服務，可以編輯並重新提交以發佈新的預覽。 有關更多更改的詳細資訊，請參閱文章["更新現有市場產品/服務](#update-existing-marketplace-offers)"。

如果您的優惠已即時且可供市場公眾使用，則在"[**優惠概述"**](https://partner.microsoft.com/dashboard/commercial-marketplace/overview)儀表板上選擇 **"上線**"之前，您進行的任何更新都將無法上線。

### <a name="publish-offer-to-the-public"></a>向公眾發佈優惠

登錄合作夥伴中心並訪問優惠。 您將被重定向到 **"優惠概覽**"頁。 在此頁面的頂部，您將看到 **"去現場"** 的選項。 選擇 **"去現場"，** 確認後，優惠將開始向公眾公佈。 優惠即時時，您將收到電子郵件通知。

## <a name="publish"></a>發佈

現在，您已選擇使用產品/服務**進行即時**處理，使其在市場上可用，因此將逐步完成一系列最終驗證檢查，以確保即時優惠的配置與產品/服務的預預覽版本類似。

- **提供購買流程設置（>10 分鐘）**

在此步驟中，我們確保客戶通過 Azure 門戶購買時可以完成您的產品/服務。 此步驟僅適用于通過 Microsoft 銷售的產品優惠。

- **試駕配置（約30分鐘）**

在此步驟中，我們部署和複製您的試駕實例，以便它們可供客戶使用。  此步驟僅適用于啟用了試駕的產品。

- **鉛管理驗證和註冊（>15 分鐘）**

- **最終發佈（>30 分鐘）**

在此步驟中，我們確保您的報價在市場上公開提供。

如果要更改已發佈的產品/服務，則需要首先更新現有產品/服務，然後再次發佈。
