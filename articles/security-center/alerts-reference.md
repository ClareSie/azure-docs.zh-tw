# <a name="security-alerts---a-reference-guide"></a>安全性警示 - 參考指南

本文列出您可在 Azure 資訊安全中心的威脅防護模組中看見的安全性警示。 在您的環境中顯示的警示，取決於您要保護的資源和服務，以及自訂設定。

若要了解如何管理與回應這些警示，請參閱[管理與回應 Azure 資訊安全中心的安全性警示](security-center-managing-and-responding-alerts.md)。

若要了解如何匯出警示 (和建議)，請參閱[匯出安全性警示和建議 (預覽)](continuous-export.md)。

在此頁面底部，有一個資料表描述用來分類這些警示意圖的 Azure 資訊安全中心終止鏈結。

## <a name="intentions"></a>意圖

了解攻擊的目的可協助您更輕鬆地調查和報告事件。 Azure 資訊安全中心警示中納入了「意圖」欄位，以協助您記錄這些資訊。

描述從偵察網路攻擊到資料外流之進度的一系列步驟，通常稱為「終止鏈結」。 

資訊安全中心支援的終止鏈結意圖是以 [MITRE ATT&CK&trade; 架構](https://attack.mitre.org/matrices/enterprise) 為基礎，並在下表中說明。

| Intent                  | 描述                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
|-------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **PreAttack**           | PreAttack 可能是嘗試存取特定資源 (無論是否為惡意用途)，或嘗試在惡意探索之前取得目標系統的存取權以收集資訊時失敗。 此步驟通常會被偵測為來自網路外部的嘗試，目的是掃描目標系統並識別進入點。 </br>您可以在 [MITRE 頁面](https://attack.mitre.org/matrices/pre/)中，閱讀 PreAttack 階段的進一步詳細資料。                                                                                                                                |
| **InitialAccess**       | InitialAccess 是攻擊者管理以取得受攻擊資源據點的階段。 此階段與計算主機和資源相關，例如使用者帳戶、憑證等。威脅執行者通常可以在此階段之後控制資源。 |
| **持續性**         | 持續性是系統的任何存取、動作或設定變更，可讓威脅執行者在該系統上持續存在。 威脅執行者通常需要透過中斷 (例如系統重新開機、認證遺失或其他可能需要遠端存取工具來重新啟動的失敗情況) 來維護系統的存取權，或提供替代的後門程式讓他們重新取得存取權。                                                                                                                                                                                            |
| **PrivilegeEscalation** | 權限提升是允許敵人在系統或網路上取得較高層級權限的動作結果。 某些工具或動作需要較高層級的權限才能正常運作，而且在整個作業的許多點上都有可能需要較高層級的權限。 若使用者帳戶具有可存取特定系統，或執行特定函式讓敵人達成目標的權限，也可能會被視為提升權限。                                                                                                                        |
| **DefenseEvasion**      | 防禦迴避包含敵人可用於規避偵測或避免其他防禦的技術。 有時候，這些動作與其他類別中的 (或各種不同的) 技術相同，都具有破壞特定防禦或緩和措施的附加優點。                                                                                                                                                                                                                                                                                                                    |
| **CredentialAccess**    | 認證存取代表可存取或控制系統、網域或服務認證的技術，可在企業環境中使用。 敵人可能會嘗試從使用者或管理員帳戶 (具有管理員存取權的本機系統管理員或網域使用者) 取得合法的認證，以在網路中使用。 有了足夠的網路存取權，敵人就可以建立帳戶以便稍後在環境中使用。                                                                                                      |
| **探索**           | 探索包含可讓敵人掌握系統和內部網路相關知識的技術。 當敵人取得新系統的存取權時，必須將自己導向到現在可以控制的內容，以利用可操作該系統的優點，在入侵期間達成目前目標或整體目標。 作業系統提供許多原生工具，有助於此入侵後的資訊收集階段。                                                                                                                                    |
| **LateralMovement**     | 橫向移動包含的技術可讓敵人存取和控制網路上的遠端系統，但不一定會包含在遠端系統上執行工具。 橫向移動技術可讓敵人從系統收集資訊，而不需要遠端存取工具等額外的工具。 敵人可以使用橫向移動來進行許多工作，包括遠端執行工具、切換到其他系統、存取特定資訊或檔案、存取其他認證，或造成不當影響。 |
| **執行**           | 執行策略代表會導致在本機或遠端系統上執行受敵人控制之程式碼的技術。 這種策略通常與橫向移動搭配使用，以擴充網路上遠端系統的存取權。                                                                                                                                                                                                                                                                                                                                                      |
| **集合**          | 集合包含的技術，可在資料外流之前，用來識別並收集來自目標網路的資訊，例如機密檔案。 此類別也涵蓋敵人可能在系統或網路上尋找可竊取資訊的位置。                                                                                                                                                                                                                                                                                                                            |
| **外流**        | 外流是指在入侵者從目標網路移除檔案及資訊時，產生或協助的技術與屬性。 此類別也涵蓋敵人可能在系統或網路上尋找可竊取資訊的位置。                                                                                                                                                                                                                                                                                                                                  |
| **CommandAndControl**   | 命令與控制策略代表敵人在目標網路中，與其控制下的系統通訊方式。                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| **影響**              | 影響事件主要會嘗試直接減少系統、服務或網路的可用性或完整性；包括影響商務或作業流程的操控。 這通常指的是勒索軟體、竄改、資料操控和其他技術。                                                                                                                                                                                                                                                                                                     |
|                         |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |  |
