# <a name="security-recommendations"></a>安全性建議

IoT 的 Azure 安全中心掃描 Azure 資源和 IoT 設備，並提供安全建議以減少攻擊面。 安全建議是可操作的，旨在説明客戶遵守安全最佳實踐。

在本文中，您將找到可在 IoT 中心和/或 IoT 設備上觸發的建議清單。

## <a name="recommendations-for-iot-devices"></a>物聯網設備建議

設備建議提供改進設備安全狀態的見解和建議。 

| Severity | 名稱                                                      | 資料來源 | 描述                                                                                                                                                                                           |
|----------|-----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 中   | 打開設備上的埠                                      | 代理程式       | 在設備上找到一個偵聽終結點。                                                                                                                                                        |
| 中   | 在其中一個鏈中找到的允許防火牆策略。 | 代理程式       | 找到允許的防火牆策略（INPUT/輸出）。 預設情況下，防火牆策略應拒絕所有流量，並定義規則以允許必要的通信到/從設備進行。                               |
| 中   | 在輸入鏈中找到允許的防火牆規則     | 代理程式       | 在防火牆中已發現一條規則，其中包含各種 IP 位址或埠的允許模式。                                                                                    |
| 中   | 在輸出鏈中找到允許的防火牆規則    | 代理程式       | 在防火牆中已發現一條規則，其中包含各種 IP 位址或埠的允許模式。                                                                                   |
| 中   | 作業系統基線驗證失敗           | 代理程式       | 設備不符合[CIS Linux基準](https://www.cisecurity.org/cis-benchmarks/)。                                                                                                        |

### <a name="operational-recommendations-for-iot-devices"></a>物聯網設備的操作建議

操作建議提供了改進安全代理配置的見解和建議。

| Severity | 名稱                                    | 資料來源 | 描述                                                                       |
|----------|-----------------------------------------|-------------|-----------------------------------------------------------------------------------|
| 低      | 代理髮送未使用的消息          | 代理程式       | 在過去 24 小時內，10% 或更多的安全消息小於 4 KB。  |
| 低      | 安全雙配置不理想 | 代理程式       | 安全雙配置不是最佳配置。                                        |
| 低      | 安全雙配置衝突    | 代理程式       | 在安全雙配置中識別了衝突。 |                          |
|

## <a name="recommendations-for-iot-hub"></a>物聯網中心的建議

建議警報為改善環境的安全狀態的操作提供了見解和建議。  

| Severity | 名稱                                                     | 資料來源 | 描述                                                                                                                                                                                                             |
|----------|----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 高     | 多個設備使用的相同身份驗證憑據 | IoT 中樞     | IoT 中心身份驗證憑據由多個設備使用。 這可能表示非法設備正在類比合法設備。 重複的憑據使用會增加惡意參與者類比設備的風險。 |
| 中   | 預設 IP 篩選器策略應拒絕                  | IoT 中樞     | IP 篩選器配置應具有為允許的流量定義的規則，並且預設情況下應預設拒絕所有其他流量。                                                                                                     |
| 中   | IP 篩選器規則包括較大的 IP 範圍                   | IoT 中樞     | 允許 IP 篩選器規則源 IP 範圍過大。 過度寬鬆的規則可能會將 IoT 中心暴露給惡意參與者。                                                                                       |
| 低      | 在 IoT 中心中啟用診斷日誌                       | IoT 中樞     | 啟用記錄並保留記錄最長一年。 保留日誌使您能夠在發生安全事件或網路受到威脅時重新創建活動跟蹤以進行調查。                                       |
|
