為了保護 Azure 檔共用中的資料免受資料丟失或損壞的影響，所有 Azure 檔共用在寫入每個檔時都會存儲多個副本。 根據工作負載的要求，您可以選擇額外的冗余度。 Azure 檔當前支援以下資料冗余選項：

- **本地冗余**：本地冗余存儲（通常稱為 LRS）意味著每個檔在 Azure 存儲群集中存儲三次。 這樣可以防止由於硬體故障（如磁片磁碟機損壞）而遺失資料。
- **區域冗余**：區域冗余存儲（通常稱為 ZRS）意味著每個檔在三個不同的 Azure 存儲群集中存儲三次。 三個不同的 Azure 存儲群集每個存儲檔三次，就像本地冗余存儲一樣，並且在不同的 Azure*可用性區域*中物理隔離。 可用性區域是 Azure 區域中唯一的物理位置。 每個區域皆由一或多個配備獨立電力、冷卻系統及網路的資料中心所組成。 在寫入所有三個可用性區域中的存儲群集之前，不會接受寫入存儲。 
- **異地冗余**：地理冗余存儲（通常稱為 GRS）類似于本地冗余存儲，因為檔在主區域中的 Azure 存儲群集中存儲三次。 然後，所有寫入都會非同步複製到 Microsoft 定義的次要區域。 異地冗余存儲提供 6 個數據副本，分佈在兩個 Azure 區域之間。 如果發生重大災難（如由於自然災害或其他類似事件導致 Azure 區域永久丟失），Microsoft 將執行容錯移轉，以便次要資料庫實際上成為主發操作，服務所有操作。 由於主區域和次要區域之間的複製是非同步，因此在發生重大災難時，尚未複製到次要區域的資料將丟失。 您還可以執行異地冗余存儲帳戶的手動容錯移轉。
- **地理區域冗余**：地理區域冗余存儲（通常稱為 GZRS）類似于區域冗余存儲，因為檔在主區域中的 3 個不同的存儲群集中存儲了 9 次。 然後，所有寫入都會非同步複製到 Microsoft 定義的次要區域。 異地區域冗余存儲的容錯移轉過程與對異地冗余存儲的工作方式相同。

標準 Azure 檔共用支援所有四種冗余類型，而高級 Azure 檔共用僅支援本地冗余和區域冗余存儲。

通用版本 2 （GPv2） 存儲帳戶提供了 Azure 檔不支援的另外兩個冗余選項：讀取可訪問的異地冗余存儲（通常稱為 RA-GRS）和讀取可訪問的地理區域冗余存儲（通常為稱為 RA-GZRS。 您可以使用這些選項組在存儲帳戶中預配 Azure 檔共用，但 Azure 檔不支援從次要區域讀取。 部署到可讀的地理或地理區域冗余存儲帳戶中的 Azure 檔共用將分別計費為異地冗余或地理區域冗余存儲。
