# <a name="integrate-3270-screen-driven-apps-on-ibm-mainframes-with-azure-by-using-azure-logic-apps-and-ibm-3270-connector"></a>使用 Azure Logic Apps 和 IBM 3270 連接器，整合 IBM 大型主機上的 3270 螢幕驅動應用程式與 Azure

> [!NOTE]
> 此連接器目前為 [*公開預覽*](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)狀態。 

使用 Azure Logic Apps 和 IBM 3270 連接器，您可以透過瀏覽 3270 模擬器畫面，來存取及執行通常會驅動的 IBM 大型主機應用程式。 如此一來，您就可以使用 Azure Logic Apps 建立自動化工作流程，將您的 IBM 大型主機應用程式與 Azure、Microsoft 和其他應用程式、服務和系統整合在一起。 連接器會使用 TN3270 通訊協定與 IBM 大型主機通訊，此通訊協定在 Azure Government 和 Azure China 21Vianet 以外的所有 Azure Logic Apps 區域中均有提供。 如果您不熟悉邏輯應用程式，請檢閱[什麼是 Azure Logic Apps？](../logic-apps/logic-apps-overview.md)

本文說明使用3270連接器的各方面： 

* 為何要在 Azure Logic Apps 中使用 IBM 3270 連接器，以及連接器如何執行3270螢幕驅動應用程式

* 使用3270連接器的必要條件和設定

* 將3270連接器動作新增至邏輯應用程式的步驟

## <a name="why-use-this-connector"></a>為何要使用此連接器？

若要存取 IBM 大型主機上的應用程式，您通常會使用3270終端機模擬器，通常稱為「綠色螢幕」。 這個方法是經過時間測試的方式，但是有一些限制。 雖然 Host Integration Server () 可協助您直接使用這些應用程式，但有時可能無法將螢幕和商務邏輯分開。 或者，您可能不再有主機應用程式運作方式的相關資訊。

為了延伸這些案例，Azure Logic Apps 中的 IBM 3270 連接器可搭配使用3270設計工具（用於記錄或「捕捉」）用於特定工作的主機畫面、透過您的大型主機應用程式定義該工作的流覽流程，以及使用該工作的輸入和輸出參數定義方法。 設計工具會將該資訊轉換成3270連接器從邏輯應用程式呼叫代表該工作的動作時，所使用的中繼資料。

從設計工具產生中繼資料檔案之後，請將該檔案新增至 Azure 中的整合帳戶。 如此一來，您的邏輯應用程式就可以在新增3270連接器動作時存取您應用程式的中繼資料。 連接器會從您的整合帳戶讀取中繼資料檔案、處理整個3270畫面的導覽，並以動態方式呈現3270連接器動作的參數。 然後，您可以將資料提供給主應用程式，而連接器會將結果傳回給您的邏輯應用程式。 如此一來，您就可以將繼承應用程式與 Azure Logic Apps 支援的 Azure、Microsoft 和其他應用程式、服務和系統整合。

## <a name="prerequisites"></a>Prerequisites

* Azure 訂用帳戶。 如果您沒有 Azure 訂用帳戶，請先[註冊免費的 Azure 帳戶](https://azure.microsoft.com/free/)。

* [如何建立邏輯應用程式](../logic-apps/quickstart-create-first-logic-app-workflow.md)的基本知識

* 建議： [integration service 環境 (ISE) ](../logic-apps/connect-virtual-network-vnet-isolated-environment.md) 

  您可以選取此環境作為建立和執行邏輯應用程式的位置。 ISE 可讓您從邏輯應用程式存取 Azure 虛擬網路內受保護的資源。

* 用來自動化及執行您3270螢幕驅動應用程式的邏輯應用程式

  IBM 3270 連接器沒有觸發程式，因此請使用其他觸發程式來啟動邏輯應用程式，例如 **迴圈** 觸發程式。 然後，您可以新增3270連接器動作。 若要開始使用，請 [建立空白的邏輯應用程式](../logic-apps/quickstart-create-first-logic-app-workflow.md)。 
  如果您使用 ISE，請選取該 ISE 作為邏輯應用程式的位置。

* [下載並安裝3270設計工具](https://aka.ms/3270-design-tool-download)。
唯一的先決條件是 [Microsoft .NET Framework 4.6.1](https://aka.ms/net-framework-download)。

  這項工具可協助您在應用程式中，針對您新增並以3270連接器動作來執行的工作，記錄畫面、流覽路徑、方法和參數。 此工具會產生 Host Integration Designer XML (HIDX) 檔案，該檔案會提供必要的中繼資料，以供連接器用來推動您的大型主機應用程式。
  
  下載並安裝此工具之後，請遵循下列步驟來連接至您的主機：

  1. 開啟3270設計工具。 從 [ **會話** ] 功能表選取 [ **主會話**]。
  
  1. 提供您的 TN3270 主機伺服器資訊。

* [整合帳戶](../logic-apps/logic-apps-enterprise-integration-create-integration-account.md)，也就是您將 HIDX 檔案儲存為對應的位置，讓您的邏輯應用程式可以存取該檔案中的中繼資料和方法定義。 

  請確定您的整合帳戶已連結至您所使用的邏輯應用程式。 此外，如果您使用 ISE，請確定您的整合帳戶位置與邏輯應用程式所使用的 ISE 相同。

* 存取裝載您的大型主機應用程式的 TN3270 伺服器

<a name="define-app-metadata"></a>

## <a name="create-metadata-overview"></a>建立中繼資料總覽

在3270螢幕驅動的應用程式中，畫面和資料欄位對您的案例而言是唯一的，因此3270連接器需要您應用程式的相關資訊，您可以將其提供做為中繼資料。 此中繼資料描述可協助邏輯應用程式識別和辨識畫面的資訊，描述如何在畫面之間流覽、輸入資料的位置，以及預期的結果。 若要指定並產生此中繼資料，您可以使用3270設計工具，它會逐步引導您完成這些特定 *模式*或階段，如稍後的詳細資料所述：

* **Capture**：在此模式中，您會記錄使用您的大型主機應用程式完成特定工作所需的畫面，例如，取得銀行餘額。

* **流覽**：在此模式中，您會指定要如何流覽整個大型主機應用程式畫面以進行特定工作的方案或路徑。

* **方法**：在此模式中，您會定義 `GetBalance` 描述螢幕導覽路徑的方法，例如。 您也可以選擇每個畫面上的欄位，以成為方法的輸入和輸出參數。

### <a name="unsupported-elements"></a>不支援的元素

設計工具不支援下列元素：

* 部分 IBM 基本對應支援 (BMS) 對應：如果您匯入 BMS 對應，設計工具會忽略部分螢幕定義。
* In/Out 參數：您無法定義 In/Out 參數。
* 功能表處理：預覽期間不支援
* 陣列處理：預覽期間不支援

<a name="capture-screens"></a>

## <a name="capture-screens"></a>捕捉畫面

在此模式中，您會在每個3270畫面上標示一個可唯一識別該畫面的專案。 例如，您可以指定一行文字或更複雜的條件集合，例如特定文字和非空白欄位。 您可以透過與主機伺服器的即時連線來記錄這些畫面，或從 IBM 基本對應支援 (BMS) 對應中匯入這項資訊。 即時連線會使用 TN3270 模擬器來連接到主機。 每個連接器動作都必須對應至一項工作，該工作會從連接到您的會話開始，並在中斷與會話的連線後結束。

1. 如果您還沒有這麼做，請開啟3270設計工具。 在工具列上，選擇 [ **capture** ]，讓您進入 [capture] 模式。

1. 若要開始錄製，請按 F5 鍵，或從 [ **錄製** ] 功能表中，選取 [ **開始錄製**]。 

1. 從 [ **會話** ] 功能表選取 **[連接]**。

1. 在 [ **Capture** ] 窗格中，從您應用程式中的第一個畫面開始，逐步執行您要錄製之特定工作的應用程式。

1. 完成工作之後，請登出您的應用程式，就像平常一樣。

1. 從 [ **會話** ] 功能表選取 **[中斷連線]**。

1. 若要停止錄製，請按 Shift + F5 鍵，或從 [ **錄製** ] 功能表中選取 [ **停止錄製**]。

   在您捕獲工作的畫面之後，設計工具會顯示代表這些畫面的縮圖。 這些縮圖的一些注意事項：

   * 包含在您的已捕捉畫面中，您有一個名為 "Empty" 的畫面。

     當您第一次連線到 [CICS](https://www.ibm.com/it-infrastructure/z/cics)時，必須先傳送「清除」金鑰，才能輸入您想要執行之交易的名稱。 您傳送「清除」索引鍵的畫面沒有任何辨識 *屬性*，例如，您可以使用螢幕辨識編輯器來新增的螢幕標題。 若要代表這個畫面，縮圖包含名為 "Empty" 的畫面。 您稍後可以使用此畫面來代表您輸入交易名稱的畫面。

   * 根據預設，所捕捉畫面的名稱會使用畫面上的第一個單字。 如果該名稱已存在，則設計工具會以底線和數位附加名稱，例如 "WBGB" 和 "WBGB_1"。

1. 若要為已捕捉的畫面提供更有意義的名稱，請遵循下列步驟：

   1. 在 [ **主機畫面** ] 窗格中，選取您要重新命名的畫面。

   1. 在相同窗格中的相同窗格中，尋找靠近相同窗格底部的 [ **螢幕名稱** ] 屬性。

   1. 將目前的畫面名稱變更為更具描述性的名稱。

1. 現在指定用來識別每個畫面的欄位。

   使用3270資料流程時，畫面沒有預設的識別碼，因此您必須在每個畫面上選取唯一的文字。 在複雜的情況下，您可以指定多個條件，例如唯一的文字，以及具有特定條件的欄位。

當您完成選取辨識欄位之後，請移至下一個模式。

### <a name="conditions-for-identifying-repeated-screens"></a>識別重複畫面的條件

若要讓連接器流覽和區別畫面，您通常會在螢幕上找到唯一的文字，您可以在此畫面上用來作為已捕捉畫面之間的識別碼。 針對重複的畫面，您可能需要更多的識別方法。 例如，假設您有兩個看起來相同的畫面，但一個畫面傳回有效的值，另一個畫面則傳回錯誤訊息。

在設計工具中，您可以使用螢幕辨識編輯器來新增辨識 *屬性*，例如「取得帳戶餘額」之類的畫面標題。 如果您有分支路徑，而且這兩個分支都傳回相同的畫面，但結果不同，則您需要其他辨識屬性。 在執行時間，連接器會使用這些屬性來判斷最新分支和分支。 以下是您可以使用的條件：

* 特定值：這個值符合指定位置的指定字串。
* 不是特定值：此值不符合指定位置的指定字串。
* 空白：此欄位是空的。
* 不是空的：此欄位不是空的。

若要深入瞭解，請參閱本主題稍後的 [範例導覽計畫](#example-plan) 。

<a name="define-navigation"></a>

## <a name="define-navigation-plans"></a>定義流覽計畫

在此模式中，您可以定義流程，或針對您的特定工作流覽您的大型主機應用程式畫面的步驟。 例如，有時候，您的應用程式可能會有一個以上的路徑，讓某個路徑產生正確的結果，而另一個路徑產生錯誤。 針對每個畫面，指定移至下一個畫面所需的按鍵，例如 `CICSPROD <enter>` 。

> [!TIP]
> 如果您要將使用相同 [連接] 和 [中斷連接] 畫面的數個工作自動化，設計工具會提供特殊的連接和中斷連接計畫類型。 當您定義這些方案時，您可以將它們新增至流覽計畫的開頭和結尾。

### <a name="guidelines-for-plan-definitions"></a>計畫定義的指導方針

* 包含所有畫面，從連線開始並結束中斷連接。

* 您可以建立獨立的方案，或使用連接和中斷連接計畫，讓您重複使用一系列的畫面來處理所有交易。

  * Connect 方案中的最後一個畫面必須與您導覽計畫中第一個畫面的畫面相同。

  * 中斷連線計畫中的第一個畫面，必須與流覽計畫中的最後一個畫面相同。

* 您所捕獲的螢幕可能包含許多重複的畫面，因此請在您的方案中選取並只使用任何重複畫面的一個實例。 以下是重複畫面的一些範例：

  * 登入畫面，例如， **MSG-10** 畫面
  * CICS 的歡迎畫面
  * 「清除」或 **空白** 畫面

<a name="create-plans"></a>

### <a name="create-plans"></a>建立方案

1. 在3270設計工具的工具列上，選擇 [ **流覽** ]，讓您進入瀏覽模式。

1. 若要開始您的方案，請在 **流覽** 窗格中選擇 [ **新增方案**]。

1. 在 **[選擇新方案名稱**] 底下，輸入方案的名稱。 從 [ **類型** ] 清單中選取 [方案類型]：

   | 方案類型 | 說明 |
   |-----------|-------------|
   | **處理** | 適用于獨立或合併的計畫 |
   | **[連接]** | 適用于 Connect 方案 |
   | **斷開** | 針對中斷連線計畫 |
   |||

1. 從 **主畫面** 窗格，將已捕捉的縮圖拖曳至 **流覽** 窗格中的導覽計畫介面。

   若要代表您輸入交易名稱的空白畫面，請使用「空白」畫面。

1. 以描述您正在定義之工作的順序排列畫面。

1. 若要定義畫面之間的流程路徑（包括分支和聯結），請在設計工具的工具列上，選擇 [ **流程**]。

1. 選擇流程中的第一個畫面。 將連接拖曳至流程中的下一個畫面並進行繪製。

1. 針對每個畫面，提供 **輔助按鍵** 屬性的值 (注意識別碼) 以及將流程移至下一個畫面的 **固定文本** 屬性。

   您可能只會有輔助按鍵，或是輔助按鍵和固定文字。

完成流覽計畫之後，您可以 [在下一個模式中定義方法](#define-method)。

#### <a name="example-identify-repeated-screens"></a>範例：識別重複畫面

若要讓連接器流覽和區別畫面，您通常會在螢幕上找到唯一的文字，您可以在整個捕獲的畫面上，用來做為識別碼。 針對重複的畫面，您可能需要更多的識別方法。 範例方案有一個分支，您可以在其中取得看起來類似的畫面。 一個畫面會傳回帳戶餘額，而另一個畫面會傳回錯誤訊息。

設計工具可讓您使用螢幕辨識編輯器來新增辨識屬性，例如名為「取得帳戶平衡」的畫面標題。 在類似畫面的情況下，您需要其他屬性。 在執行時間，連接器會使用這些屬性來決定分支和分叉。

* 在傳回有效輸入的分支中，也就是具有帳戶餘額的畫面，您可以新增具有「非空白」條件的欄位。

* 在傳回錯誤的分支中，您可以新增具有「空白」條件的欄位。

<a name="define-method"></a>

## <a name="define-methods"></a>定義方法

在此模式中，您會定義與您的流覽計畫相關聯的方法。 針對每個方法參數，您可以指定資料類型，例如字串、整數、日期或時間等等。 當您完成時，您可以在即時主機上測試方法，並確認該方法如預期般運作。 然後，您會產生中繼資料檔案或 Host Integration Designer XML (HIDX) 檔案，此檔案現在有方法定義可用來建立和執行 IBM 3270 連接器的動作。

1. 在3270設計工具的工具列上，選擇 [ **方法** ]，讓您輸入方法模式。 

1. 在 **流覽** 窗格中，選取具有您想要之輸入欄位的畫面。

1. 若要為您的方法新增第一個輸入參數，請遵循下列步驟：

   1. 在 [ **捕捉** ] 窗格的 [3270 模擬器] 畫面上，選擇您想要做為第一個輸入的整個欄位，而不只是欄位內的文字。

      > [!TIP]
      > 若要顯示所有欄位，並確定您選取 [完整] 欄位，請在 [ **View** ] 功能表上選取 [ **所有欄位**]。

   1. 在設計工具的工具列上，選擇 [ **輸入欄位**]。 

   若要新增更多輸入參數，請針對每個參數重複上述步驟。

1. 若要為您的方法新增第一個輸出參數，請遵循下列步驟：

   1. 在 [ **捕捉** ] 窗格的 [3270 模擬器] 畫面上，選擇您想要做為第一個輸出的整個欄位，而不只是欄位內的文字。

      > [!TIP]

   若要加入更多輸出參數，請針對每個參數重複上述步驟。

1. 在您新增所有方法的參數之後，請為每個參數定義這些屬性：

   | 屬性名稱 | 可能值 | 
   |---------------|-----------------|
   | **資料類型** | Byte、Date Time、Decimal、Int、Long、Short、String |
   | **欄位填滿技術** | 參數支援這些填滿類型，如有必要，請填滿空白： <p><p>- **類型**：依序在欄位中輸入字元。 <p>- **填滿**：以字元取代欄位的內容，必要時以空白填滿。 <p>- **EraseEofType**：清除欄位，然後將字元依序輸入到欄位中。 |
   | **格式字串** | 某些參數資料類型會使用格式字串，此字串會通知3270連接器如何將畫面中的文字轉換成 .NET 資料類型： <p><p>- **Datetime**： datetime 格式字串會遵循 [.net 自訂日期和時間格式字串](/dotnet/standard/base-types/custom-date-and-time-format-strings)。 例如，日期會 `06/30/2019` 使用格式字串 `MM/dd/yyyy` 。 <p>- **Decimal**： decimal 格式字串使用 [COBOL Picture 子句](https://www.ibm.com/support/knowledgecenter/SS6SG3_5.2.0/com.ibm.cobol52.ent.doc/PGandLR/ref/rlddepic.html)。 例如，數位 `100.35` 使用格式字串 `999V99` 。 |

## <a name="save-and-view-metadata"></a>儲存和查看中繼資料

在您定義方法之後，但在測試方法之前，請將您到目前為止所定義的所有資訊儲存至 RAP ( 的 rap) 檔。
在任何模式中，您都可以隨時儲存至此 RAP 檔案。 設計工具也包含範例 RAP 檔案，您可以在此位置流覽至設計工具的安裝資料夾，並開啟 "Microsoft.samples.biztalk.woodgrovebank.stubpendingtransactions" 檔案，以進行開啟和審核：

`..\Program Files\Microsoft Host Integration Server - 3270 Design Tool\SDK\WoodgroveBank.rap`

但是，如果您嘗試將變更儲存至範例 RAP 檔案，或從範例 RAP 檔案產生 HIDX 檔案，但檔案仍在設計工具的安裝資料夾中，您可能會收到「拒絕存取」錯誤。 依預設，設計工具會在 [Program Files] 資料夾中安裝，而不會有較高的許可權。 如果您收到錯誤，請嘗試下列其中一個解決方案：

* 將範例檔案複製到不同的位置。
* 以系統管理員身分執行設計工具。
* 讓自己成為 SDK 資料夾的擁有者。

## <a name="generate-and-upload-hidx-file"></a>產生並上傳 HIDX 檔案

當您準備好時，請產生 HIDX 檔案，以便您可以上傳至整合帳戶。 3270設計工具會在您儲存 RAP 檔案的新子資料夾中建立 HIDX 檔案。

1. 在3270設計工具的工具列上，選擇 [ **產生程式碼**]。

1. 移至包含您的 RAP 檔案的資料夾，然後開啟工具在產生 HIDX 檔案之後所建立的子資料夾。 確認工具已建立 HIDX 檔。

1. 登入 [Azure 入口網站](https://portal.azure.com)，並尋找您的整合帳戶。

1. [遵循下列類似步驟來新增對應](../logic-apps/logic-apps-enterprise-integration-liquid-transform.md)，以將 HIDX 檔案新增至整合帳戶的對應，但當您選取對應類型時，請選取 [ **HIDX**]。

稍後在本主題中，當您第一次將 IBM 3270 動作新增至邏輯應用程式時，系統會提示您提供連接資訊（例如您的整合帳戶和主機伺服器的名稱），以在邏輯應用程式與主機伺服器之間建立連線。 建立連線之後，您可以選取先前新增的 HIDX 檔、要執行的方法，以及要使用的參數。

當您完成所有這些步驟時，您可以使用您在邏輯應用程式中建立的動作來連線至 IBM 大型主機、為您的應用程式驅動畫面、輸入資料、傳回結果等等。 您也可以繼續將其他動作新增至邏輯應用程式，以便與其他應用程式、服務和系統整合。
