# <a name="ibm-workloads-on-azure"></a>Azure 上的 IBM 工作負荷

許多基於 z/OS 的 IBM 大型機工作負載可以在 Azure 中複製，而不會丟失功能，使用者甚至不會注意到其基礎系統中的更改。 在 Azure 上重新託管應用程式可為您提供所需的類似大型機的功能，以及雲的彈性、可用性和潛在成本節約。

Azure 支援與現有 IBM 大型機環境集成，使您能夠遷移有意義的應用項，根據需要運行混合解決方案，並隨著時間的推移進行遷移。 儘管您可以完全重寫 Azure 的現有基於大型機的程式，但重新託管它們更為常見。 重寫會增加遷移專案的成本、複雜性和時間。 通過重新託管，您可以：

- 將應用程式移動到基於雲的模擬器。

- 將資料庫移轉到基於雲的資料庫。

- 使用代碼轉換引擎替換模組和代碼。

此外，IBM 軟體（包括 WebSphere 和 MQ）現在位於 Azure 應用商店中。 借助 IBM 軟體許可證，可以利用 Azure 提供的按需基礎結構擴展快速啟動虛擬機器。

廣泛的合作夥伴生態系統可説明您將 IBM 大型機系統移轉到 Azure。 在開始分階段部署重寫或替換應用程式之前，大多數公司會盡可能採用務實的重用方法。 在[Azure 大型機遷移中心](https://azure.microsoft.com/migration/mainframe/)從合作夥伴處獲取更多指導和説明。
