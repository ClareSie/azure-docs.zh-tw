# <a name="power-bi-app-offer-prerequisites"></a>Power BI 應用程式供應項目必要條件

本文提供了在將 Power BI 應用產品/服務發佈到 AppSource 市場之前需要滿足的技術性和業務需求。  如果尚未這樣做，請查看[Office 365、動態 365、PowerApps 和 Power BI 產品發佈指南](../../appsource-offer-publishing-guide.md)。


## <a name="technical-requirements"></a>技術需求

此產品/服務類型需要的主要技術資產是[Power BI 應用程式](https://go.microsoft.com/fwlink/?linkid=2028636)。 如需詳細資訊，請參閱[建立 Power BI 技術資產](./cpp-create-technical-assets.md)。

在 Power BI 中創建和測試服務應用程式後，應保存 Power BI 生成的應用程式安裝 URL。 您需要它來[創建 Power BI 應用產品/](./cpp-create-offer.md)

## <a name="business-requirements"></a>商業需求

業務需求包括程式、合同和法律義務。  您必須：

- 成為註冊的雲市場發行者。 如果您尚未註冊，請按照["成為雲市場發行者](https://docs.microsoft.com/azure/marketplace/become-publisher)"中的步驟操作。

    > [!NOTE] 
    > 使用相同的 Microsoft 開發人員中心註冊帳戶登錄到[雲合作夥伴門戶](https://cloudpartner.azure.com)。 您的 AppSource 供應項目應該只會共有一個 Microsoft 帳戶。 帳戶不應該適用於個別服務或供應項目。
 
- 以合乎商業行為的方式為客戶提供技術支援。
- 提供符合在 AppSource 上列出的產品標準的內容。 有關詳細資訊，請參閱在[AppSource 上列出應用？下面是如何](https://appsource.microsoft.com/blogs/have-an-app-to-list-on-appsource-here-s-how)。
- 遵守[微軟隱私權聲明](https://privacy.microsoft.com/privacystatement)。


## <a name="next-steps"></a>後續步驟

滿足所有要求後，即可在雲合作夥伴門戶中[創建 Power BI 應用產品/](./cpp-create-offer.md)
