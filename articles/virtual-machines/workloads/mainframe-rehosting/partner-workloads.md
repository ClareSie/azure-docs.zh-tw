# <a name="mainframe-workloads-supported-on-azure"></a>Azure 上支援的大型機工作負載

此處列出的解決方案可説明您將大型機工作負荷遷移到 Microsoft Azure。 某些工作負載比較容易移轉。 可以重新託管依賴于舊系統軟體的其他工作負載。 

有關大型機模擬和服務的更多説明，請參閱[Azure 大型機遷移中心](https://azure.microsoft.com/migration/mainframe/)。

## <a name="migrate-mainframe-closer-to-azure"></a>將大型機遷移到更靠近 Azure 的位置

- [微聚焦視覺COBOL](https://www.microfocus.com/products/visual-cobol/)開發和集成工具。
- 用於 .NET 平臺的[Micro Focus PL/I](https://www.microfocus.com/campaign/download/pli-modernization/)舊編譯器，支援大型機 PL/I 語法、資料類型和行為。
- [微聚焦企業伺服器](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)主機集成平臺。
- [富士通NetCOBOL](https://www.fujitsu.com/global/products/software/developer-tool/netcobol/)開發和集成工具。
- [NTT 資料企業 COBOL](https://us.nttdata.com/en/digital/application-development-and-modernization)開發和集成工具。
- NTT 為 .NET 平臺[打開 PL/I](https://us.nttdata.com/en/digital/application-development-and-modernization)舊編譯器，支援大型機 PL/I 語法、資料類型和行為。
- [雨碼COBOL編譯器](https://www.raincode.com/products/cobol/)開發和集成工具。
- .NET 平臺的[Raincode PL/I 編譯器](https://www.raincode.com/products/pli/)支援大型機 PL/I 語法、資料類型和行為。
- 用於大型機彙編器 370 和 HLASM 語法的[雨碼 ASM370 編譯器](https://www.raincode.com/technical-landscape/asm370/)。
- 通過 Visual Studio 外掛程式為 .NET 框架的 .NET 編譯器的[ASNA Visual RPG。](https://asna.com/us/products/visual-rpg)
- [現代系統 CTU（COBOL-對通用）](https://modernsystems.com/automatic-cobol-to-java-conversion/)開發和集成工具。

## <a name="deploy-an-emulation-environment-for-online-and-batch-processing"></a>部署用於連線和批次處理的模擬環境

- [微聚焦企業伺服器](https://www.microfocus.com/products/enterprise-suite/enterprise-server/)主機集成平臺。
- [NTT DATA 主機重新託管開發環境](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf)本機交易處理環境。
- [NTT DATA 批次處理環境](https://us.nttdata.com/en/-/media/assets/white-paper/apps-mainframe-re-hosting-development-environment-whitepaper.pdf)（BPE），包括 JCL 事務功能。
- 用於 .NET 和 Azure 平臺的[雨碼 CICS](https://www.raincode.com/technical-landscape/cics/)模擬器。
- [雨碼 JCL](https://www.raincode.com/products/jcl/)插頭相容 JCL 解譯器。

## <a name="code-conversion"></a>代碼轉換

- [Asysco](https://www.asysco.com/azure-cloud/)原始程式碼轉換。
- [LzLabs 軟體定義大型機](https://www.lzlabs.com/)管理軟體容器，用於將大型機應用程式遷移到 Linux 電腦或專用、公共和混合雲環境。
- 用於數位化傳統商務應用程式和資料庫的[藍光時代](https://www.bluage.com/)工具。
- [海寶計算](https://www.heirloomcomputing.com/tag/convert-cobol-to-java/)服務可將主機 COBOL、CICS 和 VSAM 轉換為 JAVA。

Microsoft 與全球系統集成商 （GSIs） 合作，他們可以説明大型組織設計、構建和管理解決方案。
