# <a name="what-is-azure-service-health"></a>什麼是 Azure 服務健康狀態？

Azure 提供了一組可讓您隨時掌握雲端資源健康情況的體驗。 這些資訊包括目前和未來可能發生的問題，例如服務影響事件、計劃性維護，和其他可能影響可用性的各項變更。

Azure 服務健康狀態由三個不同的小型服務所組成。

[Azure 狀態](azure-status-overview.md)會在 **[Azure 狀態頁面](https://status.azure.com)** 上通知您有 Azure 服務中斷的狀況。 您可以在此頁面上通盤檢視所有 Azure 服務在各個 Azure 區域的健康情況。 您可以參考狀態頁面，了解事件所造成的影響，但我們強烈建議目前的 Azure 使用者利用 Azure 服務健康狀態，以便及時收到關於 Azure 事件和維護的各項通知。

[服務健康狀態](service-health-overview.md)會提供個人化檢視，讓您了解所用 Azure 服務和區域的健康情況。 由於受驗證的服務健康狀態體驗知道您目前使用的服務及資源，因此您可以在此尋找各項受影響服務的通訊，包括服務中斷、計劃性維護活動以及其健康情況諮詢。 您可以設定服務健康狀態警示，以充分利用服務健康狀態的功能，並透過慣用的通訊管道收到關於服務問題、計劃性維護或其他可能影響 Azure 服務和使用區域之變更的相關資訊。

[資源健康狀態](resource-health-overview.md)提供個別雲端資源的健康情況資訊，例如特定的虛擬機器。 使用 Azure 監視器，您也可以設定警示，以便在雲端資源可用性變更時收到通知。 資源健康狀態搭配 Azure 監視器通知會協助您及時收到資源可用性的相關通知，並快速評估問題根源是出自於您，或與 Azure 平台事件相關。

這些經驗值加在一起，會提供您更全面細微且最相關的 Azure 健康情況檢視。
