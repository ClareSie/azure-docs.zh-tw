# <a name="chat-client-library-overview"></a>聊天用戶端程式庫概觀

[!INCLUDE [Public Preview Notice](../../includes/public-preview-include.md)]

您可以使用 Azure 通訊服務聊天用戶端程式庫，將豐富的即時文字聊天新增至您的應用程式。

## <a name="chat-client-library-capabilities"></a>聊天用戶端程式庫功能

下列清單是目前通訊服務聊天用戶端程式庫中可用的一組功能。

| 功能群組 | 功能                                                                                                          | JS  | Java | .NET | Python |
| ----------------- | ------------------------------------------------------------------------------------------------------------------- | --- | ----- | ---- | -----  |
| 核心功能 | 建立 2 個或多個使用者之間的聊天對話 (最多 250 個使用者)                                                       | ✔️   | ✔️  | ✔️    | ✔️   |
|                   | 更新聊天對話的主題                                                                              | ✔️   | ✔️ | ✔️    | ✔️   |
|                   | 在聊天對話中新增或移除成員                                                                           | ✔️   | ✔️  | ✔️    | ✔️  |
|                   | 選擇是否要與新加入的成員共用聊天訊息記錄 - 全部/無/到特定時間點為止 | ✔️   | ✔️   | ✔️    | ✔️  |
|                   | 取得所有聊天成員對話的清單                                                                          | ✔️   | ✔️  | ✔️ | ✔️ |
|                   | 刪除聊天對話                                                                                              | ✔️   | ✔️  | ✔️    | ✔️  |
|                   | 取得使用者聊天對話成員資格的清單                                                                  | ✔️   | ✔️  | ✔️    | ✔️  |
|                   | 取得特定聊天對話的資訊                                                                              | ✔️   | ✔️  | ✔️ | ✔️ |
|                   | 在聊天對話中傳送和接收訊息                                                                            | ✔️   | ✔️   | ✔️    | ✔️  |
|                   | 編輯已傳送出去的訊息內容                                                                   | ✔️   | ✔️  | ✔️ | ✔️ |
|                   | 刪除訊息                                                                                                       | ✔️   | ✔️  | ✔️ | ✔️ |
|                   | 在傳送時將訊息標示為一般或高優先順序                                               | ✔️   | ✔️  | ✔️    | ✔️   |
|                   | 傳送及接收成員已讀取訊息時的讀取回條 <br/> 聊天對話中的成員超過 20 位時無法使用    | ✔️   | ✔️  | ✔️    | ✔️   |
|                   | 當成員正在聊天對話中輸入訊息時，傳送和接收輸入通知 <br/> 聊天對話中的成員超過 20 位時無法使用      | ✔️   | ✔️   | ✔️    | ✔️    |
|                   | 取得聊天對話中的所有訊息 <br/> 支援 Unicode 表情符號                                                  | ✔️   | ✔️  | ✔️    | ✔️  |
|                   | 傳送表情符號作為訊息內容的一部分                                                                              | ✔️   | ✔️  | ✔️    | ✔️  |
|即時信號 (由專屬的信號套件啟用)| 當使用者在其所屬的聊天對話中收到新訊息時，可收到通知                                     | ✔️   | ❌    | ❌  | ❌  |
|                    | 當所屬聊天對話中的另一個成員編輯訊息時，可收到通知                | ✔️   | ❌    | ❌    | ❌  |
|                    | 當所屬聊天對話中的另一個成員刪除訊息時，可收到通知                | ✔️   | ❌    | ❌    | ❌  |
|                    | 當另一個聊天對話成員正在輸入文字時，可收到通知                                                             | ✔️   | ❌    | ❌    | ❌  |
|                    | 當聊天對話中的另一個成員讀取訊息時，可收到通知 (讀取回條)                               | ✔️   | ❌    | ❌    | ❌  |
| 事件             | 使用事件方格來訂閱聊天對話中發生的使用者活動，以及整合自訂通知服務或商務邏輯     | ✔️   | ✔️  | ✔️    | ✔️  |
| 監視        | 根據傳送的訊息監視使用狀況                                                                               | ✔️   | ✔️  | ✔️    | ✔️  |
|                    | 監視應用程式所提 API 要求的品質和狀態，並透過入口網站設定警示                                                          | ✔️   | ✔️  | ✔️    | ✔️  |
|其他功能 | 使用[認知服務 API](https://docs.microsoft.com/azure/cognitive-services/) 及聊天用戶端程式庫來啟用智慧型功能 - 用戶端上傳入訊息的語言翻譯及情感分析、在成員說話時以語音轉換文字撰寫訊息等等。                                                                                         | ✔️   | ✔️  | ✔️    | ✔️  |
