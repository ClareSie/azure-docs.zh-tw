# <a name="machine-learning-features"></a>機器學習功能

在機器學習中， *功能*   是您的系統觀察及學習之資料的區別特性或屬性。

機器學習功能提供 LUIS 重要的提示，讓您在何處尋找可區分概念的專案。 這些是 LUIS 可以使用的提示，但它們不是困難的規則。 LUIS 會將這些提示與標籤搭配使用，以尋找資料。

功能可以描述為函式，例如 f （x） = y。 在範例語句中，此功能會告訴您要在何處尋找區別的特性。 使用此資訊來協助建立您的架構。

## <a name="types-of-features"></a>功能類型

LUIS 支援片語清單和模型做為特徵：

* 片語清單功能 
* 模型（意圖或實體）做為功能

應該將功能視為架構設計的必要部分。

## <a name="find-features-in-your-example-utterances"></a>在您的範例語句中尋找功能

因為 LUIS 是以語言為基礎的應用程式，所以這些功能是以文字為基礎。 選擇文字，指出您想要區別的特性。 針對 LUIS，最小單位是*token*。 對於英文語言，權杖是連續的字母和數位，不含空格或標點符號。

因為空格和標點符號不是標記，所以請將焦點放在您可以用來做為特徵的文字線索上。 請記得包含單字的變化，例如：

* 複數形式
* 動詞時態
* 縮寫
* 拼寫與拼寫錯誤

判斷文字是否區分特性，必須：

* 符合完全相同的單字或片語：考慮將正則運算式實體或清單實體新增為實體或意圖的功能。
* 比對知名概念，例如日期、時間或人員名稱：使用預先建立的實體做為實體或意圖的功能。
* 瞭解一段時間的新範例：使用概念的一些範例的片語清單，做為實體或意圖的功能。

## <a name="combine-features"></a>結合功能

您可以使用一個以上的功能來描述特徵或概念。 常見的配對是使用片語清單功能，以及常用來做為功能的實體類型：

 * 預先建立的實體
 * 正則運算式實體
 * 清單實體

### <a name="ticket-booking-entity-example"></a>票證預約實體範例

在第一個範例中，請考慮使用航班保留意圖和票證預約實體來預約航班的應用程式。

「票證預約」實體是航班目的地的機器學習實體。 若要協助解壓縮該位置，請使用兩個功能來協助：

* 相關單字的片語清單，例如、**平面**、**航班**、**預約**或**票證**
* 預先建立的**geographyV2**實體，做為實體的功能

### <a name="pizza-entity-example"></a>比薩實體範例

另舉一個例子，請考慮一個應用程式來訂購具有建立比薩訂單意圖和比薩實體的比薩餅。

比薩實體是適用于比薩詳細資料的機器學習實體。 若要協助解壓縮詳細資料，請使用兩個功能來協助：

* 相關單字的片語清單，例如，**乳酪**、 **crust**、 **pepperoni**或**pineapple**
* 預先建立的**數位**實體，做為實體的功能

## <a name="create-a-phrase-list-for-a-concept"></a>建立概念的片語清單

片語清單是描述概念的單字或片語清單。 在權杖層級上，片語清單會套用為不區分大小寫的相符項。

新增片語清單時，您可以將此功能設定為**[全域](#global-features)**。 全域功能適用于整個應用程式。

### <a name="when-to-use-a-phrase-list"></a>使用片語清單的時機

當您需要 LUIS 應用程式來一般化並識別概念的新專案時，請使用片語清單。 片語清單就像是特定領域的詞彙。 它們會增強對意圖和實體的瞭解品質。

### <a name="how-to-use-a-phrase-list"></a>如何使用片語清單

在片語清單中，LUIS 會考慮 coNtext 和一般化來識別類似的專案，但不會有完全相符的文字。 請遵循下列步驟以使用片語清單：

1. 從機器學習實體開始：
    1. 新增範例語句。
    1. 具有機器學習服務實體的標籤。
1. 新增片語清單：
    1. 新增具有類似意義的文字。 請勿新增每個可能的單字或片語。 請改為一次加入幾個單字或片語。 然後重新定型和發佈。
    1. 審查並新增建議的文字。

### <a name="a-typical-scenario-for-a-phrase-list"></a>片語清單的一般案例

片語清單的一般案例是要提升與特定概念相關的單字。

醫療術語是一個很好的範例，可能需要片語清單來提升其重要性。 這些詞彙可以具有特定的實體、化學、ekko-wave 治療或抽象意義。 LUIS 不會知道詞彙對您的主旨網域而言很重要，而沒有片語清單。

若要將醫療詞彙解壓縮：

1. 建立範例語句，並在這些語句中為醫療詞彙加上標籤。
2. 建立包含主旨網域內條款範例的片語清單。 此片語清單應包含您標示的實際字詞，以及描述相同概念的其他詞彙。
3. 將片語清單新增至實體或列，以解壓縮片語清單中所使用的概念。 最常見的案例是機器學習實體的元件（子系）。 如果片語清單應套用到所有意圖或實體，請將片語清單標記為全域片語清單。 **EnabledForAllModels**旗標會控制 API 中的此模型範圍。

## <a name="a-model-as-a-feature-helps-another-model"></a>當做功能的模型可協助另一個模型

您可以將模型（意圖或實體）當做功能加入另一個模型（意圖或實體）。 藉由加入現有的意圖或實體做為功能，您就可以新增已加上標籤範例的定義完善概念。

將模型新增為功能時，您可以將此功能設定為：
* **[必要](#required-features)**。 必須找到必要的功能，才能從預測端點傳回模型。
* **[全域](#global-features)**。 全域功能適用于整個應用程式。

### <a name="when-to-use-an-entity-as-a-feature-to-an-intent"></a>何時使用實體做為意圖的功能

當實體的偵測對意圖而言很重要時，請將實體當做功能新增至意圖。

例如，如果目的是用來預約航班（例如**BookFlight**），而實體是票證資訊（如基座數目、來源和目的地），則尋找票證資訊實體應該會在**BookFlight**意圖的預測中增加相當的權數。

### <a name="when-to-use-an-entity-as-a-feature-to-another-entity"></a>使用實體做為另一個實體之功能的時機

當實體（A）的偵測對實體（B）的預測很重要時，應該將實體（A）當做功能加入另一個實體（B）。

例如，如果寄件位址實體包含在街道位址列中，則尋找街道位址列會為出貨位址實體的預測增加大量權數。

* 交貨位址（機器學習服務實體）：

    * 街道號碼（列）
    * 街道位址（列）
    * 城市（列）
    * 州或省（列）
    * 國家/地區（列）
    * 郵遞區號（列）

## <a name="nested-subentities-with-features"></a>具有功能的嵌套子實體

機器學習清單示存在於父實體的概念。 父系可以是另一個列或最上層實體。 列的值會作為其父系的功能。

列可以同時有片語清單和模型（另一個實體）做為功能。

當列有片語清單時，它會提升概念的詞彙，但不會將任何資訊新增至預測的 JSON 回應。

當列具有另一個實體的功能時，JSON 回應會包含該其他實體的已解壓縮資料。


## <a name="required-features"></a>必要的功能

必須找到必要的功能，才能從預測端點傳回模型。 當您知道傳入的資料必須符合此功能時，請使用必要的功能。

如果語句文字不符合所需的功能，則不會將它解壓縮。

所需的功能會使用非機器學習實體：

* 規則運算式實體
* 清單實體
* 預建實體

如果您確信會在資料中找到您的模型，請視需要設定此功能。 如果找不到必要功能，則不會傳回任何專案。

繼續進行寄送位址的範例：

交貨位址（機器學習的實體）

 * 街道號碼（列） 
 * 街道位址（列） 
 * 街道名稱（列） 
 * 城市（列） 
 * 州或省（列） 
 * 國家/地區（列） 
 * 郵遞區號（列）

### <a name="required-feature-using-prebuilt-entities"></a>使用預先建立的實體所需的功能

城市、州和國家/地區通常是一組封閉的清單，這表示它們不會隨著時間而改變。 這些實體可能會有相關的建議功能，而這些功能可能會標示為必要。 這表示如果找不到具有所需功能的實體，就不會傳回整個運送位址。

如果城市、州或國家/地區位於語句中，但它們位於某個位置或俚語 LUIS 不會預期，該怎麼辦？ 如果您想要提供一些 post 處理來協助解決實體，因為 LUIS 的低信賴分數，請不要將此功能標示為必要。

送貨位址的另一個必要功能範例是將街道號碼設為必要的[預建](luis-reference-prebuilt-entities.md)編號。 這可讓使用者輸入「1個 Microsoft 的方式」或「一位 Microsoft 的方式」。 兩者都會解析為街道編號列的數位 "1"。

### <a name="required-feature-using-list-entities"></a>使用清單實體所需的功能

[清單實體](reference-entity-list.md)會用來做為正式名稱的清單及其同義字。 作為必要功能，如果語句不包含正式名稱或同義字，則實體不會當做預測端點的一部分傳回。

假設您的公司只會寄送給一組有限的國家/地區。 您可以建立清單實體，其中包含數個可讓您的客戶參考國家/地區的方式。 如果 LUIS 在語句的文字內找不到完全相符的專案，則預測中不會傳回實體（具有清單實體的必要功能）。

|正式名稱|同義字|
|--|--|
|美國|美式英文<br>美國<br>美國<br>USA<br>0|

用戶端應用程式（例如聊天機器人）可以要求後續追蹤問題來提供協助。 這可協助客戶瞭解國家/地區選取範圍受限且*必要*。

### <a name="required-feature-using-regular-expression-entities"></a>使用正則運算式實體的必要功能

當做必要功能使用的[正則運算式實體](reference-entity-regular-expression.md)會提供豐富的文字比對功能。

在寄送位址範例中，您可以建立正則運算式來捕捉國家/地區郵遞區號的語法規則。

## <a name="global-features"></a>全域功能

雖然最常見的用法是將功能套用至特定模型，但您可以將此功能設定為**全域功能**，以將它套用至整個應用程式。

全域功能最常見的用途，就是將其他詞彙新增至應用程式。 例如，如果您的客戶使用主要語言，但預期能夠在同一個語句中使用另一種語言，您可以新增包含次要語言文字的功能。

因為使用者預期會在任何意圖或實體上使用次要語言，所以請將次要語言的文字新增至片語清單。 將片語清單設定為全域功能。
