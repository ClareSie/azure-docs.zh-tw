# <a name="connectivity-architecture-in-azure-database-for-mariadb"></a>MariaDB Azure 資料庫中的連接體系結構
本文介紹了 MariaDB 連接體系結構的 Azure 資料庫，以及如何從 Azure 內外的用戶端將流量定向到 MariaDB 實例的 Azure 資料庫。

## <a name="connectivity-architecture"></a>連線架構

MariaDB 的 Azure 資料庫連接是通過閘道建立的，閘道負責將傳入連接路由到群集中伺服器的物理位置。 下圖說明瞭流量流。

![連接體系結構概述](./media/concepts-connectivity-architecture/connectivity-architecture-overview-proxy.png)

當用戶端連接到資料庫時，它們會收到連接到閘道的連接字串。 此閘道具有偵聽埠 3306 的公共 IP 位址。 在資料庫群集中，流量將轉發到 MariaDB 的相應 Azure 資料庫。 因此，為了連接到您的伺服器（例如來自公司網路），必須打開用戶端防火牆，以便出站流量能夠到達我們的閘道。 下面您可以找到我們每個區域的閘道使用的 IP 位址的完整清單。
