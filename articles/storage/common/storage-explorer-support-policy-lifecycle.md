# <a name="azure-storage-explorer-support-lifecycle-and-policy"></a>Azure 儲存體總管支援週期和原則

本檔涵蓋 Azure 儲存體總管的支援週期和原則。

## <a name="update-schedule-and-process"></a>更新排程和進程

Azure 儲存體總管每年發行四到六次。 Microsoft 也可能會略過此排程來發行重大問題的修正程式。

儲存體總管會每小時檢查更新，並在可用時加以下載。 不過，需要使用者接受，才能開始安裝程式。 如果使用者選擇在不同時間更新，他們可以手動檢查是否有更新。 在 Windows 和 Linux 上，使用者可以**從 [說明**] 功能表中選取 [**檢查更新**] 來檢查更新。 在 macOS 上，您可以在 [**應用程式] 功能表**底下找到**更新**。 使用者也可以直接移至[儲存體總管](https://azure.microsoft.com/features/storage-explorer/)下載頁面，下載並安裝最新版本。

我們強烈建議您一律使用最新版本的儲存體總管。 如果問題導致您無法更新至最新版本的儲存體總管，請在我們的[GitHub](https://github.com/microsoft/AzureStorageExplorer)上開啟問題。

## <a name="support-lifecycle"></a>支援週期

儲存體總管受[現代化生命週期原則](https://support.microsoft.com/help/30881/modern-lifecycle-policy)所規範。 使用者必須將其安裝儲存體總管保持在最新狀態。 保持最新狀態可確保使用者擁有最新的功能、效能增強功能、安全性和服務可靠性。

從版本1.14.1 開始，任何大於12個月的儲存體總管版本都會被視為不支援。 從2021年7月14日起，1.14.1 之前的所有發行都會被視為不支援。 不受支援的版本不再保證能夠完全依照設計和預期的方式運作。 如需所有版本、其發行日期和其支援日期結束的清單，請參閱[版本](#releases)。

從版本1.13.0 開始，一旦某個版本大約一個月，而不支援時，就會顯示應用程式內警示。 此警示會鼓勵使用者更新至最新版本的儲存體總管。 一旦版本不支援，每次啟動時就會顯示應用程式內警示。
