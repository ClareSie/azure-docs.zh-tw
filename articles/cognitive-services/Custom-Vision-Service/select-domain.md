# <a name="select-a-domain-for-a-custom-vision-project"></a>為自訂願景專案選擇域

從自訂視覺專案的設置邊欄選項卡中，可以為專案選擇域。 選擇最接近您的方案的域。

## <a name="image-classification"></a>影像分類

|網域|目的|
|---|---|
|__泛型__| 已針對廣泛的影像分類工作進行最佳化。 如果沒有其他適用的領域，或您不確定要選擇哪一個領域，請選取「泛型」領域。|
|__Food__|已針對菜餚相片進行最佳化，如同您在餐廳菜單上看見的一樣。 如果您想要將個別水果或蔬菜的相片分類，請使用「食物」領域。|
|__地標__|已針對可辨識的地標 (包括自然和人工) 進行最佳化。 地標在相片中清楚顯示時，此領域的效果最佳。 即使地標前面的人稍微阻擋到該地標，此領域還是能夠運作。|
|__零售__|已針對在購物目錄或購物網站上找到的影像進行最佳化。 如果您想在連衣裙、褲子和襯衫之間進行高精確度的分類，請使用此領域。|
|__精簡領域__| 針對邊緣設備上的即時分類約束進行了優化。|

## <a name="object-detection"></a>物件偵測

|網域|目的|
|---|---|
|__一般__| 已針對廣泛的物件偵測工作進行最佳化。 如果沒有其他適用的領域，或您不確定要選擇哪一個領域，請選取「泛型」領域。|
|__標誌__|已針對尋找影像中的品牌標誌進行最佳化。|
|__貨架上的產品__|針對檢測和分類貨架上的產品進行了優化。|
|__精簡領域__| 針對邊緣設備上的即時物件檢測約束進行了優化。|

## <a name="compact-domains"></a>精簡領域

精簡領域所產生的模型可以匯出到本機執行。 模型性能因所選域而異。 在下表中，我們報告英特爾桌上型電腦 CPU 和 NVidia GPU \[1\]上的型號大小和推理時間。 

> [!NOTE]
> 這些數位不包括預處理和後處理時間。

|Task|網域|模型大小|CPU 推理時間|GPU 推理時間|
|---|---|---|---|---|
|分類|一般 (精簡)|5 MB|13 毫秒|5 毫秒|
|物件偵測|一般 (精簡)|45 MB|35 毫秒|5 毫秒|
|物件偵測|常規（壓縮） [S1]|14 MB|27 毫秒|7 毫秒|

## <a name="vaidk-vision-ai-dev-kit"></a>VAIDK（視覺 AI 開發套件）

選擇緊湊域時，會提供一個額外的選項"匯出功能"，以便區分"基本平臺"和"視覺 AI 開發套件"。

在 _"匯出功能"_ 下，兩個選項是：

- 基本平臺（天流、核心ML、ONNX等）
- 願景 AI 開發套件。

當_選擇視覺 AI 開發套件_時，_通用_、_地標_和_零售_域，但_食品_緊湊域不可用於圖像分類，而_通用（壓縮）_ 和_常規（壓縮）[S1]_ 可用於物件檢測。

>[!NOTE]
>物件檢測__的一般（緊湊）__ 域需要特殊的後處理邏輯。 有關詳細資訊，請參閱匯出的 zip 包中的示例腳本。 如果需要沒有後處理邏輯的模型，請使用__常規（壓縮）[S1]__。

>[!IMPORTANT]
>不能保證匯出的模型給出的結果與雲上的預測 API 完全相同。 運行平臺或預處理實現的細微差異可能會導致模型輸出中更大的差異。 有關預處理邏輯的詳細資訊，請參閱[本文檔](python-tutorial.md)。

\[1\]英特爾至強 E5-2690 CPU 和 NVIDIA 特斯拉 M60
