# <a name="back-up-system-state-and-restore-to-bare-metal-by-using-azure-backup-server"></a>使用 Azure 備份伺服器備份系統狀態並還原到裸機

Azure 備份伺服器可備份系統狀態及提供裸機復原 (BMR) 保護。

* **系統狀態備份**：備份作業系統檔。 此備份允許您在電腦啟動時恢復，但系統檔和註冊表丟失。 系統狀態備份包括以下元素：
  * 網域成員：開機檔案、COM+ 類別註冊資料庫、登錄
  * 網域控制站：Windows Server Active Directory (NTDS)、開機檔案、 COM+ 類別註冊資料庫、登錄、系統磁碟區 (SYSVOL)
  * 執行叢集服務的電腦：叢集伺服器中繼資料
  * 執行憑證服務的電腦：憑證資料
* **裸機備份**：備份作業系統檔和關鍵卷上的所有資料，使用者資料除外。 根據定義，BMR 備份包括系統狀態備份。 當電腦無法啟動，而且您必須復原所有項目時，裸機備份便可提供保護。

下表摘要說明您可以備份及復原的項目。 有關系統狀態和 BMR 可以保護的應用版本的資訊，請參閱[Azure 備份伺服器備份什麼？](backup-mabs-protection-matrix.md)

## <a name="how-system-state-backup-works"></a>系統狀態備份的運作方式

當系統狀態備份執行時，備份伺服器會與 Windows Server Backup 通訊以要求伺服器系統狀態的備份。 根據預設，備份伺服器和 Windows Server Backup 會使用擁有最多可用空間的磁碟機。 有關此磁碟機的資訊保存在*PSDataSourceConfig.xml*檔中。 

您可以自訂備份伺服器用於系統狀態備份的磁碟機： 

1. 在受保護的伺服器上，轉到*C：\程式檔\微軟資料保護管理器\MABS_資料來源*。 
1. 打開*PSDataSourceConfig.xml*檔進行編輯。 
1. 將 \<FilesToProtect\> 值變更為磁碟機代號。 
1. 儲存並關閉檔案。 

如果保護組設置為保護電腦的系統狀態，則運行一致性檢查。 如果生成警報，則選擇警報中的 **"修改保護組**"，然後在嚮導中完成頁面。 接著執行另一個一致性檢查。

如果保護伺服器位於群集中，則可能會選擇群集磁碟機作為具有最大可用空間的磁碟機。 如果該磁碟機擁有權切換到另一個節點，並且系統狀態備份運行，則磁碟機不可用，備份失敗。 在這種情況下，修改*PSDataSourceConfig.xml*以指向本地磁碟機。

接下來，Windows 伺服器備份在還原資料夾的根目錄中創建一個名為*WindowsImageBackup*的資料夾。 當 Windows Server Backup 建立備份時，所有資料都會放在這個資料夾中。 備份完成後，檔將傳輸到備份伺服器電腦。 請注意下列資訊︰

* 備份或傳輸完成後，不會清理此資料夾及其內容。 考慮這一點的最佳方法是，在下次備份完成時，空間將保留。
* 為每個備份創建該資料夾。 其日期和時間戳記會反映最後一次進行系統狀態備份的時間。

## <a name="how-bmr-backup-works"></a>BMR 備份的工作原理

在進行 BMR (包括系統狀態備份) 時，備份作業會直接儲存到備份伺服器電腦上的共用， 它未保存到受保護伺服器上的資料夾。

備份伺服器會呼叫 Windows Server Backup，並共用該 BMR 備份的複本磁碟區。 在這種情況下，它不需要 Windows 伺服器備份來使用具有最大可用空間的磁碟機。 相反地，它會使用針對該作業所建立的共用。

備份完成後，檔將傳輸到備份伺服器電腦。 日誌存儲在*C：\Windows_Logs_WindowsServer 備份*中。

## <a name="prerequisites-and-limitations"></a>必要條件和限制

* 執行 Windows Server 2003 的電腦或執行用戶端作業系統的電腦皆不支援 BMR。

* 您無法針對位於不同保護群組的同一部電腦保護 BMR 和系統狀態。

* 備份伺服器電腦無法讓自己受到 BMR 保護。

* BMR 不支援對磁帶（磁片到磁帶或 D2T）的短期保護。 支援對磁帶的長期存儲（磁片到磁片到磁帶或 D2D2T）。

* 若要獲得 BMR 保護，您必須在受保護的電腦上安裝 Windows Server Backup。

* 對於 BMR 保護，與系統狀態保護不同，備份伺服器在受保護的電腦上沒有空間要求。 Windows Server Backup 會將備份直接傳輸到備份伺服器電腦。 備份傳輸作業不會出現在備份伺服器的 [作業]**** 檢視中。

* 備份伺服器會在複本磁碟區上保留 30 GB 的空間，以供執行 BMR。 您可以在"修改保護組嚮導"中的 **"磁片分配**"頁上更改此空間分配。 或者，您可以使用獲取資料來源磁片分配和集資料來源磁片分配 PowerShell Cmdlet。 在復原點磁碟區上，BMR 保護需要大約 6 GB 的空間來將資料保留五天。
  * 不能將副本卷大小減少到小於 15 GB。
  * 備份伺服器不會計算 BMR 資料來源的大小。 它會假設所有伺服器都有 30 GB。 請根據您預期環境中所具有的 BMR 備份大小來變更此值。 您可以粗略地計算 BMR 備份的大小，作為所有關鍵卷上已用空間的總和。 重要磁碟區 = 開機磁碟區 + 系統磁碟區 + 裝載系統狀態資料的磁碟區，例如 Active Directory。

* 如果從系統狀態保護更改為 BMR 保護，則 BMR 保護需要較少的*復原點卷*上的空間。 但是，卷上的額外空間不會回收。 您可以在"修改保護組嚮導 **"的"修改磁片分配**"頁上手動縮小卷大小。 或者，您可以使用獲取資料來源磁片分配和集資料來源磁片分配 PowerShell Cmdlet。

    如果從系統狀態保護更改為 BMR 保護，則 BMR 保護需要在*副本卷*上增加空間。 系統會自動擴充磁碟區。 如果要更改預設空間分配，請使用修改磁片分配 PowerShell Cmdlet。

* 如果從 BMR 保護更改為系統狀態保護，則復原點卷上需要更多的空間。 備份伺服器可能會嘗試自動增加磁碟區。 如果存儲池沒有足夠的空間，則會發生錯誤。

    如果從 BMR 保護更改為系統狀態保護，則需要受保護電腦上的空間。 您需要空間，因為系統狀態保護首先將副本寫入本地電腦，然後它將副本傳輸到備份伺服器電腦。

    * [Azure 備份伺服器的系統需求](https://docs.microsoft.com/system-center/dpm/install-dpm#setup-prerequisites)
    * [備份伺服器保護對照表](backup-mabs-protection-matrix.md)

1. **設置存儲**。 您可以將備份資料儲存在磁碟上、磁帶上以及 Azure 雲端中。 如需詳細資訊，請參閱[準備資料儲存體](https://docs.microsoft.com/system-center/dpm/plan-long-and-short-term-data-storage)。

1. **設定保護代理程式**。 在您要備份的電腦上安裝保護代理程式。 如需詳細資訊，請參閱[部署 DPM 保護代理程式](https://docs.microsoft.com/system-center/dpm/deploy-dpm-protection-agent)。

## <a name="back-up-system-state-and-bare-metal"></a>備份系統狀態和裸機

要備份系統狀態和裸機：

1. 要打開"創建新保護組嚮導"，請在"備份伺服器管理員主控台"中選擇 **"保護** > **操作** > **創建保護組**"。

1. 在 [選取保護群組類型]**** 頁面上，選取 [伺服器]****，然後選取 [下一步]****。

1. 在 [選取群組成員]**** 頁面上展開電腦，然後選取 [BMR]**** 或 [系統狀態]****。

    請記住，您無法針對位於不同群組的同一部電腦同時保護 BMR 和系統狀態。 此外，當您選取 BMR 時，系統會自動啟用系統狀態。 如需詳細資訊，請參閱[部署保護群組](https://docs.microsoft.com/system-center/dpm/create-dpm-protection-groups)。

1. 在 **"選擇資料保護方法"** 頁上，選擇如何處理短期備份和長期備份。 

    短期備份始終首先用於磁片，可以選擇使用 Azure 備份（短期或長期）從磁片備份到 Azure。 若要以雲端作為目的地進行長期備份，替代方式為將長期備份的目的地設為獨立磁帶裝置或連線到備份伺服器的磁帶媒體櫃。

1. 在 **"選擇短期目標**"頁上，選擇如何備份到磁片上的短期存儲：
    * 對於**保留範圍**，選擇將資料保留在磁片上的時間。
    * 對於**同步頻率**，選擇運行磁片增量備份的頻率。 如果不想設置備份間隔，可以選擇**在復原點之前**。 備份伺服器將在計畫每個復原點之前運行快速完整備份。

1. 如果要將資料存儲在磁帶上以進行長期存儲，請在 **"指定長期目標"** 頁上選擇保留磁帶資料的時間（1 到 99 年）。
    1. 對於**備份頻率**，選擇運行備份到磁帶的頻率。 頻率基於您選擇的保留範圍：
        * 當保留期為 1 到 99 年時，您可以每天、每週、每兩周、每月、每季度、每半年或每年備份一次。
        * 當保留範圍為 1 到 11 個月時，您可以每天、每週、每兩周或每月備份一次。
        * 當保留範圍為 1 到 4 周時，您可以每天或每週備份。

    1. 在 **"選擇磁帶和庫詳細資訊"** 頁上，選擇要使用的磁帶和庫。 還要選擇是否應壓縮和加密資料。

1. 在 **"查看磁片分配**"頁上，查看可用於保護組的存儲池磁碟空間。

    * [資料大小總計]**** 是您要備份的資料大小。
    * [要在 Azure 備份伺服器上佈建的磁碟空間]**** 是備份伺服器所建議的保護群組空間。 備份伺服器使用這些設置來選擇理想的備份卷。 您可以在**磁片分配詳細資訊**中編輯備份卷選項。
    * 針對工作負載，請在下拉式功能表中選取您偏好的儲存體。 您的編輯內容會變更 [可用磁碟儲存體]**** 窗格中 [儲存體總計]**** 和 [可用儲存體]**** 的值。 未預配的空間是備份伺服器建議您添加到卷以確保順利備份的存儲量。

1. 在 **"選擇複本建立方法"** 頁上，選擇如何處理初始完整資料複製。 

    如果您選擇透過網路進行複寫，建議您選擇離峰時間。 如果資料量較大或網路狀況不理想，請考慮使用卸除式媒體來離線複寫資料。

1. 在 **"選擇一致性檢查選項"** 頁上，選擇如何自動執行一致性檢查。 

    您可以在複本資料變得不一致時或按照排程的時間執行檢查。 如果不想配置自動一致性檢查，則可以隨時運行手動檢查。 若要執行手動檢查，請在備份伺服器管理員主控台的 [保護]**** 區域，以滑鼠右鍵按一下保護群組，然後選取 [執行一致性檢查]****。

1. 如果選擇使用 Azure 備份備份備份到雲，請在 **"指定連線保護資料"** 頁上選擇要備份到 Azure 的工作負荷。

1. 在 **"指定連線備份計畫"** 頁上，選擇增量備份到 Azure 的頻率。 

    您可以安排備份每天、每週、每月和年運行。 您還可以選擇運行備份的時間和日期。 一天最多可以備份兩次。 每次運行備份時，都會在 Azure 中從存儲在備份伺服器磁片上的備份資料的複本建立資料復原點。

1. 在 **"指定連線保留原則"** 頁上，選擇從每日、每週、每月和每年備份中創建的復原點如何保存在 Azure 中。

1. 在 [選擇線上複寫]**** 頁面上，選取要如何進行首次的資料完整複寫。 

    您可以通過網路複製或離線備份（離線種子設定）。 離線備份使用 Azure 導入功能。 如需詳細資訊，請參閱 [Azure Backup 中的離線備份工作流程](offline-backup-azure-data-box.md)。

1. 在 [摘要]**** 頁面上檢閱您的設定。 在選取 [建立群組]**** 後，系統就會開始進行首次的資料複寫。 資料複製完成後，在 **"狀態"** 頁上，保護組狀態**正常**。 然後根據保護組設置進行備份。

## <a name="recover-system-state-or-bmr"></a>復原系統狀態或 BMR

您可以將 BMR 或系統狀態復原到網路位置。 如果備份了 BMR，則使用 Windows 修復環境 （WinRE） 啟動系統並將其連接到網路。 然後使用 Windows Server Backup 從網路位置復原。 如果備份系統狀態，則只需使用 Windows 伺服器備份從網路位置恢復。

### <a name="restore-bmr"></a>還原 BMR

要在備份伺服器電腦上運行恢復：

1. 在 **"恢復"** 窗格中，查找要恢復的電腦。 然後選擇**裸機復原**。

1. 行事曆上會以粗體表示可用的復原點。 選取您要使用之復原點的日期和時間。

1. 在 **"選擇恢復類型"** 頁上，選擇 **"複製到網路資料夾**"。

1. 在 **"指定目標"** 頁上，選擇複製資料的目標。 

    請記住，目的地需要有足夠的空間來容納資料。 我們建議您為目標創建新資料夾。

1. 在 **"指定恢復選項"** 頁上，選擇安全設置。 然後選擇是否使用基於存放區域網路 （SAN） 的硬體快照，以便更快地恢復。 此選項僅在：
    * 您有一個提供此功能的 SAN。 
    * 您可以創建和拆分克隆，使其可寫。
    * 受保護的電腦和備份伺服器電腦連接到同一網路。

1. 設定通知選項。 
1. 在 [確認]**** 頁面上，選取 [復原]****。

要設置共用位置：

1. 在還原位置中移至該備份的所在資料夾。

1. 共用*WindowsImageBackup*上方一個級別的資料夾，以便共用資料夾的根目錄是*WindowsImageBackup*資料夾。 

    如果不共用此資料夾，還原將無法找到備份。 要使用 WinRE 進行連接，您需要一個共用，您可以使用正確的 IP 位址和憑據在 WinRE 中訪問該共用。

要還原系統：

1. 使用要還原的系統使用 Windows DVD 開機要還原映射的電腦。

1. 在第一頁上，驗證語言和地區設定。 在 [安裝]**** 頁面上，選取 [修復您的電腦]****。

1. 在 [系統修復選項]**** 頁面上，選取 [使用您先前建立的系統映像還原電腦]****。

1. 在 [選取系統映像備份]**** 頁面上，選取 [選取系統映像]**** > [進階]**** > [搜尋網路上的系統映像]****。 如果出現警告，請選取 [是]****。 移至共用路徑，輸入認證，然後選取復原點。 系統掃描該復原點中可用的特定備份。 選取您要使用的復原點。

1. 在 **"選擇如何還原備份頁"上**，選擇 **"格式化"和"重新分區磁片**"。 在下一頁上，驗證設置。

1. 若要開始還原，請選取 [完成]****。 系統需要重新啟動。

### <a name="restore-system-state"></a>還原系統狀態

要在備份伺服器中運行恢復：

1. 在 [復原]**** 窗格中找到您要復原的電腦，然後選取 [裸機復原]****。

1. 行事曆上會以粗體表示可用的復原點。 選取您要使用之復原點的日期和時間。

1. 在 **"選擇恢復類型"** 頁上，選擇 **"複製到網路資料夾**"。

1. 在 **"指定目標"** 頁上，選擇複製資料的位置。 

    請記住，您選擇的目標需要有足夠的空間用於資料。 我們建議您為目標創建新資料夾。

1. 在 **"指定恢復選項"** 頁上，選擇安全設置。 然後選擇是否使用基於 SAN 的硬體快照，以便更快地恢復。 此選項僅在： 
    * 您有一個提供此功能的 SAN。
    * 您可以創建和拆分克隆，使其可寫。 
    * 受保護的電腦和備份伺服器連接到同一網路。

1. 設定通知選項。 
1. 在 [確認]**** 頁面上，選取 [復原]****。

要運行 Windows 伺服器備份：

1. 選擇**操作** > **恢復** > **此伺服器** > **下一個**。

1. 選取 [其他伺服器]****，選取 [指定位置類型]**** 頁面，然後選取 [遠端共用資料夾]****。 輸入復原點所在資料夾的路徑。

1. 在 [選取復原類型]**** 頁面上，選取 [系統狀態]****。

1. 在 [選取系統狀態復原的位置]**** 頁面上，選取 [原始位置]****。

1. 在 [確認]**** 頁面上，選取 [復原]****。 

1. 在還原之後，請重新啟動伺服器。

您還可以在命令提示符下運行系統狀態還原： 

1. 在要恢復的電腦上啟動 Windows 伺服器備份。 

1. 若要取得版本識別碼，請在命令提示字元輸入：

   ```wbadmin get versions -backuptarget \<servername\sharename\>```

1. 使用版本識別碼來開始系統狀態還原。 在命令提示字元中，輸入： 

    ```wbadmin start systemstaterecovery -version:<versionidentified> -backuptarget:<servername\sharename>```

1. 確認您想要開始復原。 您可以在 [命令提示字元] 視窗中看到復原程序。 系統會建立還原記錄。 

1. 在還原之後，請重新啟動伺服器。
