# <a name="412002-devicemessagelocklost"></a>412002 DeviceMessageLockLost

本文說明**412002 DeviceMessageLockLost**錯誤的原因和解決方案。

## <a name="symptoms"></a>徵狀

嘗試傳送雲端到裝置訊息時，要求失敗，並出現錯誤**412002 DeviceMessageLockLost**。

## <a name="cause"></a>原因

當裝置從佇列接收雲端到裝置訊息時（例如，使用 [`ReceiveAsync()`](https://docs.microsoft.com/dotnet/api/microsoft.azure.devices.client.deviceclient.receiveasync?view=azure-dotnet) ），訊息會被 IoT 中樞鎖定一分鐘的鎖定超時時間。 如果裝置嘗試在鎖定超時到期後完成訊息，IoT 中樞會擲回此例外狀況。

## <a name="solution"></a>解決方案

如果 IoT 中樞不會在一分鐘的鎖定超時時間內收到通知，則會將訊息設回已加入*佇列*的狀態。 裝置可以嘗試再次接收訊息。 為避免未來發生此錯誤，請執行裝置端邏輯，在接收訊息的一分鐘內完成訊息。 這一分鐘的超時時間無法變更。
