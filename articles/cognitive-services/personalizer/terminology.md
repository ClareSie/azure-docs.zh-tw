# <a name="terminology"></a>詞彙

個人化器使用強化學習中的術語。 這些術語在 Azure 門戶和 API 中使用。

## <a name="conceptual-terminology"></a>概念術語

* **學習迴圈**：您為應用程式的每個部分創建一個個人化資源，稱為_學習迴圈_，這些部分可以從個人化中受益。 如果您有多個要個人化的經驗，請為每個經驗各建立一個迴圈。

* **模型**：個人化模型捕獲有關使用者行為的所有資料，從發送到 Rank 和 Reward 調用的參數的組合以及學習策略確定的培訓行為中獲取訓練資料。

## <a name="personalizer-configuration"></a>個人化配置

個人化器是從 Azure[門戶](https://portal.azure.com)配置的。

* **獎勵**：配置獎勵等待時間、預設獎勵和獎勵聚合策略的預設值。

* **探索**：配置用於勘探的排名調用的百分比

* **模型更新頻率**：重新訓練模型的頻率。

* **資料保留**：要存儲的資料價值多少天。 這可能會影響離線評估，這些評估用於改善學習迴圈。

## <a name="use-rank-and-reward-apis"></a>使用排名和獎勵 API

* **排名**： 給定具有功能和上下文功能的操作，請使用流覽或利用來返回頂部操作（內容項）。

    * **操作**：操作是要選擇的內容項，如產品或促銷。 個人化者選擇頂級操作（返回的獎勵操作 ID）通過排名 API 向使用者顯示。

    * **上下文**：要提供更準確的排名，請提供有關上下文的資訊，例如：
        * 您的使用者。
        * 其所使用的裝置。
        * 目前的時間。
        * 其他關於目前情況的資料。
        * 關於使用者或內容的歷史資料。

        特定應用程式可能會有不同的內容資訊。

    * **[功能](concepts-features.md)**：有關內容項或使用者上下文的資訊單元。 請確保僅使用聚合的功能。 請勿將特定時間、使用者專用時或其他非聚合資料用作功能。

        * _操作功能_是關於內容的中繼資料。
        * _上下文功能_是關於顯示內容的上下文的中繼資料。

* **探索**：個人化服務正在探索何時，而不是返回最佳操作，而是為使用者選擇不同的操作。 個人化工具服務可避開變化、停滯，並可藉由探索來適應不斷發展的使用者行為。

* **利用**： 個人化服務使用當前模型根據過去的資料決定最佳操作。

* **實驗持續時間**：從為該事件進行排名調用的那一刻起，個人化服務等待獎勵的時間量。

* **非活動事件**：非活動事件是您調用 Rank 的事件，但您不確定使用者是否會看到結果，因為用戶端應用程式決策。 非作用中事件可讓您建立並儲存個人化結果，並在不會影響機器學習模型的情況下，於之後決定捨棄這些結果。


* **獎勵**：使用者如何回應 Rank API 返回的獎勵操作 ID 的度量，分數介於 0 到 1 之間。 0 到 1 的值是由商務邏輯根據所做選擇對協助達成個人化商務目標有多大幫助而做出設定。 學習迴圈不會將此獎勵存儲為單個使用者歷史記錄。

## <a name="offline-evaluations"></a>離線評估

* **評估**：離線評估根據迴圈的資料確定迴圈的最佳學習策略。

* **學習策略**：個人化器如何在每個事件上訓練模型將由影響機器學習演算法工作方式的一些參數決定。 新的學習迴圈以預設**的學習策略**開頭，它可以產生中等的性能。 運行[評估](concepts-offline-evaluation.md)時，個人化程式會創建新的學習策略，專門針對迴圈的用例進行優化。 使用針對每個特定迴圈優化的策略（在評估期間生成）個人化程式的性能將顯著提高。 學習策略在 Azure 門戶中的"**個人化"資源模型和學習設置**上命名為_學習設置_。
