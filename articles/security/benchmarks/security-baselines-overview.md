# <a name="security-baselines-for-azure"></a>Azure 的安全性基準

適用于 Azure 的安全性基準可協助您透過改良的工具、追蹤和安全性功能來加強安全性。 它們也會在保護您的環境時提供一致的體驗。

Azure 的安全性基準著重在以雲端為中心的控制領域。 這些控制項與已知的安全性效能評定一致，例如 (CIS) 的網際網路安全性中心所描述的控制項。 我們的基準會針對 [Azure 安全性基準測試](overview.md)中所列的控制區域提供指引。

每個建議都包含下列資訊：

- **AZURE 識別碼**：對應至建議的 Azure 安全性基準測試識別碼。
- **建議**：緊接在 Azure 識別碼之後，建議提供控制項的高階描述。
- **指導**方針：建議的原理，以及如何執行的指引連結。 如果 Azure 資訊安全中心支援此建議，則也會列出該資訊。
- **責任**：負責執行控制項的人員。 可能的案例是客戶責任、Microsoft 責任或共同責任。
- **Azure 資訊安全中心監視**：控制項是否由 Azure 資訊安全中心監視，並連結至參考。

基準中包含所有建議，包括不適用於此特定服務的建議，可讓您完整瞭解 Azure 安全性基準測試與每項服務之間的關聯性。 有時可能會有基於各種原因而不適用的控制項，例如，以 IaaS/計算為中心的控制項 (例如作業系統設定管理的特定控制項，) 可能不適用於 PaaS 服務。


歡迎您提供 Azure 服務安全性基準的意見反應。 我們建議您在下方的意見反應區域中提供意見。 或者，如果您想要在 Azure 安全性基準小組中更私下地共用您的輸入，歡迎您填寫表單 https://aka.ms/AzSecBenchmark 。
