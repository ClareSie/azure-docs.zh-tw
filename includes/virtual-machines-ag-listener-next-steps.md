除了自動將用戶端連線至主要複本，接聽程式可用來將唯讀工作負載重新導向至次要複本。 此用法可以改善整體解決方案的效能和延展性。 如需詳細資訊，請參閱[使用 ReadIntent 路由搭配 Azure AlwaysOn 可用性群組接聽程式](https://go.microsoft.com/fwlink/?LinkId=522515)。

> [!NOTE]
> 如需 Azure 接聽程式的疑難排解秘訣，請參閱 AlwaysOn 支援小組[部落格](https://blogs.msdn.com/b/alwaysonpro/)中的[針對 Azure 中可用性群組接聽程式的問題進行移難排解](https://blogs.msdn.microsoft.com/alwaysonpro/2017/02/22/troubleshooting-internal-load-balancer-listener-connectivity-in-azure)。
