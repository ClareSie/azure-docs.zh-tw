# <a name="saas-offer-creation-checklist"></a>SaaS 供應項目建立檢查清單

SaaS 產品創建過程將帶您流覽多個頁面。 以下是您可以在每個頁面上提供的詳細資訊，以及用於瞭解有關每個專案的詳細資訊的連結。

下面列出了您需要提供或指定的專案。 某些區域是可選的，或者提供了預設值，您可以根據需要進行更改。 您不必按此處列出的連續處理這些部分。

| **專案**    | **目的**  |
| :---------- | :-------------------|
| [**新報價模式**](#new-offer-modal) | 收集提供身份資訊。  |
| [優惠設置頁面](#offer-setup-page) | 允許您選擇使用關鍵功能，並選擇如何通過 Microsoft 銷售您的產品/服務。  |
| [屬性頁](#properties-page) | 定義用於在市場對產品/服務進行分組的類別和行業、支援產品/服務的法律合同以及應用版本。 |
| [優惠清單頁面](#offer-listing-page) | 定義要在市場上顯示的要約詳細資訊，包括您的報價和行銷資產的描述。 |
| [預覽頁面](#preview-page) | 在將優惠即時發佈給更廣泛的市場受眾之前，定義一個有限的預覽受眾，以便發佈您的優惠。 |
| [提供技術配置頁面](#technical-configuration-page)  | 僅當您選擇通過 Microsoft 銷售產品/服務時，才可用。 定義用於連接到產品/服務的技術詳細資訊（URL 路徑、Webhook、租戶 ID 和應用 ID）。 |
| [**新計畫模式**](#plan-identity-modal) | 收集計畫標識資訊。  |
| [計畫清單頁](#plan-listing-page)  | 僅當您選擇通過 Microsoft 銷售產品/服務時，才可用。 定義用於在市場上列出計畫的詳細資訊。  |
| [計畫定價&可用性頁面](#plan-pricing--availability-page)  | 僅當您選擇通過 Microsoft 銷售產品/服務時，才可用。  收集每個計畫（版本）的業務特徵（定價模式）、受眾和市場可用性。  |
| [試駕清單頁面](#test-drive-listing-page)  | 僅在您選擇提供產品/服務試駕時才可用。 定義用於在市場中列出試駕的詳細資訊。  |
| 試駕技術配置頁面  | 僅在您選擇提供產品/服務試駕時才可用。 定義演示（或"試駕"）的技術細節，使客戶能夠在承諾購買之前試用您的產品/服務。  |
| [查看和發佈頁面](#review-and-publish-page)  | 選擇要發佈的更改，查看每個頁面的狀態，並向認證團隊提供注釋。  |


## <a name="new-offer-modal"></a>新報價模式 

您將被要求提供的第一條資訊是您的優惠的 ID 和別名。 

| **欄位名稱**    | **注意**   |  
| :---------------- | :-----------| 
| 供應項目識別碼  | 必需，在創建後無法更改。 最大 50 個字元，必須僅包含小寫、字母數位字元、破折號或底線。 |
| 優惠別名  | 必要。 |

## <a name="offer-setup-page"></a>優惠設置頁面

產品/服務設置頁面是您可以選擇進入不同管道和銷售運動，以及聲明使用關鍵功能（如試駕和客戶潛在客戶）的地方。 

| **欄位名稱**    | **注意**   | 
| :---------------- | :-----------|  
| 您想通過微軟銷售嗎？  | 必要。 預設：是 |
| 您希望潛在客戶如何與產品/服務清單進行交互？ （行動號召）  | 如果無法通過 Microsoft 銷售，則是必需的。 預設：免費試用，選項："立即獲取"，"免費試用"，"聯繫我"。 |
| 試用版 URL  | 如果選擇"免費試用"，則需要作為客戶與產品/服務清單交互的方式。 |
| 提供 URL  | 如果選擇"立即獲取"，則是必需的，因為客戶應與產品/服務清單進行交互 |
| 聲道  | 選擇性。 預設值：未選擇進入 CSP（轉銷商）通道。  |
| 試用產品 | 選擇性。 預設值：未啟用試駕。  |
| 試用產品的類型 | 如果啟用了試磁碟機，則是必需的。 預設值：未選擇。 選項：Azure 資源管理器、面向業務中心的動態 365、客戶參與的動態 365、操作動態 365、邏輯應用、Power BI。  |
| 潛在客戶管理 - 連接到 CRM 系統 | 如果通過 Microsoft 銷售，或者如果上市報價為"與我聯繫"，則需要這樣做。 預設值：未連接 CRM 系統。 CRM 選項：Azure 表、Azure Blob、動態 CRM 連線、HTTP 的終結點、市場、銷售隊伍  |

## <a name="properties-page"></a>屬性頁面

屬性頁是定義用於在市場對產品/服務進行分組的類別和行業、支援產品/服務的法律合同以及應用版本。 請務必在此頁面上提供有關您的優惠的完整和準確的詳細資訊，以便適當顯示並提供給正確的客戶。 

| **欄位名稱**    | **注意**   | 
| :---------------- | :-----------|  
| 類別與子類別 | 要求 1 和最大 3。 預設值：未選擇。 |
| 工業和子工業 | 選擇性。 每個 L1 行業中最多 2 個 L1 工業和最多 2 個子行業，預設值：未選擇任何 |
| 應用程式版本  | 選擇性。 預設值：無。 |
| 使用標準合同  | 選擇性。 預設值：未選中。  | |
| 使用規定  | 如果未選擇標準合同，則為必填項。  |

## <a name="offer-listing-page"></a>優惠清單頁

清單頁是提供客戶在市場中查看產品/服務清單時看到的文本和圖像的位置。 

| **欄位名稱**    | **注意**   |
| :---------------- | :-----------| 
| 名稱  | 需要，最多 50 個字元。 |
| 總結  | 需要，最多 100 個字元。 | 
| 描述  | 需要，最多 3000 個字元。 |
| 開始使用指示  | 需要，最多 3000 個字元。 |
| 開始使用指示  | 需要，最多 3000 個字元。 |
| 搜尋關鍵字  | 可選、推薦、最多 3 個關鍵字。 |
| 隱私權原則 URL  | 必要。 |
| CSP 計畫行銷材料 URL  | 選擇性。 |
| 有用的連結 標題和 URL  | 選擇性。 |
| 支援文檔標題和檔  | 需要，最小 1 和最大 3。 必須為 PDF 檔案格式。 |
| 螢幕擷取畫面  | 需要，最小 1 螢幕截圖和最多 5 個;建議使用四個或更多。 必須為 1280 X 720，採用 PNG 格式。 |
| 商店徽標（小、中、大、寬、英雄）  | 需要小 （48 X 48） 和大 （216 X 216）;其他尺寸可選，但推薦： 中等 （90 x 90）， 寬 （255 x 115）， 英雄 （815 x 290）. 必須採用 PNG 格式。 |
| 視頻名稱 + URL + 縮略圖  | 可選，推薦，最多4個視頻。 縮略圖必須為 1280 x 720，採用 PNG 格式。 視頻必須託管在 YouTube 或 Vimeo 中。 |
| 連絡人（CSP 計畫、工程、支援）  | 工程和支援連絡人（姓名、電子郵件和電話號碼）;CSP 計畫連絡人可選，但建議。 |
| 支援 URL  | 必要。 |

## <a name="preview-page"></a>預覽頁面

預覽頁面是指定受眾訪問產品/服務預覽的位置，以驗證產品/服務在上線前是否滿足您的所有要求。 

| **欄位名稱**    | **注意**   | 
| :---------------- | :-----------| 
| AAD/MSA 電子郵件 + 描述 | 需要，最小 1 和最多 10（如果手動輸入）;如果上傳 CSV 檔，則最多為 20。 |

## <a name="technical-configuration-page"></a>技術配置頁面 

技術配置頁面是指定 Microsoft 用於連接到產品/服務的技術詳細資訊的位置。 如果您決定不通過 Microsoft 銷售，則此頁面不可見。

| **欄位名稱**    | **注意**   |  
| :---------------- | :-----------| 
| 登陸頁面 URL | 通過 Microsoft 銷售，則需要。 |
| 連接網鉤 | 通過 Microsoft 銷售，則需要。 |
| Azure AD 租用戶識別碼 | 通過 Microsoft 銷售，則需要。 |
| Azure AD 應用程式識別碼 | 通過 Microsoft 銷售，則需要。 |

## <a name="plan-identity-modal"></a>規劃標識模式

要求您提供的第一條資訊是計畫的名稱和 ID。 如果您決定不通過 Microsoft 銷售，則此頁面不可見。

| **欄位名稱**    | **注意**   |  
| :---------------- | :-----------| 
| 計畫 ID  | 通過 Microsoft 銷售，則需要。 創建後無法更改它。 最大 50 個字元，必須僅包含小寫、字母數位字元、破折號或底線。 |
| 方案名稱  | 通過 Microsoft 銷售，則需要。 在優惠中的所有計劃中必須是唯一的。 最多 50 個字元。 |

## <a name="plan-listing-page"></a>計畫清單頁

計畫清單頁是您提供文本供客戶在市場中查看計畫時查看的文本的位置。 如果您決定不通過 Microsoft 銷售，則此頁面不可見。

| **欄位名稱**    | **注意**   |  
| :---------------- | :-----------| 
| 計畫描述   | 通過 Microsoft 銷售，則需要。 最大 500 個字元。 | |

## <a name="plan-pricing--availability-page"></a>計畫定價&可用性頁面

計畫定價和可用性頁面是定義每個計畫（版本）的業務特徵、受眾和市場可用性的位置。 如果您決定不通過 Microsoft 銷售，則此頁面不可見。

| **欄位名稱**    | **注意**   | 
| :---------------- | :-----------| 
| 市場可用性  | 需要，最小 1 和最大 141。 |
| 定價模型  | 必要。 預設值：統一費率。 選項：每個使用者的統一費率。 |
| 最小和最大座位  | 可選，僅在選擇基於座位的定價模型時可用。 |
| 計費期限  | 必要。 預設值：每月。 選項：每月，每年。 |
| Price  | 每月所需美元，如果選擇每月計費期限;或每年美元，如果選擇年度計費期限。 |
| 規劃受眾  | 選擇性。 預設值：公共計畫。 選項：公共、按租戶 ID 進行 |
| 受限計畫受眾（租戶 ID + 說明）  | 選擇私人計畫時是必需的。 如果手動輸入最小 1 和最多 10 個租戶 ID。 如果 CSV 檔導入，則最大 20000。 |

## <a name="test-drive-listing-page"></a>試駕清單頁

僅在您選擇提供產品/服務試駕時才可用。 定義用於在市場中列出試駕的詳細資訊。

| **欄位名稱**    | **注意**   | 
| :---------------- | :-----------| 
| 描述  | 必要。 |
| 使用者手動名稱 + 檔  | 必需，最多 1 個文檔必須為 PDF 格式。 |
| 視頻名稱、URL + 縮略圖  | 可選，建議使用。 縮略圖必須為 533 x 324（JPGP 或 PNG 格式）。 視頻必須託管在 YouTube 或 Vimeo 中。 |
