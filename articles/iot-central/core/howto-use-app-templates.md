# <a name="export-your-application"></a>匯出您的應用程式



本文介紹作為解決方案管理器，如何匯出 IoT 中央應用程式以能夠重用它。

您有兩個選擇：

- 如果只需要創建應用程式的重複副本，則可以創建應用程式的副本。
- 如果計畫創建多個副本，則可以從應用程式創建應用程式範本。

## <a name="copy-your-application"></a>複製應用程式

您可以建立任何應用程式複本，並減去其中的任何裝置執行個體、裝置資料歷程記錄和使用者資料。 副本使用標準定價計畫，您將為此收費。 不能通過複製應用程式來創建使用免費定價計畫的應用程式。

選取 [複製]****。 在對話方塊中，輸入新應用程式的詳細資訊。 然後選擇 **"複製"** 以確認要繼續。 要瞭解有關表單中欄位的更多詳細資訊，請參閱[創建應用程式](quick-deploy-iot-central.md)快速入門。

![[應用程式設定] 頁面](media/howto-use-app-templates/appcopy2.png)

應用複製操作成功後，可以使用連結導航到新應用程式。

![[應用程式設定] 頁面](media/howto-use-app-templates/appcopy3a.png)

複製應用程式還會複製規則和電子郵件操作的定義。 某些操作（如流和邏輯應用）通過規則 ID 綁定到特定規則。 當規則複製到其他應用程式時，它將獲取其自己的規則 ID。 在這種情況下，使用者必須創建新操作，然後將新規則與它關聯。 通常，最好檢查規則和操作，以確保它們在新應用中是最新的。

> [!WARNING]
> 如果儀表板包含顯示特定設備資訊的磁貼，則這些磁貼將顯示在新應用程式中**找不到請求的資源**。 必須重新配置這些磁貼以顯示有關新應用程式中設備的資訊。

## <a name="create-an-application-template"></a>建立應用程式範本

創建 Azure IoT 中心應用程式時，可以選擇內置示例範本。 您還可以從現有的 IoT 中央應用程式創建自己的應用程式範本。 然後，您可以在創建新應用程式時使用自己的應用程式範本。

創建應用程式範本時，它包含現有應用程式中的以下項：

- 預設應用程式儀表板，包括儀表板佈局和已定義的所有磁貼。
- 設備範本，包括度量值、設置、屬性、命令和儀表板。
- 規則。 包括所有規則定義。 但是，除了電子郵件操作之外，不包括操作。
- 設備集，包括其條件和儀表板。

> [!WARNING]
> 如果儀表板包含顯示特定設備資訊的磁貼，則這些磁貼將顯示在新應用程式中**找不到請求的資源**。 必須重新配置這些磁貼以顯示有關新應用程式中設備的資訊。

創建應用程式範本時，它不包括以下項：

- 裝置
- 使用者
- 作業定義
- 連續資料匯出定義

將這些專案手動添加到從應用程式範本創建的任何應用程式。

要從現有 IoT 中央應用程式創建應用程式範本，請執行：

1. 轉到應用程式中的 **"管理**"部分。
1. 選擇**應用程式範本匯出**。
1. 在 **"應用程式範本匯出"** 頁上，輸入範本的名稱和說明。
1. 選擇 **"匯出**"按鈕以創建應用程式範本。 您現在可以複製**可共用連結**，使使用者可以從範本創建新應用程式：

![建立應用程式範本](media/howto-use-app-templates/create-template.png)

### <a name="use-an-application-template"></a>使用應用程式範本

要使用應用程式範本創建新的 IoT 中心應用程式，您需要以前創建的**可共用連結**。 將**可共用連結**粘貼到瀏覽器的網址列中。 選擇自訂應用程式範本時顯示"**創建應用程式**頁"：

![從範本創建應用程式](media/howto-use-app-templates/create-app.png)

選擇您的定價計畫並填寫表單上的其他欄位。 然後選擇 **"創建"** 以從應用程式範本創建新的 IoT 中心應用程式。

### <a name="manage-application-templates"></a>管理應用程式範本

在 **"應用程式範本匯出"** 頁上，您可以刪除或更新應用程式範本。

如果刪除應用程式範本，則無法再使用以前生成的可共用連結創建新應用程式。

要更新應用程式範本，請更改**應用程式範本匯出**頁上的範本名稱或說明。 然後再次選擇 **"匯出**"按鈕。 此操作將生成新的**可共用連結**，並宣佈任何以前的**可共用連結**URL 無效。
