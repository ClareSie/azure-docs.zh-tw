# <a name="convert-to-indicator-values"></a>轉換為指標值
本文介紹了 Azure 機器學習設計器的模組。

使用 Azure 機器學習設計器中的 **"轉換為指標值"** 模組將包含分類值的列轉換為一系列二進位指標列。  

此模組還輸出用於轉換為指標值的轉換的定義。 通過使用[應用轉換](apply-transformation.md)模組，可以在具有相同架構的其他資料集上重用此轉換。

## <a name="how-to-configure-convert-to-indicator-values"></a>如何配置轉換為指標值

1.  查找 **"轉換為指標值"** 並將其拖動到管道草稿。 您可以在 **"資料轉換**"類別下找到此模組。
    > [!NOTE]
    > 您可以在 **"轉換為稻斷器值"** 模組之前使用["編輯中繼資料](edit-metadata.md)"模組將目標列標記為分類。

1. 將 **"轉換為指標值"** 模組連接到包含要轉換的列的資料集。 

1. 選擇 **"編輯"列**以選擇一個或多個分類列。

1. 如果**只想輸出新的**布林列，請選擇 **"覆蓋分類列"** 選項。 根據預設，這個選項為關閉狀態。
    

    > [!TIP]
    >  如果選擇覆蓋選項，則實際上不會刪除或修改源列。 相反，將生成新列並在輸出資料集中顯示，並且源列在工作區中仍然可用。 如果需要查看原始資料，您可以隨時使用["添加列"](add-columns.md)模組將源列重新添加。

1. 提交管道。

## <a name="results"></a>結果

假設您有一個列，其分數指示伺服器是否具有高、中或低失敗概率。  

| 伺服器識別碼 | 失敗分數 |
| --------- | ------------- |
| 10301     | 低           |
| 10302     | 中        |
| 10303     | 高          |

當您應用 **"轉換為指標值"** 時，設計器會將單個標籤列轉換為包含布林值的多個列：  

| 伺服器識別碼 | 失敗分數 - 低 | 失敗分數 - 中等 | 失敗分數 - 高 |
| --------- | ------------------- | ---------------------- | -------------------- |
| 10301     | 1                   | 0                      | 0                    |
| 10302     | 0                   | 1                      | 0                    |
| 10303     | 0                   | 0                      | 1                    |

轉換的工作原理：  

-   在描述風險**的"失敗評分**"列中，只有三個可能的值（高值、中值和低值），並且沒有缺失值。 因此，正好創建了三個新列。  

-   使用此模式：*\<源列>-\<資料值>，* 根據源列的列標題和值命名新的指標列。  

-   一個指標列中應有 1，所有其他指標列中應有 0，因為每個伺服器只能有一個風險評級。  

現在，您可以在機器學習模型中使用三個指標列作為功能。

模組返回兩個輸出：

- **結果資料集**：具有轉換指標值列的資料集。 未選擇用於清潔的列也會"通過"。
- **指標值轉換**：用於轉換為指標值的資料轉換，可以保存在工作區中，以後應用於新資料。

## <a name="apply-a-saved-indicator-values-operation-to-new-data"></a>將保存的指標值操作應用於新資料

如果需要經常重複指標值操作，可以將資料操作步驟另存為*轉換*，以便在同一資料集中重用它。 如果必須經常重新導入，然後清除具有相同架構的資料，則此功能非常有用。

1. 將["應用轉換](apply-transformation.md)"模組添加到管道中。

1. 添加要清理的資料集，並將資料集連接到右側輸入埠。

1. 在設計器的左側窗格中展開**資料轉換**組。 找到保存的轉換並將其拖動到管道中。

1. 將保存的轉換連接到[應用轉換](apply-transformation.md)的左側輸入埠。

   應用保存的轉換時，無法選擇要轉換的列。 這是因為轉換已定義並自動應用於原始操作中指定的資料類型。

1. 提交管道。
 
## <a name="technical-notes"></a>技術說明  

本節包含實現詳細資訊、提示和對常見問題的解答。

### <a name="usage-tips"></a>使用提示

-   只有標示為類別的資料行才能轉換成指標資料行。 如果看到以下錯誤，則所選列之一很可能不是分類的：  

     錯誤 0056：名稱\<列名稱>列不在允許的類別中。  

     預設情況下，大多數位符串列都作為字串要素處理，因此必須使用[編輯中繼資料](edit-metadata.md)顯式將它們標記為分類。  

-   可以轉換為指標列的列數沒有限制。 但是，由於每一列值可以生成多個指標列，因此您可能希望一次僅轉換和查看幾列。  

-   如果列包含缺少值，則為缺少的類別創建單獨的指示器列，該名稱為：*\<源列> - 缺少*  

-   如果轉換為指標值的列包含數位，則必須將其標記為分類，就像任何其他要素列一樣。 完成此操作後，數位將被視為離散值。 例如，如果您有一個數位列，其 MPG 值範圍為 25 到 30，則將為每個離散值創建新的指標列：  

    | 請確定       | 高速公路 mpg -25 | 高速公路 mpg -26 | 高速公路 mpg -27 | 高速公路 mpg -28 | 高速公路 mpg -29 | 高速公路 mpg -30 |
    | ---------- | --------------- | --------------- | --------------- | --------------- | --------------- | --------------- |
    | 孔托索汽車 | 0               | 0               | 0               | 0               | 0               | 1               |

- 避免向資料集添加過多維度。 我們建議您首先檢查列中的值數，並適當地對資料進行 bin 或量化。
