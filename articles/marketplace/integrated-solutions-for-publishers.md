# <a name="publish-an-integrated-solution"></a>發表整合解決方案

在單個捆綁產品/服務中,整合解決方案將軟體供應商的點解決方案技術與服務合作夥伴的部署和支援相結合。 服務合作夥伴在 Microsoft AppSource 或 Azure 應用商店中列出產品/服務作為[諮詢服務](./consulting-services.md)。

集成解決方案旨在滿足特定的客戶需求或方案。 提供商首先了解行業挑戰或痛點。 然後,他們集成了兩種或兩種以上同類最佳的技術,為客戶創建統一且可重複的部署體驗。 提供商使客戶快速啟動並運行。 他們為客戶提供單一聯繫點,提供相關的定制、諮詢和解決方案支援。 結果是一個整合的解決方案,包括技術和服務。

有關整合式解決方案計畫的概述,請觀看[此視訊](https://aka.ms/AA5qos4)。

## <a name="benefits-of-integrated-solutions-in-the-marketplace"></a>整合式解決方案在市場上的優勢

* 客戶可以通過市場找到解決端到端業務挑戰的打包解決方案。 由於技術是預先集成的,因此他們獲得加速的價值時間。 客戶避免單獨參與以零碎方式部署的多個應用。
* IP 與其他解決方案和服務一起打包,以提高專家服務合作夥伴提供的技術的價值。
* 服務合作夥伴使用同類最佳的技術創建預先定義的預整合解決方案,以擴展、建立和擴展更多客戶帳戶。
* Microsoft 可以使用這些預先集成、與行業一致的解決方案,以便更輕鬆地共同銷售給客戶。 定價和條款提前排序。

## <a name="eligibility-criteria"></a>適用性準則

* 參與合作夥伴必須具有 MPN ID。
* 參與的服務提供者應做好共同銷售準備。
* 參與的軟體供應商應做好智慧財產權共同銷售準備。
* 至少有一個參與合作夥伴必須在以下領域之一擁有金牌能力:

    * 應用程式開發
    * 應用程式整合
    * 應用程式生命週期管理
    * 雲端平台
    * 資料分析
    * 資料中心
    * 資料平台
    * DevOps

* 該解決方案必須至少包括兩個合作夥伴組織,將IP和服務整合在一起,以獲得完整的客戶解決方案和體驗。
* 合作夥伴 IP 必須建構在 Azure、動態 365 或電源平臺上。
* 集成解決方案的 IP 需要與軟體供應商的 IP 和基本服務提供者的產品區別開來。
* 所有合作夥伴都共用創建可重複產品的目標,該產品可通過 Microsoft AppSource、Azure 應用商店或兩者進行最終交易。

## <a name="submit-a-nomination"></a>提交提名

希望參與該計劃的合作夥伴應確定要使用的其他合作夥伴。 他們應填寫[綜合解決方案提名表](https://aka.ms/AA5qicu)。 集成解決方案計劃團隊審核提名並在 10 個工作日內跟進。 團隊提供批准和後續步驟。

## <a name="develop-an-integrated-solution"></a>開發整合解決方案

獲得提名後,請使用連結的計畫資源開發集成解決方案。 請遵循下列步驟：

1. 以研討會格式完成關鍵業務和技術討論,以定義解決方案和集成方法。

    有關解決方案的討論應包括以下幾點:
    * 影響範圍
    * 目標客戶
    * 價值主張
    * 定價
    * 授權
    * 條款和條件
    * 支援方法
    * 行銷活動
    * 銷售流程
    * 活動

    您關於整合方法的討論應包括以下幾點:
    * 整合點
    * 方案架構
    * 驗證
    * 使用者介面
    * 監視
    * 部署機制
    * API
    * 統一資料儲存庫

    在開發過程中使用這些資源:

    * [商務決策研討會討論指南](https://aka.ms/AA5qicx)
    * [技術決策研討會討論指南](https://aka.ms/AA5qid1)
    * [快速入門視頻:商務和技術研討會](https://aka.ms/AA5qos9)

1. 確定跨合作夥伴項目經理,以推動您在業務和技術研討會中制定的專案計劃和時程表。

1. 開發解決方案的完整技術整合。

1. 在 Microsoft AppSource 或 Azure 應用商店中確定解決方案定價和單個價格點。

1. 完成 Microsoft AppSource 或 Azure 應用商店列表的行銷宣傳資料,包括:

    * 組合解決方案名稱。
    * 集成解決方案的清單說明。 遵循[報價清單最佳實務](./gtm-offer-listing-best-practices.md)。
    * 向客戶宣傳一頁。 使用[單頁機樣本](https://aka.ms/AA5s08a)。
    * 客戶演示文稿。 使用[簡報範本](https://aka.ms/AA5s7ql)。
    * 組合解決方案的徽標。

## <a name="publish-your-integrated-solution"></a>發表整合解決方案

完成技術整合和行銷宣傳資料後,請參閱[Microsoft AppSource 和 Azure 應用商店諮詢服務的](./consulting-services.md)發行者指南。 使用此資源可確定解決方案將在 Microsoft AppSource 或 Azure 應用商店中發佈。 還使用本指南準備發佈專案並完成發佈過程。

儘管諮詢服務提供有五種服務類型,但集成解決方案必須是概念證明或完全實現。

## <a name="go-to-market-with-your-integrated-solution"></a>使用整合式解決方案進入市場

當解決方案位於 Microsoft AppSource 或 Azure 應用商店中時,您需要透過行銷活動來提高知名度和需求。 查看[專案計劃和時程表](https://aka.ms/AA5qiuc)中列出的活動。 此資源包括合作夥伴可以在自己的團隊中相互推動的活動。 然後,為在市場上發佈的合作夥伴獲取[上市優惠](./gtm-your-marketplace-benefits.md#list-trial-and-consulting-benefits)。 所有整合解決方案產品均有資格獲得試用級權益。
