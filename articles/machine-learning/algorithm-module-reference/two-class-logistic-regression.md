# <a name="two-class-logistic-regression-module"></a>雙類邏輯回歸模組

本文介紹 Azure 機器學習設計器（預覽）中的模組。

使用此模組可以創建邏輯回歸模型，該模型可用於預測兩個（和僅兩個）結果。 

邏輯回歸是一種眾所周知的統計技術，用於對多種問題進行建模。 該演算法是一種*監督學習*方法; 因此，必須提供已包含結果以訓練模型的資料集。  

### <a name="about-logistic-regression"></a>關於邏輯回歸  

邏輯回歸是統計中用於預測結果概率的已知方法，尤其適用于分類任務。 該演算法通過將資料擬合到物流函數來預測事件發生的概率。
  
在此模組中，分類演算法針對二分變數或二進位變數進行了優化。 如果需要對多個結果進行分類，請使用[多類邏輯回歸](./multiclass-logistic-regression.md)模組。

##  <a name="how-to-configure"></a>如何設定  

要訓練此模型，必須提供包含標籤或類列的資料集。 由於此模組適用于兩類問題，因此標籤或類列必須只包含兩個值。 

例如，標籤列可能是 [投票]，可能的值為"是"或"否"。 或者，它可能是[信用風險]，可能的值為"高"或"低"。 
  
1.  將**雙類邏輯回歸**模組添加到管道中。  
  
2.  通過設置 **"創建培訓師模式"** 選項，指定如何訓練模型。  
  
    -   **單一參數**：如果您知道如何配置模型，則可以提供一組特定的值作為參數。  

    -   **參數範圍**：如果您不確定最佳參數，則可以使用[Tune 模型 Hyper 參數](tune-model-hyperparameters.md)模組找到最佳參數。 提供一些值範圍，培訓師會對多個設置組合進行遍舍，以確定生成最佳結果的值的組合。
  
3.  對於**優化容差**，指定在優化模型時要使用的閾值。 如果反覆運算之間的改進低於指定的閾值，則演算法被視為已融合在解決方案上，並且訓練將停止。  
  
4.  對於**L1 正化權重**和**L2 正規化權重**，鍵入用於正則參數 L1 和 L2 的值。 非零值建議用於兩者。  
     *正規化*是一種通過懲罰具有極端係數值的模型來防止過度學習的方法。 正規化的工作原理是將與係數值相關的懲罰添加到假設的錯誤中。 因此，具有極端係數值的準確模型將受到更多懲罰，但具有更保守值的較不精確的模型將受到的懲罰更少。  
  
     L1 與 L2 regularization 有不同的效果，並使用。  
  
    -   L1 可以套用到疏鬆的模型，使用高維度資料時，這是很有用。  
  
    -   相較之下，L2 regularization 是不是疏鬆的資料。  
  
     此演算法支援 L1 和 L2 正規化值的線性組合：<code>x = L1</code>即<code>y = L2</code>，如果<code>ax + by = c</code>和 ，然後定義正規化術語的線性範圍。  
  
    > [!NOTE]
    >  想要瞭解有關 L1 和 L2 正規化的更多詳細資訊？ 以下本文討論了 L1 和 L2 正則如何不同，以及它們如何影響模型擬合，以及邏輯回歸和神經網路模型的代碼示例：[用於機器學習的 L1 和 L2 正則](https://msdn.microsoft.com/magazine/dn904675.aspx)  
    >
    > 為邏輯回歸模型設計了不同的L1和L2術語的線性組合：例如，[彈性淨正則](https://wikipedia.org/wiki/Elastic_net_regularization)化。 我們建議您參考這些組合來定義在模型中有效的線性組合。
      
5.  對於**L-BFGS 的記憶體大小**，指定用於*L-BFGS*優化的記憶體量。  
  
     L-BFGS 代表"有限記憶體布羅伊登-弗萊徹-戈德法布-尚諾"。 這是常用於參數估計的最佳化演算法。 此參數指出要儲存的通過位置和梯度數目，以計算下一個階梯。  
  
     此最佳化參數限制用來計算下一個步驟和方向的記憶體數量。 當您指定較少的記憶體時，訓練是更快，但較不精確。  
  
6.  對於**亂數種子**，鍵入整數值。 如果希望結果在同一管道的多次運行中可重現，則定義種子值非常重要。  
  
  
8. 向管道委任標記的資料集，並連接其中一個[訓練模組](module-reference.md)。  
  
    -   如果將 **"創建訓練器模式"** 設置為 **"單一參數**"，請使用[訓練器模型](./train-model.md)模組。  
  
9. 提交管道。  
  
## <a name="results"></a>結果

培訓完成後：
 
  
+ 要對新資料進行預測，請使用經過訓練的模型和新資料作為[分數模型](./score-model.md)模組的輸入。
