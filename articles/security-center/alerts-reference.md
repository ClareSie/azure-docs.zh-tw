# <a name="security-alerts---a-reference-guide"></a>安全警報 - 參考指南

本文列出了在 Azure 安全中心的威脅保護模組中看到的安全警報。 環境中顯示的警報取決於您保護的資源和服務以及自訂配置。

要瞭解如何回應這些警報，請參閱[在 Azure 安全中心管理和回應安全警報](security-center-managing-and-responding-alerts.md)。

要瞭解如何匯出警報（和建議），請參閱[匯出安全警報和建議（預覽）。](continuous-export.md)

警報表下方是一個表，用於對這些警報的意圖進行分類的 Azure 安全中心終止鏈。

## <a name="intentions"></a>意圖

瞭解攻擊的意圖可以説明您更輕鬆地調查和報告事件。 Azure 安全中心警報包括"意圖"欄位，以説明進行這些努力。

描述網路攻擊從偵察到資料外泄過程的一系列步驟通常被稱為"殺傷鏈"。 

安全中心支援的殺滅鏈意圖基於[MITRE&trade; ATT&CK 框架](https://attack.mitre.org/matrices/enterprise)，下表對此進行了說明。

|Intent|描述|
|------|-------|
|**攻擊前**</br>（替換探測）|Pre攻擊可能是嘗試訪問特定資源，而不管惡意意圖如何，也可以是嘗試訪問目標系統以在利用之前收集資訊的失敗嘗試。 此步驟通常被檢測為嘗試，源自網路外部，以掃描目標系統並標識進入點。</br>有關攻擊前階段的更多詳細資訊，請閱讀[MITRE 的頁面](https://attack.mitre.org/matrices/pre/)。|
|**初始訪問**</br>（取代剝削）|初始訪問是攻擊者設法在受攻擊的資源上站穩腳跟的階段。 此階段與計算主機和資源（如使用者帳戶、證書等）相關。在此階段之後，威脅參與者通常能夠控制資源。|
|**持續性**|持久性是對系統的任何訪問、操作或配置更改，該更改使威脅參與者在該系統上具有持久性。 威脅參與者通常需要通過系統重新開機、憑據丟失或其他需要遠端存取工具才能重新開機或提供備用後門才能重新獲得存取權限的中斷來保持對系統的訪問。|
|**許可權升級**|許可權升級是允許攻擊者在系統或網路上獲取更高級別許可權的操作的結果。 某些工具或操作需要更高級別的特權才能工作，並且在整個操作的多個點可能是必需的。 具有訪問特定系統的許可權或執行對手實現其目標所需的特定功能的使用者帳戶也可能被視為許可權的升級。|
|**防禦逃生**|防禦規避包括攻擊者可能用來逃避檢測或避免其他防禦的技術。 有時，這些操作與其他類別的技術相同（或變化），這些技術具有顛覆特定防禦或緩解的附加好處。|
|**憑據訪問**|憑據訪問表示導致訪問或控制企業環境中使用的系統、域或服務憑據的技術。 對手可能會嘗試從使用者或管理員帳戶（本地系統管理員或具有管理員存取權限的域使用者）獲取合法憑據，以便在網路上使用。 在網路中具有足夠的存取權限，攻擊者可以創建帳戶，以便以後在環境中使用。|
|**發現**|發現包括允許攻擊者獲得有關系統和內部網路的知識的技術。 當對手獲得新系統時，他們必須將自己定位到他們現在所掌握的，以及該系統在入侵期間運營時對其當前目標或總體目標有何好處。 作業系統提供了許多本機工具，有助於在此洩露後的資訊收集階段。|
|**橫向運動**|橫向移動包括使對手能夠訪問和控制網路上的遠端系統的技術，並且可以但不一定包括在遠端系統上執行工具。 橫向移動技術允許攻擊者從系統收集資訊，而無需其他工具，如遠端存取工具。 攻擊者可以將橫向移動用於多種目的，包括遠端執行工具、轉向其他系統、訪問特定資訊或檔、訪問其他憑據或造成效果。|
|**執行**|執行策略表示導致在本地或遠端系統上執行對手控制代碼的技術。 這種策略通常與橫向移動結合使用，以擴大對網路上的遠端系統的訪問。|
|**收集**|集合包括用於在滲出之前從目標網路識別和收集資訊（如敏感檔）的技術。 此類別還涵蓋系統或網路上的攻擊者可能查找資訊進行滲透的位置。|
|**滲漏**|滲漏是指導致或説明對手從目標網路中刪除檔和資訊的技術和屬性。 此類別還涵蓋系統或網路上的攻擊者可能查找資訊進行滲透的位置。|
|**命令和控制**|命令和控制策略表示對手如何與目標網路中其控制系統通信。|
|**影響**|影響事件主要嘗試直接降低系統、服務或網路的可用性或完整性;包括操縱資料以影響業務或運營流程。 這通常是指勒索軟體、汙損、資料操作等技術。|
