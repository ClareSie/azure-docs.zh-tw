# <a name="security-recommendations"></a>安全性建議

適用于 IoT 的 Defender 會掃描您的 Azure 資源和 IoT 裝置，並提供安全性建議，以減少受攻擊面。
安全性建議是可採取動作的，旨在協助客戶遵循安全性最佳作法。

在本文中，您會找到可在 IoT 中樞和/或 IoT 裝置上觸發的建議清單。

## <a name="recommendations-for-iot-devices"></a>IoT 裝置的建議

裝置建議會提供見解和建議，以改善裝置安全性狀態。

| Severity | 名稱                                                      | 資料來源 | 描述                                                                                                                                                                                           |
|----------|-----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 中   | 開啟裝置上的埠                                      | 代理程式       | 在裝置上找到接聽端點。                                                                                                                                                        |
| 中   | 在其中一個鏈中找到寬鬆的防火牆原則。 | 代理程式       |  (輸入/輸出) 找到允許的防火牆原則。 防火牆原則預設應拒絕所有流量，並定義規則以允許裝置的必要通訊。                               |
| 中   | 在輸入鏈中找到寬鬆的防火牆規則     | 代理程式       | 在防火牆中找到一個規則，其中包含廣泛 IP 位址或埠的寬鬆模式。                                                                                    |
| 中   | 在輸出鏈中找到寬鬆的防火牆規則    | 代理程式       | 在防火牆中找到一個規則，其中包含廣泛 IP 位址或埠的寬鬆模式。                                                                                   |
| 中   | 作業系統基準驗證失敗           | 代理程式       | 裝置不符合 [CIS Linux 基準](https://www.cisecurity.org/cis-benchmarks/)檢驗。                                                                                                        |

### <a name="operational-recommendations-for-iot-devices"></a>IoT 裝置的操作建議

操作建議可提供深入解析和建議，以改善安全性代理程式設定。

| Severity | 名稱                                    | 資料來源 | 描述                                                                       |
|----------|-----------------------------------------|-------------|-----------------------------------------------------------------------------------|
| 低      | 代理程式傳送未運用訊息          | 代理程式       | 過去24小時內10% 或更多的安全性訊息小於 4 KB。  |
| 低      | 安全性對應項設定不是最佳的 | 代理程式       | 安全性對應項設定不是最佳的。                                        |
| 低      | 安全性對應項設定衝突    | 代理程式       | 安全性對應項設定中已識別出衝突。 |                          |
|

## <a name="recommendations-for-iot-hub"></a>IoT 中樞的建議

建議警示提供改進環境安全性狀態的動作見解和建議。

| Severity | 名稱                                                     | 資料來源 | 描述                                                                                                                                                                                                             |
|----------|----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 高     | 多個裝置使用的相同驗證認證 | IoT 中樞     | IoT 中樞驗證認證適用于多個裝置。 這可能表示模擬合法裝置的非法裝置。 重複的認證使用會提高惡意執行者裝置模擬的風險。 |
| 中   | 預設 IP 篩選原則應為拒絕                  | IoT 中樞     | IP 篩選器設定應該已針對允許的流量定義規則，而且預設應拒絕所有其他流量。                                                                                                     |
| 中   | IP 篩選規則包含大型 IP 範圍                   | IoT 中樞     | 允許 IP 篩選規則的來源 IP 範圍過大。 過度寬鬆的規則可能會將您的 IoT 中樞公開給惡意執行者。                                                                                       |
| 低      | 在 IoT 中樞中啟用診斷記錄                       | IoT 中樞     | 啟用記錄並保留記錄最長一年。 保留記錄可讓您在發生安全性事件或網路遭到入侵時，重新建立活動線索供調查之用。                                       |
|
