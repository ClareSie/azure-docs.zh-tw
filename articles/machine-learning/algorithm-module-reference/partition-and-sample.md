# <a name="partition-and-sample-module"></a>資料分割和範例模組

本文描述 Azure Machine Learning 設計工具中的模組。

使用資料分割和範例模組來對資料集執行取樣，或從您的資料集建立資料分割。

取樣是重要的工具，在機器學習，因為它可讓您同時維持相同值的比例減少資料集的大小。 此模組支援在機器學習中有幾個重要的相關工作： 

- 將您的資料分割成相同大小的多個子區段。 

  您可以使用資料分割進行交叉驗證，或將案例指派給隨機群組。

- 將資料分隔成群組，然後使用特定群組中的資料。 

  將案例隨機指派給不同的群組之後，您可能需要修改僅與一個群組相關聯的功能。

- 採樣。 

  您可以將資料的百分比解壓縮、套用隨機取樣，或選擇要用來平衡資料集的資料行，並對其值執行分層取樣。

- 建立較小的資料集進行測試。 

  如果您有大量資料，您可能只想在設定管線時使用前 *n* 個數據列，然後在建立模型時切換到使用完整的資料集。 您也可以使用取樣來建立較小的資料集，以供開發使用。

## <a name="configure-the-module"></a>設定模組

此模組支援下列將資料分割成資料分割或進行取樣的方法。 先選擇方法，然後再設定該方法所需的其他選項。

- Head
- 取樣
- 指派給折迭
- 挑選摺疊

### <a name="get-top-n-rows-from-a-dataset"></a>從資料集取得前 N 個數據列

使用此模式可取得的第一個*n*資料列。 如果您想要在少量的資料列上測試管線，而不需要以任何方式平衡或取樣資料，這個選項就很有用。

1. 將資料 **分割和範例** 模組新增至介面中的管線，並串連資料組。  

1. **分割或取樣模式**：將此選項設定為 **Head**。

1. **要選取的資料列數目**：輸入要傳回的資料列數目。

   資料列數目必須為非負整數。 如果選取的資料列數目大於資料集中的資料列數目，則會傳回整個資料集。

1. 提交管線。

此模組會輸出單一資料集，其中只包含指定的資料列數目。 永遠會從資料集的頂端讀取資料列。

### <a name="create-a-sample-of-data"></a>建立資料範例

此選項支援簡單的隨機取樣或階層式隨機取樣。 如果您想要建立較小的代表性範例資料集來進行測試，這會很有用。

1. 將 **分割區和範例** 模組新增至您的管線，並串連資料組。

1. **分割或取樣模式**：將此選項設定為 [  **取樣**]。

1. **取樣率**：輸入介於0和1之間的值。 這個值會指定應包含在輸出資料集中之源資料集的資料列百分比。

   例如，如果您只想要取得原始資料集的一半，請輸入， `0.5` 表示取樣率應為50%。

   輸入資料集的資料列會根據指定的比例，隨機放置並選擇性地放入輸出資料集。

1. **取樣的隨機種子**：選擇性地輸入要做為種子值的整數。

   如果您想要每次都以相同的方式分割資料列，這個選項就很重要。 預設值為 **0**，表示會根據系統時鐘產生起始種子。 每次執行管線時，此值可能會產生稍微不同的結果。

1. **分層分割**：若要進行取樣，請選取此選項，如果資料集內的資料列在取樣之前，必須平均除以某些索引鍵資料行，請選取此選項。

   針對 [ **取樣的分層索引鍵資料行**]，選取要在分割資料集時使用的單一分層資料 *行* 。 然後，資料集中的資料列會分割如下：

   1. 所有輸入資料列都會依指定的分層資料行中的值， (分層) 群組。

   1. 將每個群組內的資料列打散

   1. 選擇性地將每個群組加入至輸出資料集，以符合指定的比率。


1. 提交管線。

   使用這個選項時，模組會輸出單一資料集，其中包含資料的代表性取樣。 資料集的其餘、unsampled 部分不會輸出。 

## <a name="split-data-into-partitions"></a>將資料分割成資料分割

當您想要將資料集分割成資料的子集時，請使用此選項。 當您想要建立自訂的折迭數目來進行交叉驗證，或將資料列分割成多個群組時，這個選項也很有用。

1. 將 **分割區和範例** 模組新增至您的管線，並串連資料組。

1. 若為數據 **分割或取樣模式**，請選取 [ **指派給**折迭]。

1. **在資料分割中使用取代**：如果您想要將取樣的資料列放回資料列集區，以避免重複使用，請選取此選項。 因此，相同的資料列可能會指派給數個折迭。

   如果您未使用取代 (預設選項) ，則取樣的資料列不會放回資料列集區，以供潛在重複使用。 因此，每個資料列只能指派給一個折迭。

1. **隨機化分割**：如果您想要將資料列隨機指派給折迭，請選取此選項。

   如果您未選取此選項，則會透過迴圈配置資源方法，將資料列指派給折迭。

1. **隨機種子**：選擇性地輸入要作為種子值的整數。 如果您想要每次都以相同的方式分割資料列，這個選項就很重要。 否則，預設值為 **0** 表示將使用隨機啟動種子。

1. **指定**分割區方法：使用下列選項，指出您要如何將資料分配至每個資料分割：

   - **平均分割**：使用此選項可在每個分割區中放置相等數目的資料列。 若要指定輸出分割區的數目，請在 [ **指定要平均分割** 的折迭數目] 方塊中輸入整數。

   - **具有自訂比例的分割**區：使用此選項可將每個分割區的大小指定為逗點分隔清單。

     例如，假設您想要建立三個磁碟分割。 第一個分割區會包含50% 的資料。 其餘兩個分割區會包含25% 的資料。 在以 **逗號分隔的比例清單** 中，輸入下列數位： **.5、25、25**。

     所有資料分割大小的總和必須最多加1。

     如果您輸入的數位加上 *小於 1*，則會建立額外的資料分割來保存其餘的資料列。 例如，如果您輸入值 **.2** 和 **.3**，則會建立第三個數據分割來保存所有資料列的剩餘50%。
     
     如果您輸入的數位加上 *多達1個以上*，當您執行管線時，就會引發錯誤。

1. **分層分割**：如果您想要在分割時分層資料列，然後選擇 [ _分層_] 資料行，請選取此選項。

1. 提交管線。

   使用這個選項時，模組會輸出多個資料集。 系統會根據您指定的規則來分割資料集。

### <a name="use-data-from-a-predefined-partition"></a>使用預先定義之分割區中的資料  

當您將資料集分割成多個資料分割，而現在想要載入每個資料分割以進行進一步的分析或處理時，請使用此選項。

1. 將 **分割區和範例** 模組新增至管線。

1. 將模組連接到先前的資料 **分割與範例**的輸出。 該實例必須使用 [ **指派給** 折迭] 選項來產生一些磁碟分割。

1. **分割或取樣模式**：選取 [ **挑選折**迭]。

1. **指定要取樣的折**迭：選取要使用的資料分割，方法是輸入其索引。 分割區索引是以1為基礎。 例如，如果您將資料集分成三個部分，資料分割會有索引1、2和3。

   如果您輸入不正確索引值，則會引發設計階段錯誤：「錯誤0018：資料集包含不正確資料」。

   除了以折迭分組資料集之外，您還可以將資料集分成兩個群組：目標折迭，以及其他專案。 若要這樣做，請輸入單一折迭的索引，然後選取 [ **挑選所選折** 迭的選項] 選項，以取得指定之折迭中的資料。

1. 如果您正在使用多個資料分割，則必須加入更多資料 **分割和範例** 模組的實例來處理每個資料分割。

   例如，第二個數據列中的資料 **分割和範例** 模組設定為 [ **指派給**折迭]，而第三個數據列中的模組設定為 [ **挑選折**迭]。   

   ![資料分割和取樣](./media/module/partition-and-sample.png)

1. 提交管線。

   使用這個選項時，模組會輸出單一資料集，其中只包含指派給該折迭的資料列。

> [!NOTE]
>  您無法直接查看折迭的稱號。 它們只存在於中繼資料中。
