# <a name="integrate-3270-screen-driven-apps-on-ibm-mainframes-with-azure-by-using-azure-logic-apps-and-ibm-3270-connector"></a>通過使用 Azure 邏輯應用和 IBM 3270 連接器，在 IBM 大型機上集成 3270 螢幕驅動應用

> [!NOTE]
> 此連接器處於[*公共預覽版*](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)中。 

借助 Azure 邏輯應用和 IBM 3270 連接器，您可以訪問並運行 IBM 大型機應用，這些應用通常通過 3270 個模擬程式螢幕導航來驅動。 這樣，您可以通過創建與 Azure 邏輯應用的自動化工作流，將 IBM 大型機應用與 Azure、Microsoft 和其他應用、服務和系統集成。 連接器使用 TN3270 協定與 IBM 大型機通信，並且在所有 Azure 邏輯應用區域都可用，Azure 政府和 Azure 中國 21Vianet 除外。 如果您是邏輯應用的新增功能，請查看什麼是[Azure 邏輯應用？](../logic-apps/logic-apps-overview.md)

本文介紹了使用 3270 連接器的以下幾個方面： 

* 為什麼在 Azure 邏輯應用中使用 IBM 3270 連接器以及該連接器如何運行 3270 螢幕驅動應用

* 使用 3270 連接器的先決條件和設置

* 向邏輯應用添加 3270 連接器操作的步驟

## <a name="why-use-this-connector"></a>為什麼要使用此連接器？

要訪問 IBM 大型機上的應用，通常使用 3270 終端模擬器，通常稱為"綠屏"。 此方法是經過時間測試的方法，但有限制。 儘管主機整合服務器 （HIS） 可説明您直接使用這些應用，但有時可能無法分離螢幕和業務邏輯。 或者，您可能不再瞭解主機應用程式的工作原理。

為了擴展這些方案，Azure 邏輯應用中的 IBM 3270 連接器與用於記錄或"捕獲"用於特定任務的主機螢幕的 3270 設計工具配合使用，通過大型機應用定義該任務的導航流，並定義具有該任務的輸入和輸出參數的方法。 設計工具將該資訊轉換為 3270 連接器在調用邏輯應用中表示該任務的操作時使用的中繼資料。

從設計工具生成元資料檔案後，將該檔添加到 Azure 中的集成帳戶。 這樣，邏輯應用可以在添加 3270 連接器操作時訪問應用的中繼資料。 連接器從集成帳戶讀取元資料檔案，處理 3270 螢幕的導航，並動態顯示 3270 連接器操作的參數。 然後，您可以將資料提供給主機應用程式，並且連接器將結果返回到邏輯應用。 這樣，可以將舊應用與 Azure、Microsoft 以及 Azure 邏輯應用支援的其他應用、服務和系統集成。

## <a name="prerequisites"></a>Prerequisites

* Azure 訂用帳戶。 如果您沒有 Azure 訂用帳戶，請先[註冊免費的 Azure 帳戶](https://azure.microsoft.com/free/)。

* [有關如何創建邏輯應用](../logic-apps/quickstart-create-first-logic-app-workflow.md)的基本知識

* 推薦：[整合服務環境 （ISE）](../logic-apps/connect-virtual-network-vnet-isolated-environment.md) 

  您可以選擇此環境作為創建和運行邏輯應用的位置。 ISE 提供從邏輯應用到 Azure 虛擬網路中受保護的資源的訪問。

* 用於自動和運行 3270 螢幕驅動應用的邏輯應用

  IBM 3270 連接器沒有觸發器，因此請使用另一個觸發器啟動邏輯應用，如 **"重複"** 觸發器。 然後，您可以添加 3270 個連接器操作。 要開始，[請創建一個空白邏輯應用](../logic-apps/quickstart-create-first-logic-app-workflow.md)。 
  如果使用 ISE，請選擇該 ISE 作為邏輯應用的位置。

* [下載並安裝3270設計工具](https://aka.ms/3270-design-tool-download)。
唯一的先決條件是[微軟.NET框架4.6.1。](https://aka.ms/net-framework-download)

  此工具可説明您記錄應用中添加和運行為 3270 連接器操作的任務的螢幕、導航路徑、方法和參數。 該工具生成主機集成設計器 XML （HIDX） 檔，該檔為連接器提供必要的中繼資料，用於驅動大型機應用。
  
  下載並安裝此工具後，請按照以下步驟連接到主機：

  1. 打開 3270 設計工具。 在 **"會話"** 功能表中，選擇 **"主機會話**"。
  
  1. 提供 TN3270 主機伺服器資訊。

* [集成帳戶](../logic-apps/logic-apps-enterprise-integration-create-integration-account.md)，它是將 HIDX 檔存儲為地圖的位置，以便邏輯應用可以訪問該檔中的中繼資料和方法定義。 

  確保您的集成帳戶已連結到您正在使用的邏輯應用。 此外，如果您使用 ISE，請確保集成帳戶的位置與邏輯應用使用的 ISE 相同。

* 訪問承載大型機應用的 TN3270 伺服器

<a name="define-app-metadata"></a>

## <a name="create-metadata-overview"></a>創建中繼資料概述

在 3270 螢幕驅動的應用中，螢幕和資料欄位對於您的方案是唯一的，因此 3270 連接器需要有關應用的資訊，您可以將這些資訊作為中繼資料提供。 此中繼資料描述有助於邏輯應用識別和識別螢幕、描述如何在螢幕之間導航、輸入資料的位置以及預期結果的位置的資訊。 要指定和生成此中繼資料，請使用 3270 設計工具，該工具將引導您完成這些特定的*模式*或階段，詳情稍後所述：

* **捕獲**：在此模式下，使用大型機應用記錄完成特定任務所需的螢幕，例如，獲取銀行餘額。

* **導航**：在此模式下，您可以指定如何流覽大型機應用螢幕的計畫或路徑，以便執行特定任務。

* **方法**：在此模式下，定義描述螢幕導航路徑的方法，例如`GetBalance`，描述螢幕導航路徑。 您還可以選擇每個螢幕上成為方法的輸入和輸出參數的欄位。

### <a name="unsupported-elements"></a>不支援的元素

設計工具不支援以下元素：

* 部分 IBM 基本映射支援 （BMS） 映射：如果導入 BMS 地圖，則設計工具將忽略部分螢幕定義。
* 輸入/出參數：無法定義輸入/出參數。
* 功能表處理：預覽期間不支援
* 陣列處理：預覽期間不支援

<a name="capture-screens"></a>

## <a name="capture-screens"></a>捕獲螢幕

在此模式下，在每個 3270 螢幕上標記唯一標識該螢幕的專案。 例如，您可以指定一行文本或更複雜的條件集，如特定文本和非空欄位。 您可以通過與主機伺服器的即時連接記錄這些螢幕，也可以從 IBM 基本映射支援 （BMS） 地圖導入此資訊。 即時連接使用 TN3270 模擬器連接到主機。 每個連接器操作都必須映射到單個任務，該任務從連接到會話開始，到斷開會話結束。

1. 如果尚未打開 3270 設計工具。 在工具列上，選擇 **"捕獲**"，以便進入捕獲模式。

1. 要開始錄製，請按 F5 鍵或"**錄製"** 功能表，選擇 **"開始錄製**"。 

1. 在 **"會話"** 功能表中，選擇 **"連接**"。

1. 在 **"捕獲"** 窗格中，從應用中的第一個螢幕開始，針對要錄製的特定任務單一步驟應用。

1. 完成任務後，請像通常一樣從應用登出。

1. 在 **"會話"** 功能表中，選擇 **"斷開連接**"。

1. 要停止錄製，請按"移位 + F5"鍵，或從 **"錄製"** 功能表中，選擇 **"停止錄製**"。

   捕獲任務的螢幕後，設計器工具將顯示表示這些螢幕的縮略圖。 有關這些縮略圖的一些說明：

   * 包含在捕獲的螢幕中，您有一個名為"空"的螢幕。

     首次連接到[CICS](https://www.ibm.com/it-infrastructure/z/cics)時，必須先發送"清除"金鑰，然後才能輸入要運行的事務的名稱。 發送"清除"鍵的螢幕沒有任何*識別屬性*，例如螢幕標題，您可以使用螢幕識別編輯器添加該屬性。 為了表示此螢幕，縮略圖包括名為"空"的螢幕。 以後，可以使用此螢幕來表示輸入事務名稱的螢幕。

   * 預設情況下，捕獲的螢幕的名稱使用螢幕上的第一個單詞。 如果該名稱已存在，則設計工具會用底線和數位（例如"WBGB"和"WBGB_1"）附加名稱。

1. 要為捕獲的螢幕指定更有意義的名稱，請按照以下步驟操作：

   1. 在 **"主機螢幕"** 窗格中，選擇要重命名的螢幕。

   1. 在同一窗格中，在同一窗格中靠近底部，查找 **"螢幕名稱"** 屬性。

   1. 將當前螢幕名稱更改為更具描述性的名稱。

1. 現在指定用於標識每個螢幕的欄位。

   使用 3270 資料流程時，螢幕沒有預設識別碼，因此您需要在每個螢幕上選擇唯一文本。 對於複雜方案，您可以指定多個條件，例如，唯一文本和具有特定條件的欄位。

選擇完識別欄位後，轉到下一個模式。

### <a name="conditions-for-identifying-repeated-screens"></a>識別重複螢幕的條件

對於在螢幕之間導航和區分的連接器，您通常在螢幕上找到唯一文本，您可以將其用作捕獲螢幕之間的識別碼。 對於重複的螢幕，您可能需要更多標識方法。 例如，假設您有兩個螢幕看起來相同，但一個螢幕返回有效值，而另一個螢幕返回一條錯誤訊息。

在設計工具中，可以使用螢幕識別編輯器添加*識別屬性*，例如，螢幕標題，如"獲取帳戶餘額"。 如果您有分叉路徑，並且兩個分支返回相同的螢幕，但結果不同，則需要其他識別屬性。 在運行時，連接器使用這些屬性來確定當前分支和分叉。 以下是您可以使用的條件：

* 特定值：此值與指定位置的指定字串匹配。
* 不是特定值：此值與指定位置的指定字串不匹配。
* 空：此欄位為空。
* 不為空：此欄位不為空。

要瞭解更多資訊，請參閱本主題後面的[示例導航計畫](#example-plan)。

<a name="define-navigation"></a>

## <a name="define-navigation-plans"></a>定義導航計畫

在此模式下，您可以定義用於在大型機應用螢幕中導航特定任務的流或步驟。 例如，有時，應用可以獲取多個路徑，其中一條路徑生成正確的結果，而另一個路徑則建置錯誤。 對於每個螢幕，指定移動到下一個螢幕所需的擊鍵，例如`CICSPROD <enter>`。

> [!TIP]
> 如果要自動執行多個使用相同的連接和斷開連接螢幕的任務，則設計工具提供特殊的連接和斷開連接計畫類型。 定義這些計畫時，可以將它們添加到導航計畫的開始和結束。

### <a name="guidelines-for-plan-definitions"></a>計畫定義準則

* 包括所有螢幕，從連接開始，從斷開連接結束。

* 您可以創建獨立計畫或使用"連接和斷開連接"計畫，這允許您重用所有事務共有的一系列螢幕。

  * "連接"計畫中的最後一個螢幕必須與導航計畫中的第一個螢幕的螢幕相同。

  * 斷開連接計畫中的第一個螢幕必須與導航計畫中的最後一個螢幕相同。

* 捕獲的螢幕可能包含許多重複的螢幕，因此選擇並使用計畫中任何重複螢幕的一個實例。 下面是重複螢幕的一些示例：

  * 登錄螢幕，例如 **，MSG-10**螢幕
  * CICS 的歡迎畫面
  * "清除"或 **"空**屏"

<a name="create-plans"></a>

### <a name="create-plans"></a>建立方案

1. 在 3270 設計工具的工具列上，選擇 **"導航**"以進入導航模式。

1. 要啟動計畫，在 **"導航"** 窗格中，請選擇 **"新建計畫**"。

1. 在 **"選擇新計畫名稱**"下，輸入計畫的名稱。 從 **"類型"** 清單中，選擇計畫類型：

   | 計畫類型 | 描述 |
   |-----------|-------------|
   | **處理程序** | 對於獨立或組合計畫 |
   | **連接** | 對於連接計畫 |
   | **中斷連接** | 對於斷開連接計畫 |
   |||

1. 從 **"主機螢幕"** 窗格中，將捕獲的縮略圖拖動到**導航**窗格中的導航計畫曲面。

   要表示輸入交易記錄名稱的空白螢幕，請使用"空"螢幕。

1. 按描述要定義的任務的順序排列螢幕。

1. 要在設計工具的工具列上定義螢幕（包括分叉和聯接）之間的流路徑，請選擇**Flow**。

1. 選擇流中的第一個螢幕。 拖動並繪製到流中下一個螢幕的連接。

1. 對於每個螢幕，提供**AID 鍵**屬性（注意識別碼）和**固定文本**屬性的值，後者將流移動到下一個螢幕。

   您可能只有 AID 金鑰，或者輔助鍵和固定文本。

完成導航計畫後，可以在[下一個模式下定義方法](#define-method)。

#### <a name="example-identify-repeated-screens"></a>示例：識別重複的螢幕

對於要導航和區分螢幕的連接器，您通常在螢幕上找到唯一文本，您可以將其用作捕獲的螢幕的識別碼。 對於重複的螢幕，您可能需要更多標識方法。 示例計畫有一個分叉，您可以在其中獲取看起來相似的螢幕。 一個螢幕返回帳戶餘額，而另一個螢幕返回錯誤訊息。

設計工具允許您通過使用螢幕識別編輯器添加識別屬性，例如，名為"獲取帳戶餘額"的螢幕標題。 對於類似的螢幕，您需要其他屬性。 在運行時，連接器使用這些屬性來確定分支和分叉。

* 在返回有效輸入的分支（即具有帳戶餘額的螢幕）中，可以添加具有"不為空"條件的欄位。

* 在返回的分支中，可以添加具有"空"條件的欄位。

<a name="define-method"></a>

## <a name="define-methods"></a>定義方法

在此模式下，定義與導航計畫關聯的方法。 對於每個方法參數，您可以指定資料類型，如字串、整數、日期或時間等。 完成後，您可以在即時主機上測試方法，並確認該方法是否按預期方式工作。 然後生成元資料檔案或主機集成設計器 XML （HIDX） 檔，該檔現在具有用於創建和運行 IBM 3270 連接器操作的方法定義。

1. 在 3270 設計工具的工具列上，選擇 **"方法**"以進入"方法"模式。 

1. 在 **"導航"** 窗格中，選擇具有所需輸入欄位的螢幕。

1. 要添加方法的第一個輸入參數，請按照以下步驟操作：

   1. 在 **"捕獲"** 窗格中，在 3270 模擬器螢幕上，選擇整個欄位，而不僅僅是欄位內的文本，這是您希望的第一個輸入。

      > [!TIP]
      > 要顯示所有欄位並確保選擇完整欄位，**請在"查看"** 功能表上選擇 **"所有欄位**"。

   1. 在設計工具的工具列上，選擇 **"輸入欄位**"。 

   要添加更多輸入參數，請為每個參數重複上述步驟。

1. 要添加方法的第一個輸出參數，請按照以下步驟操作：

   1. 在 **"捕獲"** 窗格中，在 3270 模擬器螢幕上，選擇整個欄位，而不僅僅是欄位內的文本，作為第一個輸出。

      > [!TIP]
      > 要顯示所有欄位並確保選擇完整欄位，**請在"查看"** 功能表上選擇 **"所有欄位**"。

   1. 在設計工具的工具列上，選擇 **"輸出欄位**"。

   要添加更多輸出參數，請為每個參數重複上述步驟。

1. 添加所有方法的參數後，為每個參數定義這些屬性：

   | 屬性名稱 | 可能值 | 
   |---------------|-----------------|
   | **資料類型** | 位元組， 日期時間， 十進位， int， 長， 短， 字串 |
   | **欄位填充技術** | 參數支援這些填充類型，必要時用空白填充： <p><p>- **類型**：在欄位中按順序輸入字元。 <p>- **填充**：將欄位的內容替換為字元，必要時用空格填充。 <p>- **擦除類型**：清除欄位，然後按順序在欄位中輸入字元。 |
   | **格式字串** | 某些參數資料類型使用格式字串，該格式字串通知 3270 連接器如何將文本從螢幕轉換為 .NET 資料類型： <p><p>- **日期時間**： 日期時間格式字串遵循[.NET 自訂日期和時間格式字串](https://docs.microsoft.com/dotnet/standard/base-types/custom-date-and-time-format-strings)。 例如，日期`06/30/2019`使用格式字串`MM/dd/yyyy`。 <p>- **十進位**： 十進位格式字串使用[COBOL 圖片子句](https://www.ibm.com/support/knowledgecenter/SS6SG3_5.2.0/com.ibm.cobol52.ent.doc/PGandLR/ref/rlddepic.html)。 例如，數位`100.35`使用格式字串`999V99`。 |
   |||

## <a name="save-and-view-metadata"></a>保存和查看中繼資料

定義方法後，但在測試方法之前，將到目前為止定義的所有資訊保存到 RAP （.rap） 檔中。
您可以在任何模式下隨時保存到此 RAP 檔。 設計工具還包括一個示例 RAP 檔，您可以通過流覽此位置的設計工具的安裝資料夾並打開"WoodgroveBank.rap"檔來打開和查看該檔：

`..\Program Files\Microsoft Host Integration Server - 3270 Design Tool\SDK\WoodgroveBank.rap`

但是，如果您嘗試保存對示例 RAP 檔的更改，或者在檔保留在設計工具的安裝資料夾中時從示例 RAP 檔生成 HIDX 檔，則可能會收到"訪問被拒絕"錯誤。 預設情況下，設計工具在程式檔資料夾中安裝，而不具有較高的權限。 如果出現錯誤，請嘗試以下解決方案之一：

* 將示例檔案複製到其他位置。
* 以管理員身份運行設計工具。
* 使自己成為 SDK 資料夾的擁有者。

## <a name="test-your-method"></a>測試方法

1. 要對即時主機運行方法，而仍處於"方法"模式時，請按 F5 鍵或從設計工具的工具列，選擇 **"運行**"。

   > [!TIP]

1. 要繼續下一個螢幕，請選擇 **"下一個**"。

## <a name="generate-and-upload-hidx-file"></a>生成和上傳 HIDX 檔

準備就緒後，生成 HIDX 檔，以便您可以上載到集成帳戶。 3270 設計工具在保存 RAP 檔的新子資料夾中創建 HIDX 檔。

1. 在 3270 設計工具的工具列上，選擇**生成代碼**。

1. 轉到包含 RAP 檔的資料夾，然後打開該工具在生成 HIDX 檔後創建的子資料夾。 確認該工具創建了 HIDX 檔。

1. 登錄到 Azure[門戶](https://portal.azure.com)，然後查找集成帳戶。

1. [按照以下類似步驟添加地圖](../logic-apps/logic-apps-enterprise-integration-liquid-transform.md)，將 HIDX 檔作為映射添加到集成帳戶，但當您選擇地圖類型時，請選擇**HIDX**。

在本主題的後面部分，當您首次向邏輯應用添加 IBM 3270 操作時，系統會通過提供連接資訊（如集成帳戶和主機伺服器的名稱）來在邏輯應用和主機伺服器之間創建連接. 創建連接後，可以選擇以前添加的 HIDX 檔、要運行的方法以及要使用的參數。

完成所有這些步驟後，可以使用在邏輯應用中創建的操作連接到 IBM 大型機、應用的磁碟機螢幕、輸入資料、返回結果等。 您還可以繼續向邏輯應用添加其他操作，以便與其他應用、服務和系統集成。
