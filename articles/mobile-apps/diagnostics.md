# <a name="monitor-failures-and-errors-in-real-time-for-your-mobile-application"></a>即時監控移動應用程式的故障和錯誤
應用程式被數千個使用者使用後，它很可能出現 Bug。 您的使用者可能會遇到意外的應用程式失敗。 從使用者的角度來看，他們只想是一個工作和可靠的應用程式。 錯誤和失敗的應用程式不能提供良好的使用者體驗。 失望的使用者將卸載應用程式，留下不好的評論，或公開批評和抱怨他們的經驗。

瞭解實際故障行為對於移動應用程式至關重要。 發佈到應用商店後，開發人員的工作不會立即完成。 持續監控和衡量性能以確保應用穩定性並保持使用者滿意度非常重要。 您需要的是一種監視應用程式的方法。 您需要瞭解出現問題時發生的情況，以便確定優先順序並修復對業務需求至關重要的問題。

## <a name="importance-of-failure-monitoring"></a>故障監控的重要性
作為開發人員，您需要：
- 瞭解有關應用程式運行狀況的詳細資訊，以及應用程式如何為使用者執行。
- 查找使用者設備上發生故障的根本原因和原因以及發生位置。
- 主動地對問題進行會審，解決故障，使應用程式穩定。
- 構建更好的應用程式並更快地反覆運算，以提供出色的使用者體驗，以便使用者繼續使用您的應用程式。

利用以下服務來監視應用程式故障、診斷問題並快速修復它們，從而提供穩定和高品質的移動應用程式體驗。

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[應用中心診斷](/appcenter/diagnostics/)服務可説明您監視應用程式的運行狀況。 它提供了詳細的見解，以便您可以瞭解發生了什麼以及應用程式失敗的原因。 應用中心診斷還可説明您確定故障報告的優先順序和管理。

**主要功能**
- 即時、自動故障和錯誤，巧妙分組，以突出您需要的最嚴重問題和相關資訊。
- 完整的堆疊追蹤和設備資料，提供上下文和清晰度，以説明您確定根本原因。
- 強大的搜索功能，説明您找到您最關心的問題。
- 分析集成，以便您可以瞭解使用者的行為並查看導致故障的事件。
- 自訂屬性、使用者 ID 和附件，以便您可以添加更多上下文以説明診斷問題。
- 警報、通知和錯誤跟蹤器集成，以便您可以隨時瞭解影響使用者的新故障和意外故障。
- 單行式 SDK 集成，可在幾分鐘內開始使用此服務。
- 對 iOS、Android、macOS、tvOS、Xamarin、反應原生、統一、科爾多瓦、WPF 和 WinForms 的平臺支援。
