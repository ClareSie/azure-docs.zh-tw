# <a name="comparing-microsoft-appsource-and-azure-marketplace"></a>比較 Microsoft AppSource 和 Azure Marketplace

Microsoft 商用 marketplace 包含 [Microsoft AppSource](https://appsource.microsoft.com) 和 [Azure Marketplace](https://azuremarketplace.microsoft.com)。 每個線上商店都提供獨特的客戶需求和不同的目標物件，讓您可以確定您的解決方案或服務可供適當的客戶使用。

## <a name="understanding-the-differences-between-online-stores"></a>瞭解線上商店之間的差異

若要選擇線上商店，請先找出您供應專案的目標物件。 如果您將目標設為多個線上商店的物件，您可以發佈一次，以在多個商店中銷售。

| Azure Marketplace | AppSource |
|:--- |:--- |
| IT 專業人員和開發人員 | 企業使用者與企業決策者 |

下表說明使用線上商店的優點：

| 優點 | Azure Marketplace | AppSource |
|:--- |:--- | :--- |
| **計費彈性** | Azure Marketplace 支援免費、試用、BYOL 和交易供應專案類型。 交易供應專案是以耗用量為基礎或以訂用帳戶為基礎。 以耗用量為基礎的供應專案會依每小時使用量計費，並可提供初始的免費期間。 以訂用帳戶為基礎的服務會每月或每年以每一基座或一般費率計費。| AppSource 支援免費、試用、BYOL 和交易供應專案類型。 交易供應專案是以訂用帳戶為基礎的 SaaS 產品，每月或每年可依每一基座或一般費率計費。 |
|**與其他合作夥伴的往來**  | 您目前無法將服務提供者或交付合作夥伴與供應項目連結。 | 將獨立軟體廠商、系統整合商及受控服務提供者與特定實作案例連結。 您可以合作向新客戶進行銷售。 |
| **自動化** | 您目前無法將服務提供者或交付合作夥伴與供應項目連結。 透過附加元件佈建利用自動化軟體即服務 (SaaS)。 使用「Azure 應用程式：解決方案」範本發佈選項，將 SaaS 型資料收集和部署案例自動化。 | 將獨立軟體廠商、系統整合商及受控服務提供者與特定實作案例連結。 您可以合作向新客戶進行銷售。 |
| **多個雲端類型** | 使用下列類型，發佈適用於公用雲端與內部部署環境的解決方案：<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>區域雲端，包括中國和德國</li></ul> | 目前不支援 Azure Stack、Azure Government 或區域雲端。 |
| **以內容呈現給客戶** | 在內容相關式搜尋的 Azure 入口網站內體驗中提供您的解決方案。 使用「虛擬機器」或「Azure 應用程式：解決方案範本」發佈選項。 | 透過 Microsoft 產品 (例如 Dynamics 365、Power BI 和 Office 365) 的應用程式內體驗，來觸及更多客戶。 |
