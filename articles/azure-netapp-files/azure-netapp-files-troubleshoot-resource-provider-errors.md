# <a name="troubleshoot-azure-netapp-files-resource-provider-errors"></a>針對 Azure NetApp Files 資源提供者錯誤進行疑難排解 

本文介紹了常見的 Azure NetApp 檔資來源提供者錯誤、其原因、解決方案和解決方法（如果可用）。

## <a name="common-azure-netapp-files-resource-provider-errors"></a>常見 Azure NetApp 檔資來源提供者錯誤

***無法更改裸機租戶Id。***  

當您嘗試更新或修補卷且屬性具有更改的值時，`BaremetalTenantId`將發生此錯誤。

* 原因：   
您正在嘗試更新卷，`BaremetalTenantId`並且該屬性的值與存儲在 Azure 中的值不同。
* 解決方案：   
不要在修補程式中包含`BaremetalTenantId`更新（放置）請求。 或者，請確保`BaremetalTenantId`請求中相同。

***無法更改服務等級。***  

當您嘗試更新或修補具有不同服務等級的容量池時，當容量池中已具有卷時，將發生此錯誤。

* 原因：   
當您嘗試更新容量池服務等級時，池包含卷。
* 解決方案：   
從容量池中刪除所有卷，然後更改服務等級。
* 因應措施：   
創建另一個容量池，然後在新的容量池中再次創建卷。

***無法更改池 Id***  

當您嘗試使用已更改`PoolId`的屬性更新或修補容量池時，將發生此錯誤。

* 原因：   
您正在嘗試更新容量池`PoolId`屬性。 屬性`PoolId`是唯讀屬性，無法更改。
* 解決方案：   
不要在修補程式中包含`PoolId`更新（放置）請求。  或者，請確保`PoolId`請求中相同。

***無法更改創建權杖。***

創建卷後嘗試更改檔路徑 （`CreationToken`） 時，將發生此錯誤。 創建卷時`CreationToken`必須設置檔路徑 （ ， 以後不能更改它。

* 原因：   
您正在嘗試在創建卷後更改檔`CreationToken`路徑 （ ）， 這不是受支援的操作。 
* 解決方案：   
如果不需要更改檔路徑，請考慮從請求中刪除參數以關閉錯誤訊息。
* 因應措施：   
如果需要更改檔路徑 （），`CreationToken`可以使用新檔路徑創建新卷，然後將資料移轉到新卷。

***創建權杖必須至少為 16 個字元長。***

當檔路徑 （`CreationToken`） 不符合長度要求時，將發生此錯誤。 檔路徑的長度必須至少為一個字元的長度。

* 原因：   
檔路徑為空。  使用 API 創建卷時，需要創建權杖。 如果使用 Azure 門戶，將自動生成檔路徑。
* 解決方案：   
輸入至少一個字元作為檔路徑 （`CreationToken`。

***無法更改功能變數名稱。***

當您嘗試更改活動目錄中的功能變數名稱時，將發生此錯誤。

* 原因：   
您正在嘗試更新功能變數名稱屬性。
* 解決方案：    
無。 您不能更改功能變數名稱。
* 因應措施：   
使用活動目錄配置刪除所有卷。 然後刪除活動目錄配置並重新創建卷。

***物件匯出策略的重複值錯誤。***

當匯出策略未使用唯一索引定義時，將發生此錯誤。 定義匯出策略時，所有匯出策略規則必須具有介於 1 和 5 之間的唯一索引。

* 原因：   
定義的匯出策略不符合出口策略規則的要求。 您必須至少有一個匯出策略規則，最多有五個匯出策略規則。
* 解決方案：   
確保索引尚未使用，並且索引在 1 到 5 之間。
* 因應措施：   
對嘗試設置的規則使用不同的索引。

***錯誤 [操作] [資源類型名稱]***

當其他錯誤處理在資源執行操作時未能處理錯誤時，將顯示此錯誤。   它包括文本"錯誤"。 `{action}`可以是 的任意 （`getting` `creating`、 `updating`、 `deleting`、 或 。  `{resourceTypeName}`是`resourceTypeName`（例如 ， `netAppAccount`、 `capacityPool` `volume`、 、 等）。

* 原因：   
此錯誤是未處理的異常，原因不知道。
* 解決方案：   
請與 Azure 支援中心聯繫，在日誌中報告詳細原因。
* 因應措施：   
無。

***檔路徑名稱只能包含字母、數位和連字號（"-"）。***

當檔路徑包含不受支援的字元（例如，句點（"."）、逗號（""）、底線（"*"）或貨幣符號 （"$"）時，將發生此錯誤。

* 原因：   
檔路徑包含不支援的字元，例如句點（"."）、逗號（""）、底線（"*"），或貨幣符號（"$"）。
* 解決方案：   
從輸入的檔路徑中刪除不是字母字母、數位或連字號 （"-"） 的字元。
* 因應措施：   
您可以將底線替換為連字號，或使用大寫而不是空格來指示新單詞的開頭。  例如，使用"新卷"而不是"新卷"。

***無法更改檔案系統 Id。***

當您嘗試更改`FileSystemId`時，將發生此錯誤。  更改`FileSystemdId`不是受支援的操作。 

* 原因：   
創建卷時設置檔案系統的 ID。 `FileSystemId`以後無法更改。
* 解決方案：   
不要在修補程式中包含`FileSystemId`更新（放置）請求。  或者，請確保請求`FileSystemId`中相同。

***帶 ID 的 ActiveDirectory 不存在。"字串"不存在。***

該`{string}`部分是您在活動目錄連接的屬性中`ActiveDirectoryId`輸入的值。

* 原因：   
使用 Active Directory 配置創建帳戶時，已輸入該值，該`ActiveDirectoryId`值應為空。
* 解決方案：   
不要在創建（放置`ActiveDirectoryId`）請求中包含。

***不正確 api 版本。***

API 版本未提交或包含無效值。

* 原因：   
查詢參數`api-version`中的值包含無效值。
* 解決方案：   
使用正確的 API 版本值。  資來源提供者支援許多 API 版本。 該值以 yyyy-mm-dd 格式。

***收到{1}的無效值"{值}"。***

`RuleIndex`此消息指示 欄位中的錯誤， `AllowedClients`、 `UnixReadOnly`、 `UnixReadWrite` `Nfsv3`、 和`Nfsv4`。

* 原因：   
輸入驗證請求`RuleIndex`至少未通過以下欄位之一： 、 `AllowedClients` `UnixReadOnly`、 、 `UnixReadWrite`、 `Nfsv`、`Nfsv4`和 。
* 解決方案：   
請確保在命令列上設置所有必需和非衝突參數。 例如，不能同時設置 和`UnixReadOnly``UnixReadWrite`參數。
* 因應措施：   
請參閱上面的解決方案。

***vlan {0} {2} {1}的 IP 範圍已在使用中***

出現此錯誤的原因是已使用 IP 範圍的內部記錄與新分配的 IP 位址發生衝突。

* 原因：   
為批量創建分配的 IP 位址已註冊。
原因可能是卷創建失敗。
* 解決方案：   
請與 Azure 支援中心聯繫。

***缺少"屬性"的值。***

此錯誤指示請求中缺少必需的屬性。 字串 [屬性] 包含缺少屬性的名稱。

* 原因：   
至少一個屬性的輸入驗證請求失敗。
* 解決方案：   
請確保在請求中設置所有必需和非衝突屬性，特別是錯誤訊息中的屬性。

***無法更改"裝載目標"。***

當使用者嘗試更新或修補卷 MountTargets 屬性時，將發生此錯誤。

* 原因：   
您正在嘗試更新卷`MountTargets`屬性。 不支援更改此屬性。
* 解決方案：   
不要在修補程式中包含`MountTargets`更新（放置）請求。  或者，請確保請求`MountTargets`中相同。

***名稱已在使用中。***

此錯誤指示資源的名稱已在使用中。

* 原因：   
您正在嘗試創建名稱用於現有資源的資源。
* 解決方案：   
創建資源時使用唯一名稱。

***檔路徑已在使用中。***

此錯誤指示卷的檔路徑已在使用中。

* 原因：   
您正在嘗試創建檔路徑與現有卷相同的卷。
* 解決方案：   
創建卷時使用唯一的檔路徑。

***名稱太長。***

此錯誤指示資源名稱不符合最大長度要求。

* 原因：   
資源名稱太長。
* 解決方案：   
對資源使用較短的名稱。

***檔路徑太長。***

此錯誤指示卷的檔路徑不符合最大長度要求。

* 原因：   
卷檔路徑太長。
* 解決方案：   
使用較短的檔路徑。

***名稱太短。***

此錯誤指示資源名稱不符合最小長度要求。

* 原因：   
資源名稱太短。
* 解決方案：   
對資源使用較長的名稱。

***檔路徑太短。***

此錯誤指示卷檔路徑不符合最小長度要求。

* 原因：   
卷檔路徑太短。
* 解決方案：   
增加卷檔路徑的長度。

***Azure NetApp 檔 API 無法訪問。***

Azure API 依賴于 Azure NetApp 檔 API 來管理卷。 此錯誤表示 API 連接出現問題。

* 原因：   
基礎 API 未回應，從而導致內部錯誤。 此錯誤可能是臨時的。
* 解決方案：   
這個問題可能是暫時的。 請求應在一段時間後成功。
* 因應措施：   
無。 基礎 API 對於管理卷至關重要。

***未找到 ''{0}的操作結果 ID。***

此錯誤指示內部錯誤阻止操作完成。

* 原因：   
發生內部錯誤，阻止操作完成。
* 解決方案：   
此錯誤可能是臨時的。 請等候幾分鐘後再試。 如果問題仍然存在，請創建一個票證，讓技術支援調查問題。
* 因應措施：   
等待幾分鐘，檢查問題是否仍然存在。

***不允許混合協定類型 CIFS 和 NFS***

當您嘗試創建卷時，會出現此錯誤，並且卷屬性中同時存在 CIFS （SMB） 和 NFS 協定類型。

* 原因：   
在卷屬性中都使用 CIFS （SMB） 和 NFS 協定類型。
* 解決方案：   
刪除其中一種協定類型。
* 因應措施：   
將協定類型屬性留空或為 null。

***項數：物件的 [值]：匯出策略.規則[規則索引]超出最小最大值範圍。***

當匯出策略規則不符合最小或最大範圍要求時，將發生此錯誤。 如果定義匯出策略，它必須至少有一個匯出策略規則，最多有五個匯出策略規則。

* 原因：   
您定義的匯出策略不符合所需的範圍。
* 解決方案：   
確保索引尚未使用，並且該索引在 1 到 5 之間。
* 因應措施：   
對卷使用匯出策略不是強制性的。 如果不需要使用匯出策略規則，則可以完全省略匯出策略。

***只允許一個活動目錄***

當您嘗試創建活動目錄配置時，將發生此錯誤，並且該區域中的訂閱已存在一個配置。 當您嘗試創建多個活動目錄配置時，也會發生該錯誤。

* 原因：   
您正在嘗試創建（不更新）活動目錄，但已存在一個目錄。
* 解決方案：   
如果 Active Directory 配置未使用，則可以首先刪除現有配置，然後重試創建操作。
* 因應措施：   
無。 只允許一個活動目錄。

***不支援操作"{操作"。。***

此錯誤指示該操作不適用於活動訂閱或資源。

* 原因：   
該操作不適用於訂閱或資源。
* 解決方案：   
請確保正確輸入操作，並且該操作可用於您正在使用的資源和訂閱。

***無法更改擁有者 Id***

* 原因：   
創建`OwnerId`卷時設置該屬性。 屬性隨後無法更改。
* 解決方案：   
不要在修補程式中包含`OwnerId`更新（放置）請求。 或者，請確保請求`OwnerId`中相同。

***找不到父池***

當您嘗試創建卷且找不到要在其中創建卷的容量池時，將發生此錯誤。

* 原因：   
找不到正在創建卷的容量池。
* 解決方案：   
最有可能的是，池尚未完全創建或在創建卷時已被刪除。

***此資源類型不支援修補程式操作。***

當您嘗試更改裝載目標或快照時，將發生此錯誤。

* 原因：   
裝載目標在創建時定義，以後無法更改。
快照不包含任何可以更改的屬性。
* 解決方案：   
無。 這些資源沒有任何可以更改的屬性。

***池大小太小，無法占總卷大小。***

更新容量池大小時，將發生此錯誤，並且大小小於該容量池中所有卷`usedBytes`的總值。  當您創建新卷或調整現有卷的大小，並且新卷大小超過容量池中的可用空間時，也會發生此錯誤。

* 原因：   
您正在嘗試將容量池更新為比容量池中所有卷中使用的位元組小的大小。  或者，您正在嘗試創建大於容量池中可用空間的卷。  或者，您嘗試調整卷的大小，並且新大小超過了容量池中的可用空間。
* 解決方案：   
將容量池大小設置為較大的值，或為卷創建較小的卷。
* 因應措施：   
刪除足夠的卷，以便容量池大小可以更新到此大小。

***屬性：快照的位置必須與卷相同***

當您創建具有擁有快照的卷以外的位置的位置的快照時，將發生此錯誤。

* 原因：   
快照的位置屬性中的無效值。
* 解決方案：   
在"位置"屬性中設置有效字串。

***[資源類型] 名稱必須與資源識別碼名稱相同。***

創建資源時發生此錯誤，並且使用 的名稱屬性以外的其他值填充 name 屬性`resourceId`。

* 原因：   
創建資源時，名稱屬性中的無效值。
* 解決方案：   
將名稱屬性留空，或允許它使用與 中的名稱屬性相同的值（在 上一個反斜線"/"和問號"？"`resourceId`之間）

***協定類型 [值] 不知道***

創建具有未知協定類型的卷時，將發生此錯誤。  有效值為"NFSv3"、"NFSv4"和"CIFS"。

* 原因：   
您正在嘗試在卷`protocolType`屬性中設置無效值。
* 解決方案：   
在 中`protocolType`設置有效的字串。
* 因應措施：   
設置為`protocolType`null。

***無法更改協定類型***

當您嘗試更新或修補`ProtocolType`卷時，將發生此錯誤。  更改協定類型不受支援的操作。

* 原因：   
創建`ProtocolType`卷時設置該屬性。  無法更新它。
* 解決方案：   
無。
* 因應措施：   
使用新的協定類型創建另一個卷。

***創建類型 [資源類型] 的資源將超過 [資源類型] 的 [資源類型] 類型的 [資源類型] 類型的 [配額] 資源的配額。當前資源計數為 [當前Count]，請在創建新資源之前刪除此類型的某些資源。***

當您嘗試`NetAppAccount`創建資源 （、、、`CapacityPool``Volume`或`Snapshot`）但配額已達到其限制時，將發生此錯誤。

* 原因：   
您正在嘗試創建資源，但已達到配額限制（例如：`NetAppAccounts`每個訂閱或`CapacityPools`每個`NetAppAccount`）。
* 解決方案：   
增加配額限制。
* 因應措施：   
刪除相同類型的未使用資源，然後再次創建它們。

***收到唯讀屬性"屬性名稱"的值。***

當您為無法更改的屬性定義值時，將發生此錯誤。 例如，您不能更改卷 ID。

* 原因：   
您正在嘗試修改無法更改的參數（例如卷 ID）。
* 解決方案：   
不要修改屬性的值。

***未找到請求的 {資源}。***

當您嘗試引用不存在的資源（例如卷或快照）時，將發生此錯誤。 資源可能已被刪除或具有錯字資源名稱。

* 原因：   
您正在嘗試引用已刪除或資源名稱拼寫錯誤的不存在的資源（例如卷或快照）。
* 解決方案：   
檢查請求中是否存在拼寫錯誤，以確保正確引用該錯誤。
* 因應措施：   
請參閱上面的解決方案部分。

***服務等級"[批量服務等級]"高於父級"[池服務等級]"***

創建或更新卷時發生此錯誤，並且已將服務等級設置為比包含該卷的容量池更高的級別。

* 原因：   
您正在嘗試創建或更新具有比父容量池更高的服務等級的卷。
* 解決方案：   
將服務等級設置為與父容量池相同的或更低的級別。
* 因應措施：   
在另一個容量池中創建具有正確服務等級的卷。 或者，從容量池中刪除所有卷，並將容量池的服務等級設置為更高的級別。

***SMB 伺服器名稱不能超過 10 個字元。***

在為帳戶創建或更新活動目錄配置時，將發生此錯誤。

* 原因：   
SMB 伺服器名稱的長度超過 10 個字元。
* 解決方案：   
使用較短的伺服器名稱。 最大長度為 10 個字元。
* 因應措施：   
無。  請參閱上面的解決方案。 

***無法更改子網Id。***

當您嘗試在創建卷`subnetId`後更改 時，將發生此錯誤。  `SubnetId`必須在創建卷時設置，以後不能更改。

* 原因：   
您正在嘗試更改創建卷`subnetId`後，該卷不是受支援的操作。 
* 解決方案：   
如果不需要更改`subnetId`， 則請考慮從請求中刪除參數以關閉錯誤訊息。
* 因應措施：   
如果需要更改 ，`subnetId`可以使用 創建新的 卷`subnetId`，然後將資料移轉到新卷。

***子網Id 的格式無效。***

當您嘗試創建新卷，但`subnetId`不是子網的 時`resourceId`，將發生此錯誤。

* 原因：   
當您嘗試創建新卷時，將發生此錯誤，`subnetId`但 不是子網的`resourceId`。 
* 解決方案：   
檢查 的值`subnetId`，以確保它包含用於的子網的`resourceId`。
* 因應措施：   
無。 請參閱上面的解決方案。 

***子網必須具有"Microsoft.NetApp/卷"委派。***

創建卷且所選子網未委派給`Microsoft.NetApp/volumes`時，將發生此錯誤。

* 原因：   
您嘗試創建卷，並選擇了未委派到`Microsoft.NetApp/volumes`的子網。
* 解決方案：   
選擇委派給`Microsoft.NetApp/volumes`的另一個子網。
* 因應措施：   
向子網添加正確的委派。

***指定的資源類型未知/不適用。***

當對不適用的資源類型或未知資源類型請求名稱檢查時，將發生此錯誤。

* 原因：   
要求的對未知或不受支援的資源類型進行名稱檢查。
* 解決方案：   
檢查您執行請求的資源是否受支援或不包含拼寫錯誤。
* 因應措施：   
請參閱上面的解決方案。

***未知 Azure NetApp 檔錯誤。***

Azure API 依賴于 Azure NetApp 檔 API 來管理卷。 該錯誤指示與 API 的通信中出現問題。

* 原因：   
基礎 API 發送未知錯誤。 此錯誤可能是臨時的。
* 解決方案：   
問題可能是暫時的，請求應在一段時間後成功。 如果問題仍然存在，請創建一個支援票證以調查問題。
* 因應措施：   
無。 基礎 API 對於管理卷至關重要。

***為未知屬性"[屬性名稱]"接收的值。***

當為資源（如卷、快照或裝載目標）提供不存在的屬性時，將發生此錯誤。

* 原因：   
請求具有一組可用於每個資源的屬性。 不能在請求中包含任何不存在的屬性。
* 解決方案：   
確保所有屬性名稱拼寫正確，並且屬性可用於訂閱和資源。
* 因應措施：   
減少請求中定義的屬性數，以消除導致錯誤的屬性。

***此資源類型不支援更新操作。***

只能更新卷。 當您嘗試執行不受支援的更新操作（例如，更新快照）時，將發生此錯誤。

* 原因：   
您嘗試更新的資源不支援更新操作。 只有卷可以修改其屬性。
* 解決方案：   
無。 您嘗試更新的資源不支援更新操作。 因此，無法更改它。
* 因應措施：   
對於卷，創建具有更新的新資源並遷移資料。

***無法在未成功處於狀態的池中創建卷。***

當您嘗試在未處於成功狀態的池中創建卷時，將發生此錯誤。 最有可能的是，容量池的創建操作由於某種原因失敗。

* 原因：   
包含新卷的容量池處於失敗狀態。
* 解決方案：   
檢查容量池是否已成功創建，以及容量池未處於失敗狀態。
* 因應措施：   
創建新容量池並在新池中創建卷。

***正在創建卷，目前無法刪除。***

當您嘗試刪除仍在創建的卷時，將發生此錯誤。

* 原因：   
當您嘗試刪除該卷時，仍在創建卷。
* 解決方案：   
等待卷創建完成，然後重試刪除。
* 因應措施：   
請參閱上面的解決方案。

***卷正在刪除，目前無法刪除。***

當您嘗試刪除卷時，該卷已被刪除時，將發生此錯誤。

* 原因：   
當您嘗試刪除該卷時，卷已被刪除。
* 解決方案：   
等待，直到當前刪除操作完成。
* 因應措施：   
請參閱上面的解決方案。

***卷正在更新，目前無法刪除。***

當您嘗試刪除正在更新的卷時，將發生此錯誤。

* 原因：   
當您嘗試刪除卷時，將更新卷。
* 解決方案：   
等待更新操作完成，然後重試刪除。
* 因應措施：   
請參閱上面的解決方案。

***未找到或未成功創建卷。***

當卷創建失敗，並且您嘗試更改卷或為卷創建快照時，將發生此錯誤。

* 原因：   
卷不存在，或者創建失敗。
* 解決方案：   
檢查更改的卷是否正確，以及卷的創建是否成功。 或者，檢查要為其創建快照的卷是否確實存在。
* 因應措施：   
無。  請參閱上面的解決方案。 

***指定的創建權杖已存在***

當您嘗試創建卷，並指定已存在的卷的創建權杖（匯出路徑）時，將發生此錯誤。

* 原因：   
在卷創建期間指定的創建權杖（匯出路徑）已與另一個卷關聯。 
* 解決方案：   
選擇其他創建權杖。  或者，刪除其他卷。

***保留指定的創建權杖***

當您嘗試創建卷，並將"預設"或"無"指定為檔路徑（創建權杖）時，將發生此錯誤。

* 原因：    
您正在嘗試創建卷，並將"預設"或"無"指定為檔路徑（創建權杖）。
* 解決方案：   
選擇其他檔路徑（創建權杖）。
 
***活動目錄憑據正在使用中***

當您嘗試從至少一個 SMB 卷存在的帳戶中刪除 Active Directory 配置時，將發生此錯誤。  SMB 卷是使用您嘗試刪除的活動目錄配置創建的。

* 原因：   
您嘗試從帳戶中刪除 Active Directory 配置，但至少仍然存在一個最初使用活動目錄配置創建的 SMB 卷。 
* 解決方案：   
首先，刪除使用活動目錄配置創建的所有 SMB 卷。  然後重試配置刪除。

***如果憑據正在使用中，則無法修改組織單位分配***

當您嘗試更改活動目錄配置的組織單位時，將發生此錯誤，但至少仍然存在一個 SMB 卷。  SMB 卷是通過使用您嘗試刪除的活動目錄配置創建的。

* 原因：   
您正在嘗試更改活動目錄配置的組織單位。  但是，至少一個 SMB 卷仍然存在，這些卷最初是使用活動目錄配置創建的。
* 解決方案：   
 首先，刪除使用活動目錄配置創建的所有 SMB 卷。  然後重試配置刪除。 

***活動目錄更新已在進行中***

當您嘗試編輯已為其進行編輯操作的活動目錄配置時，將發生此錯誤。

* 原因：   
您正在嘗試編輯活動目錄配置，但另一個編輯操作已在進行中。
* 解決方案：   
等待當前正在運行的編輯操作完成。

***首先使用所選憑據刪除所有卷***

當您嘗試刪除活動目錄配置時，將發生此錯誤，但至少仍然存在一個 SMB 卷。  SMB 卷是使用您嘗試刪除的活動目錄配置創建的。

* 原因：   
您正在嘗試刪除活動目錄配置，但至少仍然存在一個最初使用活動目錄配置創建的 SMB 卷。
* 解決方案：   
首先，刪除使用活動目錄配置創建的所有 SMB 卷。  然後重試配置刪除。 

***在區域中找不到活動目錄憑據***

當您嘗試創建 SMB 卷時，將發生此錯誤，但尚未將活動目錄配置添加到該區域的帳戶中。

* 原因：   
您正在嘗試創建 SMB 卷，但帳戶中未添加活動目錄配置。 
* 解決方案：   
在創建 SMB 卷之前，將活動目錄配置添加到帳戶。

***無法查詢 DNS 伺服器。驗證網路設定是否正確，以及 DNS 伺服器是否可用。***

當您嘗試創建 SMB 卷時，將發生此錯誤，但無法訪問 DNS 伺服器（在 Active Directory 配置中指定）。 

* 原因：   
您正在嘗試創建 SMB 卷，但無法訪問 DNS 伺服器（在 Active Directory 配置中指定）。
* 解決方案：   
查看活動目錄配置，並確保 DNS 伺服器 IP 位址正確且可到達。
如果 DNS 伺服器 IP 位址沒有問題，請驗證沒有防火牆阻止訪問。

***併發作業太多***

當您嘗試在為訂閱創建其他三個快照創建操作時，將發生此錯誤。

* 原因：   
當您正在為訂閱創建其他三個快照創建操作時，您正在嘗試創建快照。 
* 解決方案：   
快照創建作業最多需要幾秒鐘才能完成。  等待幾秒鐘，然後重試快照創建操作。

***無法生成其他作業。請等待正在進行的作業完成，然後重試***

當您嘗試在特定情況下創建或刪除卷時，可能會發生此錯誤。

* 原因：   
您正在嘗試在特定情況下創建或刪除卷。
* 解決方案：   
等待一分鐘左右，然後重試該操作。

***卷已在狀態之間轉換***

當您嘗試刪除當前處於轉換狀態的卷（即當前處於創建、更新或刪除狀態）時，可能會發生此錯誤。

* 原因：   
您正在嘗試刪除當前處於轉換狀態的卷。
* 解決方案：   
等待當前正在運行的（狀態轉換）操作完成，然後重試該操作。

***無法從源卷快照拆分新卷***

 當您嘗試從快照創建卷時，可能會發生此錯誤。  

* 原因：   
您嘗試從快照創建卷，卷以錯誤狀態結束。
* 解決方案：   
刪除卷，然後從快照重試卷創建操作。
