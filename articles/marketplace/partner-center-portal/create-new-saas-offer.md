# <a name="create-a-new-saas-offer"></a>建立新的 SaaS 供應項目

要開始創建"軟體即服務"（SaaS）優惠，請確保首先[創建合作夥伴中心帳戶](./create-account.md)並打開["商業市場"儀表板](https://partner.microsoft.com/dashboard/commercial-marketplace/offers)，並選中 **"概述**"選項卡。

![合作夥伴中心上的商業市場儀表板](./media/new-offer-overview.png)

>[!Note]
> 產品/服務發佈後，只有在重新發佈後，才會在系統和商店前面更新對合作夥伴中心中提供的優惠進行編輯。 請務必在進行更改後提交要發佈的產品/服務。

選擇 +**新產品/服務...** 按鈕，然後選擇 **"軟體作為服務"** 功能表項目。

如果選擇其他產品/服務類型，則可以重定向到較舊的[雲合作夥伴門戶](https://cloudpartner.azure.com/)。 目前合作夥伴中心的商業市場門戶中僅提供 SaaS 和 Dynamics 365 優惠。

![在合作夥伴中心創建產品/服務視窗](./media/new-offer-click.png)

將顯示 **"新建產品/服務**"對話方塊。

![新優惠對話方塊](./media/new-offer-popup.png)

## <a name="offer-id-and-alias"></a>優惠 ID 和別名

- **優惠 ID**：帳戶中每個產品/服務的唯一識別碼。 此 ID 將在市場產品/服務的 URL 位址和 Azure 資源管理器範本（如果適用）中向客戶可見。 優惠 ID 必須是小寫字母數位（包括連字號和底線，但沒有空格）。 **優惠 ID**限制為 50 個字元，在選擇"*創建*"後無法更改。  
示例：測試-提供-1
<br>導致 URL：`https://azuremarketplace.microsoft.com/marketplace/../test-offer-1`

- **產品/服務別名**：用於在合作夥伴中心門戶中引用產品/服務的名稱。 此名稱不會在市場中使用，並且與向客戶顯示的 *"產品/服務名稱*"和其他值不同。 選擇 "*創建*"後，無法更改此值。

<br>示例：測試優惠 1&#8482;

選取 [建立]****。  為此產品/服務創建**產品/服務概述**頁面。

### <a name="sell-through-microsoft"></a>透過 Microsoft 銷售

通過 Microsoft 銷售可提供更好的客戶發現和獲取，允許 Microsoft 代表您託管市場事務，並利用 Microsoft 的全球可用商務功能。

#### <a name="saas-offer-requirements"></a>SaaS 提供要求

為了在合作夥伴中心的商業市場列出軟體即服務 （SaaS） 優惠，必須滿足以下條件：

- 您的產品/服務必須使用[Azure 活動目錄 （Azure AD）](https://azure.microsoft.com/services/active-directory/)進行標識管理和身份驗證。
- 您的產品/服務必須使用[SaaS 履行 API](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2)與 Azure 應用商店集成。
- 有關更廣泛的要求，請參閱[SaaS 產品發佈指南](https://docs.microsoft.com/azure/marketplace/marketplace-saas-applications-technical-publishing-guide)。

#### <a name="saas-pricing-and-billing-options"></a>SaaS 定價和計費選項
在發行者 Azure 訂閱中運行 SaaS 解決方案後，客戶支付的許可費包括部署軟體的基礎結構的成本。 Azure 基礎結構使用方式將直接管理和計費給合作夥伴。 客戶不會看到實際的基礎結構使用量費用。 發行者應將 Azure 基礎結構使用費捆綁到其軟體許可證定價中。 

SaaS 根據固定費用、每個使用者或使用按流量計費服務的消耗費提供每月或年度計費支援。 微軟的商業市場以代理模式運作，即出版商設定價格，微軟向客戶收費，微軟在扣繳代理費的同時向出版商支付收入。

下表顯示了用於演示代理模型的成本和支出的示例。

|**授權成本**|**每月 100 美元**|
|:---|:---|
|Azure 使用量成本 (D1/1 核心)|直接向發行者收費，不是向客戶收費|
|由 Microsoft 向客戶收取的費用|每月 $100.00 （Publisher 必須考慮許可證費中發生的任何或傳遞的基礎設施成本）|

|**Microsoft 收取的費用**|**每月 100 美元**|
|:---|:---|
|Microsoft 向您支付授權成本的 80% <br>**對於合格的 SaaS 應用，Microsoft 支付 90% 的許可證成本*|每月 $80.00 <br>*$* 每月 90.00 美元*|

- 在此示例中，Microsoft 向客戶收取軟體許可證的 100.00 美元，並向客戶支付 80.00 美元給發行者。
- 符合**減少市場服務費**條件的合作夥伴將在 2019 年 5 月至 2020 年 6 月期間降低 SaaS 優惠的交易費用。 在這種情況下，Microsoft 會為您的軟體許可證支付 100.00 美元，然後向發行者支付 90.00 美元。

> [!NOTE]
> **降低市場服務費**：對於您在我們的商業市場上發佈的某些 SaaS 優惠，Microsoft 將將其市場服務費從 20%（如 Microsoft 發佈商協定中所述）降至 10%。 為了使您的產品/服務符合條件，您的優惠中至少有一個必須被 Microsoft 指定為 IP 共同銷售就緒或 IP 共同銷售優先。  每個日曆月結束之前必須至少五 （5） 個工作日滿足資格，才能獲得該月此降低的市場服務費。  減少的市場服務費不適用於 VM、託管應用或通過我們的商業市場提供的任何其他產品。  減少的市場服務費僅適用于 Microsoft 在 2019 年 5 月 1 日至 2020 年 6 月 30 日期間收取的合格許可費用優惠。  在此時間之後，市場服務費將恢復到正常金額。

### <a name="standard-contract-for-the-microsoft-commercial-marketplace"></a>微軟商業市場的標準合同

微軟提供了標準合約範本。

- **使用微軟商業市場的標準合同？**

為了簡化客戶的採購流程並降低軟體供應商的法律複雜性，Microsoft 為 Microsoft 商業市場提供了標準合同，以説明促進市場交易。 商業市場出版商可以選擇根據標準合同提供他們的軟體，而不是制定自訂條款及條件，客戶只需審查並接受一次。 標準合同可以在這裡找到： https://go.microsoft.com/fwlink/?linkid=2041178.

您可以選擇使用標準合同，而不是通過選擇"為商業市場使用標準合同"核取方塊來提供您自己的自訂條款及條件。

![使用標準合同核取方塊](./media/use-standard-contract.png)

> [!NOTE]
> 使用 Microsoft 商業市場的標準合約發佈產品/服務後，您將無法使用自己的自訂條款及條件。 這是一個"或"方案。 您要麼根據標準合同**或**您自己的條款及條件提供您的解決方案。 如果您想修改標準合同的條款，您可以通過標準合同修訂來修改。

#### <a name="standard-contract-amendments"></a>標準合同修訂

標準合同修訂允許發佈商選擇標準合同條款，以便簡單，並自訂其產品或業務的條款。 客戶只需審核並接受 Microsoft 標準合同，就只需查看合同的修訂。

商業市場出版商有兩種修正案：

- 普遍修改：這些修訂適用于所有客戶的標準合同。 通用修訂顯示在採購流程中向產品/服務的每個客戶。 客戶必須接受標準合同的條款和修改，然後才能使用您的優惠。
- 自訂修訂：這些修訂是標準合同的特別修訂，僅通過 Azure 租戶 ID 針對特定客戶。 發行者可以選擇要定位的租戶。 只有租戶的客戶才會在優惠的採購流程中收到自訂修改條款。  客戶必須接受標準合同的條款和修改，然後才能使用您的優惠。

>[!NOTE]
> 這兩種類型的修正相互疊加。 以定制修訂為目標的客戶在購買過程中還將獲得標準合同的普遍修訂。

**微軟商業市場標準合同的通用修正條款**：在此框中輸入通用修訂條款。 您可以提供每個報價的單一通用修訂。 您可以在此框中輸入無限數量的字元。 在發現和購買流期間，這些術語在 AppSource、Azure 應用商店和/或 Azure 門戶中向客戶顯示。

**微軟商業市場標準合同的自訂修改條款**：從選擇**添加自訂修改條款**開始。 每個優惠最多可以提供 10 個自訂修訂條款。

- **自訂修改條款**：在自訂修改條款框中輸入您的自訂修訂條款。 您可以在此框中輸入無限數量的字元。 只有您為這些自訂條款指定的租戶 ID 的客戶才會在 Azure 門戶中向產品/服務的採購流中顯示自訂修改條款。  
- **租戶 ID（** 必需）：每個自訂修訂的目標可多達 20 個租戶 ID。 如果添加自訂修訂，則必須至少提供一個租戶 ID。 租戶 ID 標識 Azure 中的客戶。 您可以向客戶詢問此 ID，並且他們可以通過導航到portal.azure.com> Azure 活動目錄>屬性來找到它。 目錄 ID 值是租戶 ID（例如，50c464d3-4930-494c-963c-1e951d15360e）。 您還可以使用我的[Microsoft Azure 和 Office 365 租戶 ID](https://www.whatismytenantid.com)上的功能變數名稱 URL 查找組織的客戶的租戶 ID。
- **說明**（可選）：可選地為租戶 ID 提供友好說明，説明您通過修訂確定目標客戶。

## <a name="offer-listing"></a>優惠清單

"產品/服務"清單選項卡顯示您的優惠可用的語言（和市場），當前英語（美國）是唯一可用的位置。 此外，此頁顯示特定于語言的清單的狀態以及添加該清單的日期/時間。 您需要為每個語言/市場定義市場詳細資訊（報價名稱、說明、搜索詞等）。

> [!NOTE]
> 報價清單內容（如產品/服務說明、文檔、螢幕截圖、使用條款和隱私政策）不需要以英文表示，只要產品/服務說明以短語開頭，"此應用程式僅以 [非英語語言]提供。 提供*有用的連結 URL*以提供產品/服務清單內容中所用語言以外的語言提供內容也是可接受的。

## <a name="technical-configuration"></a>技術配置

"**技術配置**"選項卡定義用於連接到產品/服務的技術詳細資訊（URL 路徑、Webhook、租戶 ID 和應用 ID）。 此連接使我們能夠為最終客戶提供您的報價，如果他們選擇購買。 描述收集欄位使用方式的圖表可用於[SaaS 履行 API 的文檔](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2)。

- **著陸頁 URL（** 必需）：定義客戶從市場獲取您的報價後將登錄的網站 URL。 此 URL 將是在客戶路由到頁面時接收權杖的終結點。 該權杖可以使用履行 API 中的解析交換以進行預配詳細資訊。 這些詳細資訊和您收集的任何其他詳細資訊都可以用作您體驗中構建的客戶互動式網頁的一部分，以完成註冊並啟動其購買。

- **連接 Webhook（** 必需）：對於 Microsoft 需要代表客戶發送給您的所有非同步事件（例如：SaaS 訂閱已無效），我們要求您提供連接 Webhook。 如果還沒有 Webhook 系統，最簡單的配置是擁有一個 HTTP 終結點邏輯應用，用於偵聽發佈到它的任何事件，然後適當地處理它們（例如，HTTPs：/prod-1westus.logic.azure.com:443/work）。\/ 如需詳細資訊，請參閱[在邏輯應用程式中透過 HTTP 端點呼叫、觸發或巢狀處理工作流程](https://docs.microsoft.com/azure/logic-apps/logic-apps-http-endpoint)。

- **Azure AD 租戶 ID（** 必需）：在 Azure 門戶中，我們要求您[創建 Azure 活動目錄 （AD） 應用](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal)，以便我們可以驗證兩個服務之間的連接位於經過身份驗證的通信後面。 要查找[租戶 ID，](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#get-values-for-signing-in)請轉到 Azure 活動目錄並選擇**屬性**，然後查找列出的**目錄 ID**號（例如，50c464d3-4930-494c-963c-1e951d15360e）。

- **Azure AD 應用 ID（** 必需）：還需要[應用程式 ID](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#get-values-for-signing-in)和身份驗證金鑰。 要獲取這些值，請轉到 Azure 活動目錄並選擇**應用註冊**，然後查找列出的**應用程式 ID**號（例如，50c464d3-4930-494c-963c-1e951d15360e）。 要查找身份驗證金鑰，請轉到 **"設置"** 並選擇 **"金鑰**"。 您需要提供說明和持續時間，然後提供數位值。

>[!Note]
>Azure 應用程式 ID 與發行者 ID 相關聯，因此請確保在所有產品/服務中使用相同的應用程式 ID。

## <a name="plan-overview"></a>計畫概述

"**計畫概述**"選項卡使您能夠在同一產品/服務範圍內提供各種計畫選項。 這些計畫（有時稱為 SKU）在版本、貨幣化或服務層方面可能有所不同。 您必須至少設置一個計畫才能在市場上銷售您的報價。

創建後，您將看到計畫名稱、ID、定價模型、可用性（公共或私有）、當前發佈狀態以及任何可用操作。

**計畫概述**中可用的**操作**因計畫的目前狀態而異，可能包括：

- 如果計畫狀態為**草稿**- 刪除草稿
- 如果計畫狀態為 **"即時**- 停止銷售計畫"或同步私人受眾

**創建新計畫**（為選擇通過 Microsoft 銷售的使用者至少一個計畫）

- **計畫 ID：** 為此產品/服務中的每個計畫創建唯一的計畫 ID。 此 ID 將在產品 URL 和 Azure 資源管理器範本中對客戶可見（如果適用）。 僅使用小寫、字母數位字元、破折號或底線。 此計畫 ID 最多允許 50 個字元。 選擇創建後無法修改 ID。
- **計畫名稱：** 客戶在決定在您的產品/服務中選擇哪個計畫時，將看到此名稱。 為此產品/服務中的每個計畫創建唯一的優惠名稱。 計畫名稱用於區分可能是同一產品/服務一部分的軟體計畫（例如，產品名稱：Windows 伺服器;計畫：Windows 伺服器 2016，Windows Server 2019）。

### <a name="plan-listing"></a>計畫清單

**"計畫清單**"選項卡顯示您的計畫可用的語言（和市場），當前英語（美國）是唯一可用的位置。 此外，此頁顯示特定于語言的清單的狀態以及添加該清單的日期/時間。 您需要為每個語言/市場定義市場詳細資訊（報價名稱、說明、搜索詞等）。

#### <a name="plan-listing-details"></a>計畫清單詳細資訊

選擇其中一種計畫語言將顯示**計畫清單**資訊，包括**名稱**和**說明。**

- **名稱**：根據預覽 **"新建計畫**"條目預填充，並將作為市場上顯示的產品/服務"軟體計畫"的標題顯示。
- **描述：** 此說明是一個解釋是什麼讓此軟體計畫獨一無二以及您產品/服務中其他軟體計畫的任何差異的機會。 可能最多包含 500 個字元。

完成這些欄位後選擇 **"保存**"。

#### <a name="plan-pricing-and-availability"></a>規劃定價和可用性

通過 **"定價和可用性**"選項卡，您可以配置此計畫可用的市場、所需的貨幣化模型、價格和計費期限。 此外，還可以指示是使計畫對所有人可見，還是僅對特定客戶（私人受眾）可見。

##### <a name="enabling-free-trials"></a>啟用免費試用

通過商業市場提供 SaaS 功能，使您能夠在通過 Microsoft 銷售時提供一個月的免費試用版。 對於除計量計畫之外的所有計費模型和條款，支援免費試用。 此選項允許客戶通過一個月的免費訪問獲得低門檻進入。  如果您選擇在產品/服務範圍內為計畫啟用免費試用，則客戶將無法在初始一個月期限結束前轉換為付費訂閱。  在此期間，購買您的優惠的客戶可以試用任何支援的計畫，這些計畫啟用了免費試用，並在它們之間轉換。  付費訂閱的轉換在期限結束時自動完成。

>[!Note]
>如果客戶選擇在沒有免費試用的情況下轉換為計畫，轉換將發生，但免費試用將立即丟失。  此外，一旦客戶開始為計畫付費，他們就不能再在同一訂閱上獲得免費試用，即使他們轉換為支援免費試用的 SKU 也是如此。

對於優惠中的每個計畫，均可配置免費試用版。 只需導航到每個產品/服務的定價和可用性，然後選中此框即可進行一個月的試用。

![一個月免費試用核取方塊](./media/free-trial-enable.png)

>[!Note]
>一旦您的可交易優惠發佈免費試用，則無法為該計畫禁用。 確保此設置對於第一個發佈是正確的，以避免重新創建計畫。

要獲取有關當前參與免費試用的客戶訂閱的資訊，請使用新的 API 屬性，該`isFreeTrial`屬性將標記為真或假。 有關詳細資訊，請參閱[SaaS 獲取訂閱 API](https://docs.microsoft.com/azure/marketplace/partner-center-portal/pc-saas-fulfillment-api-v2#get-subscription)。

>[!Note]
>對於利用市場計量服務的計畫，不支援免費試用。

#### <a name="markets"></a>市場

- **編輯市場**（可選）

每個計畫必須至少在一個市場可用。 選擇您希望提供此計畫的任何市場位置的核取方塊。 包含用於選擇"免稅"國家/地區的搜索框和按鈕，其中 Microsoft 代表您匯出銷售和使用稅，以説明。

如果您已為計畫設定價格（美元），並添加其他市場位置，則新市場的價格將根據當前匯率計算。 在發佈之前，您應該始終查看每個市場的價格。 在保存更改後，可以使用"出口價格 （xlsx）"連結查看定價。

#### <a name="pricing"></a>定價

- **定價模式**：基於固定費率或座位

**統一費率：** 以單月或年價格統一價格啟用對產品/服務的訪問。 這有時稱為基於網站的定價。 使用此定價模型，您可以選擇定義使用市場計量服務 API 按非標準單位向客戶收費的計量計畫。  有關計量計費的詳細資訊，請參閱[使用市場計量服務計量計費。](./saas-metered-billing.md)

**每個使用者：** 根據訪問優惠或佔據席位的使用者數量，以價格啟用對優惠的訪問。 此基於使用者的模型使您能夠根據價格設置允許的最小和最大使用者數。 這樣，可以通過配置多個計畫來根據使用者數量配置不同的價格點。  這些欄位為選擇性。 如果未選中，則使用者數將被解釋為沒有限制（最小為 1，並且最大為系統可以支援的數量）。 這些欄位可能會作為計畫更新的一部分進行編輯。

發佈後，無法更改計費定價模型選擇。 此外，同一報價的所有計劃必須共用相同的定價模型。

- **計費期限**：每月或每年

選擇客戶必須支付所列價格的頻率。 必須至少提供一個每月或年價，或者兩個選項都可以提供給客戶。

- **價格**：每月美元或美元/年

以當地貨幣（美元 = 美元）設置的價格使用設置期間可用的當前匯率轉換為所有選定市場的當地貨幣。 通過匯出定價試算表並查看每個市場的價格，在發佈之前驗證這些價格。 如果要在單個市場設置自訂價格，請修改並導入定價試算表。 您負責驗證此定價並擁有這些設置。
*\*您必須首先保存定價更改，以啟用定價資料的匯出。*

在發佈之前，請仔細查看您的價格，因為在計畫發佈後，可能會更改哪些內容有一些限制：

- 計畫發佈後，無法更改定價模型。
- 一旦為計畫發佈了計費期限，以後無法刪除它。
- 一旦計畫中市場的價格公佈，以後就改變不了。

#### <a name="privacy"></a>隱私權

- **這是一個私人計畫**（可選核取方塊）

選中此框可使您的計畫成為私有的，並且僅對您選擇的受限受眾可見。 作為私人計畫發佈後，您可以更新受眾或選擇使計畫可供所有人使用。 一旦計畫發佈為對每個人都可見，它必須保持對每個人都可見。 （無法再次將計畫配置為私有計劃）。

- **受限受眾（租戶 ID）**

分配有權訪問此私人計畫的訪問群體。 使用租戶 ID 分配存取權限，並選擇包括所分配的每個租戶 ID 的說明。 最多可以添加 10 個租戶 ID，如果導入 .csv 試算表檔，最多可以添加 20，000 個客戶租戶 ID。

租戶是組織的表示形式，ID 表示為 GUID（全域唯一識別碼，用於標識資源的 128 位整數）。 它是組織或應用程式開發人員在與 Microsoft 建立關係 (例如註冊 Azure、Microsoft Intune 或 Microsoft 365) 時收到的 Azure AD 專屬執行個體。 每個 Azure AD 租用戶都不同，並與其他 Azure AD 租用戶分開。 若要檢查租用戶，請使用您要用來管理應用程式的帳戶登入 Azure 入口網站。 如果您有租用戶，系統會自動將您登入，而且您會在您帳戶名稱正下方看到租用戶名稱。 將滑鼠停留在 Azure 入口網站右上方的帳戶名稱上，以查看您的姓名、電子郵件、目錄/租用戶識別碼 (一個 GUID) 與您的網域。 如果帳戶與多個租用戶相關聯，您可以選取帳戶名稱開啟功能表，以切換租用戶。 每個租用戶都有自己的租用戶識別碼。 您還可以使用功能變數名稱 URL 在 ：[https://www.whatismytenantid.com](https://www.whatismytenantid.com)查找組織的租戶 ID。

雖然 SaaS 提供使用租戶 ID 來定義專用受眾，但其他產品/服務類型可能使用 Azure 訂閱 ID（也表示為 GUID）。

> [!NOTE]
> 私人觀眾（或受限觀眾）與預覽觀眾不同。 在**["預覽"](#preview)** 選項卡中，您可以定義預覽受眾。 在產品/服務在市場上即時發佈*之前*，預覽受眾可以訪問您的優惠。 雖然私人受眾指定僅適用于特定計劃，但預覽觀眾可以查看所有計劃（私有或是否），但只能在計畫測試和驗證的有限預覽期間查看。

## <a name="publish"></a>發佈

完成產品/服務的所有必需部分後，選擇門戶右上角的**發佈**。 您將被重定向到 **"審閱"和"發佈**"頁。

#### <a name="submit-offer-to-preview"></a>提交優惠以預覽

如果這是您第一次發佈此產品/服務，您可以：

- 請參閱產品/服務的每個部分的完成狀態。
    - *未啟動*- 表示該部分尚未觸摸且需要完成。
    - *不完整*- 表示該節有需要修復的錯誤或需要提供更多資訊。 您需要返回該部分並更新它。
    - *完成*- 表示部分已完成，提供了所有必需的資料，並且沒有錯誤。 報價的所有部分都必須處於完全狀態，然後才能提交報價。
- 除了任何有助於瞭解應用的補充說明外，向認證團隊提供測試說明，以確保正確測試你的應用。
- 通過選擇 **"提交**"提交要發佈的產品/ 我們將向您發送一封電子郵件，告知您何時可以審核和批准優惠的預覽版本。 您必須返回合作夥伴中心，選擇 **"Go-live"** 作為優惠，向公眾發佈您的優惠（如果是私人優惠，則向私人受眾發佈）。
