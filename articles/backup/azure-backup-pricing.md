# <a name="azure-backup-pricing"></a>Azure 備份定價

若要了解 Azure 備份定價，請瀏覽 [Azure 備份定價頁面](https://azure.microsoft.com/pricing/details/backup/)。

## <a name="download-detailed-estimates-for-azure-backup-pricing"></a>下載 Azure 備份定價的詳細估算

如果您想要估算成本以便編列預算或比較成本，請下載詳細的 [Azure 備份定價估算器](https://aka.ms/AzureBackupCostEstimates)。  

### <a name="what-does-the-estimator-contain"></a>估算器包含哪些內容？

Azure 備份成本估算器工作表有選項可讓您估算您想要使用 Azure 備份來備份的所有可能工作負載。 這些工作負載包括：

- Azure VM
- 內部部署伺服器
- Azure VM 中的 SQL
- Azure VM 中的 SAP Hana
- Azure 檔案共用

## <a name="estimate-costs-for-backing-up-azure-vms-or-on-premises-servers"></a>估算 Azure VM 或內部部署伺服器的備份成本

若要估算使用 Azure 備份來備份 Azure VM 或內部部署伺服器的成本，您需要下列參數：

- 您所嘗試備份的 VM 或內部部署伺服器大小
  - 輸入需要備份的磁碟或伺服器「已用大小」

- 具有該大小的伺服器數目

- 這些伺服器上預期的資料變換量為何？<br>
  變換指的是資料的變更量。 例如，如果您的 VM 有 200 GB 的資料需要備份，且這些資料中有 10 GB 會每天變更，則每日變換率為 5%。

  - 變換率越高表示您備份的資料越多

  - 針對檔案伺服器，請挑選 [低] 或 [中]，如果您執行的是資料庫，請挑選 [高]

  - 如果您知道自己的 **變換 %** ，則可以使用 [輸入自己的 %] 選項

- 選擇備份原則

  - 您預期會保留「每日」備份多久？ (以天為單位)

  - 您預期會保留「每週」備份多久？ (以週為單位)

  - 您預期會保留「每月」備份多久？ (以月為單位)

  - 您預期會保留「每年」備份多久？ (以年為單位)

  - 您預期會保留「立即還原快照集」多久？ (1 到 5 天)

    - 此選項可讓您使用儲存在磁碟上的快照集快速地進行還原，最久可還原回到 7 天前。

- **選用** - 選擇性磁碟備份

  - 如果您在備份 Azure VM 時使用 [選擇性磁碟備份] 選項，請選擇 [排除磁碟] 選項，並輸入要從備份中排除的磁碟百分比 (以大小為單位)。 例如，如果您有一個 VM 連線到三個磁碟，每個磁碟使用了 200 GB，而且您想要將其中的兩個磁碟從備份中排除，請輸入 66.7%。

- **選用** - 備份儲存體備援

  - 這代表您的備份資料將會採用的儲存體帳戶備援。 我們建議使用 **GRS** 以獲得最高的可用性。 GRS 可確保備份資料的複本會保留在不同區域，因此可協助您符合多個合規性標準。 如果您要備份的開發或測試環境不需要企業級備份，請將備援變更為 [LRS]。 如果您想要在已為備份啟用 [跨區域還原](backup-azure-arm-restore-vms.md#cross-region-restore) 時了解成本，請選取工作表中的 [RAGRS] 選項。

- **選用** - 修改區域定價或申請折扣費率

  - 如果您想要了解不同區域或折扣費率下的估算值，請在 [嘗試估算不同區域?] 選項中選取 [是]，並輸入您想要用來執行估算的費率。

## <a name="estimate-costs-for-backing-up-sql-servers-in-azure-vms"></a>估算在 Azure VM 中備份 SQL 伺服器的成本

若要估算使用 Azure 備份在 Azure VM 中備份 SQL 伺服器的成本，您需要下列參數：

- 所嘗試備份的 SQL 伺服器大小

- 具有上述大小的 SQL 伺服器數目

- SQL 伺服器備份資料預期的壓縮率為何？

  - 大部分的 Azure 備份客戶會發現， **已啟用** SQL 壓縮時，備份資料相較於 SQL 伺服器大小會壓縮 80%。

  - 如果您預期會有不同的壓縮率，請在此欄位中輸入該數字

- 預期的記錄備份大小為何？

  - % 會以 SQL 伺服器大小 % 的形式來指出每日記錄大小

- 這些伺服器上預期的每日資料變換量為何？

  - 一般而言，資料庫會有「高」變換率

  - 如果您知道自己的 **變換 %** ，則可以使用 [輸入自己的 %] 選項

- 選擇備份原則

  - 備份類型

    - 針對每週/每月/每年完整備份，您所能選擇的最有效原則是 [每日差異]。 Azure 備份也可以透過單鍵點擊的方式進行差異還原。

    - 您也可以選擇擁有每日/每週/每月/每年完整備份的原則。 此選項所取用的儲存體會比第一個選項略多一些。

  - 您預期會保留「記錄」備份多久？ (以天為單位) [7-35]

  - 您預期會保留「每日」備份多久？ (以天為單位)

  - 您預期會保留「每週」備份多久？ (以週為單位)

  - 您預期會保留「每月」備份多久？ (以月為單位)

  - 您預期會保留「每年」備份多久？ (以年為單位)

- **選用** - 備份儲存體備援

  - 這代表您的備份資料將會採用的儲存體帳戶備援。 我們建議使用 **GRS** 以獲得最高的可用性。 GRS 可確保備份資料的複本會保留在不同區域，因此可協助您符合多個合規性標準。 如果您要備份的開發或測試環境不需要企業級備份，請將備援變更為 [LRS]。

- **選用** - 修改區域定價或申請折扣費率

  - 如果您想要了解不同區域或折扣費率下的估算值，請在 [嘗試估算不同區域?] 選項中選取 [是]，並輸入您想要用來執行估算的費率。

## <a name="estimate-costs-for-backing-up-azure-file-shares"></a>估算 Azure 檔案共用的備份成本

若要估算使用 Azure 備份所提供的[快照集式備份解決方案](azure-file-share-backup-overview.md)來備份 Azure 檔案共用的成本，您需要下列參數：

- 所要備份的檔案共用大小 ( **以 GB 為單位** )。

- 如果您想要備份的檔案共用分散於多個儲存體帳戶，請指定有上述大小的檔案共用裝載所在的儲存體帳戶數目。

- 所要備份的檔案共用上預期的資料變換量。 <br>變換指的是資料的變更量，其會直接影響快照集儲存體大小。 例如，如果您的檔案共用有 200 GB 的資料需要備份，且這些資料中有 10 GB 會每天變更，則每日變換率為 5%。
  - 變換率越高表示檔案共用內容每天的資料變更量很高，因此增量快照集 (只擷取資料變更) 大小也會比較大。
  - 請根據您的檔案共用特性和使用量來選取 [低] (1%)、[中] (3%) 或 [高] (5%)。
  - 如果您知道檔案共用的確切 **變換 %** ，則可以從下拉式清單中選取 [輸入自己的 %] 選項。 指定適用於每日、每週、每月和每年變換的值 (以 % 表示)。

- 儲存體帳戶的類型 (標準或進階)，以及所備份檔案共用裝載所在儲存體帳戶的儲存體備援設定。 <br>在 Azure 檔案共用的目前備份解決方案中，快照集會儲存在與所備份檔案共用相同的儲存體帳戶中。 因此，與快照集相關聯的儲存體成本會根據帳戶類型的快照集定價以及所備份檔案共用和快照集裝載所在儲存體帳戶的備援設定，隨 Azure 檔案的帳單一起計費。

- 不同備份的保留期
  - 您預期會保留「每日」備份多久？ (以天為單位)
  - 您預期會保留「每週」備份多久？ (以週為單位)
  - 您預期會保留「每月」備份多久？ (以月為單位)
  - 您預期會保留「每年」備份多久？ (以年為單位)

  如需每個類別中支援的保留值上限，請參閱 [Azure 檔案共用支援矩陣](azure-file-share-support-matrix.md#retention-limits)。

- **選用** - 修改區域定價或申請折扣費率。
  - 估算器中針對每 GB 的快照集儲存體成本和受保護的執行個體成本所設定的預設值適用於「美國東部」區域。 如果您想要了解不同區域或折扣費率下的估算值，請在 [嘗試估算不同區域?] 選項中選取 [是]，並輸入您想要用來執行估算的費率。
