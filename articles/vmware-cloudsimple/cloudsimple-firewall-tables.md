# <a name="firewall-tables-overview"></a>防火牆表概述

防火牆表列出了用於篩選往來私有雲資源的網路流量的規則。 您可以將防火牆表應用於 VLAN/子網。 規則控制源網路或 IP 位址與目標網路或 IP 位址之間的網路流量。

## <a name="firewall-rules"></a>防火牆規則

下表描述了防火牆規則中的參數。

| 屬性 | 詳細資料 |
| ---------| --------|
| **名稱** | 唯一標識防火牆規則及其用途的名稱。 |
| **優先順序** | 數位介於 100 和 4096 之間，其中 100 是最高優先順序。 規則會依照優先順序進行處理。 當流量遇到規則匹配時，規則處理將停止。 因此，不會處理優先順序較低的規則，這些規則的屬性與優先順序較高的規則相同。  小心避免衝突的規則。 |
| **狀態跟蹤** | 跟蹤可以是無狀態的（私有雲、互聯網或 VPN）或有狀態（公共 IP）。  |
| **協定** | 選項包括"任意"、TCP 或 UDP。 如果需要 ICMP，請使用"任意"。 |
| **方向** | 規則是否套用至輸入或輸出流量。 |
| **動作** | 允許或拒絕規則中定義的流量類型。 |
| **來源** | IP 位址、無類域間路由 （CIDR） 塊 （例如 10.0.0.0/24）或 Any。  指定範圍、服務標記或應用程式安全性群組使您能夠創建較少的安全規則。 |
| **源埠** | 網路流量源自的埠。  您可以指定單個埠或埠範圍，例如 443 或 8000-8080。 指定範圍可讓您建立較少的安全性規則。 |
| **目的地** | IP 位址、無類域間路由 （CIDR） 塊 （例如 10.0.0.0/24）或 Any。  指定範圍、服務標記或應用程式安全性群組使您能夠創建較少的安全規則。  |
| **目標埠** | 網路流量流向的埠。  您可以指定單個埠或埠範圍，例如 443 或 8000-8080。 指定範圍可讓您建立較少的安全性規則。|

### <a name="stateless"></a>無狀態

無狀態規則僅查看單個資料包，並根據規則對其進行篩選。  
對於相反方向的流量，可能需要其他規則。  對以下點之間的流量使用無狀態規則：

* 私有雲子網
* 本地子網和私有雲子網
* 來自私有雲的互聯網流量

### <a name="stateful"></a>可設定狀態

 有狀態的規則知道通過它的連接。 系統會為現有連線建立流程記錄。 允許或拒絕通訊都會以此流程記錄的連線狀態為依據。  對於公共 IP 位址使用此規則類型來篩選來自 Internet 的流量。

### <a name="default-rules"></a>預設規則

每個防火牆表上都創建了以下預設規則。

|優先順序|名稱|狀態跟蹤|方向|流量類型|通訊協定|來源|來源連接埠|Destination|目的地連接埠|動作|
|--------|----|--------------|---------|------------|--------|------|-----------|-----------|----------------|------|
|65000|允許全互聯網|可設定狀態|輸出|公共 IP 或互聯網流量|全部|任意|任意|任意|任意|Allow|
|65001|拒絕從互聯網所有|可設定狀態|輸入|公共 IP 或互聯網流量|全部|任意|任意|任意|任意|拒絕|
|65002|允許所有到內聯網|無狀態|輸出|私有雲內部或 VPN 流量|全部|任意|任意|任意|任意|Allow|
|65003|允許所有來自內聯網|無狀態|輸入|私有雲內部或 VPN 流量|全部|任意|任意|任意|任意|Allow|
