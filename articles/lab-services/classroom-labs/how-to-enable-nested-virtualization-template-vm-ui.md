# <a name="enable-nested-virtualization-on-a-template-virtual-machine-in-azure-lab-services-manually"></a>在 Azure 實驗室服務中的範本虛擬機器上手動啟用嵌套虛擬化

嵌套虛擬化使您能夠在實驗室的範本虛擬機器內創建多 VM 環境。 發佈範本將為實驗室中的每個使用者提供一個虛擬機器，其中設置多個 VM。  有關嵌套虛擬化和 Azure 實驗室服務的詳細資訊，請參閱在[Azure 實驗室服務中的範本虛擬機器上啟用嵌套虛擬化](how-to-enable-nested-virtualization-template-vm.md)。

本文介紹如何使用 Windows 角色和工具在 Azure 實驗室服務中的範本電腦上設置嵌套虛擬化。  要使類能夠使用嵌套虛擬化，需要一些操作。  以下步驟將介紹如何使用 Hyper-V 手動設置實驗室服務電腦範本。  步驟適用于 Windows 伺服器 2016 或 Windows 伺服器 2019。  

>[!IMPORTANT]
>創建實驗室時，為虛擬機器大小選擇 **"大型（嵌套虛擬化）"** 或 **"中型"（嵌套虛擬化）。**  嵌套虛擬化將不起作用。  

## <a name="enable-hyper-v-role"></a>啟用 Hyper-V 角色

以下步驟描述了使用任一伺服器管理員在 Windows 伺服器上啟用 Hyper-V 所需的操作。  安裝成功後，Hyper-V 管理器將可用於添加、修改和刪除用戶端虛擬機器。

1. 在 **"伺服器管理員**"中，按一下"**添加角色和功能**"。
2. 在 [在您開始前]**** 頁面上，按一下 [下一步]****。
3. 在 **"選擇安裝類型"** 頁上，保留基於角色或基於功能的安裝的預設選擇，然後按一下"**下一步**"。
4. 在 **"選擇目標伺服器"** 頁上，從伺服器池中選擇伺服器。   當前伺服器將已被選中。  按 [下一步]。
5. 在 [選取伺服器角色]**** 頁面上，選取 [Hyper-V]****。  
6. 將顯示"**添加角色和功能嚮導"** 快顯視窗。  選擇 **"包括管理工具（如果適用）**"。  按一下"**添加功能"** 按鈕。
7. 在 [選取伺服器角色]**** 頁面上，按 [下一步]****。
8. 在 **"選擇功能"頁上**，按一下 **"下一步**"。
9. 在 [Hyper-V]**** 頁面上，按 [下一步]****。
10. 在 **"創建虛擬切換"** 頁上，接受預設值，然後按一下"**下一步**"。
11. 在 **"虛擬機器遷移**"頁上，接受預設值，然後按一下"**下一步**"。
12. 在 **"預設存儲"** 頁上，接受預設值，然後按一下"**下一步**"。
13. 在 **"確認安裝選擇"** 頁上，根據需要**自動選擇重新開機目標伺服器**。
14. 當"**添加角色和功能嚮導"** 快顯視窗出現時，按一下"**是**"。
15. 按一下 **[安裝]**。
16. 等待 **"安裝進度"** 頁以指示 Hyper-V 角色已完成。  機器可能在安裝中重新開機。
17. 按一下 **關閉**。

## <a name="enable-dhcp-role"></a>啟用 DHCP 角色

創建的任何 Hyper-V 用戶端虛擬機器都需要 NAT 網路中的 IP 位址。  稍後我們將創建 NAT 網路。  分配 IP 位址的一種方法是將主機（本例中為實驗室虛擬機器範本）設置為 DHCP 伺服器。  以下是啟用 DHCP 角色所需的步驟。

1. 在 **"伺服器管理員**"中**Dashboard**，按一下"**添加角色和功能**"。
2. 在 [在您開始前]**** 頁面上，按一下 [下一步]****。
3. 在 [選取安裝類型]**** 頁面上，選取 [角色型或功能型安裝]****，然後按 [下一步]****。
4. 在 **"選擇目標伺服器"** 頁上，從伺服器池中選擇當前伺服器，然後按一下"**下一步**"。
5. 在 **"選擇伺服器角色**"頁上，選擇**DHCP 伺服器**。  
6. 將顯示"**添加角色和功能嚮導"** 快顯視窗。  選擇 **"包括管理工具（如果適用）**"。  按一下 **[新增功能]**。

    >[!NOTE]
    >您可能會看到一個驗證錯誤，指出未找到靜態 IP 位址。  對於我們的方案，可以忽略此警告。

7. 在 [選取伺服器角色]**** 頁面上，按 [下一步]****。
8. 在 [選取功能]**** 頁面上，按 [下一步]****。
9. 在**DHCP 伺服器**頁上，按一下 **"下一步**"。
10. 在 [確認安裝選項]**** 頁面上，按一下 [安裝]****。
11. 等待 **"安裝進度"頁**以指示 DHCP 角色已完成。
12. 按一下 [關閉]。

## <a name="enable-routing-and-remote-access-role"></a>啟用路由和遠端存取角色

1. 在 **"伺服器管理員**"中**Dashboard**，按一下"**添加角色和功能**"。
2. 在 [在您開始前]**** 頁面上，按一下 [下一步]****。
3. 在 [選取安裝類型]**** 頁面上，選取 [角色型或功能型安裝]****，然後按 [下一步]****。
4. 在 **"選擇目標伺服器"** 頁上，從伺服器池中選擇當前伺服器，然後按一下"**下一步**"。
5. 在 **"選擇伺服器角色**"頁上，選擇 **"遠端存取**"。 按一下 [確定]****。
6. 在 [選取功能]**** 頁面上，按 [下一步]****。
7. 在 **"遠端存取"** 頁上，按一下"**下一步**"。
8. 在**角色服務**頁上，選擇 **"路由**"。
9. 將顯示"**添加角色和功能嚮導"** 快顯視窗。  選擇 **"包括管理工具（如果適用）**"。  按一下 **[新增功能]**。
10. 按 [下一步]****。
11. 在 [網頁伺服器 (IIS) 角色]**** 頁面上，按一下 [下一步]****。
12. 在 [選取角色服務]**** 頁面上，按 [下一步]****。
13. 在 [確認安裝選項]**** 頁面上，按一下 [安裝]****。
14. 等待 **"安裝進度"** 頁以指示遠端存取角色已完成。  
15. 按一下 **關閉**。

## <a name="create-virtual-nat-network"></a>創建虛擬 NAT 網路

現在，所有必要的角色都已安裝完畢，是時候創建 NAT 網路了。  創建過程將涉及創建交換器和 NAT 網路本身。  NAT（網路位址轉譯）網路將公共 IP 位址分配給私人網路絡上的一組 VM，以便連接到互聯網。  在我們的例子中，私有 VM 組將是嵌套 VM。  NAT 網路將允許嵌套 VM 相互通信。 交換器是處理網路中流量接收和路由的網路設備。

### <a name="create-a-new-virtual-switch"></a>創建新虛擬交換器

1. 從 Windows 管理工具打開**超 V 管理器**。
2. 在左側導航功能表中選擇當前伺服器。
3. 按一下**虛擬交換器管理器...** 從**Hyper-V 管理器**右側的 **"操作"** 功能表。
4. 在**虛擬交換器管理器**快顯視窗上，為要創建的交換器類型選擇 **"內部**"。  按一下 [建立虛擬交換器]****。
5. 對於新創建的虛擬交換器，將名稱設置為令人難忘的內容。  在此示例中，我們將使用"實驗室服務交換器"。  按一下 [確定]****。
6. 將創建新的網路介面卡。  該名稱將類似于"v乙太網（實驗室服務交換器）"。  要驗證打開**控制台**，請按一下 **"網路和互聯網**"，按一下"**查看網路狀態和任務**"。  在左側，按一下"**更改配接器設置**"。

### <a name="create-a-nat-network"></a>創建 NAT 網路

1. 從 Windows 管理工具打開**路由和遠端存取**工具。
2. 在左側導航頁中選擇本機伺服器。
3. 選擇**操作** -> **配置和啟用路由和遠端存取**。
4. 當**出現路由和遠端存取服務器設置嚮導時**，按一下 **"下一步**"。
5. 在 **"配置"** 頁上，選擇**網路位址轉譯 （NAT）** 配置。  按 [下一步]****。

    >[!WARNING]
    >不要選擇"虛擬私人網路絡 （VPN） 訪問和 NAT"選項。

6. 在**NAT 互聯網連接**頁面上，選擇"乙太網"。  不要選擇我們在 Hyper-V 管理器中創建的"vEthernet（實驗室服務交換器）"連接。 按 [下一步]****。
7. 按一下嚮導最後一頁上的 **"完成**"。
8. 當出現 **"開始服務**"對話方塊時，按一下"**開始服務**"。
9. 等待服務啟動。

## <a name="update-network-adapter-settings"></a>更新網路介面卡設置

網路介面卡將與用於之前創建的 NAT 網路的預設閘道 IP 的 IP 相關聯。  在此示例中，我們創建一個 IP 位址 192.168.0.1，子網路遮罩為 255.255.255.0。  我們將使用前面創建的虛擬交換器。

1. 打開**控制台**，按一下 **"網路和互聯網**"，按一下 **"查看網路狀態和任務**"。
2. 在左側，按一下"**更改配接器設置**"。  
3. 在 **"網路連接**"視窗中，按兩下"v乙太網（實驗室服務交換器）"以顯示**vEthernet（實驗室服務交換器）狀態**詳細資訊對話方塊。
4. 按一下"**屬性"** 按鈕。
5. 選擇**Internet 協定版本 4 （TCP/IPv4）** 專案，然後按一下"**屬性**"按鈕。
6. 在**Internet 協定版本 4 （TCP/IPv4） 屬性**對話方塊中，選擇 **"使用以下 IP 位址**"。  對於 ip 位址，請輸入 192.168.0.1。 對於子網路遮罩，輸入 255.255.255.0。  將預設閘道留空。  將 DNS 伺服器留空。

    >[!NOTE]
    > 我們的 NAT 網路範圍將以 CIDR 符號標記法 192.168.0.0/24 表示。  這將創建一系列可用的 IP 位址，從 192.168.0.1 到 192.168.0.254。  按照慣例，閘道具有子網範圍內的第一個 IP 位址。

7. 按一下 [確定]。

## <a name="create-dhcp-scope"></a>建立 DHCP 領域

以下步驟是添加 DHCP 作用域的說明。  在本文中，我們的 NAT 網路在 CIDR 符號中為 192.168.0/24。  這將創建一系列可用的 IP 位址，從 192.168.0.1 到 192.168.0.254。  創建的範圍必須位於該可用位址範圍內，不包括之前已創建的 IP 位址。

1. 打開**管理工具**並打開**DHCP**管理工具。
2. 在**DHCP**工具中，展開當前伺服器的節點並選擇**IPv4**。
3. 在"操作"功能表中，選擇 **"新範圍..."...**
4. 出現 **"新範圍"嚮導**時，按一下 **"歡迎**頁面的 **"下一步**"。
5. 在 **"範圍名稱**"頁上，輸入"LabServicesDhcpScope"或其他令人難忘的名稱。  按 [下一步]****。
6. 在**IP 位址範圍**頁上，輸入以下值。

    - 192.168.0.100 用於啟動 IP 位址
    - 192.168.0.200 用於最終 IP 位址
    - 長度 24 表示
    - 255.255.255.0 用於子網路遮罩

7. 按 [下一步]****。
8. 在"**添加排除項和延遲"** 頁上，按一下 **"下一步**"。
9. 在 **"租約持續時間"** 頁上，按一下 **"下一步**"。
10. 在 **"配置 DHCP 選項**"頁上，選擇 **"是"，我現在要配置這些選項**。 按 [下一步]****。
11. 在**路由器上（預設閘道）**
12. 添加 192.168.0.1（如果尚未完成）。 按 [下一步]****。
13. 在**功能變數名稱和 DNS 伺服器**頁面上，添加 168.63.129.16 作為 DNS 伺服器 IP 位址（如果尚未完成）。  168.63.129.16 是 Azure 靜態 DNS 伺服器的 IP 位址。 按 [下一步]****。
14. 在**WINS 伺服器**頁面上，按一下 **"下一步**"。
15. 一個 **"啟動範圍**"頁，選擇 **"是"，我現在要啟動此作用域**。  按 [下一步]****。
16. 在 **"完成新範圍嚮導"** 頁上，按一下"**完成**"。
