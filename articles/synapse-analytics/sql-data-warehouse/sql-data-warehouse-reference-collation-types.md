# <a name="database-collation-support-for-azure-synapse-analytics-sql-pool"></a>Azure 同步分析 SQL 池的資料庫排序規則支援

創建新的 Azure Synapse SQL 池資料庫時，可以從 Azure 門戶更改預設資料庫排序規則。 此功能使得使用 3800 支援的資料庫排序規則之一創建新資料庫變得更加容易。 

排序規則為基於字元的資料類型提供地區設定、字碼頁、排序次序和字元敏感度規則。 選擇後，需要排序規則資訊的所有列和運算式都將從資料庫設置繼承所選排序規則。 可以通過顯式聲明基於字元的資料類型的不同排序規則來覆蓋預設繼承。

## <a name="changing-collation"></a>更改排序規則
要更改預設排序規則，請更新到預配體驗中的"拼貼"欄位。

例如，如果要將預設排序規則更改為區分大小寫，只需將排序規則從SQL_Latin1_General_CP1_CI_AS重命名為SQL_Latin1_General_CP1_CS_AS。

## <a name="checking-the-current-collation"></a>檢查當前排序規則

要檢查資料庫的當前排序規則，可以運行以下 T-SQL 程式碼片段：

```sql
SELECT DATABASEPROPERTYEX(DB_NAME(), 'Collation') AS Collation;
```

當傳遞"排序規則"作為屬性參數時，DatabasePropertyEx 函數將返回指定的資料庫的當前排序規則。 有關詳細資訊，請參閱[資料庫屬性Ex。](/sql/t-sql/functions/databasepropertyex-transact-sql)
