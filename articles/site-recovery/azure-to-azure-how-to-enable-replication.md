# <a name="replicate-azure-vms-to-another-azure-region"></a>將 Azure VM 複製到另一個 Azure 區域


本文說明如何將 Azure VM 從一個 Azure 區域複寫到另一個區域。

## <a name="before-you-start"></a>開始之前

本文假定您已為網站恢復部署做好準備，如[Azure 到 Azure 災害復原教程](azure-to-azure-tutorial-enable-replication.md)中所述。

先決條件應該就位，並且您應該已經創建了恢復服務保存庫。


## <a name="enable-replication"></a>啟用複寫

啟用複寫。 此程序假設主要 Azure 區域為東亞，而次要區域為東南亞。

1. 在保存庫中，按一下 [+複寫]****。
2. 請注意以下欄位：
   - **來源**：VM 的起點，在此案例中為 **Azure**。
   - **源位置**：要保護 VM 的 Azure 區域。 在此圖例中，來源位置是 [東亞]
   - **部署模型**：來源機器的 Azure 部署模型。
   - **源訂閱**：源 VM 所屬的訂閱。 這可以是您的復原服務保存庫所在的相同 Azure Active Directory 租用戶內的任何訂用帳戶。
   - **資源群組**：來源虛擬機器所屬的資源群組。 下一個步驟會列出所選取資源群組下的所有 VM，以供保護。

     ![啟用複寫](./media/site-recovery-replicate-azure-to-azure/enabledrwizard1.png)

3. 在 [虛擬機器] > [選取虛擬機器]**** 中，按一下並選取您要複寫的每部 VM。 您只能選取可以啟用複寫的機器。 然後按一下 **[確定]**。
    ![啟用複寫](./media/site-recovery-replicate-azure-to-azure/virtualmachine_selection.png)

4. 在 [設定]**** 中，您可以選擇性地設定目標網站設定：

   - **目標位置**：將複寫來源虛擬機器資料的位置。 端視您選取的機器位置而定，Site Recovery 將提供適當目標區域的清單。 建議您讓目標位置與復原服務保存庫位置保持相同。
   - **目標訂用帳戶**：用於災害復原的目標訂用帳戶。 根據預設，目標訂用帳戶會與來源訂用帳戶相同。
   - **目標資源群組**：所有已複寫虛擬機器所屬的資源群組。
       - 預設情況下，網站恢復在目的地區域中創建一個新資源組，名稱中具有"asr"尾碼。
       - 如果網站恢復創建的資源組已存在，則重用它。
       - 您可以自訂資源組設置。
       - 目標資源組的位置可以是任何 Azure 區域，但承載源 VM 的區域除外。
   - **目標虛擬網路**：預設情況下，網站恢復在目的地區域中創建一個新的虛擬網路，名稱中帶有"asr"尾碼。 這會對應至您的來源網路，並用來進行任何未來的保護。 [深入了解](site-recovery-network-mapping-azure-to-azure.md)網路對應。
   - **目標存儲帳戶（源 VM 不使用託管磁片）：** 預設情況下，網站恢復創建一個新的目標存儲帳戶，模仿源 VM 存儲配置。 如果儲存體帳戶已經存在，就會重複使用。
   - **副本託管磁片（源 VM 使用託管磁片）：** 網站恢復在目的地區域中創建新的複製副本託管磁片，以鏡像源 VM 的託管磁片，其存儲類型（標準或高級）與源 VM 的託管磁片相同。
   - **快取儲存體帳戶**：Site Recovery 需要在來源區域中有額外的儲存體帳戶 (稱為快取儲存體)。 在來源 VM 上發生的所有變更都會受到追蹤，並傳送到快取儲存體帳戶，然後複寫到目標位置。 此存儲帳戶應為標準帳戶。
   - **目標可用性集**：預設情況下，網站恢復在目的地區域中創建一個新的可用性集，名稱中帶有"asr"尾碼，用於作為源區域中可用性集一部分的 VM。 如果網站恢復創建的可用性集已存在，則重複使用。
   - **目標可用性區域**：根據預設，如果目標區域支援可用性區域，則 Site Recovery 會在目標區域中指派與來源區域相同的區域編號。

     如果目標區域不支援可用性區域，則目標 VM 會預設為單一執行個體。 如有需要，您可以按一下 [自訂]，將這類 VM 設定為目標區域中可用性設定組的一部分。

     >[!NOTE]
     >啟用複寫之後，便無法變更可用性類型 - 單一執行個體、可用性設定組或可用性區域。 您需要停用後再啟用複寫，以變更可用性類型。
     >

   - **複寫原則**：它定義復原點保留歷史記錄和應用程式一致的快照頻率的設置。 預設情況下，Azure 網站恢復會創建一個新的複寫原則，預設設置為"24 小時"，用於復原點保留，為應用一致的快照頻率創建"4 小時"。

     ![啟用複寫](./media/site-recovery-replicate-azure-to-azure/enabledrwizard3.PNG)

### <a name="enable-replication-for-added-disks"></a>為添加的磁片啟用複製

如果將磁片添加到啟用複製的 Azure VM，則會發生以下情況：
-   VM 的複製運行狀況顯示警告，並且注釋會通知您一個或多個磁片可供保護。
-   如果為添加的磁片啟用保護，則該警告將在磁片的初始複製後消失。
-   如果選擇不為磁片啟用複製，則可以選擇關閉該警告。


    ![添加了新磁片](./media/azure-to-azure-how-to-enable-replication/newdisk.png)

要啟用添加磁片的複製，執行以下操作：

1.  在複製**專案**>保存庫中，按一下向其添加磁片的 VM。
2.  按一下**磁片**，然後選擇要為其啟用複製的資料磁片（這些磁片的狀態**未受保護**）。
3.  在**磁片詳細資訊**中，按一下啟用**複製**。

    ![為添加的磁片啟用複製](./media/azure-to-azure-how-to-enable-replication/enabled-added.png)

啟用複製作業運行並完成初始複製後，將刪除磁片問題的複製運行狀況警告。



## <a name="customize-target-resources"></a>自訂目標資源

您可以修改 Site Recovery 所使用的預設目標設定。

1. 按一下「目標訂用帳戶」旁的 [自訂]**** 來修改預設目標訂用帳戶。 在同一個 Azure Active Directory (AAD) 租用戶中，從所有可用的訂用帳戶清單中選取訂用帳戶。

2. 按一下 [自訂:]**** 以修改預設設定：
    - 在 [目標資源群組]**** 中，從訂用帳戶目標位置上的所有資源群組清單中選取資源群組。
    - 在 [目標虛擬網路]**** 中，從目標位置上的所有虛擬網路清單中選取網路。
    - 在 [可用性設定組]**** 中，如果 VM 是來源區域中可用性設定組的一部分，即可將可用性設定組新增至其中。
    - 在 [目標儲存體帳戶]**** 中，選取您要使用的帳戶。

        ![啟用複寫](./media/site-recovery-replicate-azure-to-azure/customize.PNG)
3. 按一下 [自訂:]**** 以修改複寫設定。
4. 在**多 VM 一致性**中，選擇要一起複製的 VM。
    - 在容錯移轉時，複寫群組中的所有機器都會共用當機時保持一致和應用程式一致復原點。
    - 啟用多 VM 一致性可能會影響工作負載性能（因為它佔用 CPU。 僅當電腦運行相同的工作負載並且需要跨多台電腦保持一致性時，才應啟用它。
    - 例如，如果應用程式具有 2 個 SQL Server 虛擬機器和兩個 Web 服務器，則應僅將 SQL Server VM 添加到複製組。
    - 您可以選擇複製組中最多有 16 個 VM。
    - 如果您啟用多部 VM 一致性，則複寫群組中的機器會透過連接埠 20004 彼此通訊。
    - 確保沒有防火牆設備阻止 VM 通過埠 20004 之間的內部通信。
    - 如果希望 Linux VM 成為複製組的一部分，請確保根據特定 Linux 版本的指南手動打開端口 20004 上的出站流量。
![啟用複寫](./media/site-recovery-replicate-azure-to-azure/multivmsettings.PNG)

5. 按一下 **"創建目標資源** > **啟用複製**"。
6. 啟用 VM 以進行複寫之後，您就可以在 [複寫的項目]**** 下方檢查 VM 健康情況的狀態。

>[!NOTE]
>在初始複寫期間，狀態可能需要一些時間才會重新整理，期間不會有任何進展。 按一下 [重新整理]**** 按鈕來取得最新狀態。
>
