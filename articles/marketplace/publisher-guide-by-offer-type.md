# <a name="publishing-guide-by-offer-type"></a>依供應項目類型的發行指南

一旦[您決定發布選項](https://docs.microsoft.com/azure/marketplace/determine-your-listing-type),您就可以選擇將用於顯示產品/服務的報價類型。

 「供應項目類型」** 會定義供應項目結構 (包括用來在市集中展示供應項目的中繼資料、成品和其他內容)。

您必須先選擇供應項目類型，才能建立供應項目。 供應項目類型會對應至您想要發佈的解決方案、應用程式服務供應項目類型，以及對等的 Microsoft 產品和服務。

可以以不同方式配置單個產品/服務類型,以啟用不同的發佈選項、操作調用、預配或定價。 發佈選項和供應項目類型的組態，也會符合供應項目資格和技術需求。

在您建立供應項目之前，請務必檢閱店面和供應項目類型資格需求，以及技術發佈需求。

## <a name="list-of-offer-types"></a>供應項目類型的清單

下表列出 Azure Marketplace 的供應項目類型。

| **優惠類型**    | **說明**  |
| :------------------- | :-------------------|
| [**虛擬機器**](./marketplace-virtual-machines.md) | 當您要將虛擬設備部署到與客戶相關的訂用帳戶時，請使用「虛擬機器」供應項目類型。 |
| [**解決方案範本**](./marketplace-solution-templates.md) | 當解決方案需要單個 VM 以外的其他部署和配置自動化時,請使用解決方案範本(也稱為 Azure 應用程式)產品類型。 解決方案範本可以使用許多不同類型的 Azure 資源,包括但不限於 VM。  |
| [**託管應用程式**](./marketplace-managed-apps.md) | 必須符合下列條件時，請使用「Azure 應用程式：受控應用程式」供應項目類型： <br> <ul> <li> 您使用 VM 或整個 IaaS 型解決方案為客戶部署訂用帳戶型解決方案。 </li> <li>您或您的客戶要求由合作夥伴管理解決方案。 </li> <ul> |
| [**SaaS 應用程式**](./marketplace-saas-applications-technical-publishing-guide.md) | 您可以使用「SaaS 應用程式」供應項目類型，讓客戶以訂用帳戶的形式購買您的 SaaS 型技術解決方案。 |
| [**容器供應項目**](./marketplace-containers.md) | 當您的解決方案是佈建成 Kubernetes 型 Azure 容器服務的 Docker 容器映像時，請使用「容器」供應項目類型。 |
| [**Azure IoT 邊緣模組**](./iot-edge-module.md) | Azure IoT Edge 模組是 IoT Edge 所管理的最小計算單位，並可包含 Microsoft 服務 (例如「Azure 串流分析」)、第三方服務，或您自己的解決方案特定程式碼。 |
| [**諮詢服務**](./consulting-services.md) | 諮詢服務有助於將客戶與支援和擴展其使用 Azure、Dynamics 365 或 Power Suite 服務的服務聯繫起來。|
| [**Office 365、動態 365 和 Power BI**](./appsource-offer-publishing-guide.md) | 您可以發佈基於動態 365、Office 365、Power BI 和電源應用構建或擴展的 AppSource 產品。|
| [**整合式解決方案**](./integrated-solutions-for-publishers.md) | 您可以發佈集成的、與行業一致的解決方案,將技術和服務作為單一產品群組。|

有關單一登入要求的資訊,請參閱清單選項和產品/服務類型,請參閱[**Azure AD 要求**](./enable-appsource-marketplace-using-azure-ad.md)。
