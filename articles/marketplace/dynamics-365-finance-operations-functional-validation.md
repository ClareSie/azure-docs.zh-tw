# <a name="appsource-dynamics-365-finance-and-operations-functional-validation"></a>AppSource Dynamics 365 財務和作業功能驗證

若要在[合作夥伴中心](https://partner.microsoft.com/dashboard/home)完成第一次發佈，Dynamics 365 財務和作業的優惠需要兩項功能驗證：

- 上傳顯示基本功能的 Dynamics 365 環境示範影片。
- 提供示範解決方案[生命週期服務](https://lcs.dynamics.com/)（LCS）環境的螢幕擷取畫面。

> [!NOTE]
> 後續的重新認證發佈不需要示範。 若要深入瞭解，請參閱[AppSource 原則檔](https://docs.microsoft.com/legal/marketplace/certification-policies#1440-dynamics-365-finance-ops)。

## <a name="how-to-validate"></a>如何驗證

功能驗證有兩個選項：

- 在太平洋標準時間（PST）上班時間舉辦30分鐘的會議電話，以示範並記錄[LCS](https://lcs.dynamics.com/)環境和解決方案，或
- 在合作夥伴中心，移至 [[商業 Marketplace](https://partner.microsoft.com/dashboard/commercial-marketplace/overview)  >  ]**[總覽**]，並在供應專案的 [補充內容] 索引標籤上上傳示範影片 URL 和 LCS 螢幕擷取畫面。

Microsoft 認證小組會回顧影片和檔案，然後核准解決方案，或透過電子郵件傳送您的後續步驟。

### <a name="option-1-30-minute-conference-call"></a>選項 1:30-分鐘會議通話

若要排程最後的審查呼叫，請與 [appsourceCRM@microsoft.com](mailto:appsourceCRM@microsoft.com) 您的供應專案名稱和一些可能的時間位置（介於上午8點之間）聯繫。 和下午5點 太平洋時間。

### <a name="option-2-upload-a-demo-video-and-lcs-screenshots"></a>選項2：上傳示範影片和 LCS 螢幕擷取畫面

1. 錄製影片，並將位址上傳至您選擇的主控網站。 請遵循這些方針：

    - 由 Microsoft 認證小組觀看。
    - 長度少於20分鐘。
    - 在 Dynamics 365 環境中包含最多三個核心功能重點。

    > 如果符合方針，就可以使用現有的行銷影片。
