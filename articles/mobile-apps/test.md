# <a name="perform-automated-testing-on-thousands-of-mobile-devices-simultaneously"></a>同時在數千台行動裝置上執行自動測試
由於設備碎片、應用程式複雜性、快速發佈週期、短會話和較高的使用者期望，測試可能具有挑戰性且越來越複雜。 您需要定期確保您提供經過全面測試的應用程式，從而提供卓越的客戶體驗。 測試不是一項簡單的任務，而且使用開發人員時間手動測試每個版本也不是很好。

測試挑戰可以有效應對。 您可以使用自動測試在各種設備上測試移動應用程式。 您可以使用雲服務加快移動應用的發佈，並克服為應用程式開發和部署添加瓶頸的測試挑戰。

## <a name="importance-of-automated-testing"></a>自動測試的重要性
- 通過定位不同的外形規格、連線類型和作業系統版本，提高測試覆蓋率。
- 通過在不同的設備和作業系統覆蓋範圍上進行測試，節省時間、資金和資源。
- 通過避免重複的手動測試來提高準確性，並有更多時間創建新的自動化軟體測試並處理複雜的功能。
- 持續測試，分析應用程式性能，並在使用者遇到問題並快速發佈之前查找和修復問題。
- 通過確保客戶在各種設備上獲得最佳體驗，從而提供高品質的應用程式。

## <a name="visual-studio-app-center"></a>Visual Studio App Center
[App Center 測試](/appcenter/test-cloud/)是一種基於雲的服務，可用於在雲中的數千個真實設備上運行針對本機和混合移動應用的自動 UI 測試。 使用 2，000 多個設備，您可以編寫可自動執行的測試腳本，使測試變得簡單。

**主要功能**
   - 在真實設備上運行的測試不是根植或越獄，以提供使用者環境的最佳近似值。
   - 螢幕截圖會自動捕獲並附加到每個測試步驟。 可在試運行中對所有目標設備並排查看它們。
   - 測試回合中所有目標設備的設備日誌和失敗測試步驟的堆疊追蹤。
   - 併發測試同時在多個設備上運行，以加快結果的時間。
   - 流行的測試框架支援，如適用于Android和iOS的Appium（JAVA與JUnit），適用于Android的Espresso（JAVA），適用于Android和iOS的卡拉巴什，適用于Android和iOS的Xamarin.UITest，以及適用于iOS的XCUITest。
   - 對 iOS、Android Xamarin 和反應本機的平臺支援。
