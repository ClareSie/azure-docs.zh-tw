# <a name="container-publishing-prerequisites"></a>容器發佈先決條件

> [!IMPORTANT]
> 從2020年4月13日開始，我們會開始將您的 Azure 容器供應專案管理移至合作夥伴中心。 在遷移之後，您將在合作夥伴中心建立和管理您的供應專案。 請依照[建立 Azure 容器供應](https://docs.microsoft.com/azure/marketplace/partner-center-portal/create-azure-container-offer)專案中的指示來管理您的已遷移優惠。

本文說明在 Azure Marketplace 上發佈容器供應項目的先決條件。  如果您尚未這麼做，請參閱[容器供應專案發佈指南](../../marketplace-containers.md)。

## <a name="publishing-prerequisites"></a>發佈先決條件

若要發佈新的容器映像，您必須符合下列先決條件：

- 可存取 Cloud Partner 入口網站。 如需詳細資訊，請參閱《Azure Marketplace 和 AppSource 發行指南》。
- 同意 Azure Marketplace 條款
- 將您的容器技術資產裝載於 Azure Container Registry 中。
- 備妥容器中繼資料以供使用。 例如，下列非完整清單：
  - 標題
  - 描述 (HTML 格式)
  - 標誌影像 (PNG 格式)，並採用以下的固定影像大小：40x40 px、90x90 px、115x 115 px 和 255x115 px。
- *使用條款*和*隱私權原則*聲明
- 容器解決方案檔
- 支援連絡人

## <a name="next-steps"></a>後續步驟

如果您尚未這麼做，您必須在[建立容器供應](./cpp-create-offer.md)專案之前，先[準備您的容器技術資產](./cpp-create-technical-assets.md)。
