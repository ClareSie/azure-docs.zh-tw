# <a name="comparing-microsoft-appsource-and-azure-marketplace"></a>比較微軟應用程式源和 Azure 應用商店

微軟的商業市場包括[微軟應用程式源](https://appsource.microsoft.com)和[Azure 市場](https://azuremarketplace.microsoft.com)。 每個網店都提供獨特的客戶需求和不同的目標受眾,因此您可以確保您的解決方案或服務可供合適的客戶使用。

## <a name="understanding-the-differences-between-storefronts"></a>了解店面之間的差異

要選擇網店,請先確定產品/服務的目標受眾。 如果您鎖定多個店面的對象，您可發佈一次以在多個店面中銷售。

| Azure Marketplace | AppSource |
|:--- |:--- |
| IT 專業人員和開發人員 | 企業使用者與企業決策者 |

下表說明使用店面的優點：

| 優點 | Azure Marketplace | AppSource |
|:--- |:--- | :--- |
| **計費靈活性** | Azure 應用商店支援免費、試用、BYOL 和交易產品類型。 交易優惠既基於消費,要麼基於訂閱。 基於消費的優惠按小時使用量計費,並提供初始免費期限。 基於訂閱的服務按月或按按座位或固定費率按月計費。| AppSource 支援免費、試用、BYOL 和交易產品類型。 Transact 產品是基於訂閱的 SaaS 產品,可以按月計費,也可以按每個座位或固定費率按年計費。 |
|**與其他合作夥伴的往來**  | 您目前無法將服務提供者或交付合作夥伴與供應項目連結。 | 將獨立軟體廠商、系統整合商及受控服務提供者與特定實作案例連結。 您可以合作向新客戶進行銷售。 |
| **自動化** | 您目前無法將服務提供者或交付合作夥伴與供應項目連結。 透過附加元件佈建利用自動化軟體即服務 (SaaS)。 使用「Azure 應用程式：解決方案」範本發佈選項，將 SaaS 型資料收集和部署案例自動化。 | 將獨立軟體廠商、系統整合商及受控服務提供者與特定實作案例連結。 您可以合作向新客戶進行銷售。 |
| **多種雲類型** | 使用下列類型，發佈適用於公用雲端與內部部署環境的解決方案：<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>區域雲端，包括中國和德國</li></ul> | 目前不支援 Azure Stack、Azure Government 或區域雲端。 |
| **向客戶介紹上下文內** | 在內容相關式搜尋的 Azure 入口網站內體驗中提供您的解決方案。 使用「虛擬機器」或「Azure 應用程式：解決方案範本」發佈選項。 | 透過 Microsoft 產品 (例如 Dynamics 365、Power BI 和 Office 365) 的應用程式內體驗，來觸及更多客戶。 |
