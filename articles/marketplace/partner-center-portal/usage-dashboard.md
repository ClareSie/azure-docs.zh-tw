# <a name="usage-dashboard-in-commercial-marketplace-analytics"></a>商業 marketplace 分析中的使用量儀表板

本文提供合作夥伴中心內 [使用量] 儀表板的相關資訊。 此儀表板會在兩個不同的索引標籤中顯示所有 VM 提供使用量和計量付費計量： VM 使用量和計量付費使用量。

若要存取使用量儀表板，請開啟 [商業 Marketplace] 底下的 [**[分析](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)**] 儀表板。

>[!NOTE]
> 如需分析術語的詳細定義，請參閱[商業 marketplace 分析的常見問題和術語](./faq-terminology.md)。

## <a name="usage-dashboard"></a>使用量儀表板

[使用量] 儀表板代表所有虛擬機器（VM）提供使用量和計量付費使用量的計量。 這些可在兩個不同的索引標籤中找到： VM 使用量和計量付費使用量。

在 [VM 使用量] 索引標籤中，有下列專案的圖形表示：

- [使用量摘要](#usage-summary)
- [依地理位置的使用量](#usage-by-geography)
- [使用方式（依優惠）](#usage-by-offers)
- [依供應專案和 Sku 的使用趨勢](#usage-trend-by-offers-and-skus)
- [依供應專案類型的使用量](#usage-by-offer-type)
- [依 VM 大小的使用量](#usage-by-vm-size)
- [依銷售通路的使用量](#usage-by-sales-channel)
- [詳細使用量資料](#detailed-usage-data)

> [!NOTE]
> 分析報表在 Cloud Partner 入口網站（CPP）和合作夥伴中心會以不同的方式顯示。 CPP 中的**賣方深入**解析具有 [訂單和使用量] 索引標籤，可顯示以使用量為基礎的供應專案和非使用量型供應專案的資料。 在合作夥伴中心，使用計量會顯示在另一個頁面上。

### <a name="usage-summary"></a>使用量摘要

[使用量摘要] 表格會顯示他們已購買之所有供應專案的客戶使用時數。

- 標準化的使用時數會定義為以正規化的使用時數，以考慮 VM 核心數目（[VM 核心數目] x [小時的原始使用量]）。 指定為 "SHAREDCORE" 的 Vm 會使用1/6 （或0.1666）作為 [VM 核心數目] 乘數。
- 已將原始使用量小時定義為 Vm 以小時為單位執行的時間量。
- 百分比值代表所選日期範圍的使用量成長變更（[上個月使用量–第一個月使用量]）/第一個月使用量）。
- 向上指的綠色三角形表示成長變更。
- 指向向下的紅色三角形表示相對於上一個月的負面成長變更。
- 微橫條圖代表每月的值。 您可以將滑鼠停留在圖表內的資料行上方，以顯示每個月的值。

### <a name="usage-by-geography"></a>依地理位置的使用量

[**依地理位置的正規化使用**熱度圖] 會根據客戶國家/地區顯示對應的使用時數。 國家/地區色彩變化代表標準化的使用集中。 按下地圖上的 [**首頁**] 按鈕，還原為原始的視圖。

### <a name="usage-by-offers"></a>使用方式（依優惠）

- [依供應專案的**正規化使用量**] 圓形圖會根據選取的日期範圍，顯示依供應專案的正規化使用時數細目。 前5個供應專案會顯示在 graph 中，其餘部分則會在「全部 rest」類別中分組。
- 橫條圖會描述所選日期範圍的每月成長趨勢。 月份資料行代表供應專案的使用時數，以及個別月份的最高使用時數。 折線圖會描述在次要 Y 軸上繪製的成長百分比趨勢。
- 使用圖表頂端的滑杆，沿著 X 軸和/或專注于特定資料點的向右滾動。

### <a name="usage-trend-by-offers-and-skus"></a>依供應專案和 Sku 的使用趨勢

此圖表顯示供應專案所選 Sku 的正規化使用趨勢。 [供應專案] 排行榜會顯示最高使用量的前50優惠，並依使用量小時排序。 [SKU] 排行榜會顯示前 50 Sku，其中包含所選供應專案的最高使用量。

### <a name="usage-by-offer-type"></a>依供應專案類型的使用量

- [**依供應專案類型的使用量**] 圓形圖會根據供應專案類型來組織使用量。
- 主要的供應專案會顯示在圖表中，其餘的供應專案則會分組為「全部 Rest」。
- **趨勢**圖會顯示每個月的成長趨勢。 [月份] 資料行代表該月份中的前幾個供應專案類型的使用量。

### <a name="usage-by-vm-size"></a>依 VM 大小的使用量

此圖表代表所選 VM 大小（最多5個）的使用趨勢（您的所有供應專案/Sku）。 直條圖會以所選 VM 大小的使用時數來堆疊。

排行榜會顯示最高使用量的前 50 VM 大小，並依使用時數來排序。

### <a name="usage-by-sales-channel"></a>依銷售通路的使用量

- [依銷售通路的使用量] 圓形圖會根據銷售通路來組織使用量
- 具有最高使用量的頂尖銷售通路會顯示在圖表中，而銷售通路的其餘部分則會分組為「全部 Rest」。
- [月份] 資料行代表該月份中的頂尖銷售通路的使用量。
- 此圖表的功能與 [依供應專案使用量] 圖表相同

### <a name="detailed-usage-data"></a>詳細使用量資料

[**使用量詳細資料] 資料表**會顯示依使用量排序之前1000使用量記錄的編號清單。

- 方格中的每個資料行都是可排序的。
- 如果記錄計數小於1000，則可以將資料解壓縮至 CSV 檔案。
- 如果記錄計數超過1000，則匯出資料會以非同步方式放置在下載頁面中，這會在未來30天內提供。
- 篩選器可以套用至**詳細的使用量資料**，只顯示您感興趣的資料。 資料可以依國家/地區、銷售通路、Marketplace 授權類型、使用類型、供應專案名稱、供應專案類型、免費試用、Marketplace 訂用帳戶識別碼、客戶識別碼和公司名稱進行篩選。

> [!NOTE]
> 選取 [頁面篩選] 中的 [**使用類型**]，在 [正規化視圖] 或 [原始視圖] 中查看頁面上的圖表。 這些圖表的預設視圖為「正規化視圖」。

[**使用量] 頁面篩選**會在頁面層級套用。 您可以選取多個篩選器來呈現您選擇要查看的準則圖表，以及要顯示在 [詳細使用量資料] 方格/匯出中的資料。 篩選準則會套用至已針對您在 [訂單] 頁面右上角選取的資料範圍所解壓縮的資料上。

- **供應專案類型**和**供應專案名稱**只會針對您在選取的日期範圍內取得的優惠列出。 清單中的供應專案名稱會顯示在清單中選取的供應專案類型。
- 針對每個篩選選項，預設選項為 [全部]，但 [**使用類型**] 除外。 [**使用類型**] 的預設選取專案為 [正規化使用方式]。 若要在圖表中顯示原始使用量，請選取 [原始使用量]。
- [套用的篩選] 會針對所做的篩選準則顯示選取的計數。 預設選項不會顯示套用的篩選。

> [!NOTE]
> 「詳細訂單資料」方格中的每個欄位的詳細定義、頁面篩選和所有可能的選取專案都定義于[常見問題和術語](link needed)文章的資料字典一節中。

[**計量付費使用量**] 索引標籤會顯示供應專案類型的使用量資訊，其中使用量是依計量維度來測量。 目前提供 SaaS 供應專案類型超額部分。 此索引標籤會針對 SaaS 計量付費使用量的超額趨勢呈現圖形化標記法：

- **依計量維度的超額趨勢**：顯示供應專案之所選計量維度的每月超額趨勢。 X 軸代表月份，而 Y 軸代表使用數量。 自訂計量的測量單位也會顯示在 Y 軸上。
- **依 sku 的超額趨勢**：代表所選計量維度依 sku 的使用數量趨勢。 所顯示的 Sku 將代表所選供應專案使用量最高的前5個 Sku。
- **前50客戶的超額趨勢**：最高使用時間的前50項供應專案會顯示在***領導者面板***上，並依自訂計量的最高使用量進行排序。 選取排行榜中的客戶，以查看所選計量維度的使用趨勢。
- **由頂尖客戶所超額的趨勢**：提供對整體使用百分比貢獻的最高客戶百分位數。 頂尖客戶百分位數會沿著 X 軸顯示，並取決於客戶的使用數量。 Y 軸會顯示使用數量。 您可以將滑鼠停留在折線圖上的點上方，以顯示詳細資料。

> [!NOTE]
> 系統會針對針對頁面篩選選取的任何計量維度，顯示此頁面上的使用量詳細資料和所有圖表。
