# <a name="usage-dashboard-in-microsoft-commercial-marketplace-analytics"></a>Microsoft 商業市集分析中的使用量儀表板

本文提供合作夥伴中心內使用量儀表板的相關資訊。 此儀表板會在兩個不同的索引標籤中顯示所有 VM 供應項目使用量和計量付費度量：VM 使用量和計量付費使用量。

若要存取使用量儀表板，請開啟 [商業市集] 下方的 **[分析](https://partner.microsoft.com/dashboard/commercial-marketplace/analytics/summary)** 儀表板。

>[!NOTE]
> 如需分析術語的詳細定義，請參閱[商業市集分析的常見問題集與術語](./faq-terminology.md)。

## <a name="usage-dashboard"></a>使用量儀表板

[使用量] 儀表板代表所有虛擬機器 (VM) 供應項目使用量和計量付費使用量的度量。 這些度量位在兩個不同的索引標籤，分別是：VM 使用量和計量付費使用量。

在 VM 使用量索引標籤中，有下列項目的圖形表示：

- [使用量摘要](#usage-summary)
- [依地理位置的使用量](#usage-by-geography)
- [依供應項目的使用量](#usage-by-offers)
- [依供應專案和方案的使用量趨勢](#usage-trend-by-offers-and-plans)
- [以供應項目類型的使用量](#usage-by-offer-type)
- [依 VM 大小的使用量](#usage-by-vm-size)
- [依銷售通路的使用量](#usage-by-sales-channel)
- [詳細使用量資料](#detailed-usage-data)

在合作夥伴中心產生和報告的使用事件之間的最大延遲為48小時。

### <a name="usage-summary"></a>使用量摘要

[使用量摘要] 資料表會顯示客戶所購買所有供應項目的客戶使用量時數。

- 標準化使用量時數的定義是可負責 VM 核心數目的標準化使用時數 ([VM 核心數目] x [未處理使用時數])。 指定為「SHAREDCORE的 VM」使用 1/6 (或0.1666) 作為 [VM 核心數目] 乘數。
- 未處理使用量時數的定義是 VM 已執行多久的時間 (單位為小時)。
- 百分比值代表所選日期範圍的使用量成長變化 \(\[上個月使用量–第一個月使用量\]/第一個月使用量\)。
- 向上指的綠色三角形表示成長的變化。
- 向下指的紅色三角形表示相對於上個月呈現負成長的變化。
- 微橫條圖代表每月的值。 您可以將滑鼠停留在圖表內的資料行上方，以顯示每個月的值。

### <a name="usage-by-geography"></a>依地理位置的使用量

[ **依地理位置的正規化使用量** ] 熱度圖會根據客戶國家/地區顯示對應的使用時數。 國家/區域的色彩變化代表標準化的使用密度。 按下地圖中的 [首頁] 按鈕可還原為原始檢視。

### <a name="usage-by-offers"></a>依供應項目的使用量

- [依供應項目的標準化使用量] 圓形圖會根據所選的日期範圍，依供應項目顯示標準化使用量的明細。 前五個供應專案會顯示在圖形中，其餘部分會分組到  [全部待用] 類別。
- 橫條圖會描述所選日期範圍的每月成長趨勢。 月份資料行代表個別月份使用時數最高的供應項目的使用時數。 折線圖描述繪製在次要 Y 軸上的成長百分比趨勢。
- 使用圖表頂端的滑杆，沿著 X 軸由右向左捲動，並/或將焦點放在特定資料點。

### <a name="usage-trend-by-offers-and-plans"></a>依供應專案和方案的使用量趨勢

此圖表顯示所選方案的正規化使用量趨勢， (先前稱為供應專案的 Sku) 。 供應項目排行榜會顯示前 50 個使用量最高的供應項目，依使用時數排序。 [方案排行榜] 會顯示前50方案，其中包含所選供應專案的最高使用量。

### <a name="usage-by-offer-type"></a>以供應項目類型的使用量

- [依供應項目類型的使用量] 圓形圖會根據供應項目類型來組織使用量。
- 主要的供應項目會顯示在圖表中，剩餘的供應項目則會分組為 [全部待用]。
- [趨勢] 圖表顯示每月成長趨勢。 [月份] 資料行代表該月份中前幾個供應項目類型的使用量。

### <a name="usage-by-vm-size"></a>依 VM 大小的使用量

此圖表代表所選 VM 大小的使用趨勢， (所有供應專案/方案的最大五個) 。 直條圖會以所選 VM 大小的使用時數來堆疊。

排行榜會顯示最高使用量的前 50 個 VM 大小，並依使用時數排序。

### <a name="usage-by-sales-channel"></a>依銷售通路的使用量

- [依銷售通路的使用量] 圓形圖會根據銷售通路來組織使用量
- 使用量最高的前幾名銷售通路會顯示在圖表中，而其他的銷售通路會分組為 [全部待用]。
- [月份] 資料行代表該月份前幾名銷售通路的使用量。
- 此圖表的功能與 [依供應項目的使用量] 圖表相同

### <a name="detailed-usage-data"></a>詳細使用量資料

[使用量詳細資料表] 會顯示依使用量排序的前 1000 個使用量記錄編號清單。

- 方格中的每個資料行皆可排序。
- 如果記錄計數小於 1000 項，則可將資料擷取至 CSV 檔案。
- 如果記錄計數超過 1000，則匯出的資料在接下來 30 天可供使用。
- 將篩選套用至 **詳細的使用量資料** ，只顯示您感興趣的資料。 依國家/地區、銷售通路、Marketplace 授權類型、使用類型、供應專案名稱、供應專案類型、免費試用、Marketplace 訂用帳戶識別碼、客戶識別碼和公司名稱來篩選資料。

> [!NOTE]
> 選取頁面篩選條件中的 [使用量類型] 以在 [標準化檢視] 或 [原始檢視] 中的頁面上檢視圖表。 這些圖表的預設檢視為「標準化檢視」。

**使用量頁面篩選條件**會在頁面層級套用。 您可以選取多個篩選條件來轉譯您選擇要檢視的準則圖表，以及要在 [詳細使用資料] 格線/匯出中顯示的資料。 篩選條件會套用至在訂單頁面右上角選取的資料範圍內所擷取資料。

- 系統只會列出您在所選日期範圍所取得供應項目的 [供應項目類型] 和[供應項目名稱]。 系統會為從清單中選取的供應項目類型顯示清單中的供應項目名稱。
- [使用量類型] 除外，每個篩選條件選項預設選擇 [全部]。 [使用量類型] 的預設選項是標準化使用量。 若要在圖表中顯示原始使用量，請選取 [原始使用量]。
- 套用的篩選條件會顯示所選篩選條件的計數選擇。 系統不會為預設的選取項目顯示所套用的篩選條件。

> [!NOTE]
> [詳細訂單資料] 方格中每個欄位的詳細定義、頁面篩選條件，以及所有可能的選取項目，皆定義於[常見問題與術語](link needed)一文的資料字典一節中。

[計量付費使用量] 索引標籤會顯示供應項目類型的使用量資訊，其中使用量是根據計量維度測量而得。 目前顯示的是 SaaS 供應項目類型超額部分。 此索引標籤會將 SaaS 計量付費使用量的超額趨勢以圖形表示：

- **依計量維度的超額趨勢**：顯示供應項目其所選計量維度的每月超額趨勢。 X 軸代表月份，而 Y 軸代表使用數量。 自訂計量的測量單位也會顯示在 Y 軸上。
- **依計畫的超額趨勢**：代表所選計量維度依方案的使用數量趨勢。 顯示的方案將代表所選供應專案使用量最高的前五個計畫。
- **前 50 名客戶的超額趨勢**：在***排行榜***上顯示前 50 名使用時數最高的供應項目，依客戶計量的最高使用量排名。 選取排行榜中的客戶可檢視所選計量維度的使用趨勢。
- **高排名客戶的的超額趨勢**：列出佔總使用量百分比的高排名客戶百分位數。 沿著 X 軸會顯示高排名客戶的百分位數，由客戶的使用數量決定。 Y 軸會顯示使用數量。 將滑鼠暫留在折線圖的點上可顯示詳細資料。

如果您有多個使用自訂計量的供應專案，計量付費使用量報表會根據其自訂計量維度，顯示您所有供應專案的使用量資訊。

> [!NOTE]
> 系統會針對頁面篩選條件所選取的任何計量維度，顯示此頁面上的使用量詳細資料和所有圖表。
