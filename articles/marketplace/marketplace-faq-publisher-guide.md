# <a name="common-questions-about-the-microsoft-commercial-marketplace"></a>有關 Microsoft 商業 marketplace 的常見問題

本文會回答有關 Microsoft 商業 marketplace 的常見問題。

## <a name="general"></a>一般

### <a name="what-is-microsoft-commercial-marketplace"></a>什麼是 Microsoft 商業 marketplace？

商業市場將商業和政府機構與合作夥伴所建立的解決方案連接在一起。 我們的合作夥伴會在合作夥伴中心建立和管理供應專案，而客戶可以透過共同銷售方案，在產品內部體驗中探索並購買解決方案，例如 Microsoft AppSource、Azure Marketplace、雲端解決方案提供者方案中的轉銷商，以及我們的銷售團隊。

若要深入瞭解，請前往[Microsoft 商業 marketplace 中樞](https://partner.microsoft.com/asset/collection/commercial-marketplace#/)。

### <a name="whats-the-difference-between-microsoft-appsource-and-azure-marketplace"></a>Microsoft AppSource 和 Azure Marketplace 之間有何差異？

AppSource 包括商務和產業解決方案，例如整合到 Microsoft 365、Dynamics 365 和 Power Platform 服務的應用程式。

Azure Marketplace 包括以 Azure 為基礎或在 Azure 訂用帳戶內使用的 IT 和開發人員解決方案。

若要深入瞭解，請移至[購買者指南](https://aka.ms/MarketplaceBuyerGuide)。

### <a name="why-should-i-make-my-software-or-services-available-in-the-microsoft-commercial-marketplace"></a>為什麼我應該在 Microsoft 商業 marketplace 中提供我的軟體或服務？

商用 marketplace 是進入市場共同作業的起點，並可透過銷售通路和 Marketplace 報酬來啟用。 網站和產品中的體驗可讓您的產品看到以您自己的解決方案為基礎的數百萬個客戶。 您甚至不需要建立或購買商務功能。 

商用 marketplace 以每個計畫為基礎，以141地區提供。 

我們會管理許多國家/地區的稅務付款，並提供存取轉銷商的全球通路網路。 您可以使用客戶的 Microsoft 客戶合約或 Enterprise 合約向組織銷售，而不需要建立新的購買關係。

若要深入瞭解，請觀看[為什麼要透過 marketplace 銷售](https://partner.microsoft.com/asset/detail/why-sell-through-marketplace)的影片。

### <a name="is-there-a-listing-fee-for-the-microsoft-commercial-marketplace"></a>Microsoft 商業 marketplace 是否有列出費用？

否，在我們的商業市場中發佈供應專案並不會產生任何費用。 我們會以交易費用的形式保留少量的銷售，並將其投資到 Marketplace 獎勵方案、平臺創新和商業 Marketplace 作業。

### <a name="how-do-i-sign-up-to-be-a-publisher-in-the-microsoft-commercial-marketplace"></a>如何? 註冊為 Microsoft 商業 marketplace 中的發行者嗎？

若要在商業 marketplace 中建立供應專案，您的組織必須同意 Microsoft 合作夥伴合約並接受發行者合約，才能成為 Microsoft 合作夥伴。

若要註冊為商業 marketplace 發行者，請移至[合作夥伴中心](https://partner.microsoft.com/dashboard/account/v3/enrollment/introduction/azureisv)。

### <a name="how-can-customers-engage-with-my-offers-in-the-commercial-marketplace"></a>客戶如何在商業 marketplace 中與我的供應專案互動？

客戶可以使用您的商業 marketplace 供應專案的方式，取決於您建立供應專案的方式。

- 對於您自動提供的解決方案，不論是付費、自備授權（BYOL）或免費，客戶都必須呼籲才能*立即取得*。
- 支援試用產品的供應專案（示範）要求客戶*試用磁片磁碟機*。 
- 軟體即服務（SaaS）供應專案，可在我們的商業市場外提供試用版，例如*免費試用版*。
- 簡單的清單可讓客戶要求*聯繫我*。

### <a name="how-do-i-create-offers-for-the-commercial-marketplace"></a>如何? 為商用 marketplace 建立供應專案嗎？

在您註冊並接受發行者合約之後，您就可以存取合作夥伴中心的 [商業 marketplace] 頁面。 這是您建立及管理供應專案的位置，以及您可以在其中查看分析報告的位置。 針對 Microsoft 365 應用程式，您也必須在合作夥伴中心內啟用 Office Store 程式。

如需詳細資訊，請參閱[歡迎使用商業 marketplace](https://docs.microsoft.com/azure/marketplace/) ，並[透過 Microsoft 商業 marketplace 銷售您的 Office 365 增益集](https://docs.microsoft.com/office/dev/store/monetize-addins-through-microsoft-commercial-marketplace)。

### <a name="how-do-i-get-support-assistance-for-the-commercial-marketplace"></a>如何? 取得商用 marketplace 的支援協助？

若要與我們的 marketplace 發行者支援小組聯繫，您可以從合作夥伴中心[提交支援票證](https://partner.microsoft.com/support/v2/?stage=1)。 

您也可以[加入我們的主動式社區論壇](https://www.microsoftpartnercommunity.com/t5/Microsoft-AppSource-and-Azure/bd-p/2222)，以瞭解最佳作法和分享資訊。

### <a name="what-analytics-are-available-to-my-organization-from-the-commercial-marketplace"></a>我的組織可以從商業 marketplace 取得哪些分析？

我們會在我們的商業市場中提供您供應專案的報告。 若要存取客戶、訂單、商店參與等等的資料，請移至[合作夥伴中心的商業 Marketplace 分析](https://docs.microsoft.com/azure/marketplace/partner-center-portal/analytics)。

### <a name="what-is-microsofts-relationship-with-my-customers"></a>Microsoft 與我的客戶之間的關係為何？

商用 marketplace 會使用代理程式模型運作。 客戶透過其 Microsoft Cloud 合約或 Enterprise 合約（EA）與我們簽署購買合約。 我們負責計費、集合、適用區域中的稅務付款，以及發行者支出。 身為發行者，您可以設定價格、設定使用條款，並提供技術支援。

### <a name="what-terms-of-use-options-are-available-with-the-commercial-marketplace"></a>商用 marketplace 有哪些使用規定選項？

身為發行者，您可以設定軟體解決方案的使用規定，客戶必須接受這項產品，才能進行購買流程。 為了簡化及加速事情，我們提供了標準合約，您可以選擇將其納入您的條款及條件。 這可讓客戶一次審查標準合約，取得法律和採購團隊的核准，然後專注于每項供應專案的修訂。

您可以建立適用于所有客戶的變更，也可以對特定客戶進行特定變更。 您也可以完全自訂您的條款及條件，這可在文字方塊中輸入或裝載于將顯示為 web 位址的網站上。

若要深入瞭解，請參閱[Microsoft 商業 marketplace 的標準合約](https://docs.microsoft.com/azure/marketplace/standard-contract)。

### <a name="is-azure-marketplace-available-in-azure-government"></a>Azure Government 中是否有 Azure Marketplace 可用？

是，Azure Government Marketplace 可在 Azure Government 區域內使用，並支援 Azure 虛擬機器和 Azure 應用程式供應專案。 有一個核取方塊選項，可在標準發佈程式期間，將供應專案提供給 Azure Government Marketplace。

### <a name="what-permissions-do-i-need-in-partner-center-to-manage-my-offers"></a>在合作夥伴中心管理我的供應專案時，需要哪些許可權？

有四個相關角色可管理合作夥伴中心的存取權。

- *管理員*：提供管理目錄中合作夥伴中心使用者的許可權，以及建立和更新供應專案。
- *開發人員*：允許使用者建立和更新供應專案。
- *財務參與者*：提供許可權來查看支出報告。
- *擁有*者：可以設定付款帳戶和稅務設定檔。 根據預設，接受發行者合約的使用者會成為擁有者。

Azure Active Directory （Azure AD）需要*全域管理員*許可權，才能建立新的使用者帳戶或邀請基礎目錄中的來賓帳戶。 合作夥伴中心會將 Azure AD 套用至身分識別模型，而組織的 Azure AD 實例通常是由 IT 或安全性小組管理及控制。

若要深入瞭解，請參閱[指派使用者角色和許可權](https://docs.microsoft.com/partner-center/permissions-overview)。

### <a name="do-customers-need-a-subscription-for-microsoft-azure-cloud-services"></a>客戶需要 Microsoft Azure 雲端服務的訂用帳戶嗎？

根據您的產品傳遞模型，可能需要 Azure 雲端服務的訂用帳戶。 直接在 Azure 訂用帳戶、Microsoft 365 應用程式或 Dynamics 365 應用程式中使用的解決方案，將需要客戶擁有現有的訂用帳戶。 在 Azure Marketplace 中購買的供應專案需要 Azure 訂用帳戶，因為此訂用帳戶會當做計費機制使用。

### <a name="whos-responsible-for-supporting-customers"></a>誰負責支援客戶？

商業 marketplace 發行者負責支援其軟體，而 Microsoft 負責支援購買、帳單、付款和訂用帳戶管理體驗。

### <a name="whats-the-process-of-validating-the-end-to-end-purchase-and-setup-flow"></a>驗證端對端購買和設定流程的程式為何？

在發佈過程中，您可以取得供應專案預覽的存取權。 只有您在 [**預覽**] 索引標籤中指定的使用者才能存取，而且該供應專案無法供其他人查看。 您可以購買預覽並測試程式，但會根據您的供應專案設定向您收取完整金額。

若要以非常低廉的價格完成購買，建議您發佈私人方案，讓您單獨查看，並以您可以接受作為測試成本的價格提供。 支援 $0.00 的價格，但不會反映完整的體驗，因為付費訂用帳戶會這麼做，因此我們建議您將價格設定為 $1.00。

### <a name="what-are-leads-from-the-commercial-marketplace-and-what-should-i-do-with-them"></a>什麼是商業 marketplace 的潛在客戶，我該怎麼處理？

當客戶在 Microsoft AppSource 上使用您的供應專案，並 Azure Marketplace 店面時，就會捕捉資訊。 在客戶展現出足夠的權益之後，就會建立潛在客戶，並將商機傳送至您選擇的系統。 成功使用我們的商業 marketplace 的發行者通常會尋求潛在客戶、開發關聯性，並將重點放在交易中。

如需詳細資訊，請參閱[來自您 marketplace 供應專案的客戶潛在客戶](https://docs.microsoft.com/azure/marketplace/partner-center-portal/commercial-marketplace-get-customer-leads)。

### <a name="why-arent-my-changes-reflected-in-the-offer-on-microsoft-appsource-or-azure-marketplace"></a>為什麼我的變更不會反映在 Microsoft AppSource 或 Azure Marketplace 的供應專案中？

在合作夥伴中心內所做的變更，只有在您重新發佈供應專案之後，才會在系統和商業市場中更新。 請確定您在任何變更後已提交供應專案供發行。

## <a name="channels"></a>聲道

### <a name="where-can-customers-find-my-commercial-marketplace-offers"></a>客戶可以在哪裡找到我的商業 marketplace 供應專案？

供應專案會根據您設定的方式和客戶的動作內容，透過產品內部經驗、網站、轉銷商和 Microsoft 銷售小組呈現給客戶。 如果您的軟體直接與 Azure、Microsoft 365、Dynamics 365 或 Power Platform 服務搭配使用，客戶就可以在產品中找到它。 Microsoft AppSource 和 Azure Marketplace 依供應專案類型和物件進行分割，而轉銷商只有在您已加入宣告雲端解決方案提供者（CSP）計畫後，才能存取您的供應專案。

### <a name="can-my-offer-appear-in-both-microsoft-appsource-and-azure-marketplace"></a>我的供應專案可以同時出現在 Microsoft AppSource 和 Azure Marketplace 嗎？

Microsoft AppSource 是商務和產業解決方案的目的地，而 Azure Marketplace 則是針對 IT 和開發人員解決方案所設計。 根據您在供應專案建立期間選取的類別和產業，每個店面都會提供供應專案。 在極少數的情況下，此供應專案與兩個店面的使用者有關，它可以同時出現在兩者中。

### <a name="what-is-the-cloud-solution-provider-program"></a>什麼是雲端解決方案提供者方案？

我們的全球轉銷商網路包含數十個參與雲端解決方案提供者計畫的組織。 CSP 計畫可讓組織從商業 marketplace 轉售我們的產品和供應專案，並提供他們可成功執行此工作的工具。

如需詳細資訊，請參閱[雲端解決方案提供者](https://docs.microsoft.com/azure/marketplace/cloud-solution-providers)。

### <a name="how-can-i-find-resellers-to-establish-partnerships"></a>如何找出轉銷商以建立合作關係？

藉由使用[解決方案提供者目錄](https://www.microsoft.com/solution-providers/home)，您可以依位置、產品、服務、技能、產業或組織名稱來搜尋我們的合作夥伴。

### <a name="can-i-limit-the-resellers-who-are-allowed-to-sell-my-offer"></a>我可以限制允許銷售我的供應專案的轉售商嗎？

是，您可以將供應專案提供給雲端解決方案提供者方案中的所有轉銷商，或僅供您指定的轉銷商使用。

如需詳細資訊，請參閱[雲端解決方案提供者](https://docs.microsoft.com/azure/marketplace/cloud-solution-providers)。

### <a name="how-can-i-pay-margin-to-resellers"></a>我要如何支付轉銷商的邊界？

商用 marketplace 無法指定轉售商的邊界，或直接或自動支付。 不過，您可以在商業 marketplace 系統外建立合約、銷售比賽或利潤。

### <a name="how-does-geographic-availability-work-with-resellers"></a>地理位置的可用性如何與轉銷商搭配運作？

供應專案和方案的可用性是由客戶的帳單位置所定義。 轉銷商會先選取要銷售的客戶，然後選取您的供應專案。 如果客戶位於不受支援的位置，您的供應專案將無法供他們使用。

## <a name="billing-capabilities"></a>計費功能

### <a name="whats-the-difference-between-transaction-and-monetization"></a>交易和營收之間的差異為何？

- 交易是貨物的金錢交換。 供應專案在商業 marketplace 中會被視為 transactable，如果它有價格，而且是由客戶透過我們的帳單和集合來支付。 
- 營收是為您的智慧財產收取費用的能力，這可能會發生在商業平臺內部或外部。 例如，您的自備授權（BYOL）小組應用程式可使用 transactable SaaS 授權服務來販賣。

### <a name="what-offer-types-are-transacted-through-microsoft"></a>哪些供應專案類型會透過 Microsoft 進行交易？

*現在*，您可以透過 Microsoft 針對某些傳遞模型來處理供應專案。 目前支援下列傳遞模型：
- VM 映像
- Azure 應用程式（受控應用程式）
- SaaS 應用程式

### <a name="what-billing-models-are-supported-in-the-commercial-marketplace"></a>商用 marketplace 支援哪些計費模型？

- VM 映射會根據每小時使用量計費，而且價格會設定為每小時。
- 受控應用程式的計費依據是選擇性的每月固定費用，加上計量付費和應用程式所使用 VM 映射的任何使用成本。
- SaaS 應用程式的計費依據是以固定費率、每位使用者為基礎，或依每月或每年的自訂計量收費。

### <a name="can-i-monetize-offer-types-that-are-not-transactable"></a>我可以銷售未 transactable 的供應專案類型嗎？

雖然並非所有供應專案類型都是 transactable 的，但您仍可觸及數百萬名客戶，並銷售您的智慧財產。 藉由在解決方案內新增授權檢查，您可以收集授權的付款。 銷售授權可以透過 Microsoft 或分開完成。

### <a name="can-i-offer-a-software-trial-through-the-commercial-marketplace"></a>我可以透過商業 marketplace 提供軟體試用版嗎？

是，有多種方式可以提供免費試用：

- 慣用的方法是在付費 SaaS 供應專案（一個月）和 VM 映射供應專案（一或三個月）的開頭新增免費期間。 客戶將會收到 $0.00 的費用，而且在試用期結束後，訂用帳戶會自動變更為 [付費]。 然後，合約詞彙會從該點開始。
- 您可以建立 SaaS 供應專案清單，讓客戶前往您選擇的網址，以在商業 marketplace 外部啟動試用版。 某些供應專案類型支援*試用產品，可*讓您向客戶提供示範。

### <a name="who-pays-for-the-underlying-azure-consumption"></a>誰支付基礎 Azure 耗用量？

如果解決方案直接用於客戶的 Azure 訂用帳戶，客戶須負責支付 Azure 成本。 如果您的 Azure 訂用帳戶中使用解決方案，例如 SaaS 應用程式，您必須負責 Azure 成本。 您應該以客戶的 SaaS 訂用帳戶成本來說明該費用。

### <a name="can-i-sell-professional-services-through-the-commercial-marketplace"></a>我可以透過商業 marketplace 銷售專業服務嗎？

針對雲端服務建立或建立的軟體可以透過商業 marketplace 進行交易。 尚未 transactable 專業服務。

## <a name="purchasing"></a>購買

### <a name="whats-required-to-purchase-solutions-from-microsoft-appsource-or-azure-marketplace"></a>從 Microsoft AppSource 或 Azure Marketplace 購買解決方案需要什麼？

Azure Marketplace 要求客戶必須擁有 Azure Active Directory 識別碼和有效的 Azure 訂用帳戶。 如果沒有或只有一個需求，則會要求客戶在購買流程中建立。

Microsoft AppSource 需要客戶擁有 Azure Active Directory 識別碼。 如果識別碼不存在，系統會要求客戶在購買流程中建立。

### <a name="what-payment-methods-are-supported-with-the-commercial-marketplace"></a>商用 marketplace 支援哪些付款條件？

Microsoft AppSource 支援信用卡付款。 Azure Marketplace 支援信用卡付款或發票。 透過 Azure Marketplace 購買的供應專案會自動新增至客戶的 Azure 帳單，並根據與訂用帳戶相關聯的付款方法收費。

### <a name="can-customers-apply-azure-credits-or-sponsorship-toward-purchases"></a>客戶可以將 Azure 點數或贊助套用於購買嗎？

沒有、預付或贊助的 Azure 支出無法用來從 Azure Marketplace 購買合作夥伴解決方案。

### <a name="what-azure-subscription-permissions-are-required-to-complete-a-purchase"></a>需要哪些 Azure 訂用帳戶許可權才能完成購買？

使用者帳戶必須具有 Azure 訂用帳戶的*擁有*者或*參與者*許可權。

### <a name="in-what-countries-can-customers-purchase-from-microsoft-appsource-and-azure-marketplace"></a>客戶可以從哪些國家/地區購買 Microsoft AppSource 和 Azure Marketplace？

客戶可以從位於世界各地141位置的商業市場取得解決方案。

若要查看位置的清單，請移至[商業 marketplace 發行者的稅務詳細資料](https://docs.microsoft.com/azure/marketplace/partner-center-portal/tax-details-paid-transactions)。

交易可以用下列17種貨幣來處理：

若要查看貨幣清單，請移至 [[地理位置] 和 [商業 marketplace 的貨幣支援](https://docs.microsoft.com/azure/marketplace/marketplace-geo-availability-currencies)]。

### <a name="can-customers-change-subscriptions-or-license-volumes-mid-term"></a>客戶是否可以定期變更訂用帳戶或授權量？

客戶可以隨時增加或減少每位使用者 SaaS 訂閱的使用者計數。 無論任何磁片區或計畫升級，授權訂閱條款都會維持不變。 透過應用程式開發介面（Api），現有的訂閱也支援計畫變更。 發行者可以使用 API 呼叫來變更客戶的訂用帳戶方案。 我們鼓勵獨立軟體廠商（Isv）在升級和降級情況的 SaaS 解決方案內考慮客戶的使用者體驗。

如果已針對現有的訂用帳戶變更計畫，此期限會維持不變，而且成本會根據剩餘詞彙的持續時間按比例計算。

### <a name="what-could-block-a-customer-from-completing-a-purchase"></a>哪些專案會封鎖客戶完成購買？

針對 Microsoft AppSource，客戶必須在與 Azure AD 實例相同的國家/地區中擁有信用卡，而且客戶的 Azure AD 系統管理員必須確保系統不會防止驗證 Microsoft AppSource。

針對 Azure Marketplace，客戶必須擁有有效的 Azure 訂用帳戶，以及具有該訂用帳戶之*擁有*者或*參與者*許可權的使用者帳戶。 客戶的 Azure 系統管理員必須確定 Azure 原則不會與所使用的供應專案發生衝突，而且無法設定成本管理來停用 Azure Marketplace。

## <a name="collection-and-payouts"></a>集合和支出

### <a name="when-are-customers-billed-for-their-purchases-in-the-commercial-marketplace"></a>客戶何時會針對其在商業 marketplace 中的購買付費？

客戶通常會按月計費，以在先前的帳單週期中進行任何購買。 計量付費供應專案（例如 VM 映射、SaaS 應用程式和受控應用程式）的耗用量供應專案會在使用後計費。 以訂用帳戶為基礎的方案會在期限一開始收費。

### <a name="how-often-will-i-receive-payouts-for-sales-through-the-commercial-marketplace"></a>我會多久透過商業 marketplace 收到支出的銷售資訊？

支出每月發生一次。 在上個月中處理的集合會一次付費，而銷售必須有最小的結合值 $50.00。

信用卡訂閱和購買會按月計費，而針對 Microsoft 客戶合約或 Enterprise 合約的發票可能較不頻繁。 從客戶計費的時間開始，我們會提供30天的費用。 在我們收到付款之後，信用卡交易就會進入30天的證書處理期間。

如需詳細資訊，請參閱[付款臨界值、方法和時間](https://docs.microsoft.com/azure/marketplace/partner-center-portal/payment-thresholds-methods-timeframes)範圍。

### <a name="in-what-countriesregions-are-taxes-managed-on-my-behalf"></a>哪些國家/地區會代表我管理稅金？

客戶可以從世界各地的商業市場尋找解決方案，並在許多位置管理稅金。 

若要查看完整的位置清單，請移至[商業 marketplace 發行者的稅務詳細資料](https://docs.microsoft.com/azure/marketplace/partner-center-portal/tax-details-paid-transactions)。

### <a name="where-can-i-find-payout-reporting-for-the-commercial-marketplace"></a>哪裡可以找到商業 marketplace 的支出報告？

在合作夥伴中心內，具有「*擁有*者」或「*財務參與者*」許可權的使用者可以查看支出報告。 這些使用者可以在右上方存取報表，方法是選取顯示保留帳單的圖示。

如需詳細資訊，請參閱[支出報告](https://docs.microsoft.com/azure/marketplace/partner-center-portal/payout-summary)。

### <a name="how-do-i-configure-my-payout-method-and-destination"></a>如何? 設定我的支出方法和目的地？

使用「合作夥伴中心」時，具有「*擁有*者」許可權的使用者可以設定付款帳戶和稅務表單。 使用者可以選取右上方的 [**設定**] 按鈕（齒輪圖示）來執行這項操作。

如需詳細資訊，請移至付款[帳戶和稅務表單](https://docs.microsoft.com/azure/marketplace/partner-center-portal/set-up-your-payout-account-tax-forms)。

### <a name="are-returns-and-refunds-supported-with-the-commercial-marketplace"></a>退貨和退款是否支援商業 marketplace？

若要接收退款，客戶可以在指定的時間範圍內取消訂用帳戶。 使用解決方案後根據使用量計費的供應專案不適合退款。
