# <a name="connect-windows-security-events"></a>連線 Windows 安全性事件 

安全事件連接器允許您將所有安全事件（伺服器和工作站、物理和虛擬）資料流到 Azure Sentinel 工作區。 這使您能夠在儀表板中查看 Windows 安全事件，在創建自訂警報時使用它們，並依靠這些事件改進調查，讓您更深入地瞭解組織的網路並擴展安全操作能力。 您可以從以下集中選擇要資料流的事件：<a name="event-sets"></a>

- **所有事件**- 所有 Windows 安全和 AppLocker 事件。
- **通用**- 用於審核目的的標準事件集。 此集中包含完整的使用者審核跟蹤。 例如，它包含使用者登錄和使用者登出事件（事件標識 4624、4634）。 還有審核操作，如安全性群組更改、金鑰網域控制站 Kerberos 操作以及符合已接受最佳做法的其他類型的事件。

    **公共**事件集可能包含某些類型的不太常見的事件。  這是因為**公共**集的要點是將事件量減少到更易於管理的水準，同時仍然保持完整的審核跟蹤功能。

- **最小**- 一組可能指示潛在威脅的事件。 此集不包含完整的審核跟蹤。 它只涵蓋可能指示成功違規的事件，以及發生率非常低的其他重要事件。 例如，它包含成功和失敗的使用者登錄（事件標識 4624， 4625），但它不包含登出資訊 （4634），這些資訊雖然對審核很重要，但對違規檢測沒有意義，並且具有相對較高的容量。 此集的大部分資料量由登錄事件和進程創建事件（事件 ID 4688）組成。

- **無**- 無安全或 AppLocker 事件。 （此設置用於禁用連接器。

> [!NOTE]
> 可以從 Azure 安全中心或 Azure Sentinel 配置單個工作區上下文中的安全事件集合，但不能同時配置這兩個集合。 如果在已在運行 Azure 安全中心的工作區中載入 Azure Sentinel，並且設置為收集安全事件，則有兩個選項：
> - 將安全事件集合保留在 Azure 安全中心中。 您將能夠在 Azure Sentinel 和 Azure 安全中心中查詢和分析這些事件。 但是，您將無法監視連接器的連接狀態或在 Azure Sentinel 中更改其配置。 如果這對您很重要，請考慮第二個選項。
>
> - 禁用 Azure[安全中心中的安全事件集合](../security-center/security-center-enable-data-collection.md)，然後僅在 Azure Sentinel 中添加安全事件連接器。 與第一個選項一樣，您將能夠在 Azure Sentinel 和 Azure 安全中心中查詢和分析事件，但現在您將能夠監視連接器的連接狀態或更改其配置（並且僅在 Azure Sentinel 中）。

## <a name="set-up-the-windows-security-events-connector"></a>設置 Windows 安全事件連接器

要在 Azure 哨兵中收集 Windows 安全事件，請執行：

1. 在 Azure Sentinel 導航功能表中，選擇 **"資料連線器**"。 從連接器清單中，按一下**安全事件**，然後在右下角的 **"打開連接器"頁面**按鈕上按一下。 然後按照 **"說明"** 選項卡下的螢幕上的說明進行操作，如本節其餘部分所述。

1. 驗證您是否具有"**先決條件**"所述的適當許可權。

1. 在要將安全事件資料流到 Azure Sentinel 的電腦上下載並安裝[日誌分析代理](../azure-monitor/platform/log-analytics-agent.md)（也稱為 Microsoft 監視代理或 MMA）。

    對於 Azure 虛擬機器：
    
    1. 按一下**Azure Windows 虛擬機器上的安裝代理**，然後按一下下面顯示的連結。
    1. 對於要連接的每個虛擬機器，按一下右側清單中的名稱，然後按一下"**連接**"。

    對於非 Azure Windows 電腦（物理、虛擬上部或另一個雲中的虛擬）：

    1. 按一下**非 Azure Windows 電腦上的安裝代理**，然後按一下下面顯示的連結。
    1. 按一下右側顯示在**Windows 電腦**下的相應下載連結。
    1. 使用下載的可執行檔，在您選擇的 Windows 系統上安裝代理，並使用上面顯示的下載連結下方的**工作區 ID 和金鑰**進行配置。

    > [!NOTE]
    >
    > 要允許沒有必要互聯網連接的 Windows 系統將事件仍資料流到 Azure Sentinel，請使用右下角的連結在單獨的電腦上下載並安裝**OMS 閘道**，以充當代理。  您仍然需要在每個要收集其事件的 Windows 系統上安裝日誌分析代理。
    >
    > 有關此方案的詳細資訊，請參閱[**日誌分析閘道**文檔](../azure-monitor/platform/gateway.md)。

    有關其他安裝選項和更多詳細資訊，請參閱[**日誌分析代理**文檔](../azure-monitor/platform/agent-windows.md)。

1. 選擇要資料流的事件集（[全部、通用或最小](#event-sets)）。

1. 按一下 [更新]****。

1. 要在日誌分析中為 Windows 安全事件使用相關架構`SecurityEvent`，請鍵入查詢視窗。
