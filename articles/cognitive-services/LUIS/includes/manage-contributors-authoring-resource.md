如果應用需要由其他人創作，則需要在 Azure 門戶的創作資源中添加關聯的電子郵件地址。

如果您是 Azure**訂閱的擁有者或管理員**，則可以向資源添加參與者。

如果您不是**Azure 訂閱的擁有者或管理員**，則 Azure 帳戶需要具有`Microsoft.Authorization/roleAssignments/write`許可權。

如果在此角色指派時遇到問題，請查看：

* [Azure 角色指派](../../../role-based-access-control/role-assignments-portal.md)
* [Azure 存取控制故障排除](../../../role-based-access-control/troubleshooting.md#problems-with-azure-role-assignments)
